Degree of Completion

The degree of completion in the Realized % Based on cost field is a very important ratio when considering a job. It is normally very hard for the system to calculate for jobs and so it is usually estimated.

cc|project integrates a concept for calculating individual degrees of realization based on the job lines. This concept also enables global degrees of completion (at the assembly level or for the job) to be presented. The completion level for each WBS position is determined automatically by the system, but it can also be performed manually by the user. This allows the user to have a direct influence on WBS positions which do not themselves have any substructures. The completion level is displayed at WBS position level in the Realized % Based on cost field.

Note

For linked WBS positions, the completion level is determined via the substructures.

Realized % fields are displayed in the Job Planning and Processing view and the Job Rolling Calculation view.

Field

Description

Realized % (Base)

Determines if the Realized % value is based on Quantity or Cost.

Realized % Based on cost

Non-editable field that is calculated over substructures.

Realized % Finalized

Provides the ability to manually overwrite automatic calculated values.

Signal %

Provides an index for variance on calculated values; used with the Signal Monitor Tolerance % field in Jobs Setup.

Signal Monitor

Displays variances in three defined statuses (refer to topic “Signal Monitor” on page 1).
