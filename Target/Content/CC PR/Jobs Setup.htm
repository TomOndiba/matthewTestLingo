Jobs Setup

The general setup for cc|project is entered in the Jobs Setup page, accessed from the Departments menu by clicking cc|project > Administration > Jobs Setup.

: Jobs Setup page, General and Numbering FastTabs

Note

Although the Job Setup Wizard provides many default settings, there are additional fields that may need to be set up in the Jobs Setup page. The default settings can also be reviewed and changed in this page.

Job FastTab

On this FastTab, the default fields for the Job Card are configured.

: Jobs Setup, Job FastTab

The fields on the Job FastTab are described in the following table.

Field

Description

Job Version Start

Initializes the format for job versions. Each new job is marked as a first version.

New Version per Status change

Prompts for a new job version when a job status changes. The options are:

  • Never – Option will never be used
  • Optional – Option will be used on request
  • Always – Option will always be used

Default Sales Code

Identities the Sales Code that will be used as the default for all new jobs.

Default Unit of Measure Code

Controls which Unit of Measure is used in calculation lines as a default. The options are:

  • Base Unit of Measure
  • Sales Unit of Measure
  • Purchase Unit of Measure

WBS Format

Determines how WBS positions are formatted in the job structure. The options are:

  • Without Level 1 – no WBS code will be created for level 1. Numbering is starting with level 2.
  • Components only – WBS codes are only created for lines with components (resources, items, resources, work/machine centers and general components).

Realized % (Base)

Used to automatically calculate the Realized % field. The options are:

  • Quantity – Posted / calculated quantity
  • Cost – Posted / calculated cost

This can also be set up by using the field on a component (with higher priority).

Location for Inv. Requests

Identifies the default location for inventory request related procurement planning.

Location for Purchase

Identifies the default location for purchase related procurement planning.

Procurement Planning Required

If selected, the planning and release of a procurement process is mandatory.

Job Dimension

Defines the dimension for automatically generated default dimension values for the Job No. field.

Autom. Job Dimension

Determines if the dimension value should be inherited from the Job No. of the current job or from the main job.

Job Type Dimension

Allocates the dimension depending on the job type.

The default dimension values are assigned in the Job Types Setup page.

Dim. from Cust. - Copy

If selected, default dimensions from the customer are copied to the job.

Dim. from Cust. - Overwrite

If selected, default dimensions from the customer are copied to the job. Existing default dimensions on the job will be overwritten.

Dim. from Person Responsible

If selected, default dimensions from the person responsible are copied to the job.

Copy Dimension to Purchase

If selected, job dimensions are copied to the document dimensions in purchase documents.

Copy Dimension to Sales

If selected, job dimensions are copied to the document dimensions in sales documents.

Job Line Dimension 1 /  Job Line Dimension 2 /  Job Line Dimension 3

Job line dimensions should be used if the dimensions are dependent on the job structure, i.e. structure line dimensions are inherited by the upper structure.

In addition to defining a single dimension for the job structure, it is possible to differentiate three dimensions on the job lines.

The following rules apply for the use of dimensions on the job line:

  1. The job line dimension is taken over from the component in the job line as a default value.
  2. The job line dimension is extracted from a higher level, if no dimensions are attached to the individual component.
  3. An alteration of a job line dimension results to a subsequent alteration within the sub-structures.
  4. In the process of downgrading the project line, the dimension value is taken over from the highest position.

The dimensions are transferred from the job line to the document that is generated from this line.

Default WIP Basis

Indicates the WIP Basis that will be defaulted to the Job Card each time a Job is created. The options are:

  • Sales Code
  • WIP Position
  • Job

Change Sales Code After Posting

Determines if Sales Codes can be changed after posting to the job line. The options are:

  • Never – Sales Code cannot be changed
  • Optional – Sales Code can be changed after confirmation
  • Always – Sales Code can be changed

Procurement FastTab

On this FastTab, all settings relating to Procurements are defined.

: Jobs Setup, Procurement FastTab

The fields on the Procurement FastTab are described in the following table.

Field

Description

Procurement Cost Source

Determines the pricing method for the Direct Unit Cost in the procurement process.

Signal Monitor Tolerance %

Indicates a buffer for controlling differences in the Realized % field.

Fixed Job Line Purchase

Can be used to implement a function for automatic WBS finding in Purchase Lines.

Job Journal Time Entry / Job Jnl. Batch Ext. Time Entry

Identifies the default job journal templates for internal and (optionally) external time entries.

Note

A special module is required for external time entry.

Terminal Posting Allowed

If selected, the person recording time can also post their time in the Terminal.

Post Released Quantity only

If selected, released procurement lines are set to Finished automatically when the procurement documents (purchase order, inventory request) are posted.  With this option, it is possible to post up to the released quantity, and when that quantity is reached, the procurement lines will be closed. An error will occur if there is an attempt to post more than the release quantity.

Note

This option cannot be used with unplanned travel expenses if it is not possible to release a quantity for an unplanned cost; therefore use the Auto. Close Proc. Line field to handle automatic posting (refer to next field).

Auto. Close Proc. Line

If selected, released procurement lines are set to Finished automatically when the procurement documents (purchase order, inventory request) are posted. With this option, it is possible to post as much as needed with no limit to the released quantity. However, the procurement lines will be closed if the full released quantity or more are posted.

Note

Only this option can be used with unplanned travel expenses because it is possible to post unplanned costs.

Req. Wksh. Template / Req. Wksh. Name

Identifies the template and name of the requisition worksheet; required to use the purchase procurement channel is used.

External Services Code / External Trav. Exp. Code

Allows the use of resources on purchase documents using a link to a G/L account.

Order Confirmation Required

If selected, the Order Confirmation Required check box is selected in Purchase Documents.

Job Journal Inventory Requests

Indicates the job journal to be used for inventory requests.

Inv. Request per Location

If selected, multiple inventory requests will be created for a procurement covering several positions.

Populate Qty. in Inv. Requests

If selected, the picking amount will not be confirmed on order completion. 

Check Quantity in Inv. Req.

If selected, an inventory audit of the respective storage location will be checked during posting.

Def. Prod. Order Status

Determines the default production order to be used if production procurement is used.
