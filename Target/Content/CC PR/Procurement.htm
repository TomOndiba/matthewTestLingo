Procurement

Procurement is displayed in the Job Planning and Processing view. All functions described below are executed or accessed from this view.

Process Planning Defaults

After the process planning has been completed, the following data is available for the purchasing process:

  • Process start date (will be inherited from the top level structure down to the bottom level)
  • Process end date (will be inherited from the top level structure down to the bottom level)
  • Planned hours (calendar entries, simulated inclusively)
  • Planned resources (existing calendar entries)

Procurement Planning

To create purchasing activities from the calculated lines, at least the following two fields must be defined:

  • Procurement Channel – defines which purchasing process is used.
  • Procurement Finish Date – defines the due date. The date is transferred from the process planning. If no date is defined in the process, then the current date will be used.

The following optional data can be displayed in this view:

  • Procurement Vendor No. – can be a different from the Vendor No. used for the original calculation.
  • Procurement Location Code – can be modified according to the inventory requirements. If nothing else is selected the default setting from the Job Card will apply.
  • Procurement Cost Source – defines which wholesale price will be used for the purchase order.

The “Procurement Planning” function is accessed on the Navigate tab of the Job Planning and Processing view by clicking Procurement > Planning in the Processing action group.

: Job Planning and Processing view with Procurement fields

The planned positions can be easily identified after a successful procurement planning run (click the Yes value in the Procurement Planning field for an automated procurement planning). Automated Planning will only apply for those positions holding unplanned remaining quantities. Missing dates will be supplemented.

Individual lines can be amended by clicking on the value in the Procurement Planning field.

The procurement planning for all job lines can be viewed in the Job Procurement Planning page, accessed from the cc|project menu by clicking Planning > Procurement Planning.

Procurement Release

After the planning process is complete, procurement WBS positions with a status of Planning can be released by clicking Navigate > Procurement > Release.

Released procurement documents can be created during the release process. Created documents are displayed per procurement channel.

: Released procurement lines for job J000001.01

Rolling Calculation after Procurement Release

Values from released procurement lines are displayed in the Total Direct Cost Released (LCY) field on the Job Rolling Calculation view.

: Rolling Calculation for job J000001.01

Released amounts in procurement documents can vary from calculated values for any of the following reasons:

  • Release is not complete.
  • Released value is higher than calculated.
  • Procurement Direct Unit Cost is different in procurement lines and calculation lines.
  • Procurement values are moved to other WBS positions (Summarize Position).

Time Entry

Time entry refers to the recording of an employee’s actual time spent and expenses incurred while performing a job-related task. In cc|project, an employee can enter their time in a personal Job Journal batch or a stand-alone Terminal can be set up for multiple user time entry.

In this section, the use of the Job Journal and a Terminal are explained for entering job-related tasks and expenses.

Note

Select the Job Time Entry Administration check box in User Setup to allow a user can see all Job Journal Batches. If not set, a user can only see their own time entry journal. Refer to topic “User/Profile Setup” on page 1 for more information.

Time Entry Job Journal

The Time Entry Job Journal is accessed from the navigation pane by clicking cc|project > Processing > Time Entry Job Journals.

The Time Entry Job Journal is a worksheet used for the direct recording of: 

  • Hours spent (for service lines)
  • Travel expenses (for travel expense lines)
  • Consumables (i.e. for maintenance jobs)

Each person should have an individual worksheet (batch) for time recording purposes.

Note

Job Journal Batches are automatically created when persons are created as resources and the Person for Time Entry field is populated with the corresponding Resource No.

: Time Entry Job Journal for employee P001

Once the Job No. field is populated, a particular job line can be chosen in one of the following ways:

  • Select the WBS position then the Job Procurement No.
  • Select the Job Procurement No. directly.

The first option will display the positions that have been released for time recording (independent of the allocated person). A released procurement line will be taken automatically if only one exists; multiple lines need to be selected separately.

For the second option, leave the WBS field empty and click the drop-down arrow in the Job Procurement No. field. This will display only the lines that have been assigned to the person logged in or which do not have a dedicated assignment.

Fig. 8 illustrates a job journal with recorded positions.

: Job lines entered in the Job Journal for employee P001

During the posting process, job entries are generated with Type = Usage. These values can be found in the Total Direct Cost Usage RU (LCY) field in the Job Planning and Processing view.

: Posted job entries as displayed in the Total Direct Cost Usage RU (LCY) field

Note

The Time Entry field is still set to Yes because the Procurement Planning Lines have not been set to Finished yet.

Time Entry Terminal

A terminal can also be used for time entry. The terminal enables a single workstation dedicated to time entry to be made available for several users.

Note

Posting availability for the terminal can be optionally allocated on the Procurement FastTab in the Jobs Setup page.

: Job Terminal

After logging in, an employee enters their time the same as they would in the Job Journal. Afterwards, they can log off so that another employee can then enter their hours.

Purchase Processes

Direct purchases can be triggered in the following ways:

  • Create a Purchase Quote
  • Create a Requisition Worksheet line
  • Create a Purchase Order directly

As Fig. 18 illustrates, a quote, requisition line and order have been created for the purchased items job lines.

: Job Planning Processing view for purchased items

Purchase Quotes and Requisition Worksheet lines will create Purchase Orders. To open the Purchase Quote or Order from the job line, click the Yes value in the relevant document field or in the Procurement Release field. Requisition Worksheets cannot be processed from the job views and therefore must be opened from the navigation pane by clicking Departments > Purchase > Planning.

During the Purchase Order process, differences for quantity and/or costs in regards to calculation are possible.

: Costs from posting purchase-related procurement lines

Transfer Purchase Conditions from Purchase Quote

Purchase prices and line discounts can be transferred from the purchase quote to the master data (Item). In this example, the purchase quote created during Procurement has been opened and pricing has been updated according to the vendor. The price remains but a small discount has been added. Currently, there are no purchase prices or line discounts assigned to this item (as displayed in the Purchase Line Details FactBox).

: Purchase Quote created from procurement, preparing to run Transfer Purchase Conditions function

When the “Transfer Purchase Conditions” function is run, a message appears to select if this is general pricing or job-specific pricing.

  • If General Pricing is selected, the purchase price and/or line discount is transferred without any job information.
  • IfJob specific price is selected, the purchase price and/or line discount is transferred with job information so that the pricing is relevant only for the job.

In this example, the Job specific price option is selected and a confirmation message appears with the results.

: Confirmation message from running the “Transfer Purchase Conditions” function

In the Purchase Line Details FactBox, notice that both Purchase Prices and Purchase Line Discounts indicate purchasing conditions have been created. Clicking the Purchase Prices/Purchase Line Discount value opens the Get Purchase Price and Get Purchase Line Disc. pages, which show the job-specific information.

: Job-specific purchase price and line discount information

Notice that not only is the job number assigned but also the Main Job No. This means that the special pricing is valid for any job created for the main job. If this item and vendor combination is used in other jobs not related to this main job, the special pricing is not used.

The Source Quote No. and Source Version No. fields provide historical information to indicate where the pricing originated. Clicking Actions > Show Source Document opens the Purchase Quote Archive page.

: Purchase Quote Archive created from running Transfer Purchase Conditions function

Note

Ideally, all costing/pricing should be handled in the job budgeting phase and, therefore , the “Transfer Purchase Conditions” function is most beneficial when used with the cc|purchase quotes manager.

Finish Procurement

When the procurement documents (purchase orders and inventory requests) are posted, the Procurement Status on the released procurement lines is automatically set to Finished if the:

  • Post Released Quantity Only check box is selected on the Procurement FastTab in the Jobs Setup page
  • full quantity has been allocated

For time entry lines, there is no such automation because usually no direct quantity matching can be achieved. Therefore, these lines are set to finished manually by clicking Navigate > Procurement > Set Finished.

Note

This function can be used on any lines that need to be set as finished.

Finished procurement lines are indicated through the Procurement Finished field.
