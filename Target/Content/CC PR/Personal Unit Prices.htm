Personal Unit Prices

During contract negotiations, special unit prices for certain resources may be agreed upon. In these cases, the unit price on the invoice depends on if the person has posted usage.

Special Setup

The special settings that should be considered when setting up personal unit prices are described in the following table.

Setup Page

Description

Service Card (Resource Card, General Component Card)

In the Procurement Cost Source field, select Standard Costs to use the direct unit costs from the person with the posting process.

Work Types for Services (assigned as Service Work Type Code on Resource Card)

It is possible to use more Work Type Codes than the default assigned on the Resource Card for Services (resources with Type = Service).

Work Types are used to set up different prices. For example, Service Consulting has three entries in the Work Types page:

  • SV CONS is the standard work type for calculation
  • SV CON JUN and SV CON SEN are for pricing setup

Service Work Types are used to define direct unit costs and unit prices for the calculation.

Resource Card

Persons (resources with Type = Person) have default work types, which are defined in the Person Work Type Code field on the Person FastTab.

  • Person Work Types are the basis for direct unit costs in the job journal.
  • Sales prices for Person Work Types only need to be set up for personal unit prices. Otherwise unit prices are taken from the job line.

Note

Service resources and Service Work Type Codes are linked on the Job Sales Work Types page (refer to topic “Services and Persons Link” on page 1). Additionally, Resources and Work Type Codes can be linked to a unit price on the Job Card through the Job Resource Prices page (refer to topic “Pricing Setup” on page 1).

To use personal unit prices in job journal lines, a special Job Sales Code must be set up in order to select the sales position in the job line. The fields must be set up as follows:

  • Sales Channel = Usage
  • Sales Price Source = Standard Prices

: Job Sales Codes for personal unit prices selection in job journal lines

Services and Persons Link

In cc|project, the service is used as a component in the calculation and the person is used to enter time against a service position. All necessary information is linked in the Job Sales Work Types page, accessed from the cc|project menu by clicking Administration > Job Sales Work Types.

: Job Sales Work Types

Use the Job Sales Work Types page to specify which Service Work Type Code has to be taken for pricing based on the Service (component in calculation) and Person (Person Work Type Code field in time entry) combination.

Pricing Setup

There are two settings for personal unit prices:

  • Standard Unit Price on the Service Work Type Code (cc|project > Administration > Work Types > Prices)

: Standard unit price for the SV CON JUN service work type

  • Special Unit Price for a Job (Job Card > Navigate > Resource)

: Job Resource Prices (for services) assigned to job J000001.02

Notice that the standard Unit Price for SV CON JUN is 80.00 but for job J000001.02, the unit price for a junior consultant is 90.00.

Calculation

Services – but not actual person-type resources – can be used as a component in a calculation line. Therefore, the standard work type for services is adequate for calculating the price.
