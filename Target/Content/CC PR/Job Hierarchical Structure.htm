Job Hierarchical Structure

The two types of hierarchical structures in cc|project are set up in the:

  • Job Card
  • Job Budget Calculation View

This section describes the structure of each.

Job Card (Main and Sub Jobs)

Typically, a job contains a main job (general project) and multiple sub-jobs (specific projects) that are linked to the main job. In cc|project, this hierarchical structure is defined in the Job Cards.

On the main job, all information relevant for the sub-jobs is entered; some information will be transferred automatically from the default settings in the Jobs Setup page but can be adjusted as needed per job.

: Main Job

From the main job, click Create Sub-Job on the ribbon to automatically create the sub job. The first part of the new job number is the Job No. from the main job, and the second part is an extension with the format of .00. 

: List of main job and sub-jobs

The “Create Sub-Job” function also copies the following from the main job to the sub-job:

  • General information (customer, posting groups, etc.)
  • Dimensions
  • Sales Prices
  • Resources

The information on the sub-job can be changed and does not affect the main job.

: Sub-Job with comments, resources and prices

Job Budget Calculation View

Within each job, each cost or revenue line can be defined, tracked and analyzed. This is possible because of the hierarchical structure within jobs in cc|project. In this section, the hierarchical structure of a job is described.

The complete job structure is displayed by clicking Budget Calculation on the ribbon.

: Job Budget Calculation, displaying the hierarchical structure

Work Packages

Similar to a Work Breakdown Structure (WBS) in Microsoft Project, a work package is a logical group of WBS lines that performs a task on a project. It can be a single WBS line or a group of WBS lines in the job structure. In cc|project, the work package is created when the component information (Type and No. fields) is entered on a line. A work package can be set up with the following components:

  • Services
  • Travel charges
  • Items
  • Machine/Work Centers
  • Other (components)

 

All default information from the component is transferred to the work package, including component pricing. Special job prices for services based on work types are also taken into account in the pricing. Work packages can also be recorded using alternative units of measure.

Once a line is created with the default information, it can be updated with the necessary information and the structure will be extended if needed. It is possible to overwrite components after initial setup. When cc|project recognizes an intended change, the Job Line Update Rules page is displayed that asks for a more detailed specification of the changes to be applied.

: Job Line Update Rules dialog

Positions and Levels

The hierarchical structure within cc|project is organized by Positions and Levels. By default, 25 nested levels are available. This can be increased by the user as required.

The Position is the top level within the job. Depending on the number scheme selected, this will be 1., 2., 3., and so on; or 001, 002, 003, and so on. Under each Position, multiple Levels can be attached, and Levels can exist within Levels.

To understand Positions and Levels, refer to Fig. 4 for this example:

  • Line 1. (Total Price) is a Position
  • Lines 1.1. (Services), 1.2. (Conveyor System) and so on are Levels of Position 1. (Total Price)
  • Lines 1.1.1. (Project Management), 1.1.2. (Consulting) and so on are Levels of Level 1.1. (General Services)
  • Lines 1.2.1.1 (Hydraulic system) and 1.2.1.2. (Machine department) are Levels of Level 1.2.1.
  • Lines 1.2.1.1.1 (Hydraulic Ventile Type 70/B) and 1.2.1.1.3 (Machine department) are Levels of 1.2.1.1.

For information about managing the structures, refer to topic “Create Structure” on page 1.

Job Directories

As substructures are created, cc|project automatically creates directories. These directories are marked by a plus (+) or minus (-) sign in the column:

Indicates that there are substructures below this level.

Indicates that the substructures below this level have been expanded.
