Planning Processes Page

The Planning Processes page manages the processes for a job, which are derived from the calculation lines. The view contains the essential information for:

  • work and duration
  • predecessor / successor relationships
  • restriction types

: Planning Processes page

Note

This page will not contain any lines until the Planning Processes have been completed (refer to topic “Job Process Planning and Scheduling” on page 1).

In the Planning Processes page, a significant number of fields are populated with values from the Job Planning and Processing view. Therefore, the following table does not provide a complete description of each field.

Field

Description

Planning Status

Indicates the current state of the WBS position. The options are:

  • Planning
  • Finished
  • Release

 

Process ID

Indicates the unique process number assigned to the WBS position (refer to topic “Process Logic” on page 1).

Predecessor ID

Creates links between WBS positions (refer to topic “Process Logic” on page 1).

Predecessor / Follower

Indicates the hierarchical link between WBS positions (refer to topic “Process Logic” on page 1).

Description

Defaults from the Job Planning and Processing view.

Comment General

Indicates if a job comment line with Comment Type = General exists for the WBS position.

Milestone

If selected, the WBS position is set for milestone processing.

MSP Interface

Indicates a mapping to Microsoft Project (refer to topic “Microsoft Project Integration” on page 1).

Work Quantity / Work Format

Identifies the amount of time to complete the WBS position.

Duration Quantity / Duration Format

Identifies the length of time to complete the WBS position.

Constraint Type / Constraint Date / Constraint Minutes

Affect the calculation of the start and end dates for the planning process (refer to topic “Process Logic” on page 1).

Start

Indicates when processing for the WBS position is set to begin.

Finish

Indicates when processing for the WBS position is set to end.

Signal Conflict / Signal Conflicts

Indicates with symbols or letters if there is a conflict with dates or resource availability (refer to topic “Detect Conflicts” on page 1).

Work Not Planned Hours

Displays the number of hours that have not been planned, i.e., resources have not yet been allocated (refer to topic “Assign Planning Resources” on page 1).

Assigned Resource No.

Displays the resources that have been assigned to the WBS position (refer to topic “Assign Planning Resources” on page 1).

Assigned Capacity %

Displays the total capacity percentage assigned to the WBS position (refer to topic “Assign Planning Resources” on page 1).

Work Planned Hours

Displays the number of hours that have been planned, i.e., resources have been allocated (refer to topic “Assign Planning Resources” on page 1).

Effort driven

Controls if the process, start date or end date should be limited by their planned availabilities (refer to topic “Calculation of Schedule Entries” on page 1).

Schedule Type

Determines how capacity is planned (refer to topic “Scheduling and Schedule Types” on page 1).

Scheduled Hours

Displays how many hours have been scheduled for this planning process.

Scheduled Freeze

If selected, planning for the WBS position is stopped regardless of the Schedule Type. The planned times for this position will not be changed during the planning process.

% Complete

Indicates the current stage of completion for the WBS position.

Assigned Resources

Indicates if resources have been assigned to the WBS position (refer to topic “Assign Planning Resources” on page 1).

Buffer Finish Days

Indicates the number of days that can be used as a buffer for the ending date.

Buffer Start Days

Indicates the number of days that can be used as a buffer for the starting date.

Deadline

Indicates a specific date when the WBS position must be completed.

Early Finish

Displays the earliest possible time of planning for the end of the process.

Early Start

Displays the earliest possible time of planning for the start of the process.

Finish Time

Displays the finish time if start and end times are specified in planning for the WBS position.

Hours after Finish

Displays how many hours were designed after the scheduled end.

Hours Fixed

Displays how many hours are planned for this position, with Scheduling Type = Fixed.

Hours Planned

Displays how many hours are planned for this position, with Scheduling Type = Planned.

Hours Simulated

Displays how many hours are planned for this position, with Scheduling Type = Simulated.

Job No.

Defaults from the Job Planning and Processing view.

Late Finish

Displays the last possible time of planning for the end of the process.

Late Start

Displays the last possible time of planning for the start of the process.

Lead Time Calculation

Indicates the lead time calculation (e.g., 2W for two weeks) to be used for planning a component.

Minutes at Finish

Displays the time offset at the end of the process to the end date.

Minutes before Start

Displays the time delay on the start date.

No.

Defaults from the Job Planning and Processing view.

Outline Code

Defaults from the Job Planning and Processing view.

Performed Hours

Indicates the number of hours posted as usage in the Job Ledger Entries.

Person Responsible

Identifies a person (resource) that is responsible for the WBS position.

Planned Capacity %

Identifies the assignment of planned capacity.  100% corresponds to one person.

Priority

Indicates a numeric level of importance of the WBS position.

Priority Option

Indicates the level of importance of the WBS position. The options are:

  • None
  • Very high
  • Middle
  • Top
  • High
  

Process Channel

Processes are created if the calculation position is linked with a process channel. The options are:

  • Work – Processes will be created with Work and Duration.
  • Lead time – Processes are created only with Duration.
  • Structure – Processes are created without Work and Duration.

Process No.

Identifies the unique process number assigned to the process line.

Process Type Fixed …

Specifies which base to fix in calculations. The options are:

  • Units (by planning resources)
  • Work
  • Duration

Quantity

Indicates how many units of the component should be planned.

Scheduled Early Days

Displays how many days are too early for real scheduling.

Scheduled Early Start

Displays the earliest planning date for the process.

Scheduled Late Days

Displays how many days are too late for real scheduling.

Scheduled Late Finish

Displays the latest schedule for this process.

Signal Backward

Displays a signal if the planning direction is backward.

Signal Forward

Displays a signal if the planning direction is forward.

Sorting No.

Creates a custom sort order for the WBS positions.

Start Time

Displays the start time if start and end times are specified in planning for the WBS position.

Summary Finish

Displays the planned ending date for a summary task.

Summary Start

Displays the planned beginning date for a summary task.

Time Entry Hours

Indicates the number of hours that have been posted in the Time Entry Job Journal for the WBS position.

Type

Defaults from the Job Planning and Processing view.

Unit of Measure Code

Defaults from the Job Planning and Processing view.

WBS

Defaults from the Job Planning and Processing view.

Work Overplanned Hours

Display how many hours were planned in excess, i.e., too many planning resources have been assigned.

Work Type Code

Defaults from the Job Planning and Processing view.

Actions Tab

The functions on the Actions tab are described in the following table.

Function

Description

Structure

Contains functions for creating the hierarchical processing structure. The options are:

  • New Process
  • New Process ID

Refer to topic “Job Process Planning and Scheduling” on page 1 for more information.

Chain

Sets the link between predecessors and followers.

Sorting

Changes the sorting of the lines. The options are:

  • Chance sorting – changes the sorting key from the WBS Position field to the Sorting No. field and vice versa.
  • Up
  • Down

Schedule

Plans processes for WBS positions without calculating predecessors and successors. The options are:

  • Schedule Process
  • Schedule Simulated only

Refer to topic “Calculation of Schedule Entries” on page 1 for more information.

Schedule (with links)

Plans processes for linked WBS positions. The options are:

  • Schedule Processes
  • Processes effort driven
  • Processes never effort driven

Refer to topic “Calculation of Schedule Entries” on page 1 for more information.

Update Scheduling

Updates the lines in the Job Scheduled Entries page. The options are:

  • Simulated to Planned
  • Fixed to Planned
  • Fix all and release
  • Planned to Simulated
  • Planned to Fixed
  • Delete Schedule Entries

Planning Processes

Contains actions related to planning processes. The options are

  • Release – releases a WBS position for planning processes.
  • Finish – finishes the planning process for the WBS position.

Navigate Tab

The Navigate tab has two action groups: Planning and MS Project. The actions in each group are described in the following sections.

Planning Action Group

The actions in the Planning group are described in the following table.

Action

Description

Planning Setup

Opens the Job Planning Setup page, which is defaulted from Standard Planning Setup and can be changed for individual Job planning parameters.

Process Card

Opens the Job Position Card page, which provides a card view of all the information in the Planning Processes page for the selected WBS position.

Job Team

Opens the Job Team page, which displays all planning resources assigned to the job.

Resource Availability

Opens the Planning Res. Availability page, which displays availability for the resource in a matrix view (refer to topic “Availability Overview” on page 1).

Resource Allocation

Opens the Planning Res. Allocation page, where resources can be assigned to WBS planning positions (refer to topic “Resource Allocation” on page 1).

 

MS Project Action Group

The actions in the MS Project group are used with the Microsoft Project integration. Refer to topic “Microsoft Project Integration” on page 1 for more information about setting up these actions.
