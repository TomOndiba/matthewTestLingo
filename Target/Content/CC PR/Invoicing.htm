Invoicing

Invoicing is processed in the Planning and Processing view and the Sales view.

Invoice Run

The invoice run will be demonstrated using the following WBS positions:

  • 1.1.4., Installation – Fixed Price invoicing , Sales Code = FP
  • 1.1.5., Training – Time and Material (T&M) invoicing, Sales Code = LQ SP (with a quantity limitation for the line)

: Job Planning and Processing view with fixed price and T&M invoicing lines

Consumption entries are available; the lines have been booked using the time recording worksheets.

During the invoice run, the posting of the invoice can be conditional for each consumption line. Therefore, the invoice run must be planned, i.e., relevant invoice lines will be suggested by the system. On the ribbon, click Sale.

In the Job Budget Sales view, select the relevant lines and then click Planning on the ribbon.

: Job Budget Sales with planned invoicing lines

The Invoicing Planning field shows that planning lines are available and the Invoicing Total Pr. Pl. (LCY) field shows the sale total.

Each planning line can be shown and/or edited by clicking Per Block on the ribbon. It is also possible to show the invoice lines for a complete job structure by clicking Navigate > Invoicing > All Lines.

: Job Processing Invoicing view with all invoicing lines

For WBS 1.1.4, Installation, seven hours were recorded through time recording (refer to Fig. 9). However, as a fixed price job line, only six hours will be billed.

The training position will be charged according to the actual time recorded, which was nine hours (refer to Fig. 9). Here the invoice line is directly linked to the job line and an employee name can be entered.

To create a sales order from the draft invoice, release the invoice run by clicking Release on the Job Budget Sales view.

: Job Budget Sales view with invoiced sales lines

Invoice creation is indicated in the Sales Invoice field; click this value to open the Sales Invoice.

: Sales Invoice created from Job Budget Sales view

Note

To create a single invoice for multiple shipments from different sales orders, click Actions > Create collective Invoice on the Job Budget Sales view. This function opens the standard Get Shipment Lines page, which shows all shipped, but not invoiced positions from all sales orders related to the job. The selected shipment lines can be then combined into a single invoice for processing.

Before posting the invoice, click Navigate > Preview Posting to see what the posted Job Ledger Entries will be.

: Posting Preview displaying expected job ledger entries

Close the Job Ledger Entries Preview page and view other entries, or close the Posting Preview page to continue with the posting process.

After posting the invoice, the posted sums are shown in the Job Budget Sales view.

: Job Budget Sales view with updated posted sales information

Note

The Planning and Release functions can be run periodically using the Job Invoicing Planning and Job Invoicing Release reports accessed from the Departments menu by clicking cc|project> Periodic Activities. These reports allow for batch processing of multiple job.

Invoicing with Foreign Currencies

Invoices with foreign currencies are processed the same as described in the previous section, but the Invoice Currency Code field on the Job Card and Sales Currency Code field in the planning lines need to be populated accordingly.

In the Job Card, entering a currency in the Invoice Currency Code field allows for alternative currencies to be used in the sales invoice. For example, US Dollars (USD) is entered as an alternative currency for a job that uses British Pounds (GBP), which is the default currency for CRONUS International Ltd.

: Invoice Currency Code set to USD, alternate to default currency of GBP

Fixed price entries are added to the Budget Calculation.

: Job Budget Calculation entries

Click Sale to open the Job Budget Sales page for invoice planning.

: Job Budget Sales, preparing to run Planning function

With the first line selected, click Planning to prepare the line for processing and then click Per Block to open the Job Processing Invoicing page. Notice that the Sales Currency Code is set to USD, which was the currency in the Invoice Currency Code on the Job Card. Also note that the fields showing local currency (LCY) are different because the default currency for the customer is GBP.

: Job Processing Invoicing with Sales Currency Code = USD from Job Card

The Sales Currency Code can be changed as needed, if multiple currencies need to be invoiced for a job. For example, the first two entries in this job budget calculation are invoiced to currencies different than USD: Swedish Krona (SEK) and Swiss Franc (CHF), respectively.

Note that the sales fields are updated to accommodate the new currency. To prepare for invoicing, the Invoicing Status is changed to Planning.

: Job Processing Invoicing line updated with Swedish Krone

The same planning process is done for the second line, except with the Swiss Francs.

: Job Processing Invoicing line update with Swiss Francs

The third line is planned and the USD currency is unchanged.

: Job Processing Invoicing line with US Dollars

When ready to invoice, select all lines in the Job Budget Sales page and then click Release.

: Preparing to release the invoice planning lines

After release, the individual sales invoices are created.

: Job Budget Sales lines with invoices

Click Yes in the Sales Invoice field of each line to review the currencies.

: Sales Invoices for job lines with SEK, CHF and USD currencies

Sales Credit Memos

New Sales Credit Memos are generated through the Sales & Marketing menu. Posted invoices can be used as templates for the credits by using the “Copy Document” function.

: Sales Credit Memo with job lines from posted sales invoice (Copy Document function)

In addition to the standard fields on the credit memo, the Reason for Credit Memo field offers additional functionality for the re-invoicing of job lines. The options are:

  • New Invoice – determines if new invoice lines (for this position) are generated for the invoice run and a new (adjusted) invoice can be created.
  • Refund – determines if the total price for this position will be reduced by the sales credit memo with no further processing.

: Sales Credit Memo with adjusted lines for refund processing

In this example, one hour of Training will be credited without substitution. There will be no new lines generated for the invoice run and the credit memo will reduce the posted sales price.

: Job line adjustment from posting Sales Credit Memo

Job positions that are defined with Sales Code = SALESORDER can be processed directly by posting a sales order instead of through an invoice run.
