Calculation

This topic describes various setup pages relating to Calculations in the cc|project, Administration menu.

Calculation Groups

Calculation groups are used to:

  • Differentiate the extra cost from the Indirect Cost % and Extra Charge % fields in the calculation line.
  • Group extra cost with calculation schemes.

: Job Calculation Groups page

Job Charges

Job Charges are used to enter default charges for other charges.

  • Indirect Cost % – used to calculate the Unit Cost.
  • Extra Charge % – used if no Unit Price is found. If a Unit Price is found, this field is calculated.

: Job Charges page

The Job Charges are prioritized as follows:

  1. Job
  2. Customer
  3. Responsibility Center
  4. Global (all three fields blank)

Work Types

Work Types are standard in Microsoft Dynamics NAV. A Work Type indicates the various kinds of work that a resource can perform and establishes various prices and costs in resources and resource groups (refer to topic “Personal Unit Prices” on page 1).

In cc|project, the Resource Type field differentiates between the various resources used on jobs. The options relevant to cc|project are:

  • Person
  • Service
  • Traveling

: Work Types page, highlighting the Resource Type field

Job Sales Work Types

Job Sales Work Types match the Person Work Type Code and Service Work Type Code and are used to specify which of the following has to be used for pricing:

  • Service No. used component in calculation line
  • Person Work Type Code used in time entry
  • Service Work Type Code used in calculation line

: Job Sales Work Types page

Calculation Schedule Extra Charges

In the Job Calc. Extra Charges page, define the extra charges to be included in a calculation schedule.

: Job Calc. Extra Charges page

Calculation Schedule Templates

A default Job Calculation Schedule Template can be generated by running the Setup Wizard (refer to topic “Setup with Job Setup Checklist” on page 1).

: Job Calc. Schedule Template page

The fields in the Lines FastTab are described in the following table.

Field

Description

Row No.

Unique code field that is used to identify a corresponding calculation line in the Link Row No. field.

Note

With the SQL Server, it is useful to create row numbers with leading letters (R010, R020, and so on) or zeros (0100, 0200, and so on) to prevent sequencing from changing when numbers greater than 999 are used.

Description

Description of the calculation, which can be indented by clicking Functions > To the left or To the right on the ribbon.

Special Format

Allows for formatting of the text in the Description field.

Link Row No.

Refers to the row number that applies to the extra charge calculation. Used only on lines with a Totaling Type of Extra Charge and Rounding.

Totaling Type

Specifies what the calculation line is based on. The options are:

  • Calculation Group – used on calculation lines that gather totals from job calculation groups.
  • Extra Charge – used on lines that calculate an increase or decrease of another calculation line.
  • Formula – used on lines that total other calculation lines. Standard Microsoft Dynamics NAV filters can be used, such as range (R100..R199), either/or (1000|2000), or any of the following: * ? > < = @.
  • Value – used to enter a value manually.
  • Rounding – used to round an amount.

Totaling

Values relating to the selection in the Totaling Type field.

Calculation Group (e.g. MAT), an Extra Charge (e.g. MAT%), a Formula (e.g. R100..R199), a Value (e.g. leave empty to enter later), a Rounding (e.g. 100,<).

Amount Type

Indicates the source amount for the calculation. The options are:

  • Total Direct Cost
  • Total Cost
  • Total Price

Include Remaining

If selected, remaining amounts (hidden buffers) are included in the calculation.

More calculation schedule templates can be created manually by clicking New in the Job Calc. Schedule Template page.

: Job Calc. Schedule Template page
