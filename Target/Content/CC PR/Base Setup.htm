Base Setup

This topic describes setup pages located within the Base setup and Other Setup menu groups in the cc|project, Administration menu.

Job Posting Groups

Job Posting Groups are used to direct postings for Work in Progress (WIP) job activities to accounts within the General Ledger. The default Job Posting Group is set up when the initial setup is run (using the Job Setup Checklist); however, the accounts need to be specified.

: Job Posting Groups page

The fields on this page that are relevant for cc|project are described in the following table.

Field

Description

Code

Unique identifier for the posting group.

Description

Short description of the posting group.

Default WIP Method

WIP Method that will be defaulted to the Job Card for this posting group. The options are Cost Value, Sales Value, Cost of Sales, Percentage of Completion and Completed Contract.

WIP Value

Asset account, used to capture the calculated job value, i.e., for expenses that can be processed at a later time.

WIP Prospective Cost

Liabilities account, used to capture the expected future cost, i.e., for expenses that are released but will be processed at a later time.

WIP Impending Losses

Accrual account, used to capture exceeded estimated costs.

WIP Cost of Sales

Revenue account, used to capture summarized costs.

WIP IS Adjustment

General Ledger account (Income Statement), used for periodic accrual adjustments.

WIP IS Impending Losses

General Ledger account (Income Statement), used for periodic accrual adjustments of impending losses.

WIP IS Prospective Cost

General Ledger account (Income Statement), used for periodic accrual adjustments of prospective costs.

WIP IS Cost of Sales

General Ledger account (Income Statement), used for periodic revenue adjustments.

Note

Additional set up can be performed at any time, but will only apply to future job activities; they cannot be made retroactively.

Job Journal Templates

The Job Journal Templates are used to set up special information that can be used in the journal lines. There are two different kinds of Job Journal templates for the use of cc|project:

  • Job Journal Template for Job Inventory Requests
  • Job Journal Template for Job Time Entries

: Job Journal Templates page

In addition to the standard Microsoft Dynamics NAV job journal template information that can be assigned to the journal lines, the Source Channel field defines:

  • <blank>
  • Time Entry
  • Purchase
  • Inventory Request
  • Job Shipment

The Page ID field specifies which page is used for the Job Journal Templates:

  • Page ID 5292398 for Page Job Inventory Request List
  • Page ID 5292395 for Page Job Time Entry

Job Types

Job Types are used to create logical classifications among jobs. Each Job Type can have a specific number series, default dimension and posting group attached.

: Job Types Setup page

The fields on this page are described in the following table.

Field

Description

Code

Unique identifier for the job type.

Description

Short description of the job type.

Default Status

Indicates the status that will be defaulted to the Job Card each time a Job is created with this Job Type.

Job Nos.

Number series to be used for this Job Type.

Default Sales Code

Indicates the Sales Code that will be defaulted to the Job Card each time a Job is created with this Job Type.

Default Job Posting Group

Indicates the Job Posting Group that will be defaulted to the Job Card each time a Job is created with this Job Type.

Default Dimension Value

Indicates the job type dimension value that will be defaulted to the Job Card each time a Job is created with this Job Type.

Default Unit Price Base

Indicates the Unit Price Base that will be defaulted to the Job Card each time a Job is created with this Job Type.

Default QC Template

Indicates the QC Test List for the Job that will be created each time a Job is created with this Job Type. Refer to topic “Quality Checklists” on page 1 for more information.

List Sorting

Enables a logical sorting of Job Types but overrides alphabetical sorting, e.g., entering a 1 means that it will appear first in the list.

Job Type when Order

Job Type to be changed to on the Job Card when a job is converted from a quote into an order.

Note

The Job Type Setup page can be opened directly from the Job Types page (cc|project > Jobs > Job Types > Actions > Setup).

Sales Codes

Sales Codes are used to manage all sales processes. Sales documents can only be created if a link to a Sales Code exists. This link can be formed from a Sales Code in the line or in a linked upper structure.

: Job Sales Code page

Some of the fields on the Job Sales Codes page are described in the following table.

Field

Description

Sales Channel

Identifies the sales channel. The options are:

  • Fixed Price – Invoicing is independent from usage.
  • Usage – Invoicing is directly based on usage entries.
  • Realized % – Usage entries are the basis for the calculation of Realized %. Invoicing is indirectly based on Realized % of Cost usage, Quantity usage, or manual entry of Realized %.
  • Performance – Currently has no functionality.

Chargeable

If not selected on a sales position, there will be no calculation on “Sales Line Amount”.

Invoicing Trigger

Determines what prompts the invoicing process. The options are:

  • Planning Invoicing – Used at the end of the invoicing period. All transactions for this period are already posted. Use the “Planning and Processing” view or “Sale” view and then click Navigate > Invoicing (in the Processing group) > Planning to start the functionality.
  • Enter in Job Journal Line – Collects the data directly while posting through the Time Journals. The respective sales amount is shown in Invoicing Total Price (Proc.) field.
  • Posting Process – Collects the invoicing data based on the posted usage records. Postings with a limited quantity are allocated by posting sequence to invoice run.

Sales Price Source

Defines which sales price should be used for invoicing. The options are:

  • Job Line – Unit prices are taken from the sales job position.
  • Standard Prices – Unit prices are taken from the standard price findings.
  • Cost Plus – Unit prices are taken from posted costs + surcharge from the sales job position.

Limited Quantity

Distinguishes chargeable parts from non-chargeable parts. In this case, the limited quantity restriction relates to an outstanding posting or a directly linked sales position (of a higher level structure).

Check Milestones

If selected, the sales process is triggered through milestone status.

 

Job Signals

In the Job Signals page – located within the Other Setup menu group on the cc|project, Administration menu – define the colors that will be used in the various views.

After a Job Signal is defined with a unique code and a short description, assign an image file by right-clicking the Picture field and then clicking Select Picture.

: Selecting a picture in the Job Signals page

Locate and select the picture file to import and then click Open. The image file appears in the Picture field.

: Job Signals list page

Click Edit or View to open the corresponding Job Signal Card, where the same process to add job signals can be performed. The Job Signal Card is also used to provide WebClient compatibility for uploading signal pictures.
