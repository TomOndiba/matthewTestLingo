cc|configure to order

With cc|configure to order functionality, the item master can become a Master BOM and Checklists can be assigned to it. Formulas, functions and variables assigned in the Master BOM and Checklist work together to form a configuration, which is used to produce a BOM.

In cc|project, the checklists are processed in the Job Budget Calculation line by answering pre-defined questions. These answers are transferred to a Document BOM, which contains the selected configuration. Here, changes can be made prior to exploding the BOM in the Job Budget Calculation. Once exploded, budget information can be assigned and processed as usual in cc|project. The Document BOM can also be used to create Construction Orders and Production Orders in the Job Planning and Processing view.

This topic briefly describes the Master BOM, Checklist and Document BOM in cc|configure to order as well as the usage of each in the Job Budget Calculation, and the creation of Construction Orders in the Job Planning and Processing view. For detailed setup, configuration and usage information, refer to the current “cc|industry solution: discrete manufacturing” documentation.

Master BOM

The Master BOM works according to the principle of the maximum BOM, which combines all valid options in a configuration. During the BOM calculation, the variables use formulas and rules to determine quantities, setup and run time, and if positions are valid.

: Master BOM for item D-1000, Conveyor machine complete

Once set up, the Master BOM is assigned directly to the item.

: Master BOM assigned to item D-1000

Checklist

The checklist is the interface between users and system. Since the checklist can provide its own logic, it ensures that only information examined and verified by the user is available for configuration.

: Checklist for item D-1000, Conveyor machine

Once set up, the Checklist is assigned to the item.

: Checklist assigned to item D-1000

Document BOM

As previously stated, the master BOM and checklist work together to form a configuration, which is used to produce the Document BOM.

: Document BOM from Checklist and Master BOM

Creating Document BOMs in Job Budget Calculation Line

There are four ways to create Document BOMs in the Job Budget Calculation line:

  • Item with a Checklist and Master BOM
  • Item with a Master BOM
  • Item without a Master BOM
  • Item with a Production BOM

Each method is described in this section.

Create Document BOM with Checklist and Master BOM

When using cc|configure to order functionality to create a BOM in the Job Budget Calculation, the Job Line BOM Control dialog does not appear because a BOM does not exist for the line until the Document BOM is created.

The budget line will be populated with only the item. Notice that the Checklist No. field is populated as well; this is generated automatically from the Checklist assigned on the item.

: Item with Checklist but no BOM

On the ribbon, click Navigate > Checklist to open the Checklist.

Note

If the Open Immediately check box is selected in the Checklist Template, then the Checklist will open automatically when the Quantity field is populated.

: Checklist for job budget calculation line

Additional machine type components can be selected, a type of motor can be entered (kW power) and additional documentation can be included. For this example, no changes are made to the machine type, a power of 90 is entered and an English manual must also be included.

: Checklist with indicated power and additional English manual selected

The Checklist is completed by clicking OK.

Before a Document BOM can be created, a quantity must be entered in the line. Then on the Navigate tab, click Document BOM to create the Document BOM. A message will appear if no Document BOM exists, asking to create a new Document BOM.

: Message to create a new Document BOM

Click Yes to create the Document BOM. The Item - Master BOM page will open, displaying the related Master BOM. Click OK; the Document BOM page opens with the configuration from the Checklist, including the motor with 100 kW (based on settings in the Master BOM to choose this item if (POWER>=75) and (POWER<=100)) and the English documentation.

: Document BOM highlighting specific configurations from Checklist

Any changes that need to made to the Document BOM can be made here. If necessary, the original Document BOM can be recreated by deleting all existing lines in the Document BOM page and then running the "Request BOM" function from the ribbon.

Click OK to close the Document BOM. In the Job Budget Calculation view, the Document BOM No. field is populated as well as the Prod. Order from Document BOM check box.

: Document BOM No. and Prod. Order from Document BOM fields populated in Job Budget Calculation

Create Document BOM with Master BOM

Document BOMs with Master BOMs but without checklists can be used in the Job Budget Calculation line using the same process as with a checklist. When an item has no checklist, the Checklist No. field remains empty when the item is added to the Job Budget Calculation line.

: Item with no checklist added to the Job Budget Calculation

In the Job Budget Calculation line, enter a quantity and then click Navigate > Document BOM to create the Document BOM. As previously described, a message will appear asking to create a new Document BOM. Click Yes to create the Document BOM. The Item - Master BOM page opens, displaying the related Master BOM. Click OK; the Document BOM page opens.

: Document BOM created without checklist

In the Job Budget Calculation line, the Document BOM No. is assigned and the Prod. Order from Document BOM check box is selected.

: Document BOM fields updated in the Job Budget Calculation line

Create Document BOM without Master BOM

It is also possible to create a Document BOM manually, without a Master BOM, using basically the same process as previously described. Once the item without a master BOM is added to the Job Budget Calculation line, enter a quantity and then click Navigate > Document BOM to create the Document BOM. A message will appear asking to create a new Document BOM.

: Creating a Document BOM with no Master BOM

Click Yes to create the Document BOM; an empty Document BOM page opens.

: Empty Document BOM created for item with no Master BOM

In the Document BOM page, enter all BOM information and then click OK to close the Document BOM. In the Job Budget Calculation view, the Document BOM No. field is populated as well as the Prod. Order from Document BOM check box.

: Document BOM fields updated in the Job Budget Calculation line

Create Document BOM with Production BOM

It is also possible to create a Document BOM with a Production BOM, using basically the same process as previously described. Once the item with a Production BOM is added to the Job Budget Calculation line, enter a quantity and then click Navigate > Document BOM to create the Document BOM. A message will appear asking to create a new Document BOM.

: Creating a Document BOM with Production BOM

Click Yes to create the Document BOM; the Select BOM page opens with no Master BOM lines for selection.

: Select BOM page with no Master BOM lines

Select the Production BOM and Routing check boxes to transfer both types of entries to the Document BOM and then click Yes.

: Document BOM populated from Production BOM and Routing assigned to item

The Document BOM page opens and is populated with all Production BOM and Routing lines (if selected). These entries can be modified as needed. Click OK to close the Document BOM. In the Job Budget Calculation view, the Document BOM No. field is populated as well as the Prod. Order from Document BOM check box.

Processing Document BOMs in Job Budget and Planning Lines

The Document BOM is the base for exploding the Document BOM to the Job Budget Calculation line using the “Expand BOM” function or creating Production Orders in the Job Planning and Processing view. Each option is described in this section.

Expand BOM

The “Expand BOM” function explodes the Document BOM in the Job Budget Calculation line so that regular budgeting and calculation processes can be completed.

With the Document BOM line selected to expand, click Actions > Expand BOM.

: Job Line BOM Control dialog set to explode the Document BOM

Click OK to explode the Document BOM in the Job Budget Calculation line.

 

: Job Budget Calculation with exploded Document BOM

Create Production Orders from Document BOM

In the Job Planning and Procurement view, lines that have the Prod. Order from Document BOM check box selected will create production orders based on the Document BOM when the procurement processes are run. This process is done when a complete production order needs to be processed but the line was not exploded because the configuration is not used for calculation purposes in the Job Budget Calculation.

: Job Planning and Processing line for production order created from Document BOM

After planning and releasing, the Production Order is created based on the Document BOM.

: Production Order created from Document BOM in Job Planning and Processing view

Note

To create Production Orders from Document BOMs, produced items that are part of a Master BOM must have a Routing at this time; however, it can be a dummy routing that does not include any routing lines.

Create Construction Orders from Job Planning and Processing Line

In cc|configure to order, a company’s engineering or construction department can use the construction order for managing the BOM prior to manufacturing. Therefore, the construction order is considered the work order for the construction department. The design engineer has the option of modifying the document BOM created by sales or creating a new BOM. In this case, he or she has access to the checklist.

In the Job Planning and Procurement view of cc|project, the Construction Order procurement channel provides the ability to create construction orders for the job lines outside of the Job Budget Calculation, which can then be transferred to the Production module. Document BOMs that exist for the job line will be transferred to the new Construction Order.

: Job Planning and Processing line for construction order created from Document BOM

After planning and releasing, the Construction Order is created based on the Document BOM if one exists.

: Construction Order created from Document BOM in Job Planning and Processing view

The Job FastTab in the Construction Order provides information specific to the related job.

If a Document BOM does not exist for the job line, one can be created from the Construction Order.

: Job Planning and Processing line for construction order created without Document BOM

In the Construction Order, use the “Request BOM” function to create the Construction BOM from the Master BOM assigned to the item.

: Construction Order created without Document BOM in Job Planning and Processing view, requesting Master BOM

Prototypes in Job Budget Calculation

With the prototype creation function, it is possible to generate new items automatically and/or find already produced items based on their developments. Likewise, new variants of an item can be produced and/or reused.

The creation of a prototype is part of the configuration feature of cc|configure to order and is used when opening a Master BOM in the Document BOMs and the Construction Orders. The prototype item or item variant is then transferred to the Job Budget Calculation.

Prototype Setup

First, an item card must be set up as a Prototype template (refer to Fig. 29). On the Config. to Order FastTab, select the Prototype check box to make the Prototype Type and Prototype No. Series fields editable.

The Prototype Type field determines whether a new item or a new variant of the item will be applied during the creation of a new characteristic of the prototype. In the Prototype No. Series field, enter the number series to be used for the new creation of items and/or variants as a copy of the prototype.

: Templates for Prototype item and variant

Furthermore, attributes must be entered. During the calculation of the Document BOMs, the system checks whether an identical item and/or variant is already present or has to be created. It is important that the Relevant to Prototype check boxes are selected for the attributes, which are to be used for retrieving the prototypes.

: Prototype Attribute Setup

In the Checklist and Master BOM, the attributes used to find an existing or create a new item/variant are defined. In this example, the POWER attribute is used to determine when a new item or variant is created or searched.

: POWER attribute set in the Checklist

: POWER attribute set in a Master BOM of an item in the main Master BOM

Example – Prototype Item

In this example, the Conveyor Machine has been set up as a prototype item based on the type of engine power needed. When this item is added to the Job Budget Calculation, a new item will be created or an existing prototype item will be used in either scenario:

  • The power required is between 50 and 74 kW
  • The power required is between 75 and 100 kW

The prototype item is added to the Job Budget Calculation line.

: Prototype item added to Job Budget Calculation line

In the Checklist, the power is set to 54 in the Input field.

: Checklist for prototype item

The Document BOM is opened for the Job Budget Calculation line. The item that corresponds to the input power of 54 (D-7011) is used in the Master BOM.

: Document BOM with motor based on 54 kW of power indicated in Checklist

When the Document BOM is closed, the prototype item number is replaced by a new item number in the Job Budget Calculation line because this is the first instance of the configuration of the Conveyor Machine.

: New item number for prototype configuration in Job Budget Calculation line

If a prototype item is entered on the Job Budget Calculation line, and an existing item is found, the system will not recalculate the Master BOM. The Document BOM that is linked to the found prototype item will be copied.

In the Item Card for this new item, the related Document BOM is stored in the Configure to Order FastTab, as well as from which prototype item the new item was created.

: Prototype item creation references in Item Card

If the prototype item had been created from a Construction Order, the Prototype BOM Type and Prototype BOM No. fields would display Construction Order and the Construction Order number, respectively.

Prototype Variant

Prototype variants work the same as prototype items except that instead of creating a new item, item variants are created for the item. In the Item Card, the Prototype Type field is set to Variant and a number series for prototype variants is used (refer to Fig. 29).

In the Job Budget Calculation, the prototype item variant is entered. Currently, the Variant Code field is empty.

: Prototype item variant added to Job Budget Calculation line

In the Checklist, the power is set to 54 in the Input field.

: Checklist for prototype item variant

The Document BOM is opened for the Job Budget Calculation line. The item that corresponds to the input power of 77 (D-7012) is used in the Master BOM.

: Document BOM with motor based on 77 kW of power indicated in Checklist

Since no variants exist for this configuration, a variant is added to the prototype item when the Document BOM is created. When the Document BOM is closed, the new variant is entered in the Variant Code field in the Job Budget Calculation line.

: New variant for prototype item configuration in Job Budget Calculation line

If another line was added to the Job Budget Calculation for the same item using the same configuration, the same variant would be used. If another line was added to the Job Budget Calculation for the same item using a different configuration, a new variant would be created for the item and entered in the Job Budget Calculation line.

In the prototype item, the Item Variants show the related Document BOM information.

: Prototype item creation references in Item Card

If the prototype item variant had been created from a Construction Order, the Prototype BOM Type and Prototype BOM No. fields would display Construction Order and the Construction Order number, respectively.
