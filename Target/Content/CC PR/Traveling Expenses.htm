Traveling Expenses

Traveling expenses are assigned to customers when specific rates are agreed upon between the company and its customer. These expenses can then be transferred to jobs relating to the customer.

Pricing Considerations for Traveling Expenses

When determining prices for traveling expenses while using cc|project, consider these factors:

  • Prices are based on the Work Type assigned to the Traveling Expense.

Note

The Work Type is typically entered into the calculation during, not prior to, posting.  

  • Multiple currencies for traveling expenses are not supported. Thus, any entries for traveling expenses must be made in the local currency (LCY).
  • Prices can be defined by Job.

Prices can be derived from the Traveling Expenses Card; however, this is not recommended.

Setup

The following setup is required for using traveling expenses.

Components

A minimum of one resource must be set up for the Job Budget Calculation with the following settings:

  • Type = Traveling
  • Procurement Channel = Time Entry
  • Procurement Cost Source = Standard Costs

: Resource Card for Traveling Expenses

Different work types can be set up for traveling expenses and a default can be assigned to the component in the Traveling Work Type Code field on the Calculation FastTab of the Resource Card.

: Work Types for Traveling Expenses accessed from the Resource Card

General Traveling Expenses

Other setup for general traveling expenses is defined in the Job Traveling Expenses page, accessed from the cc|project menu by clicking Administration > Job Travel Codes.

: General Job Traveling Expenses

If the travel expenses should be reimbursed to the employee, the Reimbursement check box must be selected.

The travel expense can be set as a default per customer and per project by clicking Navigate > Traveling Expenses on the Customer Card or Home > Traveling Expenses on the Job Card.

: Job Traveling Expenses assigned to job J000001.01

Job Budget Calculation

To process traveling information, a minimum of one job line with Type = Traveling must be entered in the Job Budget Calculation view.

: Traveling Expenses line assigned in the Job Budget Calculation view for job J000001.01

As illustrated in Fig. 5, the travel expenses are calculated on a Time and Materials, Non-Chargeable and Flat Rate basis. The sales price is calculated based on cost plus project surcharges, non-invoicable and per usage (Sales Code = KO+, NI and USG, respectively).

Traveling Expense Processing

This section explains the procurement and sales processing of traveling expenses.

Procurement Planning

For procurement planning, it is important to consider that two procurement channels may need to be released (i.e., time recording and purchase order). 

: Traveling Expenses line set up in the Job Planning and Processing view for job J000001.01

As no quantity is entered, the lines need to be planned manually (with the “New Procurement” function in the Job Processing Procurement page) and a purchasing document is not necessary. The lines are then available for posting through purchasing and time recording.

: Released Traveling Expenses ready for time entry posting

Posting by Time Entry

To enter traveling expenses in Time Entry, use the Travel Code field to copy the information from the Job Card setup to the time entry line.

: Traveling Expenses entered in Time Entry for WBS positions 2.1.–2.3. in job J000001.01

The purchasing prices will be transferred from the predefined travel codes and the sales prices are transferred from the predefined markup of the calculation lines as well as the job traveling expenses. Both types can be manually adjusted in Time Entry as needed.

Reimbursement of Travel Expenses

Travel expenses can be reimbursed to employees through a purchase invoice created automatically or manually. Before a purchase invoice can be created for reimbursement, the employee must be set up as a vendor in Microsoft Dynamics NAV and the Vendor No. must be assigned in the Processing FastTab of the Resource Card.

: Vendor number assigned to the resource card for employee P016

Automatic Purchase Invoice

The request page for the Employee Traveling Expenses report provides the ability to create the invoice while generating the report. The report and invoice will only include job lines that have travel codes with the Reimbursement check box selected.

: Employee Traveling Expenses report request page

The fields on the Options FastTab are described in the following table.

Field

Description

Create Invoice

If selected, a Purchase Invoice will be automatically created for the resource, preliminary a vendor number needs to be created for each employee and needs to be linked to the personal card of each employee by filling the field vendor no.

With Invoiced Entries

If selected, previously invoiced entries will appear on the report with the open entries.

Vendor Invoice No.

Identifies the number of the invoice from the vendor; will be transferred to the Vendor Invoice No. field on the Purchase Invoice.

To access the report from the cc|project menu, click Reports > Employee Traveling Expenses.

: Employee Traveling Expenses report filtered on Resource No. P016

The following Purchase Invoice was created automatically to vendor V0020 – which is linked to resource P016 – because the Create Invoice check box was selected in the Employee Traveling Expenses report request page when the report was generated.

: Purchase Invoice automatically created for reimbursable expenses by running the Employee Traveling Expenses report

Additionally, the Reimbursed check box is automatically selected on the job ledger lines after the invoice is generated.

: Reimbursable expenses updated in the job ledger entries by running the Employee Traveling Expenses report with Create Invoice selected

Manual Purchase Invoice

Instead of generating the Purchase Invoice from the Employee Traveling Expenses report, the expenses can be entered directly in a Purchase Invoice and linked back to the job lines.

In the purchase invoice, enter the employee’s vendor in the Vendor field and in the lines, enter the traveling expenses to the appropriate job line(s) and purchase account(s).
