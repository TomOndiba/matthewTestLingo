Job Process Planning and Scheduling

This section described the various planning and scheduling functions performed in the Job Planning and Processes view.

Create Processes

The Process Channel field is set as a default from the component. It can be checked and changed in the Job Planning and Processing view.

: Process Channel field on the Job Planning and Processing view

For WBS line 1.2.3. Documentation, no planning is needed because this is a stock item.

To create the planning process, select the lines to include and then click Planning on the ribbon (from the Navigate tab, click Planning Process > Planning).

: Preparing to run the Process Planning function

In the dialog, click OK to process the selected lines. The process-related fields are updated accordingly.

: Job Planning and Processing view after running the Process Planning function

To display all processes in one view (relating to the current position), click Navigate > View > Per Block.

: Planning Processes page

The initial planning run can be viewed from the Planning Processes page. On the ribbon, click Navigate > Planning Setup to open the Job Planning Setup page, which is created automatically using the default settings.

: Job Planning Setup page

By creating the processes, the default data sets are processed based on the following Process Channel:

  • Work – the work and unit of the job line are calculated.
  • Lead time – the duration will be calculated using the component lead time.

Missing process channel defaults can be entered directly in the Planning Processes view.

Click New Process on the ribbon to define multiple processes for each job line, such as: 

  • Split budgeted positions into sub-processes
  • Create milestones
  • Include customer processes in the schedule

: Planning Processes page with new and adjusted entries

Process Logic

Processes can be linked by using predecessors based on the Process IDs. To recalculate IDs that are out of order due to inserted processes, click Actions > Structure > New Process ID.

In the Predecessor ID field, type the Process ID(s) to create the link between job lines. Use the semi-colon (;) to enter multiple process IDs per line. The Predecessor and Follower fields are automatically updated to indicate the hierarchical link.

Constraint Types affect the calculation of the start and end dates for the planning process. The options are:

  • ASAP – As soon as possible (the follower is planned directly behind the predecessor)
  • ALAP – As late as possible (the predecessor is planned in front of the follower)
  • MSOn / MFOn – Must start on / Must finish on
  • SNET / SNLT – Start not earlier then / Start not later then
  • FNET / FNLT – Finish not earlier then / Finish not later then
