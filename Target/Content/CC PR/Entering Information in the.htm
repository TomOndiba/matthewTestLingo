Entering Information in the Job Budget Calculation

This section includes processes for entering information in the Job Budget Calculation view.

Exercise – Create Job Calculation Lines

This exercise explains how to enter general, service or standard items in the Job Budget Calculation view. For information on entering items with BOMs, refer to topic “Entering BOMs in Job Structure” on page 1.

To create job calculation lines, follow these steps:

  1. On the navigation pane, click Departments > cc|project > Jobs > Jobs.
  2. Locate the relevant Job Card to add budget calculation lines.
  3. On the ribbon, click Budget Calculation.
  4. On the ribbon, click New Position.
  5. If the first position is a header line with no pricing, follow these steps:
    1. Leave the Type and No. fields empty.
    2. In the Description field, type a short description for the Position.
    3. Leave the remaining fields empty.
  6. If the first position is used to identify the total job price, follow these steps:
    1. In the Type field, enter General.
    2. In the No. field, enter the appropriate general component. Remember that the selection here determines which G/L account is posted to when this order is invoiced.
    3. If the Description field is not populated from the selected number, type a short description, for example, Total Price. This description will appear on the invoice.
    4. In the Quantity field, type 1 since this is a single line entry for the full cost of the job budget.
    5. In the Direct Unit Cost field, type the full cost of the job budget.
    6. In the Unit Price (LCY) field, type the full price of the job, if not already calculated.
    7. Alternatively, leave the Quantity, Direct Unit Cost and Unit Price (LCY) fields blank so that these values can be automatically calculated using the Rollup functionality.
  7. While on the first Position, insert the next line, click one of the following actions on the ribbon:
    1. New Level to insert a Level of that Position.
    2. New Position to insert a new Position.
  8. In the Type field, enter the type of job line to be created, such as a service to be performed, or an item to be purchased.
  9. In the No. field, enter the appropriate number corresponding to the selected Type.
  10. Update the Description field as needed.
  11. In the Quantity field, type the number of days, hours, and so on, budgeted for this level/position.
  12. In the Unit of Measure Code field, enter the appropriate unit of measure for this level/position.
  13. In the Direct Unit Cost field, type the unit cost in the relevant unit of measure for this level/position.
  14. In the Unit Price (LCY) field, type the unit price in the relevant unit of measure for this level/position.
  15. Repeat steps 7–14 for the remaining calculation lines.

: Job Budget Calculation view with sample entries

Entering BOMs in Job Structure

Complete item BOMs can be considered for the job structure plan.

If the system identifies that a production or assembly BOM is present, the Job Line BOM Control dialog box appears.

: Job Line BOM Control dialog box opened by adding item D-1000 to the Budget Calculation

Note

The usage of Document BOMs are described in topic “cc|configure to order” on page 1.

When a Production BOM Item is inserted as a component on a job structure, the entire Production BOM and Routing assigned to the item can be “exploded” in the Job Budget Calculation view.

As illustrated in Fig. 3, a BOM is included on item D-1000, Conveyor System. When expanded, the substructure (Production BOM and Routing) can be modified and then the modified Production BOM and Routing can be used in a Production Order created from the job.

: Conveyor System item with expanded production BOM and routings

The production BOM of the pallet rotating device can be expanded further. Here it can be decided if the BOM will be modified for the project or if it should be purchased as a whole (item).

BOMs can also be added to the Job Budget Calculation from a General Component to which an item with an assembly BOM has been assigned.

: BOM Item assigned to general component FREIGHT

When the general component is added to the Job Budget Calculation line, the Job Line BOM Control dialog box opens, identifying it as an Assembly BOM.

: Job Line BOM Control dialog box opened by adding general component with item BOM to Job Budget Calculation

The BOM behind the item can be exploded to the job line budget without creating an additional level.

: General component with BOM item added to Job Budget Calculation

Any Production BOM or Assembly BOM that was added to the Job Budget Calculation line without exploding it upon insertion (by setting BOM Levels = No BOM in the Job Line BOM Control dialog box) can be exploded at any time using the “Expand BOM” function on the Actions tab, which reopens the Job Line BOM Control dialog box.

Copy and Move Positions and Levels

This section explains how to copy and move Positions and Levels within the Job Budget Calculation view.

Copying Positions and Levels

Within cc|project, it is possible to copy Positions from one job to another. When a Position is copied, the subordinate Levels are copied with it.

Exercise – Copy Job Structure

To copy positions and levels from one job to another, follow these steps:

  1. On the navigation pane, click cc|project > Jobs > Jobs.
  2. Locate and select the job to which the structure will be copied.
  3. On the ribbon, click Budget Calculation.
  4. On the ribbon, click New Position.

Note

At least one Position must exist before a structure can be copied.  

  1. On the ribbon, click Actions > Copy from.
  2. On the Job FastTab, enter in the No. filter field the job number to be copied.
  3. Enter other filters as needed and then click OK.
    Results that match the filtering criteria display in the Search Job Lines page.

: Search Job Lines page displaying lines available for copying

  1. In the Search Job Lines page, select the line to be copied and then click OK. The Levels are copied with the Position.
  2. If additional lines need to be copied, add additional positions and then repeat steps 5–8.

Note

Complete job structures can also be copied from one job to another using the “Copy Job” function.

Moving Positions and Levels

It is possible to move Positions and Levels in the Job Budget Calculation view by clicking Actions > Move and then specifying the direction and degree of the move. The options are:

  • Up (from one Position)
  • Down (from one Position)
  • Level Up (move up within a Level)
  • Level down (move down within a Level)

Rollup Functionality

The Rollup functionality enables lines to be summarized. Lines with a Rollup Type are then summarized on the next higher level in the hierarchy according to the selected Rollup Type. The options are:

  • Value – Only amounts are passed (with a Quantity of 1)
  • Quantity+Value – Quantities and amounts are passed
  • BOM – BOM components are passed to the main BOM; the quantity in upper structure will not change

: Rollup calculations in Job Budget Calculation view

The summary of the quantity information can be applied to a different unit in the higher level. As shown in Fig. 5, WBS positions 1.1.1.–1.1.3. produce 280 hours in the summarized WBS position 1.1. This can be done because WBS position 1.1 was subsequently set up with a component that uses hours as the base unit, but can also be used with day units with a conversion factor of 8.

Note

No plausibility checks are carried out for the units when WBS positions are linked.

Budgeting

The Budgeting functionality enables fixed calculation values to be assigned to WBS positions.

: Budgeting in Job Budget Calculation view

WBS positions are fixed by selecting the Budget check box, which also selects the Budget Costs check box. Both fields can be used independently:

  • Budget – used for quantity and sales amounts.
  • Budget Costs – used for quantity, total direct cost, and total cost.

Once the check boxes are selected, the budget line can be modified, meaning that direct budgeting operations can then be performed on the WBS position.

The essential calculation data and remaining budget differences will be displayed in the WBS Calculation FactBox box for the actual position.

: Budgeting in Job Budget Calculation view

Special Direct Unit Costs from Vendors

Stored direct unit costs from vendors can be applied when the direct unit costs are calculated.

When an item is selected, the master vendor is displayed automatically in the Vendor No. field. If the system finds a special direct unit cost for the item from the vendor, this value is included in the calculation. The parent remaining budget is adjusted.

: Vendor direct costs in Job Budget Calculation view

Job-Specific Purchase Conditions

Purchase prices and line discounts can be transferred from the item to the Job Budget Calculation. This feature uses the standard Purchase Prices and Line Discounts from the Item. The difference is that pricing can be made specific to a job.

For example, item D-7015 has a direct unit cost of 2500 but also has a purchase price of 2200 for 1 PCS to vendor 40000 when used for job J00010.01.

: Job-specific Purchase Prices for item D-7015

Note that the Main Job No. and Job No. fields are populated. This means that the special pricing is used not only for this job (J00010.01) but also applies to any job relating to the main job (J00010).

When this item is added to the Job Budget Calculation, the direct unit cost is 2500.

: Item with purchase prices added to Job Budget Calculation without vendor

When the vendor with special pricing is added, the cost is updated to 2200.

: Direct Unit Cost updated from vendor purchase price

If a line discount exists for this job, perhaps a 1.5 percent discount if 2 PCS are purchased from vendor 40000, the Purchase Line Discount %/Amount is also updated in the Job Budget Calculation.

: Purchase Line Discount applied to Job Budget Calculation

It is possible to transfer purchase prices and line discounts from a purchase quote to the master data (Item). If a job is attached to the purchase quote line, the related job information is also included as well as the source document (purchase quote) information.

The “Transfer Purchase Conditions” function can be used at any time a purchase quote is created, such as during the procurement process or when using cc|purchase quotes manager.

Refer to topic “Transfer Purchase Conditions from Purchase Quote” on page 1 for more information.

Calculation with Currencies

Calculations can be performed using prices (sales and purchases) in foreign currencies.

: Purchase currency assigned to vendor 01587796 for item D-7011, Engine

Foreign currencies can be entered directly in the relevant job line or are predefined through the vendor setup. Purchase prices in foreign currencies will be converted internally for the process of job calculation. 

The Purchase Currency Date and Sales Currency Date fields are set up on the individual lines. If this field is blank, the current date is used.

Alternative currencies can be defined for sales invoices on the Job Card. Refer to topic “Invoicing with Foreign Currencies” on page 1.

: Sales currencies on Job Card

Recalculate Charges

Job Charges that are added or modified after the Job Budget Calculation has been created can be recalculated. There are two options for recalculation charges:

  • “Update Calculation” function from the Actions tab on the Job Budget Calculation view
  • “Update Extra Charges” function from the Job Charges page

Update Calculation

As described in topic “Update Calculation” on page 1, the “Update Calculation” function updates the existing calculation depending on the line type.

Fig. 11 illustrates the calculation that has been created based on the defined project surcharges.

: Job Charges in Job Budget Calculation view

For all low level calculation lines, an Indirect Cost % of 10 has been added to the unit cost, which results in the calculated unit price.

The Extra Charge % will only be calculated for the component lines if the Unit Price Base from the Job Card is Cost. In case of Price, the surcharge will be calculated based on the Unit Cost and Unit Price fields.

Job charges are set up by defining the Indirect Cost % and Extra Charge % fields on the Job Card by clicking Navigate > Charges. These charges can be changed at any time.

: Indirect Cost % and Extra Charge % values updated for Job J000001.01

In the Job Budget Calculation view, click Actions > Update Calculation to begin the update process. Depending on the line(s) selected, a selection dialog may open first to ensure the correct line(s) are included in the update.

Calculation Schedules

The Calculation Schedule is a worksheet that is used to perform various calculations on a job line to determine its Unit Price.

Calculation Schedules rely on Calculation Groups, which are set up on the components.

: Job Budget Calculation view with Calculation Groups

Calculation Schedules can only be used on lines that are hierarchically below or within itself. For example, line 1.1. can have a Calculation Schedule that is the result of the following formula: (line 1.1.1. + line 1.1.2.) * 1.05% + line 1.1.3. Notice that the formula is comprised of subsections hierarchically within line 1.1.

Extra charges can be calculated once they are set up in the Job Calc. Extra Charges page, which is accessed from the cc|project menu by clicking Administration > Calc. Schedule Extra Charges.

: Job Calculation Extra Charges

In Fig. 22, notice the Extra Charge lines and the corresponding Job Calc. Extra Charge codes in the Totaling field.

The Calculation Schedule is accessed from the Job Budget Calculation view.

With the proper job line selected, click Navigate > Calculation Schedule. If a calculation schedule does not exist, a message appears asking to create one for the WBS line (alternatively, a new schedule – or an additional version – can be created by clicking Actions > Create Calc. Schedule). If a calculation exists, the Job Calc. Schedule Job Line page opens.

: New Job Calculation Schedule for WBS 1.

To populate the calculation schedule lines with a standard template, click Import from template on the ribbon. To update the calculation in the lines, click Calculate on the ribbon.

: Job Calc. Schedule with standard calculation template and Quote Calculation

Summarize Position

Calculation structures can be different than processing structures.

It is possible to create sums for a component without the value of this sum being used in the calculation. These WBS positions are used for processing because it is sometimes easier to enter time on one WBS position instead of on detailed WBS positions.

To use the summarize WBS position feature, a new WBS position must be created. After the component to be summarized is inserted, select the Summarize Position check box to create the sums.

: Summarized Position for Assembly lines in the Job Budget Calculation view

If the Summarize Position check box is selected and WBS positions exist for the same component, the Quantity and Direct Unit Cost fields will be updated automatically.
