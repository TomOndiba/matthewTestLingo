cc|calculation

Although similar to the Calculation Structure in cc|project, calculations that are performed using the cc|calculation module have different requirements for set up and usage as well as offer different functionality.

The calculation from cc|calculation is completely independent from the budget calculation/rollup method (as is the cc|project Calculation Schedule) and is used mainly for controlling purposes, not for calculating a sales price.

Additionally, the Concurrent Calculation functionality incc|calculation provides the ability to show several calculations with different time bases next to one another and/or to put two calculations based on two different jobs into one (e.g., alternative Jobs, similar Jobs, etc.). Calculations can also be performed on Job Groups, which provides a single calculation for multiple jobs based on specified criteria.

This section describes the functionality that is specific to using cc|calculation in cc|project and does not include standard cc|calculation field descriptions, etc. For detailed setup, configuration and usage information, refer to the current “cc|calculation” documentation.

Calculation Schedule Setup

The Calculation Schedule Card is accessed from the Sales & Marketing, Administration menu (Departments > Sales & Marketing > Administration > Calculation Schedule). This card is similar to the Job Calc. Schedule Template in cc|project however there are many different fields or fields with similar functionality but different names.

: Calculation Schedule Card in cc|calculation

To use the Calculation Schedule with a job, the Totaling Type / Totaling fields must always be mapped to a Job Calculation Group, unless it is a summary row.

Note

Since the Job Calculation Groups from cc|project are also used in the calculations from cc|calculation, it is important to consider how Calculation Groups will be used in each product. If a calculation is using data from the Job Budget Calculation, the calculation groups need to be mapped correctly to the lines in both the Calculation Template (in cc|project) and the Calculation Schedule Card (in cc|calculation) to ensure proper calculation.

Rows can be indented below a main row to create a hierarchical structure using the actions in the Lines FastTab > Functions menu.

It is possible to review the rows that are part of the summaries by clicking the summary row and then Lines FastTab > Functions > Update Sum Calculation (or pressing F5).

The Calculation Schedule does not need to be certified in order to use it on a job although a message will appear if it is not, and it will be automatically certified when certain processes are performed in the Calculation Card.

Job Calculation

Once the Calculation Schedule has been created and Calculation Groups have been assigned to the Job Budget Calculation lines, a calculation can be performed on a job by clicking Calculation on the Job Card.

: Calculation action on the Job Card

Clicking Calculation on the Job Card opens the Calculation List page, which displays any existing calculations and from where new calculations can be added. Click New to create a new calculation or Edit to open an existing calculation.

In the Calculation Card page, press Enter to insert the next calculation number

: New Calculation Card with job calculation type dialog

The Select Job Calculation Type dialog opens, which determines the data that is the basis for the new calculation. The options are described in the following table.

Option

Uses data from...

Quote Calculation (Version)

Job version that is assigned to a Sales Quote

Order Calculation (Version)

Job version that is assigned to a Sales Order

Expected

Lines in the Job Budget Calculation

Actual Cost

Job Ledger Entries assigned to Job Budget Calculation

Liability

Purchase Lines assigned to Job Budget Calculation

Click OK to open the Calculation Schedule List page, which displays the list of available Calculation Schedules. Select a Calculation Schedule and then click OK to use a calculation schedule.

Note

If a schedule is not selected, one can be added after the calculation is created by assigning it manually in the Schedule FastTab.

: New Calculation Card with job-defined calculation schedule based on Expected data from Job Budget Calculation

To see calculated values, click Update Calculation on the ribbon.

: Calculated values from Expected data in the Job Budget Calculation

Calculation data can be displayed visually with different charts in the Calculation Chart FactBox (use the “Refresh” function in the FactBox if not automatically displayed).

: Calculation values visibly displayed in Calculation Chart FactBox

Concurrent Calculations

Concurrent Calculation functionality allows users to calculate from job versions (sales quotes/orders), expected/actual costs, to posted job ledger entries and purchasing, and provides a matrix view to compare the different calculations.

: Concurrent Calculation Code field populated by running Concurrent Calculation function on Calculation Card

There are numerous ways to create/assign a concurrent calculation: 

  • create automatically from a universal setting on the Calculation Schedule Card
  • create by clicking Concurrent Calculation on the Calculation Card
  • assign an existing Concurrent Calculation manually in the Concurrent Calculation Code field on the Document FastTab of the Calculation Card
  • assign an existing calculation to an existing Concurrent Calculation line

When a Concurrent Calculation is created by running the “Concurrent Calculation” function on the Calculation Card, the Concurrent Calculation page automatically opens.

: Concurrent Calculation created from Calculation Card with Expected job values

When a new calculation is created for a different job calculation type, it can be manually added to the existing Concurrent Calculation for comparison, either by filling in the field on the Calculation Card or in the Concurrent Calculation line.

: Concurrent Calculation created from Calculation Card with Expected and Actual job values

To display the concurrent calculation in a matrix view, click Concurrent Calculation Matrix on the ribbon.

: Concurrent Calculation Matrix view with Expected and Actual job values

Note

Any calculation from any job can be assigned to a concurrent calculation for comparison purposes.

Job Group Calculation

With the calculation feature for Job Groups, a single calculation can be created for all entries in a job group, which can be beneficial for having an overview of a job with multiple sub-jobs.

Click Calculation on the ribbon to create a calculation for job group. The rest of the process is the same as for a Job; refer to topic “Job Calculation” on page 1

: Accessing the Calculation function for the Job Group

Once the new calculation is created for the selected Job Calculation Type, it can be updated to show the calculations for the entire job group.

: Job Group Calculation with job-defined calculation schedule based on Expected data from Job Budget Calculation

On the Document FastTab, notice that the Reference Type is Job Group.

A concurrent calculation can be created for the Job Group as well (refer to topic “Concurrent Calculations” on page 1).

: Concurrent calculation created for Job Group with expected values

Calculations for other Job Calculation Types can be added to the concurrent calculation.

: Concurrent calculation for Job Group with expected and actual values
