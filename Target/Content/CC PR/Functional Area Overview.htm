Functional Area Overview

cc|project optimizes work allocation when processing projects by subdividing organizations into five closely linked functional areas.

: Project Views and Modules

Cost Control through Calculations

Efficient cost controls can be obtained through clear calculations based upon flexible project structures. cc|project assists in this effort by offering:

  • Hierarchical structuring of work breakdowns in as many levels as required
  • Timed and calculated linking of work breakdown structures
  • Multi-level cost calculation increases in a flexible, definable calculation schedule
  • Version management of project plans

Effective Planning and Controlling

Project structural planning easily creates and organizes, based on hierarchy, all performance work breakdown structures within an individual project. Single work breakdown structures and entire blocks can be easily moved and manipulated within cc|project.

Project managers now have all information regarding project deadlines and project development readily available, enabling them to effectively manage the economics surrounding their project. All this is supported by the Signal Monitor functionality which provides visual cost control for individual work breakdown structures or for entire work packages, dependent on the extent of project realization.

Planning and Controlling Individual Work Packages

Planning project details is critical to controlling resources and timelines. cc|project enables this by providing access to scheduling and planning within the individual work plans, integrated with resource calendars and views. cc|project offers:

  • Clear planning screens for project managers
  • Development of networked plans with bi-directional Microsoft Project interface
  • Calendars with resource availability comparisons
  • Integrated vacation planning

Simple Navigation in Complex Project Structures

Complex project structures remain clearly identified through the use of the Navigator function. cc|project provides access to project transaction-related information by drilling down to the original entry information. Thus, individual sections from the project structure plan can be separately viewed and revised, creating greater visibility into and accuracy within the financial and project information.

Optimized Execution through Efficient Management

Management screens allow constant control of internal performance contribution. All information can be gathered from cc|project with respect to:

  • Time entry
  • Travel expense recording
  • Inventory transactions (sales, production, purchasing, etc.)

Within this procedure, a distinction is made between performance contribution which can be invoiced and contributions which cannot be invoiced, such as re-work or budget overruns. This provides the ability to see all pertinent information in one place, making data collection easy and timely decisions more feasible.

Full Visibility into Procurement

All product or inventory requirements that have been released for procurement are shown in the Job Procurement view. This information is forwarded to either Purchasing or to other internal departments for production or fabrication. Screens detailing deadlines that have been set and budgeted plans are continually available for monitoring purposes from within cc|project.

By integrating procurement into projects, purchases can occur when the materials are needed, ensuring that minimal levels of inventory are on-hand.

Centralized Sales Data

All quotes and orders belonging to a given project can be created from the Job Sales view within cc|project. Monitoring, administration, and invoicing are all possible from here; and all orders, whether original or change orders, are traceable from project planning to billing.

Centralized sales data provides the ability to match bills to quotes, bills to projects, bills to actual usage, and to track margins for each line item or sub project.
