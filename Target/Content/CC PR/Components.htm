Components

This topic describes various setup pages relating to Components in the cc|project, Administration menu.

Component Types

The Component Type can be used alternative to the Type field in the job calculation line. The structure of the Component Types is individual.

Example

Instead of an Item, a component type can be used with more details, such as Electronics or Mechanism. Different types (e.g. items and resources) can be merged into one Component Type, such as Service Items and Services.

: Component Types page

The Component Type code can be assigned to Items, Resources (Services and Traveling) and General Components.

Component Groups

All components (items, services and general components) can be structured and confined hierarchically by setting up Component Groups.

: Component Groups main page

A component group hierarchy can be created by clicking Sub Groups on the ribbon.

: Pipes sub groups opened from Component Groups main page

Component groups can be used calculation-wide. An overall component list is available in the Job Budget Calculation view by leaving the Type field blank and using the lookup in the No. field.

: Component Groups Selection page accessed from the No. field on Job Budget Calculation view

Navigate Tab

The cc|project-specific actions in the Job menu group on the Navigate tab are described in this section.

Action

Description

Comments and Documents

Opens the Job Comment Lines page, which is used to record unstructured data, such as user-specific comments. The Comment Type determines where the comment appears.

Component Groups

This action contains a drop-down with the following actions:

  • Set Component Groups – Assigns the component group hierarchy that was selected in the Component Groups Selection page to the Item Card.
  • Delete Component Groups – Clears the Component Group 1 Code – Component Group 5 Code fields on the Job FastTab.

Resources

Services and traveling expenses are set up as resources (Type = Service and Traveling, respectively) and are used as components in budget calculation. Each can be calculated independently from a person resource.

The cc|project-related fields and functions on the Resource Card are described in this section.

Person FastTab

Person resources are used for Responsibilities, Planning and Time Entry.

: Resource Card, Person FastTab

The fields on this FastTab are described in the following table.

Field

Description

Person Work Type Code

Identifies a work time group that can be used to assign wage groups.

Base Calendar Code / Job Work-Hour Template Code

Together these fields assist in the person’s capacity calculation. 

E-Mail

Used for exporting with the MS Project interface.

Job Person Filter

Can only be used if the jobs to which the respective filter setting applies are to be displayed in a job list.

The information set up on the Job Card (Home tab > Job Team) is considered.

Activate Job Person Filter

If selected, activates the Job Person Filter setting.

Job Line Person Filter

Can only be used if those job lines should be shown to which the respective filter setting applies. This will only include the project team data defined for the job line.

Activate Job Line Person Fil.

If selected, activates the Job Line Person Filter setting.

User ID

Opens the Users page, where a Microsoft Dynamics NAV user can be assigned to the resource; used to personalize the Role Center activities.

Salesperson/Purchaser

Assigns a salesperson or purchase code to the resource; used to personalize the Role Center activities.

Job Journal Time Entry / Job Journal Batch Name

Identifies a unique job journal/batch for the resource.

Note

Every person must have a personal Journal for Time Entry.

Terminal Password

Currently has no functionality.

General

General Components are components that do not have the characteristics of services or items and are ultimately settled via G/L accounts. However, the BOM Item No. field provides the ability to assign an item with a BOM to a general component for use in the job line budget (refer to topic “Entering BOMs in Job Structure” on page 1 for more information).

: General Components List page

External Services are used for purchase because Resources cannot be purchased with standard Microsoft Dynamics NAV functionality.

Note

External Services and External Traveling Expenses must be specified on the Procurement FastTab of the Jobs Setup page.

The General Component Card contains similar fields and functions as the Resource Card; refer to topic “Resources” on page 1 for more information.

Machine Centers

Machine Centers are standard in Microsoft Dynamics NAV and contain no cc|project-specific information. Refer to Online Help for information about setting up new machine centers for use with cc|project.

Work Centers

Work Centers are standard in Microsoft Dynamics NAV and contain no cc|project-specific information. Refer to Online Help for information about setting up new work centers for use with cc|project.
