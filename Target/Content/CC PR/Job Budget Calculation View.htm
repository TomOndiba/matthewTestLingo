Job Budget Calculation View

In the Job Budget Calculation view, a significant number of fields are populated with default values from the work packages (Type and No. of the selected components). Many of these fields were defined in topic “Setup” on page 1 and others are defined within process sections later in this documentation. Therefore, this section does not provide a complete description of each field.

: Job Budget Calculation for job J000001.01

The fields in the Job Budget Calculation view are described in the following table.

Note

Some of the field calculations are based on special line settings, such as rollup functionality, job charges and budgeting. Therefore, field calculations described in this section are described based on general line settings.

Field

Description

+/-

Indicates if the directory levels are expanded or collapsed (refer to topic “Job Directories” on page 1).

WBS

Indicates the line position or level created using the “New Position” or “New Level” functions (refer to topic “Work Packages” on page 1 and “Positions and Levels” on page 1).

Identifies the unit of measure to be budgeted for the component.

Actions Tab

The functions on the Actions tab are described in the following sections.

Create Structure

The functions in the “Create Structure” function manage the hierarchical structure in the Job Budget Calculation view:

  • New Position – creates a new position
  • New Level – creates a new level below the position
  • Delete Position – deletes a position and all related levels

Move

Positions and Levels can be moved by direction and degree. Refer to topic “Moving Positions and Levels” on page 1 for more information.

Expand/Collapse

The directory levels can be expanded and collapsed by clicking Actions > Expand/Collapse:

  • Current Position – only relevant on Position lines; expands or collapses depending on the current state, e.g., expanded lines within the Position will be collapsed when the function is run.
  • Expand All – all levels are expanded.
  • Collapse All – all levels are collapsed to the Position level.

Update Calculation

This function updates the existing calculation depending on the line type. Also, depending on the selected line when the function is accessed, a selection dialog may open first to ensure the correct line(s) are included in the update.

: Update Calculation request page

Select the correct option:

  • Current line – only the selected line is included in the calculation
  • Current line + sub-structure – the selected line and related positions/levels are included in the calculation
  • All lines – all lines are included in the calculation
  • Selected lines – only selected lines are included in the calculation

Note

This line selection dialog appears during many different processes.

After the selection is made, click OK to open the Update Calculation request page.

: Update Calculation request page

The fields on the request page are described in the following table.

Field

Description

Reset Budget

Budget amounts can be updated in summary lines. The options are:

  • Cost
  • Sale and Cost

Reset Discount

Sales Line Discount % can be updated in summary lines.

Direct Unit Costs

Direct Unit Cost information can be updated in base lines. The options are:

  • Pricing – line is recalculated with standard price information.
  • Fix prices – changed price on the line can be fixed.

Indirect Cost %

Indirect Cost % in the base lines is recalculated with standard price information.

Extra Charge %

Extra Cost % in the base lines is recalculated with standard price information.

Unit Prices

Unit Price information can be updated in base lines. The options are:

  • Pricing – line is recalculated with standard price information.
  • Fix prices – changed price on the line can be fixed.

Unit Price Base

Determines the change to the Unit Price. The options are:

  • Unit Price – Sales Price finding is used to find an Unit Price
  • Extra Charge % – Unit Price is calculated from Unit Cost plus Extra Charge %

Target Total Price

Changes the current sales amount to a target price.

Type of Change

Determines the change to the target total price. The options are:

  • Line Amount – a change will create a line discount.
  • Total Price – a direct change without a line discount.
  • Profit – an update based on the profit value in all lines.

Total Price

Indicates the new total price based on the Target Total Price and Type of Change.

Total Price (LCY)

Indicates the new total price (LCY) based on the Target Total Price and Type of Change.

Profit (LCY)

Indicates the new profit (LCY) based on the Target Total Price and Type of Change.

Profit %

Indicates the new profit % based on the Target Total Price and Type of Change.

 

Copy from…

This function copies WBS positions from one job to another. When a WBS position is copied, the subordinate Levels are copied with it.

: Copy from function

The Search Job Line request page appears, in which the search criteria can be delimited. After clicking OK, all suitable results are displayed in the Search Job Lines page.

In the Search Job Lines page, select the WBS position which is to be copied to the new job. The WBS position and the relevant substructure are then integrated at the current job WBS position.

The actions in the PQM group are related to the Purchase Quotes Manager functionality. Refer to topic “cc|purchase quotes manager” on page 1 for more information.

MS Project Action Group

FactBoxes

The Job Budget Calculation view includes two cc|project-specific FactBoxes:
