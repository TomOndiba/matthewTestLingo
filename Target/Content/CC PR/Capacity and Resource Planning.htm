Capacity and Resource Planning

The enhanced capacity and resource planning capabilities within cc|project helps plan resources in the same way production can be planned, thus providing greater resource visibility into projects.

By using the integrated capacity and resource planning function in cc|project, resources can be allocated during the planning status of a project. This might include actual resources (individual employees and resource groups) or other assets that are relevant to the planning procedure, such as automotive vehicles and machines.

Vacation Management is also included within the resource planning function. Approval controls are integrated to manage employees that have been assigned to individual projects (refer to topic “Vacation Administration” on page 1 for more information.

This section includes information on calculating capacities, managing resource calendars, synchronizing calendar entries, object planning, scheduling times across all resources and planning using employee groups.

Provide Capacity

The capacity for the internal resources must be set up before availability can be calculated. 

Capacity is assigned to each person by running the “Create Person Capacity” function from the Resource Card.

: Create Person Capacity request page

Note

Running this function deletes existing entries and creates new capacity entries. Select the Keep History check box in the Options FastTab to retain existing capacity entries.

The calculation uses the setup from the Base Calendar Code and Job Work-Hour Template Code fields on the Person FastTab on the Resource Card.

: Resource Card, Person FastTab

The capacities for individual persons can be viewed in the Planning View Person page, accessed from the cc|project menu by clicking Planning > Person Planning.

: Planning View Person showing monthly capacity per person

Note

The values displayed in the Planning View Person page are defaulted from the Planning FastTab in the Jobs Setup page but can be changed on the View Options FastTab.

The capacity for a Resource Group (based on capacities for every person) is summarized in the Planning View Person Groups page, accessed from the cc|project menu by clicking Planning > Resource Groups Planning.

: Planning View Person Groups showing monthly capacity by resource group

Assign Planning Resources

On the Planning Processes page, planning resources can be entered in the Assigned Resource No. field. This can be done directly in the field or by clicking the AssistEdit button to open the Job Process page for entering in the lines.

: Planning Processes showing assigned resources, capacity and work planned hours

Every Planning Resource from the Job Team can be used, as well as Persons and Resource Groups.

After the assignments are done, the planned hours are allocated to the resources as displayed in the Work Planned Hours field. The allocation is equal, but it can be re-allocated manually in the Assigned Resources page.

: Resources assigned to the Consulting lines

Calculation of Schedule Entries

Capacity planning can begin once the processes are linked and the planning resources have been assigned. The following prerequisites exist:

  • Planning quantity and planning resources are known.
  • Installation and customer testing are not supposed to be calculated by effort. In all other lines, the Effort driven check box is selected, which controls if the process, start date or end date should be limited by their planned availabilities.

: Effort driven planning process lines

If capacity limitations prevent an effort driven process to be finished with its planned resources, then the finishing date of this process is automatically assigned to the last planned date.

For non-effort driven processes, the start and finish date will be calculated based on the process duration independent of their related process planning. 

For process planning without the calculation of predecessors and successors, run the “Schedule Process” function by clicking Actions > Schedule > Schedule Process.

For linked process planning, run the “Schedule Processes” function by clicking Actions > Schedule (with Links) > Schedule Processes.

: Result of running the Schedule Processes function

The calendar entries are the result of the schedule process and the calculation is always based on available capacities. If no Schedule Type is selected, the Simulated option is used as the default (refer to topic “Scheduling and Schedule Types” on page 1).

: Job Schedule Entries for the Consulting line

Scheduling and Schedule Types

The different Schedule Types are described in the following table.

Schedule Type

Description

Simulated

Simulated calendar entries are NOT used to calculate availabilities of planning resources.

Every new planning deletes all simulated entries and creates new ones.

Planned

Planned calendar entries are used for availabilities.

Every new planning deletes all planned entries and creates new ones.

Note

This is the default for all process lines.

Fixed

Fixed calendar entries are used for availabilities.

A new planning does NOT delete these entries.

From a process view, the planning can be changed rapidly. Therefore, changing the Schedule Type directly in the overview automatically changes all related processes to this planning level.

: Planning Processes highlighting the Schedule Type

The “Update Scheduling” function on the Actions tab provides several functions that can be applied to marked process lines (as seen in Fig. 10).

: Update Scheduling options

Released lines are available for selection in the Job Calendar (cc|project menu > Planning) by clicking the drop-down arrow in the Job Process No. field.

Resource Allocation

Using resource groups is recommended if a job needs to be planned but there is some uncertainty about which employee will be available for the task but the task needs to be scheduled anyway. The planning can be amended in the short term with the desired resource.

: Planning Processes highlighting the Resource Allocation action

While planning with a resource group, the scheduled dates are assigned to all members of the group and the group member availability will be reconciled with the resource group. To plan with a resource group, click Resource Allocation on the ribbon.

Detect Conflicts

Planning can become a complex issue and it can be hard at times to keep the full picture in view. It is extremely important to spot planning conflicts in the system to be able to regulate them.

cc|project supports the process of finding conflicts with the Signal Conflict field’s two types of planning conflicts:

  1. Predecessor/Successor: Start and End Date do not match
  2. Planned Date (Start or End of the process) cannot be matched (planned too early/too late)

Predecessor/Successor Conflict

There are currently two fields to indicate predecessor/successor conflicts:

  • The Signal Conflict field displays “!>” for successor conflict and “!<” for predecessor conflict.
  • The Signal Conflicts field displays (in bolded blue) an F for successor conflict and a P for predecessor conflict.

In this example, the customer processes (IDs 13 and 14) are causing a conflict because the successor (ID 14, Customer Test OK) is set to be started/finished before the predecessor (ID 13, Customer Test (ID 13).

: Predecessor/Successor Conflict on customer testing lines (showing both signal conflict options)

Planning Conflicts

There are currently two fields to indicate planning conflicts:

  • The Signal Conflict field displays “X+” for a planned after finish conflict and “-X” for a planned before start conflict.
  • The Signal Conflicts field displays (in bolded blue) “T+” for planned after finish and “-T” for planned before start.

Planning Conflicts are usually a result of differences between:

  • Capacity independent planning of processes
  • Capacity dependent planning of resources

This becomes a problem when there are not enough resources available or additional deadlines need to be set after the desired finishing date or a reverse planning must be done (refer to example in Fig. 19).
