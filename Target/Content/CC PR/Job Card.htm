Job Card

In this section, the elements of the Job Card are described.

: Job Card, General FastTab

General FastTab

The General FastTab contains basic information about the job, including the customer, job type and status, persons responsible and job links. The fields on this FastTab are described in the following table.

Field

Description

No.

Unique identifier for the job. Depending on number series setup, this number can be generated automatically or entered manually.

Description /  Description 2

Short description for the job.

Contact Company No. / Contact Company Name

Number and name of the contact company; used if the company requesting the service is not set up as a customer yet.

Contact Person No. / Contact Person Name

Number and name of the person requesting the service; used if the company requesting the service is not set up as a customer yet.

Sell-to Customer No. / Sell-to Customer Name / Sell-to Address / Sell-to Post Code / Sell-to City

Number and address information of the customer; used if the company requesting the service is set up as a customer.

Note

The selection made here is independent of the selection made in the Company Contact and Person Contact fields.

External Document No.

Informational document number that will be transferred from the job to the sales documents.

Job Type

Classification for the job.

Completion Reason Code

Indicates the reason for which the Status has been set to Completed.

Job Link

Indicates whether this is a Main or Sub job:

  • A Main job can have multiple Sub jobs.
  • A Sub job can have multiple Sub jobs and only one Main job.

Refer to topic “Job Card (Main and Sub Jobs)” on page 1 for more information.

Job Link No.

Displays the linked job number, if this is a Sub job.

Alternative of No.

Indicates that this job is an alternative to the Job No. assigned in this field. Alternative jobs can be created as a copy of an existing job using the “Copy Job” function, for example to calculate an alternative Budget Calculation.

Current Version

Displays the Version No. for this job, if multiple versions exist. The version can be locked by clicking Actions > Save Version.

Responsibility Center

Identifies a Responsibility Center for the job.

Bill-to Customer No.

Determines where the sales invoice is sent.
Defaults to the Sell-to Customer No. selected on the General FastTab. This default can be changed if the Bill-to Customer is different than the Sell-to Customer.

Bill-to Contact No.

Identifies the contact number to whom the invoice is sent.

Bill-to Name .. Bill-to Contact

Name, address and contact information of the selected Bill-to Customer/Contact No.

Search Description

Alternate search name for the job.

Person Responsible / Person Responsible 2

Person or Resource to be associated with the job, typically intended for the sales and project manager. These individuals are automatically assigned to the Job Team.

Blocked

Specifies the blocking status for actions related to the job. The options are:

  • Posting – Jobs can be used but are blocked for posting. Choosing this option means that no usage or sale can be posted on the job.
  • All – All actions are blocked. That means no usage or sale can be posted on the job and the job no. cannot be entered in any document such as purchase order, time entry, inventory request, etc.

Last Date Modified

Date on which the job was last modified.

 

Posting FastTab

The Posting FastTab contains the information relevant for invoicing the job.

: Job Card, Posting FastTab

The fields on this FastTab are described in the following table.

Field

Description

Status

Indicates the current phase of the job. The options are:

  • Planning
  • Quote
  • Order – allows posting to the job
  • Completed – prevent posting to the job

Note

Completed jobs are available from the cc|project menu by clicking History > Completed Jobs; these jobs are displayed in the Jobs list.

Job Posting Group

Determines which G/L accounts are used in posting processes.

The default setup in the Jobs Setup page is defaulted in, but can be changed.

Payment Terms Code

Inherited from the Customer and passed on to sales quotes and orders, but can be changed as needed.

Department Code

Dimension information to be assigned to the job. This field is dynamic and will match the company’s selection for Global Dimension 1.

Project Code

Assigned to the job based on the Job Dimension (PROJECT) on the Job FastTab of the Jobs Setup page. The Autom. Job Dimension options are:

  • Job No. – Project code is created with the job number.
  • Main Job No. – Project code from the main job is used.

<Shortcut Dimension 3>

Dimension information to be assigned to the job. This field is dynamic and will match the company’s selection for Shortcut Dimension 3.

<Shortcut Dimension 4>

Dimension information to be assigned to the job. This field is dynamic and will match the company’s selection for Shortcut Dimension 4.

Duration FastTab

The fields on the Duration FastTab indicate the length of the job and creation date.

: Job Card, Duration FastTab

Foreign Trade FastTab

The fields on the Foreign Trade FastTab are used for job calculation in foreign currencies.

The Invoice Currency Code field is used to define alternative currencies for sales (refer to topic “Calculation with Currencies” on page 1). The other fields are standard Microsoft Dynamics NAV; refer to Online Help for more information about these fields.

: Job Card, Foreign Trade FastTab

WIP and Recognition FastTab

The fields on the WIP and Recognition FastTab are used for the valuation of a job. The desired WIP Method and Realized % can be selected here; all other fields show calculated values from the various views. Refer to topic “Job Evaluation” on page 1 for more information about WIP processing.

: Job Card, WIP and Recognition FastTab

The fields on this FastTab are described in the following table.

Field

Description

Budget Total Price (LCY)

Sum of the Calculation Total Price (LCY) fields for all lines in the Job WIP view.

Budget Total Cost (LCY)

Sum of the Calculation Total Cost (LCY) fields for all lines in the Job WIP view.

Budget Total Profit (LCY)

Difference between the calculated values in the Calculation Total Price (LCY) and the Calculation Total Cost (LCY) fields.

Total Price Released (LCY)

Transferred from released sales orders; used to calculate the WIP Amount.

Realized % Based on cost

Automatically calculated value based on actual cost and budget cost.

Realized %

That is a statistical value in this field.

Variance (LCY)

Difference between realized budget cost and the actual cost. Uses Realized % if unequal to zero or Realized % Based on cost.

Variance %

Variance (LCY) / Actual Total Cost (LCY) * 100.

Actual Total Price (LCY)

Sum of the Total Price Posted (LCY) fields for all lines in the Job WIP view.

Actual Total Cost (LCY)

Sum of the Total Cost Usage SP (LCY) fields for all lines in the Job WIP view.

Actual Total Profit (LCY)

Difference between the calculated values in the Total Price Posted (LCY) and the Total Cost Usage SP (LCY) fields.

WIP Basis

Defaults from the Job FastTab of the Jobs Setup page.

WIP Method

Determines how WIP is calculated using standard Microsoft Dynamics NAV logic. The options are:

  • Cost Value
  • Percentage of Completion
  • Sales Value
  • Completed Contract
  • Cost of Sales
  

Refer to Online Help for descriptions of these standard options.

WIP Transfer

Determines in which job WIP is calculated; works only with WIP Basis = Job.

Last WIP Date

Indicates the last date on which WIP was calculated.

WIP Amount (LCY)

Displays the calculated WIP amount (in local currency) between the last and the current WIP date.

Impending Losses (LCY)

Displays the calculated amount (in local currency) for pending losses between the last and the current WIP date.

Prospective Cost (LCY)

Displays the calculated amount (in local currency) for future expenses between the last and the current WIP date.

Cost of Sales (LCY)

Displays the calculated amount (in local currency) for revenue-based WIP (refer to the WIP Revenue-Based (LCY) field in the Job Line WIP Entries page).

WIP Not Posted

Indicates if a calculated job value has not been posted (Finance).

 

Shipment FastTab

The Shipment FastTab contains contact shipping information for job related deliveries.

: Job Card, Shipment FastTab

The fields on this FastTab are described in the following table.

Field

Description

Ship-to Code

Identifies a ship-to location for the job.

The Ship-to Address manages shipment addresses for customer-specific addresses of construction sites or other addresses. These are often job-specific addresses. Therefore, the Job No. and Main Job No. fields are also on the Ship-to Address.

Shipment Contact No.

Identifies the person to whom the items related to this job should be shipped.

Shipment Contact Name

Displays the name of the contact selected in the Shipment Contact No. field.

 

Controlling FastTab

The Controlling FastTab displays all indicators of the job in one place. Indicators for instance are Order Value, Invoice Amount, Actual Cost, Expected Internal Cost, Purchase Commitments and the calculated Total Profit in percent and local currency. These values are also saved with a job version.

: Job Card, Controlling FastTab

The fields on this FastTab are described in the following table.

Field

Description

Total Order Value (LCY)

Indicates the values of the released sales orders, in local currency.

This field also appears on the Jobs list page.

Total Invoice Amount (LCY)

Indicates posted sales in the Job Ledger Entries, in local currency.

This field also appears on the Jobs list page.

Actual Cost (LCY)

Indicates posted usage in the Job Ledger Entries, in local currency.

This field also appears on the Jobs list page.

Purchase Commitments (LCY)

Indicates the value of open entries in the Purchase Lines, in local currency.

Exp. Internal Cost (LCY)

Indicates the value of open Inventory Requests in the Job Journal, in local currency.

Actual Total Profit (LCY)

Total profit value in local currency, calculated by Total Invoice Amount (LCY) - Actual Cost (LCY).

Actual Total Profit %

Total profit percentage calculated by Actual Total Profit (LCY) / Total Invoice Amount (LCY) * 100.

Hours Budget

Indicates the number of hours that have been budgeted for resources.

Hours Posted

Indicates the number of hours that have been posted to resources.

Hours Job Journal

Indicates the number of hours in the Job Journal that have not yet been posted to resources.

Hours Scheduled

Indicates the number of hours that have been scheduled for resources.

Hours Released

Indicates the number of hours that have been released for resources.

Sales

Amount Invoiced (LCY)
Amount Receipt (LCY)
Amount Outstanding (LCY)
Amount Due (LCY)

Indicates posted invoices and receipts in the Customer Ledger Entries, as well as the outstanding amount and amount due, in local currency.

Payments and refunds are also calculated in the Amount Receipt (LCY) field.

Purchase

Amount Invoiced (LCY)
Amount Receipt (LCY)
Amount Outstanding (LCY)
Amount Due (LCY)

Indicates posted invoices and receipts in the Vendor Ledger Entries, as well as the outstanding amount and amount due, in local currency.

Payments and refunds are also calculated in the Amount Receipt (LCY) field.

Navigate Tab

The Navigate tab includes four action groups: Job, Views, MS Project, Prices and History. The actions within each group are described in this section.

Job Action Group

This section describes the most relevant actions in the Job action group on the Navigate tab.

Dimensions

This action opens the Default Dimensions page, which displays the default dimensions assigned to this job.

: Default Dimensions page

Comments and Documents

This action opens the Job Comment Lines page, which is used to record unstructured data, such as user-specific comments. In addition, documents relating to the comment information can be attached using the standard Links functionality.

: Job Comment Lines page

Job Comments can be added to the Item Card, Service Card and Other Services Card. These comments will then be copied to the appropriate job view when the component is selected in the Job Budget Calculation view. The Comment Type field determines where the comment appears. The options are:

  • General
  • Support
  • Rough Concept
  • Purchase
  • Detailed Concept
  • Sale
  • Processing
  • Long Text
Contacts

This action opens the Job Contacts page, which is used to assign additional contacts to the job.

: External Job Contacts

Once added to the page, contacts can be moved by clicking Actions > Move > Up or Down for sorting the contacts. 

With this integration, all relevant functions within the application can be used for project management purposes, thus providing the ability to:

  • Track and monitor progress
  • Tie vendor billings to customer billings
  • Create “to-dos” and calendar appointments, including:
    • Planning deadlines
    • Task management
    • Documenting activity work breakdown structures
    • Integration with Microsoft® Outlook®
Job Team

This action opens the Job Team page. The employees or resource groups assigned in this page are used later in the budgeting and planning stages.

: Job Team page

Since responsibilities are generally assigned to both the project and the job WBS positions, cc|project provides the following two fields on the Job Card:

  • Person Responsible
  • Person Responsible 2

The resources entered in these fields are automatically transferred to the Job Team with the resource assigned to the Person Responsible field identified as the Responsible individual. Additional team members can be assigned to the job by clicking Job Team on the ribbon.

Planning resources can be single persons as well as resource groups. Each can be added manually on the lines or by these functions on the Actions tab:

  • Get new group members – Opens the Resource Groups page, from where all resources of the selected resource group can be added to the selected job team.
  • Person to job – Opens the Resource List page, from where resources can be added to the selected job team.
  • Group to job – Opens the Resource Groups page, from where resource groups can be added to the selected job team.

For more transparency of responsibilities and job assignments, the planning resources can be individually assigned to each job line or hierarchical level (refer to topic “Capacity and Resource Planning” on page 1).

Versions

This action opens the Versions page, which displays non-editable copies of the job. Each new job is set as a first version.

A new version can be created from the Job Card by clicking Actions > Save Version. In addition, each time the Job status changes, cc|project can be set up to ask whether the current version is to be saved.

The Versions page can be accessed from the Job Card by either clicking Navigate > Versions or by clicking the value in the Current Version field on the General FastTab.

: Versions page

On the ribbon of the Versions page, click Project Card to open the Version Job card. The Version Job card contains most of the same information as the Job Card as of the moment the Version is created.

: Version Job page

Click Budget Calculation to open the Version Job Budget Calculation page, which is similar in structure and style to the Job Budget Calculation. Additional fields in the Version Job Budget Calculation page can be used to compare the version Quantity, Total Cost Budget and Sales Line Amount Budget with the current values of the Budget Calculation.

: Version Job Budget Calculation page, highlighting the version and current budget fields

As illustrated in Fig. 15, the version quantity, cost and sales line amounts are different from the current values in the Job Budget Calculation in positions 1.1., 1.1.3., 1.2.2. and 1.2.2.4. The changed values are displayed in red italics to help quickly identify any differences between version and current budget values.

Milestones

This action opens the Job Milestones page, which displays all positions that have been set up as milestones.

: Job Milestones page

Guarantees

This action opens the Job Guarantee List page, which displays all guarantees relating to the job.

: Job Guarantee List page

Job Documents

This action opens the Job Documents page, which displays all documents relating to a job.

: Job Documents page

QC Test List

This action opens the QC Test List page, which displays QC Tests that have been assigned to the job.

: QC Test List page

Views Action Group

This section briefly describes the actions in the Views group on the Navigate tab; each view is described in more detail in other sections.

Budget Calculation

This action opens the Job Budget Calculation page, which is used to create the job structure and calculation, and is the basis for creating a job offer. Refer to topic “Calculation and Budgeting” on page 1 for more information.

: Job Budget Calculation page

Planning and Processing

This action opens the Job Planning and Processing page, which focuses on all operative processes for planning, purchasing and project delivery. It is not possible to do a calculation here but all calculation-related fields can be displayed. Refer to topic “Job Processing” on page 1 for more information.

: Job Planning and Processing page

Sale

This action opens the Job Budget Sales page, which shows only sales positions that are associated with a Sales Code. This view is used to review and initialize invoicing. Refer to topic “Job Processing” on page 1 for more information.

: Job Budget Sales page

Rolling Calculation

This action opens the Job Rolling Calculation page, which includes the most important sums during a job cycle. It is used for permanent job controlling.

: Job Rolling Calculation page

Calculation

This action opens the Calculation List page, where a new calculation can be created or an existing calculation can be opened for the job (refer to topic “Calculation Schedule Templates” on page 1).

WIP

This action opens the Job WIP page, which shows the information and batch operations necessary for project evaluation. Refer to topic “Job Evaluation” on page 1 for more information.

: Job WIP page

Prices Action Group

This section describes the actions in the Prices action group on the Navigate tab.

Resource

This action opens the Job Resource Prices page, where job-specific pricing/discounts are defined for resources (persons or groups).

: Job Resource Prices page

Item

This action opens the Job Item Prices page, where job-specific pricing/discounts are defined for Items.

: Job Item Prices page

Charges

This action opens the Job Charges page, where the following charges/discounts are defined for the job:

  • Purchasing charges (Indirect Cost %)
  • Surplus charges (Extra Charge %)
  • Line discount percentages

: Job Charges page

Traveling Expenses

This action opens the Job Traveling Expenses page, where the job-related costs/prices are defined for travel expenses. Refer to topic “Traveling Expenses” on page 1 for more information.

: Job Traveling Expenses page

MS Project Action Group

The actions in the MS Project group are used with the Microsoft Project integration. Refer to topic “Microsoft Project Integration” on page 1 for more information about setting up these actions.

History Action Group

The History action group contains the Ledger Entries action.

Ledger Entries

Opens the Job Ledger Entries page, which is used to view all the ledger entries for a job. Posting to jobs always creates job ledger entries. The entries are either debits or credits depending on whether they increase your costs or represent sales to a customer.

: Job Ledger Entries page

Job Journals and Purchase Receipts/Invoices/Credit Memos create Usage entries, Sales Invoices/Credit Memos create Sales entries, and Job Shipments create Job Shipment entries.

Actions Tab

The actions on the Actions tab on the Job Card are described in the following table.

Function

Description

Create Sub-Job

Opens a message dialog, which asks the user if he/she wants to create a sub-job for the job (refer to topic “Job Card (Main and Sub Jobs)” on page 1).

Copy Job

Opens the Copy Job dialog, which copies specific job information to a new job.

Save Version

Opens a message dialog, which asks the user if he/she wants to create a new version for the job (refer to section “Versions” on page 1).

When this function is run, no description is added in the Versions page but one can be added manually.

Add QC Test

Opens the QC Template List page, where new QC Templates can be created and assigned to the job by selecting the template and then clicking OK. The assigned QC Tests can be processed from the Job Card by clicking Navigate > QC Test List.

 

Job Card FactBoxes

The Job Card includes three cc|project-specific (or modified) FactBoxes:

  • Job Comment Lines – displays all comments assigned to the job.
  • Job Details - No. of Prices – displays the number of prices for resources, items and G/L accounts.
  • Job Details - WIP/Recognition – displays actual price/cost for the job and WIP information.

: FactBoxes on Job Card
