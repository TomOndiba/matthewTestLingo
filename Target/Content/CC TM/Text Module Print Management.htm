Text Module Print Management

The Text Module Print Management page creates the link between text lines and the objects to be printed and/or shown.

: Text Module Print Management for sales documents

The fields on this page are described in the following table.

FastTab

Field

Description

General

Code

Unique identifier for the print management record.

 

Description

Short description of the print management record.

Line

Output Type

Determines where text is printed or shown. The options are Report and Table.

 

Output ID

Indicates the specific ID of the selected output type. This list is filtered by reports and tables entered in the Text Module Objects Setup table.

 

Output Document Type

Used with Table objects to indicate the type of document. For example, enter Object Type = 36 with Object Document Type = 1 to have text appear on the sales order header.

 

Output Name

Displays the name of the output ID/output document type.

 

Description

User-defined description of the print management line record.

Exercise – Set Up Print Management for Sales and Purchase Documents

Your company wants to use header and line text on sales and purchases as follows:

  • Sales – header and line text printed on Order Confirmation report and pop-up pages to display header and line text during sales order creation.
  • Purchases – header and line text printed on Order report and pop-up pages to display header and line text during purchase order creation.

This exercise explains how to complete this setup, which will be used in exercises in the following sections.

Set Up Print Management for Sales Documents

Follow these steps to set up print management for sales documents:

  1. In the navigation pane, click cc|text module > Lists > Text Module Print Mgt. List.
  2. On the ribbon, click New.
  3. In the Code field, type “S-Orders”.
  4. In the Description field, type “Sales Order Documents”.
  5. In the Lines FastTab, enter 205 in the Output ID field.
  6. In the next line, click the arrow in the Output Type field and select Table.
  7. In the Output ID field, enter 36.
  8. In the Output Document Type field, enter 1.
  9. In the next line, enter 37 in the Output ID field.
  10. In the Output Document Type field, enter 1.

: Text Module Print Management set up done for sales order documents

  1. Do not close the Text Module Print Management page.

Set Up Print Management for Purchase Documents

Follow these steps to set up print management for purchase documents:

  1. In the Text Module Print Management page, click the arrow on the OK button and then click OK & New.
  2. In the Code field, type “P-Orders”.
  3. In the Description field, type “Purchase Order Documents”.
  4. In the Lines FastTab, enter 405 in the Output ID field.
  5. In the next line, click the arrow in the Output Type field and select Table.
  6. In the Output ID field, enter 38.
  7. In the Output Document Type field, enter 1.
  8. In the next line, enter 39 in the Output ID field.
  9. In the Output Document Type field, enter 1.

: Text Module Print Management set up done for purchase order documents

  1. Click OK to close the Text Module Print Management page.

Assignment to Text Records

The Print Management Code can be added to text assigned to master data or standard text in the lines of the Text Module Card (refer to topic “Lines FastTab” on page 1) or in the Edit Text page (refer to topic “Edit Text Page” on page 1), generally accessible from document headers/lines.

: Edit Text page opened from Sales Order header

As previously stated, assigning the SALES print management code to text in the sales order header means that it will appear in the Order Confirmation report. To prevent the text from appearing on the report, simply remove the value in the Print Mgt. Code field.
