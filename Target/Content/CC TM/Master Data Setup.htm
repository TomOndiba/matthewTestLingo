Master Data Setup

Depending on the master data, different text can be assigned. For example, on customers and vendors, header and footer data can be assigned; on items, single text can be assigned, which means the text is printed below the item description – similar to extended text in standard Microsoft Dynamics NAV.

Text Module Card

The setup of text is managed in the Text Module Card page. Initially, the text is entered in this page and then viewed/modified from the individual documents.

: Text Module Card accessed from the Customer Card

In this section, the Text Module Card page is described.

General FastTab

The General FastTab identifies the type of text, languages, and length of usage. Each field is described in the following table.

Field

Description

Text No.

Contains a consecutive number of the text modules linked to a certain data record (such as an item) and is allocated internally by the system.

Language Code

Indicates in which language the text is displayed. If the field is <blank>, the language code from the Local Language Code field on the Text Module Setup page is assumed.

Note

If Language Codes are defined on the customer card, then the Language Code field must be filled in and/or the All Languages check box must be selected in the Text Module Card. Changes cannot be made to the Language Code field after other fields are populated.

All Languages

If selected, the text is to be used independently from the language code of the vendor or customer if no other text was detected using the current language code.

For example, header text is created in English (ENG) and German (DEU). This text will only appear on documents for customers/vendors that have either language code. To make the text available for all customers/vendors in one of the languages, e.g., ENG, then the Text Module Card should be set up with Language Code = ENG and All Languages = Yes. Therefore, the ENG text will be available for a customer/vendor with Language Code = ESP.

Description

Short description of the text.

If left blank, this field is automatically populated from the first line of the actual text (in the Text field on the Lines FastTab).

Sorting No.

Determines the order of the text lines on the document if multiple texts are used.

Header/Footer

Distinguishes between header and footer texts.

Starting Date

Date from which the text should be active.

The system compares the date fields with the document date of the document. If no document date exists, the working date is used as basis of the comparison.

Ending Date

Date from which the text should be invalid.

 

Lines FastTab

The fields in the Lines FastTab are described in the following table.

Field

Description

Text

Contains the actual text. Multiple lines can be entered as needed, although each line can only contain 85 characters.

Italic / Bold / Underline / Color

Formatting options that determine how the text appears on reports and documents.

NOTE

When multiple text lines are entered, the formatting from the above line is copied to the next line.

Print Mgt. Code

Determines whether the text will appear on reports or in pop-up pages on documents. Leave this field empty to exclude the text from reports and pop-up pages.

Multiple lines can be added to the Text Module Card with different print management codes. Therefore, multiple lines of text can be entered, for example, on an Item Card with some text assigned to codes for sales documents and others assigned to codes for purchase documents. The relevant text will appear in the relevant documents: sales text only appears on header/footer text for sales documents, and so on.

NOTE

When multiple text lines are entered, the code from the above line is copied to the next line.

Note

The fields in the Lines FastTab are the same fields used in the Edit Text page.

Navigate Tab

The Navigate tab contains the “Get Standard Text” function. This function is used to bring standard text into the Text Module Card. Refer to topic “Standard Text” on page 1 for more information about using the Standard Text functionality.

Master Data (Customer) Header and Footer Text Setup

When text is assigned to master data, such as a customer or vendor, it can appear on a printed document as a header or footer. Header text appears before the body of the document and Footer text appears at the end – similar to stationary/letter text.

This section uses an exercise to show how header and footer text is added to a customer.

Exercise – Assign Text to a Customer

Mr. Andy Teal – the main contact at customer 10000, The Cannon Group – receives all orders. Your company has decided to add header and footer text to sales orders to greet and thank him for the order.

Assign Header Text

Follow these steps to complete the header text setup:

  1. In the navigation pane, click Sales & Marketing > Order Processing > Customers.
  2. Locate customer 10000 and open the customer card.
  3. Click Navigate > Text Module.
  4. On the ribbon, click New.
  5. In the Language Code field, enter ENG.
  6. In the Description field, type “Greeting”.
  7. In the Text field, type “Mr. Teal, please review your order below.”
  8. Select the Bold and Underline check boxes.
  9. In the Print Mgt. Code field, enter S-ORDERS.

: Completed Text Module Card for header text

  1. Do not close the Text Module Card page.

Assign Footer Text

Follow these steps to complete the footer text setup:

  1. In the Text Module Card page, click the arrow on the OK button and then click OK & New.
  2. In the Language Code field, enter ENG.
  3. In the Description field, type “Closing”.
  4. In the Header/Footer field, select Footer.
  5. In the Text field, type “We appreciate your order!”
  6. Select the Italic check box.
  7. In the Color field, click the arrow and select Blue.
  8. In the Print Mgt. Code field, enter S-ORDERS.

: Completed Text Module Card for footer text

  1. Click OK to close the Text Module Card page.
  2. Close the open pages.

Master Data (Item) Text Setup

When text is assigned to master data, such as an item, it can appear on a printed document. As opposed to customer or vendor text, item text does not distinguish between header and footer text.

This section uses an exercise to show how text is added to an item.

Exercise – Assign Text to an Item

Item 1936-S – the yellow BERLIN Guest Chair – has a new price starting January 21, 2019. Your company has decided to add text to sales orders to explain the item price change.

Assign Item Text

Follow these steps to complete the item text setup:

  1. In the navigation pane, under Sales & Marketing, click Inventory & Pricing > Items.
  2. Locate item 1936-S and then click Navigate > Text Module.
  3. On the ribbon, click New.
  4. In the Language Code field, enter ENG.
  5. In the Starting Date field, enter 01/21/19.
  6. In the first line of the Text field, type “Price Increase” and then press TAB.
    Note that the text from the first line populated the Description field in the General FastTab.
  7. Select the Bold check box.
  8. In the Color field, click the arrow and select Red.
  9. In the Print Mgt. Code field, enter S-ORDERS.
  10. In the next line, type “Due to manufacturing changes, the price of this item has increased by 15%.”
  11. Select the Italic and Underline check boxes.
  12. In the Color field, click the arrow and select Black.
  13. Ensure that the Print Mgt. Code field is set to S-ORDERS.

: Completed Text Module Card for item text

  1. Click OK to close the open pages.
