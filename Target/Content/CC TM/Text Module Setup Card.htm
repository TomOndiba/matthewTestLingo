Text Module Setup Card

The Text Module Setup Card page is accessed from the navigation pane by clicking Departments > cc|text module > Tasks > Text Module Setup. In the Local Language Code field, enter the relevant Language Code. The system equates this Language Code with a <blank> language code.

Predefined Setup

The “Predefined Setup” function creates data for standard finance, manufacturing and service areas, as well as for cc|configure to order. The Expert setup pages are populated with this default data.

Note

New areas and customized reports/tables must be added manually to the Text Module Use in Area Base List and Text Module Objects Setup pages.

Use in Area Base List

In the Text Module Use in Area Base List page, define each Microsoft Dynamics NAV module that can use the cc|text module.

: Text Module Use in Area Base List page

Object Setup

In the Text Module Objects Setup page, define all objects where the text should be printed and/or shown.

: Text Module Objects Setup page filtered on Sales and Purchase report and table objects

The value in the Object Type field determines where and how text is displayed:

  • Report – indicates that text will appear on a printed report. For example, in the header of an Order Confirmation report or the lines of a Purchase Order report.
  • Table – indicates that text will appear in a document header/lines. Additionally, the text to be copied will be displayed in a pop-up page. For example, text assigned to a customer will appear in a pop-up page when the Sell-to Customer No. field is populated in a Sales Order.

Therefore, to print text in a report or to appear as a pop-up in a page, the Use check box must be selected for that line. By default, the Use check box is selected on all reports and tables.

Note

In various versions of Microsoft Dynamics NAV, some reports from the ID range 1300 are set up in the Report Selection - Sales and Report Selection - Purchase pages. These reports are not prepared to print text from the cc|text module. Therefore, ensure that the correct reports are set up for sales (ID range 200..) and purchases (ID range 400..) in the respective Report Selection page.
