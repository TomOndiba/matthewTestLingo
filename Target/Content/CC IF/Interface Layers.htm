Interface Layers

The interface is split into four layers and an additional layer for Standard NAV and the external Application or Component. Each layer is described in more detail in the following table.

Layer

Description

Standard NAV

Represents the Microsoft Dynamics NAV standard business logic. The standard business logic must only communicate with the Process Interface layer.

Process Interface

Represents the business logic that is needed to handle a specific business process within the interface. This layer transfers data between the General Microsoft Dynamics NAV Interface and Standard Microsoft Dynamics NAV or calls standard functions of Microsoft Dynamics NAV.

General NAV Interface

Represents the general Microsoft Dynamics NAV interface. This layer connects specific business processes in the Process Interface and the General Data Transfer layer. The standard Microsoft Dynamics NAV business logic must not be used from this layer.

General Data Transfer

Represents the layer that converts the data to the target format and writes it to the transportation media like stream or files, and vice versa.

Data Transport

Represents the transport layer for the data; this can be a web service, FTP server or text file.

External Application

Represents the external 3rd party application or component that technically connects to Microsoft Dynamics NAV.
