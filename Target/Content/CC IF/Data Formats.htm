Data Formats

Every project function must have a definition of how the data is formatted and which way the data is transported. These are defined in the Data Formats page.

: Data Formats

The fields on the Data Formats page are described in the following table.

Field Name

Description

Code

Unique code of data format.

Description

Short description of the data format.

Data Format

Specifies the basic format of the data file. The options are Text, CSV and Custom.

Endpoint / Filename

Defines the endpoint for the communication like a web service address or filename. If using file names, the asterisk (*) can be used as a wild card for processing multiple files in one folder.

Delimiter

Specifies the separating character for the field in a CSV file. The options are Tab, Semicolon, Comma, Space and Other.

Other Delimiter Character

Defines the delimiting character to be used if Delimiter = Other.

Encoding

Defines the text encoding used by Microsoft Dynamics NAV. The options are MsDos, UTF-8, UTF-16 and Windows.

Transfer

Defines the transportation layer. The options are File, Web Service and FTP.

Only text file is implemented as a generic default layer. For Web Service and FTP, a custom codeunit must be implemented and assigned in the Custom Format Codeunit ID field.

Contains Field Names

If selected, the first line in a CSV file contains field names and will be ignored.

Custom Format Codeunit ID

Identifies a custom codeunit ID if a transport layer other than text file is used. This codeunit then handles the data format.
