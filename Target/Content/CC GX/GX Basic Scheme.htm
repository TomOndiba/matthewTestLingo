GX Basic Scheme

The GX Basic Scheme is the basic setup for a GX view. All GX views are based on the same basics that can be configured on the GX Basic Scheme Card page. For a specific GX view (for example, Prod. Order Planning View) an additional configuration is needed that reacts like an extension to the basic scheme (for example, Prod. Order Planning Setup).

: GX Basic Scheme Card page, General FastTab

General FastTab

The fields on the General FastTab are described in the following table.

Field

Description

Code

Unique identifier of a basic scheme.

Description

Short description for the scheme.

View Starting Date / View Ending Date

Used to build a date range that is the base of the data that will be displayed in the view. If you open a view that displays capacity profiles, only the capacity ledger entries in the date range are displayed.

In these fields, enter a Microsoft Dynamics NAV-specific date formula that is used for the date calculation based on the work date.

Visible Range

Width of the view (in time units) when the view is opened.

Job Height

Default render-height of jobs. A value of zero forces the view to use an internal default value.

Shadows

Enables or disables the rendering of shadows. It might be useful to disable shadow rendering for client PCs with a very low performance.

Scroll Duration (ms)

Animates the scrolling process when skipping to a special job, a special date, or a special row. The value in this field defines the duration of each scroll process. A value of zero disables animated scrolling and lets the view jump to the target position instantly.

Job Displacement Pixels

Vertical displacement when rendering overlapping jobs. This value affects the initial height of the rows that contain jobs. When changing the row height manually, the displacement value is recalculated internally.

Hot Track Job Pattern Code

This pattern is rendered if the mouse cursor is on a job. This pattern is drawn in addition to the job’s pattern code.

Selected Jobs Pattern Code

Multiple jobs can be selected by clicking them while pressing the CTRL key. All selected jobs are drawn with this pattern. This pattern is drawn in addition to the job’s pattern code.

Current Job Pattern Code

The current job is the last job that has been clicked. This pattern is drawn in addition to the job’s pattern code.

Sel. Res. Caption Pattern Code

Multiple resources can be selected. This pattern is drawn in addition to the resource’s pattern code.

Cur. Res. Caption Pattern Code

The current resource is the last clicked row or the row of the current job. This pattern is drawn in addition to the resource’s pattern code.

Current Order Pattern Code

Multiple jobs can be combined into an order. If you selected a job, all jobs that belong to this job’s order are rendered with this pattern. This pattern is drawn in addition to the job’s pattern code.

Selected Order Pattern Code

This pattern is drawn in addition to the job’s pattern code.
