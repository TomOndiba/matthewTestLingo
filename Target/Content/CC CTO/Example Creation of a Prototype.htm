Example – Creation of a Prototype

With the prototype creation function, it is possible to generate new items automatically and/or find already produced items based on their developments. Likewise, new variants of an item can be produced and/or regained.

The creation of a prototype is part of the configuration feature of cc|configure to order and is used when opening a Master BOM in the Document BOMs and the Construction Orders.

Prototype Item Setup

First, an item card must be set up as a Prototype template (refer to Fig. 1). On the Config. to Order FastTab, select the Prototype check box to make the Prototype Type and Prototype No. Series fields editable.

The Prototype Type field determines whether a new item or a new variant of the item will be applied during the creation of a new characteristic of the prototype. In the Prototype No. Series field, enter the number series to be used for the new creation of items and/or variants as a copy of the prototype.

: Prototype setup on the Item Card

Furthermore, attributes must be entered. During the calculation of the Document BOMs, the system checks whether an identical item and/or variant is already present or has to be created. As indicated in Fig. 2, three attributes have been defined. It is important that the Relevant to Prototype check boxes are selected for the attributes, which are to be used for retrieving the prototypes.

: Prototype Attribute Setup

Master BOM and Checklist Template Setup

If prototype items are entered in a Master BOM, attributes can be indicated and/or calculated (refer to Fig. 3 and Fig. 4).

: Definition of Prototypes in the Master BOM

: Definition of Attributes in the Master BOM

Values entered in the Description – Description 5 fields on the Item Card and Master BOM will be transferred to the new item, if one is created. Therefore, substitute syntax can be used. As illustrated in Fig. 3, the substitute syntax “#descr#” is entered in the Description 2 field, which corresponds to the DESCR variable on the Checklist Template (refer to Fig. 5).

: Checklist Template for prototype item

The new Checklist Template and Master BOM should then be assigned to the assembly item (with Part Type = Production) that uses the prototype item as component.

: Checklist Template and Master BOM assigned to assembly item

Prototype Item Usage

During the BOM calculation, the substitute syntax on the Item Card and Document BOM (refer to Fig. 7 and Fig. 8) is replaced with the value in the Input field on the corresponding document checklist line (refer to Fig. 6).

: Document Checklist setup for a prototype item with a defined description

If the Document BOM and/or the construction order are opened, the procedure is as follows:

  • For the Master BOM line, the entered characteristics are calculated first.
  • The system checks whether there is an existing item (and/or a variant), which has the same characteristics for all attributes that are defined on the item card of the prototype.
    • If such an item is found, this item will be transferred to the Document BOM.
    • If no suitable item is found, a copy of the prototype item is created (refer to Fig. 7) and the attribute/characteristics and the descriptions are transferred from the BOM. In the Document BOM, the prototype item is replaced by the new item (refer to Fig. 8).

In this example, no values were entered in the measurement fields on the checklist so a new item has been created with empty measurements. This new item is transferred to the Document BOM.

: Item created from prototype item

The Created from Prototype field is automatically populated with the originating item number from which the item was copied, thus indicating that this is a version of a prototype. In this example, item PROT00000 was created as copy of the prototype template, item number D-PROTOTYPE.

: New prototype item in the Document BOM

If values had been entered into the measurement fields on the document checklist before the Document BOM was created, then a second prototype item would have been created and assigned to the BOM, along with the attributes.

: Document Checklist set up for a prototype item with a defined description and attribute values

Attributes are transferred to the document BOM and the new item.

: New item and measurement values from document checklist transferred to Document BOM

Prototype Item on Sales Line

A prototype for a finished good item can be entered directly in the sales line as well. Therefore, the Formula Value of the prototype item’s attributes must be entered on the Item Card because when requesting the Master BOM to the Document BOM, the designated attributes on the sales lines will be calculated first.

: Formula Values assigned to a prototype item for a finished good item

If a prototype item is entered on the sales line, and an existing item is found, the system will not recalculate the master BOM. The Document BOM or the Construction Order that is linked to the found prototype item will be copied. The BOM or Construction Order information can be found in the Prototype BOM Type and Prototype BOM No. fields on the Item Card.

: New item created from D-PROTOTYPE_FG item with Document BOM and attributes and description from checklist

Example

Previously, item PROT00002 was created from the D-PROTOTYPE_FG item (refer to Fig. 12). Now, a new sales order is created with the D-PROTOTYPE_FG item on the sales line.

: Sales Order with item D-PROTOTYPE_FG on the line

The attributes entered on the Document Checklist match the attributes for item PROT00002 (refer to Fig. 12).

: Attributes on new checklist match attributes on item PROT00002

The Document BOM is requested and because the attributes match item PROT00002, the Document BOM for item PROT00002 is copied to the Document BOM.

On the sales line, item D-PROTOTYPE_FG is replaced with item PROT00002.

: Item D-PROTOTYPE_FG replaced with item PROT00002 on sales line

Note

If no or different attributes were entered on the Document Checklist, then a new item (from the prototype item’s number series) would have been created – with (or without) attributes and new Document BOM – and assigned to the sales line.

Prototype Variants

Prototype variants work the same as prototype items except that instead of creating a new item, item variants are created for the item.

In the Item Card, the Prototype Type field is set to Variant and a number series for prototype variants is used.

: Settings for prototype variant

To determine whether existing variants can be used when a Document BOM or Construction Order is calculated, other attributes must be set up.

Example

In this example, an attribute for item power is defined along with the Formula Value of POWER for calculating the variable in the checklist.

: Attribute defined for prototype variant with Formula Value

Next, the prototype item is added to a Sales Order. The POWER variable – that was added as a placeholder in the Description field on the Item Card – will be replaced with the configuration from the checklist and Document BOM, and inserted into the variable description.

: Sales Order for item with prototype variant

In the Document Checklist, a value for power is entered in the Input field. Note that Variable for Dynamic BOM is POWER, which is the same as the variable used in the item description and the attribute.

: Checklist with value added for Power

Based on the setting in the Item Card, when the Document BOM and/or the construction order is created, the procedure is as follows:

  • For the Sales Line, the entered characteristics are calculated first.
  • The system checks whether there is an existing variant, which has the same characteristics for all attributes that are defined on the item card of the prototype.
    • If such a variant is found, this item will be transferred to the Sales Line or Document BOM.
    • If no suitable variant is found, a variant is created for the prototype item (refer to Fig. 21), referencing the source document. In the Sales Line and Document BOM, the variant code is added and the description changes to the variant description (refer to Fig. 22).
  • If a prototype is used already in a Sales Line, and an existing, matching Prototype Item/Variant is found, then the Document BOM or Construction Lines that were created previously for the prototype is copied without recalculating the Master BOM.

In this example, a power value of 54 was entered in the Checklist. Since no variants exist for this characteristic, a variant is added to the prototype item when the Document BOM is created. Note that in the variant description, the POWER variable from the prototype description has been updated to match the power value entered in the Checklist.

: Variant added to prototype item, Power variable updated in variant description

In the Sales Order line, the variant is added and the Description field is updated.

: Sales Order line updated with variant from prototype

If another line was added to the sales order for the same item using the same power value, the same variant would be used. If another line was added to the sales order for the same item using a different power value, a new variant would be created for the item and entered in the sales line.

: Existing and new variants from prototype in sales order line

Prototype Item in Construction Header

The same item/variant Prototype function used in a Sales Line is available in manually created Construction Orders, as long as a reservation to the Construction Header does not exist.

With this function, it is possible to create and design a new item using Checklists and Master BOMs to calculate the BOM. Therefore, no (dummy) sales documents are needed to connect a Checklist and Master BOM to use the Prototype functionality.

Example

Item D-1000_CO is set up with an Attribute for item power and as a prototype item with a number series.

: Attribute and prototype settings for item D-1000

A Construction Order is created manually from the cc|configure to order > Order Processing menu for one unit of item D-1000. The Document Checklist is opened and a value of 90 for power is entered in the Input field.

: Power value entered in Document Checklist for Construction Order

When the Request BOM function is run, the Item No. is replaced in the Construction Header with an item number from the prototype number series because no other items with these characteristics match item D-1000_CO, and the lines are calculated from the Master BOM.

: New Prototype item in Construction Header, manual changes in Construction Lines

If a matching item had been located, it would have been used for this Construction Order. In this example, changes were made to the Construction BOM for the new prototype item.

: Changes made to Construction Lines

If a new Construction Order is manually created that has the same attribute (e.g., power = 90), then the prototype item and BOM from the Construction Order will be used.

: New Construction Order with same power value as existing prototype item

When the Request BOM function is run, the same prototype item and Construction Lines from the previous Construction Order is used in the new Construction Order.

: New Construction Order with prototype item and BOM

Note

If no or different attributes were entered on the Document Checklist, then a new item (from the prototype item’s number series) would have been created – with (or without) attributes and new Document BOM – and assigned to the Construction Header.

If a variant prototype was used instead of an item prototype, and no matching attributes were found, then a new Variant would have been created (from the prototype variant’s number series) and entered in the Construction Header. If a matching item variant was found, the same item/variant would be assigned to the Construction Order without recalculating the Master BOM.
