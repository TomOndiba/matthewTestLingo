Separate Delivery Processing

To arrange separate delivery, the output of the selected production order lines must be posted. After posting, click Navigate > Separate Delivery on the production order. A transfer order for the separate delivery lines is generated, which transfers the assembly units from the production inventory to the customer’s location.

: Transfer Order for Separate Delivery components

This transfer order can be shipped and received simultaneously. If the transfer order is posted and a user runs the “Separate Delivery” function from the production order, then a history dialog box appears with the posted transactions.

: Posted delivery transaction history

In the Prod. Order Components page, the delivery status is available in the Transfer Created and Transfer Posted fields. These fields indicate whether the transfer order was already created and whether it was posted. The Old Location Code field shows from which location the components were taken.

: Status of the Delivery

To access a report that displays the delivered components, open the Location List page, select the customer location, and then click Navigate > Inventory.

: Report showing separately delivered inventory by location

If the item is installed and finished at the customer location, the consumption of the first production order line must be posted. Because this consumption will be posted with the customer’s location code, this location will be empty.
