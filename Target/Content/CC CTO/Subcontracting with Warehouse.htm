Subcontracting with Warehouse Management

When Warehouse Management functionality is used with a location from where enclosed subcontracting components must be transferred to the vendor location, it is possible to automatically create and/or release Warehouse Shipments when the subcontracting Transfer Order is released.

The setting is defined in the Create Enclosure Whse. Shpt. field on the Manufacturing FastTab of the Configure to Order Setup page. The options are:

  • Create Whse. Shpt. Open – Creates the Warehouse Shipment with Status = Open
  • Create Whse. Shpt. Released – Creates the Warehouse Shipment with Status = Released

When the Warehouse Shipment is created, the “Autofill Qty. to Ship” function is automatically run so the shipment lines are ready for posting.

Example – Create a Warehouse Shipment with Subcontract Order

In this example, the Create Enclosure Whse. Shpt. field is set to Create Whse. Shpt. Released and the Require Shipment check box is selected in the RED location.

A sales order and construction order are created for 1 unit of item D-9005 from the RED location, where warehouse management is required. After the construction order is released, a production order is created and the Subcontracting Prod. Order page is opened. The “Create Subcontracting Worksheet” function is run and then the “Create Purchase Order” function is run.

When the Transfer Order is opened and released, the released Warehouse Shipment is created.

: Released Warehouse Shipment automatically created when the Transfer Order is released

The Status is set to Released and the Qty. to Ship field is automatically set to the order quantity. The Warehouse Shipment can now be posted and the remainder of the subcontracting process can continue.

Note

If the Transfer Order is not shipped prior to release, the message “There is nothing to post.” may appear when the Warehouse Shipment opens because the Warehouse Shipment must be posted before the Transfer Order.
