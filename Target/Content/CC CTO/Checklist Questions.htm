Checklist Questions

Checklist Questions are set up on the cc|configure to order menu by clicking Checklists > Checklist Questions.

: Checklist Question Card, General FastTab

With the selection of a checklist question number into a Checklist Template, the information entered here is copied into the template.

General FastTab

The fields on this FastTab are described in the following table.

Field

Description

Code

Unique identifier of the question that can contain up to 20 characters.

Description

Designation of the checklist question. This description appears in the checklist as the user question. The description can be a maximum of 50 characters.

Translation Code

Contains the translation code generated from the “Find/Create Translation” function. Refer to topic “Checklist Translations” on page 1 for more information.

Language

Default language of the checklist question or the translated language that is available for the question if the “Find/Create Translation” function is run; used with checklist translations (refer to topic “Checklist Translations” on page 1).

Description (T)

Contains the translated description if translations are used. Refer to topic “Checklist Translations” on page 1 for more information.

Comment

In this field, additional remarks can be entered to the question up to a maximum of 80 characters.

Variable for Dynamic BOM

The answer to the question (in the Input field in the checklist) can be stored as a variable for further use in the checklist or in the dynamic BOM. The variables have a global validity within the checklist and the Master BOMs.

Sub-Checklist Template Code

If necessary, alternative checklists can be assigned to a question or an answer. In this field, the code of the sub-checklist is entered (refer to topic “Example – Sub-Checklists” on page 1).

Description Color

Defines which color the field description is to have. The options are black, blue, red [italicized], green [bolded] and yellow.

Description Bold

If selected, the text in the Description field is bolded on the checklist.

Note

This field has no style influence when Description Color = yellow (never bolded) or green (always bolded).

More Answers Possible

If selected, several answers are possible for the question, but only if the Type field for the answer is set to Answer.

Mandatory Input

If selected, the user must fill in the Input field on the question line.

When a Master BOM is opened in a Document BOM, the system validates that the mandatory fields are filled in. If the Input field is blank on a mandatory line, an error message is generated.

Picture

Displays an uploaded picture for the Checklist Question; uses standard Microsoft Dynamics NAV picture insertion functionality.

When questions are added to a Checklist Template/Document Checklist line, assigned pictures are copied to the Checklist Picture FactBox and Picture field in the Lines FastTab.
