Config. to Order FastTab

The fields on this FastTab are described in the following table.

: Item Card, Config. to Order FastTab

Field

Description

Description 2 –Description 5

Additional description fields, each hold up to 50 characters.

Main Location Code

Main location code of the item. If a location code is filled in, the system will copy it when the item is inserted into the following documents:

  • Master BOM Line
  • Document BOM Line
  • Construction Order Line
  • Sales Line (Quotes, Orders, etc.)
  • Purchase Line (Quotes, Orders, etc.)
  • Production Order Components
  • Service Item Worksheet
  • Requisition and Planning Worksheet

Part Type

It is necessary to mark the part type when the parts have to be manufactured specifically for this order (Production) and to be included in supply planning (Standard or Purchase). The options are:

  • Standard – The BOM of this assembly group are for informational purposes only. In the Document BOM, the Information Line check box is selected. The components of this assembly group are not included in supply planning.
  • Production – This option is only possible for assembly groups. If an item is set up with this option, the Replenishment Method changes automatically to Prod. Order. With this option, the components of this assembly group are used in supply planning. In the Document BOM/Construction Order line, the Information Line check box is selected.

Note

For the Production part type, the item must have a Master BOM. When using the standard Microsoft Dynamics NAV production BOM, this option is irrelevant.

  • Purchase – Same as Part Type = Standard except that if the Status of a Document BOM and/or Construction Line is changed to Released, a new Variant Code gets inserted to the Variant Code field in the BOM line.

Separate Prod. Order

If selected, the system will create a separate production order for this item.

Spare Part

Determines whether the item is a spare part. This information is retained in the master and Document BOMs. In the service item, parts list components with this identification can be copied into the service item components.

Wear Part

Determines whether the item is a wear part. This information is retained in the master and Document BOMs. In the service item, parts list components with this identification can be copied into the service items components.

Base Material

Identifies the assignment of a Base Material to the item. The Base Material will be used for the work package (refer to topic “Work Packages” on page 1).

Separate Delivery

Indicates whether this item is set up for separate shipment. This setting is copied from the Item Card to the BOM and the production order where it can be changed as needed.

Full Text

Displays the complete text of all description fields (1–5) by joining these values together. This field is not editable.  

No. of Change Orders

Indicates the number of Changes Orders that exist for the item (refer to topic “Change Order ” on page 1).

Drawings

Indicates if the item has been assigned a design (refer to topic “Item Drawing” on page 1).

Discontinued Item

Indicates if this is a discontinued item (refer to topic “Discontinued Items” on page 1).

Replacement possible from

If a date is entered in this field, the replacement of the discontinued item will occur by this date, at the earliest (refer to topic “Discontinued Items” on page 1).

Substitutes Exist

Indicates if a substitute item has been assigned to this item using standard Microsoft Dynamics NAV substitution functionality. A replacement must be assigned to use the discontinued item functionality (refer to topic “Discontinued Items” on page 1).

Prototype

Indicates if this is a prototype item (refer to topic “Example – Creation of a Prototype” on page 1).

Prototype Type

Identifies the type of prototype that the program will use when it creates a new prototype item. The options are:

  • Item – Select this option if the item will be copied into a new prototype item.
  • Variant – Select this option if a new item variant should be created only. 

Refer to topic “Example – Creation of a Prototype” on page 1 for more information.

Prototype No. Series

Identifies the number series used for items and/or variants created from this prototype item (refer to topic “Example – Creation of a Prototype” on page 1).

Created from Prototype

Identifies the originating prototype item number that this item was created from (refer to topic “Example – Creation of a Prototype” on page 1).

Prototype BOM Type

Indicates if the prototype was created from a Document BOM or Construction Order (refer to topic “Example – Creation of a Prototype” on page 1).

Prototype BOM No.

Identifies the Document BOM or Construction Order number from which the prototype item was created (refer to topic “Example – Creation of a Prototype” on page 1).

KANBAN

Indicates if this item has been set up for Kanban processing (refer to topic “Item Setup” on page 1).
