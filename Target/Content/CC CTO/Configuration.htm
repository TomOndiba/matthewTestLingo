Configuration

The configuration feature of cc|configure to order consists of two substantial components:

  • Checklists
  • Dynamic Master BOMs

The checklists and Master BOMs can be used independently; however, a complete configuration is possible only in combination with both.

The main principle of the configuration feature is that in the first step of the configuration process – with the help of the checklist – an order is defined. The checklist is the interface between users and system. Since the checklist can provide its own logic, it ensures that only information examined and verified by the user is available for configuration.

The values entered in the checklist are stored as variables. In the second step, the Master BOM is connected with the checklist. In the Master BOM, the same variables that were entered in the checklist are addressed. The Master BOM works according to the principle of the maximum BOM, which combines all valid options in a configuration. During the BOM calculation, the variables use formulas and rules to determine quantities, setup and run time, and if positions are valid.

A highlight of the configuration feature of cc|configure to order is that the formulas defined for the rules have an easy-to-learn syntax. The configuration feature can be used without formulas; therefore, it is possible to create simple BOMs without formulas and checklists. In addition, checklists can be used without Master BOMs, for example, for creating lists of questions and answers to document the quality of the delivered items in a purchase order.

Thus the application can be adapted within the range of the configuration to the needs of the users.
