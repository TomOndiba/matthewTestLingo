Checklist Answers

Checklist Answers are set up on the cc|configure to order menu by clicking Checklists > Checklist Answers.

: Checklist Answer Card, General FastTab

With selection of a checklist answer number in a Checklist Template, the information entered here is copied into the template.

General FastTab

The fields on this FastTab are described in the following table.

Field

Description

Code

Unique identifier of the answer that can contain up to 20 characters.

Type

Definition of the kind of answer. The options are:

  • Answer – Assigned to a question but no further dependence between question and answer exists. If the More Answers Possible check box is not selected in the checklist question, only one answer for each question can be selected.
  • Selection Option – Assigned to a question and normally do not have their own variables. The value of a selection option is entered in the Input field of the question (refer to topic “Variable/Attribute Types” on page 1 for additional information about using this checklist answer type).

Description

Designation of the checklist answer. This description appears in the checklist. The description can be a maximum of 50 characters.

Translation Code

Contains the translation code generated from the “Find/Create Translation” function. Refer to topic “Checklist Translations” on page 1 for more information.

Language

Default language of the checklist answer or the translated language that is available for the answer if the “Find/Create Translation” function is run; used with checklist translations (refer to topic “Checklist Translations” on page 1).

Description (T)

Contains the translated description if translations are used. Refer to topic “Checklist Translations” on page 1 for more information.

Value for Input

This field is only relevant with a Type of Selection Option.

With this value, further calculation is then accomplished in the checklist. So the question about a color can be assigned the answer, for example, blue = value 1.

The assigned value appears in the Input field of the checklist question. If no value is assigned, the description of the answer in the Input field is assigned.

Input Description

Additional description of the answer with a maximum of 80 characters.

Input

Default value for an answer. This field is not valid when the Type field is set to Selection Option.

Input Type

Definition of the data type that is expected of an answer. The options are:

  • Text
  • Time
  • Decimal
  • Boolean
  • Date
  • Integer

Selected

Default value for an answer. This field is only valid when the Type field is set to Answer and the Input Type field is set to Boolean.

Comment

In this field, additional remarks can be entered to the question up to 80 characters.

Sub-Checklist Template Code

If necessary, an answer to a question can be assigned to a sub-checklist. In this field, the code of the sub-checklist is registered (refer to topic “Example – Sub-Checklists” on page 1).

Variable for Dynamic BOM

The answer to the question (in the Input field in the checklist) can be stored for further use in the checklist or in the dynamic BOM as a variable. The variables have a global validity within the checklist and the Master BOMs.

Rounding Precision

In this field, the exact rounding of the input number can be determined.

Rounding Method

Indicates how the input number is to be rounded. The options are:

  • Commercial
  • Round off
  • Round out

Low/Upper Limit

In these fields, assign a range of values to an answer.

Description Color

In this field, the color of the text in the Description field is specified. The options are black, blue, red [italicized], green [bolded] and yellow.

Description Bold

If selected, the text in the Description field is bolded on the checklist.

Note

This field has no style influence when Description Color = yellow (never bolded) or green (always bolded).

Mandatory Input

If selected, then input is required on this answer line.

Item Category Code

This field is used when a checklist answer is made with a Component Type of Item. Use the arrow in the Input field for this answer line to see and select from the items that belong to this item category.

Picture

Displays an uploaded picture for the Checklist Answer; uses standard Microsoft Dynamics NAV picture insertion functionality.

When answers are added to a Checklist Template/Document Checklist line, assigned pictures are copied to the Checklist Picture FactBox and Picture field in the Lines FastTab.
