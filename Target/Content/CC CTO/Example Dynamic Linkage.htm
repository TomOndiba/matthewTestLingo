Example – Dynamic Linkage of Items

During the dynamic linkage of items, a dummy or a substitute symbol item in the Dynamic Master BOM is replaced by an item that is selected in the checklist.

In the Master BOM, item D-999999 is registered as the dummy or substitute symbol item. This item must be set up with the COMP attribute, which matches the default attribute defined in the Variable for Component field in the Configure to Order Setup page (refer to topic “Dynamic BOM FastTab” on page 1). The attribute was set up on the item in the Master BOM line by clicking the Lines FastTab > Line > Attributes. In the Formula Value field, the COLOR variable is registered, thus creating the link to the checklist line.

: Functionality for Dynamic Linkage of Item in the Master BOM Template

In the Checklist Template, the corresponding line is set up with Input Type = Text, Variable for Dynamic BOM = COLOR, Component Type = Item and Item Category Code = COLOR.

: Dynamic Linkage of Item in the Checklist Template setup

Based on these settings, in the document checklist, the Input field on the “Select standard color” line will contain an arrow that opens a list of items that are set up with the Item Category Code of COLOR. The item selected is stored in the COLOR variable and will later transfer to the Document BOM.

: Item lookup in document checklist

Now, when the Document BOM is opened, item D-999999 is replaced by the item selected in the checklist (in this example, item 7003 “Ruby Red”).

: Result of the dynamic item linkage in the Document BOM
