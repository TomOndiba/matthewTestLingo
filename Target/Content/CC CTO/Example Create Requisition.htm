Example – Create Requisition Line for G/L Account from Document BOM

A Sales Order and Document BOM are created for 1 unit of item D-1000. On the last line of the Document BOM, a line is inserted for G/L Account = ACCOUNT, Exp. Quantity = 1 and Unit Cost (CC) = 100.

: Document BOM with G/L Account line

Click Actions > Purch. G/L Acc./Item on the ribbon.

: Purchase G/L Account function request page

With the Requisition Line option selected, click OK and then select the appropriate requisition worksheet from the Req. Wksh. Names page.

: Req. Wksh. Names page

A confirmation message is generated and on the G/L Account line:

  • Purchase Exists = selected
  • Purchase Status = Purchase Created

: Document BOM with G/L Account requisition line created indicated

If the “Purch. G/L Acc./Item” function is run again, the following message is generated.

: Message to open requisition line

Click Yes to open the Requisition Lines page and then click Navigate > Show Worksheet to open the Req. Worksheet page.

: Requisition Worksheet with G/L Account purchase from Document BOM

Run the “Carry Out Action Message” function, go back to the Document BOM page and run the “Purch. G/L Acc./Item” function again. Click Yes to open the Purchase Lines page and then open the Purchase Order. The Purchase Order has been created for the G/L Account for the Unit Cost entered on the Document BOM line.

: Purchase Order created from the Requisition Worksheet with G/L Account purchase from Document BOM

Note

If the value in the Direct Unit Cost Excl. VAT field is changed prior to posting the Purchase Order, the new amount will transfer to the Document BOM line.

Post the Purchase Order, go back to the Document BOM and notice on the G/L Account line:

  • Purchase Exists = not selected
  • Purchase Status = Purchase Posted

: Document BOM line updated with Purchase Order posting
