Construction Order, Actions Tab

The following menu items appear within the Function and Posting groups on the Actions tab of the Construction Order ribbon.

  • Request BOM: Enables the complete master BOM of the item in the construction header to be opened. However, to do this all the construction lines must be deleted first. If the configuration feature is being used, the entries in the document checklist are considered (refer to topic “Configuration” on page 1).
  • Schedule: With this function, scheduling occurs based on the construction order. In this case, scheduling considers all the times for resources, work centers and groups of work centers entered in the construction order. The items are always scheduled so that they must be available to an assembly at the start of the first series of operations. Limited capacities are taken into consideration. The following types of scheduling are possible:
    • Forward – Here scheduling of the construction order is performed for the future starting at the start date. The Starting Date field is available in the construction header and is transferred from the sales order. It can, however, also be entered manually.
    • Backward – Here scheduling of the construction order is performed backward starting at the end date. The Ending Date field is available in the construction header and is transferred from the sales order. It can, however, also be entered manually.
  • Transfer Due Date in Shipment Date: This function enables the scheduling result to be transferred into the Shipment Date on the construction lines.
  • Release: Changes the status of the construction order to Released. Lines with this status are relevant for requirements if this is defined in the Configure to Order Setup page. Additionally, a variant code will be created automatically for released lines with Part Type = Purchase.
  • Reverse Status: Changes the status of the construction order from Released to Open.
  • Assign New Line No.: This function enables the lines of the construction order to be renumbered. This is necessary when no further new lines can be added to the construction order using the “Auto Split Key” function of Microsoft Dynamics NAV.
  • Order Promising: This function enables, for example, critical parts or long-running items to be planned in advance. This function is based on the standard order promising functionality. The system calculates the earliest possible procurement date. If you accept the proposal, planning proposals are generated immediately which are connected with the document BOM line or the construction line via the reservation management of Microsoft Dynamics NAV.
  • Update Unit Cost & Price: Updates the unit cost and unit price of lines (Type <> G/L Account, Purchase Status <> Purchase Posted) with the current costs and prices. Refer to topic “Example – Update Unit Costs/Prices in CTO BOMs” on page 1 for more information.
  • Reserve: Enables reservations to be made manually (e.g. from the warehouse stock).
  • Item Tracking Lines: Here item tracking can be assigned to components that are subject to management in batches or by serial numbers; standard Microsoft Dynamics NAV functionality.
  • Create Prod. Order: Creates a production order for the entire Construction Order. If the production order is generated here, there are two options:
    • Create Prod. Order without Variants – This function creates a production order for the overall construction order without creating variant codes, production BOMs, or routings.
    • Create Prod. Order with Variants – This function creates a production order based on the construction order. The system generates a variant for all production items with the Part Type field set to Production and generates a manufacturing routing and a manufacturing BOM. The automatically generated production BOMs and routings are linked to the item variant.

Whether the system creates production orders with or without variant codes by default is defined on the Configure to Order Setup page (refer to topic “Manufacturing FastTab” on page 1).

When the production order is generated for the Create Prod. Order with Variants option, the following steps are performed by the system:

  1. Generation of one variant per production item with the Part Type field set to Production.
  1. Creation of a manufacturing BOM using the default number series (refer to topic “Manufacturing FastTab” on page 1).
  1. Creation of a manufacturing routing using the default number series (refer to topic “Manufacturing FastTab” on page 1).
  1. Assignment of the manufacturing BOM and the routing on the Item Variants table.
  1. Creation of the production order based on the BOM entered and the routing.
  • Transfer Changes to Prod. Order: Transfers changes that were executed after the creation of a production order from the construction order. This function compares the construction order lines with the production lines, production components, and production routings. Lines for transfer to the production order are displayed in the Changes Prod. Order Constr. page for the following modifications:
    • Changed quantities for components or times for routing lines
    • Deletion of components/routing lines
    • Insertion of new components/routing lines

As illustrated in Fig. 1, four changes were made in the Construction BOM: the quantity was changed for item E1011, item E1023 was changed, work center 200 was deleted and machine center 340 was added.

Additionally, inserting machine center 340 between other routing lines (without filling out any of the operation number fields) updated and recalculated the fields, causing work center 300 to be deleted and reinserted with the new operation codes.

: Changes to be transferred to the existing production order

If new lines were added, then the Line Inserted check box is selected, and if a line was deleted, then the Line Deleted check box is selected.

The user can decide which changes can be transferred to the production order by clearing the Transfer Change check box for the lines not to transfer.

By clicking OK, the selected changes are transferred to the production order. A message appears that indicates the number of lines that were transferred to the production order.

Note

This function cannot change the BOM structure of the Production Order. It can only transfer changes of components and routing lines – not the deletion or insertion of items that have assemblies.

  • Transfer Back to Service BOM: Updates the Service BOM if lines were changed in a construction order that was created from a Service Order (refer to topic “Create Construction Order from Service Item Worksheet” on page 1).
  • Archive Construction: Creates an archived version of the construction order. The archived versions can be opened from the construction order header by clicking the arrow in the No. of Archived Versions field.
  • Switch Const.-/Prod. View: Switches between the construction view and the production view (refer to topic “Example – Production View” on page 1).
  • Archive and Finish: Archives the current construction order and deletes the current version. Normally this function is used when the construction order has been completed.
