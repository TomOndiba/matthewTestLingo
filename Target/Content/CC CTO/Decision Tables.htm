Decision Tables

Decision Tables can be used to set up a big number of value combinations within one table and use these values in all formulas of cc|configure to order. Using the function GETVALUE, the user can extract the information from the decision table.

: Decision Table with no filtering

The fields on the General FastTab of the Decision Table page are described in the following table.

Field

Description

Code

Unique identifier for this decision table. This code will be used as first parameter of the dynamic function GETVALUE.

Description

Description of this decision table.

Status

Status of this decision table. The options are:

  • New – The decision table is new and the user can edit the lines.
  • Certified – The decision table is certified and the lines are not editable. Only this table can be used in dynamic formulas.
  • Under Development – The decision table is under development and the user can edit the lines.
  • Closed – This decision table is no longer used.

Last Status Modified Date

Date when the status was last changed.

Last Status Modified User

The ID of the user who last changed the status.

Last Date Modified

Date when the header was last changed.

Last User ID Modified

The ID of the user who last changed the header.

The fields on the Lines FastTab of the Decision Table page are described in the following table.

Field

Description

Search Sequence

Indicates the search order of the lines to be used if more than one line matches all filtering conditions.

Value Code

Value Code to which the decision table line includes, e.g. PRICE. This will be used as second parameter in function GETVALUE.

Filter Dim 1 is alphanum.

Determines the type of value that can be entered in the Dimension 1 field:

  • If selected, an alphanumeric value can be entered.
  • If not selected, only numeric values can be entered (e.g. 10..20 is possible, but BLUE|GREEN will get an error).

Dimension Value 1

First Dimension to identify the right value, e.g. LENGTH. This dimension will be used as third parameter in function GETVALUE.

Filter Dim 2 is alphanum.

Determines the type of value that can be entered in the Dimension 2 field:

  • If selected, an alphanumeric value can be entered.
  • If not selected, only numeric values can be entered (e.g. 10..20 is possible, but BLUE|GREEN will get an error).

Dimension Value 2

Second Dimension to identify the right value, e.g. WIDTH. This dimension will be used as forth parameter in function GETVALUE.

Filter Dim 3 is alphanum.

Determines the type of value that can be entered in the Dimension 3 field:

  • If selected, an alphanumeric value can be entered.
  • If not selected, only numeric values can be entered (e.g. 10..20 is possible, but BLUE|GREEN will get an error).

Dimension Value 3

Third Dimension to identify the right value, e.g. DEPTH. This dimension will be used as fifth parameter in function GETVALUE.

Value

Indicates the value that the function GETVALUE gives back as return value.

If the Decision Table has been set up like in Fig. 1 and the function GETVALUE is used with the following parameters: GETVALUE(DT00000;PRICE;100;250;350), the result of this function will be 18. If you only need two dimensions, leave the Dimension Value 3 field empty and use only four parameters in the function GETVALUE. If the system cannot find a valid decision table line, the return value will be 0.
