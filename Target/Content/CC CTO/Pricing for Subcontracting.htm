Pricing for Subcontracting

Production companies negotiate prices with subcontractors for producing an item. The price a company has to pay depends on the quantity of the item or on the time the subcontractor needs (e.g. hours or days). It is also possible to grant discounts depending on a special quantity.

To accommodate the special pricing needs for subcontracting, purchase prices and line discounts from standard Microsoft Dynamics NAV can be added to work centers for calculation in the Subcontracting Worksheet. The prices can be set for units or times (based on the Unit Cost Calculation from the Work Center Card).

Pricing by Time

Pricing by time means that capacity is bought by a subcontracting vendor. The amount of time it will take the subcontractor to complete the work is defined in the Routing plan. The Unit of Measure Code is normally defined on the Work Center Card.

In the Work Center Card, the Unit Cost Calculation is set to Time. With this setting, an entry for the Unit of Measure of the Run Time has to be available.

: Work Center Card, Unit Cost Calculation set for pricing by time

When calculated in the Subcontractor Worksheet, the time calculation is based on the Unit of Measure of the Run Time.

On the Navigate tab, click Prices to open the Purchase Prices page. The page is filtered by the Subcontractor No. entered in the Posting FastTab of the Work Center Card.

: Purchase Prices for time set up for Work Center 901

Pricing for times are entered in the Purchase Prices page like in standard Microsoft Dynamics NAV. For a vendor, various items can be set up with different prices for different quantities. As illustrated in Fig. 2, vendor 65000 has been set up with two different hourly prices for item D-9005: if it takes between 1-9 hours to produce item D-9005, vendor 65000 charges 70.00 per hour; if beyond 9 hours, they charge 60.00 per hour.

It is also possible to assign line discounts by clicking Line Discounts on the Navigate tab.

: Line Discounts set up for Work Center 901

The line discounts are also entered the same as in standard Microsoft Dynamics NAV. Various items for a vendor are set up with different discounts based on different quantities.

For this example, a routing has been assigned to item D-9005 with Work Center 901, which has a run time of 10 hours.

: Routing for item D-9005 with Work Center 901

When a released production order is created for 1 unit of item D-9005, the routing uses the base pricing from the work center, which was 45.00 direct unit cost.

: Production Order Routing with base pricing from work center

When the Subcontractor Worksheet is created, the special purchase pricing is used for vendor 65000, since it was used as the routing link in the routing.

: Special pricing and line discounts for vendor 65000

As illustrated in Fig. 6, the Direct Unit Cost of 600.00 with a 1.25% Line Discount is used because the run time of Work Center 901 is 10, and the price beyond 9 hours was 60.00 with a discount of 1.25 for vendor 65000.

When the “Create Purchase Order” function is run, the cost from the Subcontracting Worksheet, not the routing, is transferred to the purchase line.

Pricing by Unit

Pricing by unit means that special pricing per item per vendor is set up based on the number of units that the subcontractor will produce.

In the Work Center Card, the Unit Cost Calculation is set to Units. With this setting, an entry for the Unit of Measure Code of the item must be available.

: Work Center Card, Unit Cost Calculation set for pricing by units

When calculated in the Subcontractor Worksheet, the time calculation is based on the Unit of Measure of the item.

On the Navigate tab, click Prices to open the Purchase Prices page.

: Purchase Prices for units set up for Work Center 901

Pricing for units are entered in the Purchase Prices page like in standard Microsoft Dynamics NAV. However, the Unit of Measure Code must exist for the item and is entered manually.

For a vendor, various items can be set up with different prices for different quantities. As illustrated in Fig. 9, vendor 65000 has been set up to charge 1500.00 per one unit of item D-9005.

It is also possible to assign line discounts by clicking Line Discounts on the Navigate tab.

: Line Discounts set up for Work Center 901

The line discounts are also entered the same as in standard Microsoft Dynamics NAV. Various items for a vendor are set up with different discounts based on different quantities.

As shown in the example for pricing by time, a routing has been assigned to item D-9005. When a released production order is created for 1 unit of item D-9005, the routing still uses the base pricing from the work center, which was 45.00 direct unit cost.

: Production Order Routing with base pricing from work center

When the Subcontractor Worksheet is created, the special purchase pricing is used for vendor 65000, since it was used as the routing link in the routing.

: Special pricing and line discounts for vendor 65000

As illustrated in Fig. 12, the Direct Unit Cost of 1500.00 with a 1.00% Line Discount is used because one unit of item D-9005 was entered in the released production order for vendor 65000.

When the “Create Purchase Order” function is run, the cost from the Subcontracting Worksheet, not the routing, is transferred to the purchase line.

: Purchase Order with unit-based pricing for subcontractors

Pricing with Foreign Currency

Pricing for foreign currency is handled with standard Microsoft Dynamics NAV functionality in the Purchase Prices and Purchase Line Discounts pages through the use of the Currency Code field.
