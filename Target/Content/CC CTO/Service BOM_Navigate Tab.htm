Navigate Tab

The following menu items appear on the Navigate tab on the ribbon.

  • Create Checklist: Opens a function to create a checklist for the service BOM.
  • Show Checklist: Opens the checklist that is specified in the Checklist No. field on the header.
  • Structure Representation: Displays the Service BOM in a tree-structure where the user can expand or collapse the different levels.
  • Request BOM: Opens the master BOM of the item that is registered in the service BOM header and generates the entire BOM in the Service BOM lines.
  • Assign New Line No.: Renumbers all the lines in the Service BOM without losing the BOM structure. This is used when there is no longer space available for line insertion between existing lines.
  • Switch Const.-/Prod. View: Switches between the construction and the production view (refer to topic “Example – Production View” on page 1).
  • Copy BOM: Copies a processed BOM into the Service BOM structure in the following order:
    • If the Production Order No. field is populated on the Service Item Card, the system suggests this BOM first.
    • If the Production Order No. field is empty, but the Construction Order No. field is populated, then this BOM is suggested by the system.
    • If the Production Order No. and Construction Order No. fields are empty, but the Document BOM No. field is populated, then this BOM is suggested by the system.

It is also possible to copy from other service BOMs, construction orders, document BOMs, and production orders. Refer to topic “Copy Deleted Source Documents to Service BOM” on page 1 for more information about the “Copy BOM” function.

If the Quantity in the Sales Line/Construction Order/Production Order is more than 1 and components use Serial Nos., a dialog will appear to choose the Serial Nos. that are used in the current Service Item. Additionally, when a Production Order is copied to a Service BOM, and Prod. Order Components use Serial Nos., a Service BOM Line is created for each one and populates the Serial No. field accordingly.
