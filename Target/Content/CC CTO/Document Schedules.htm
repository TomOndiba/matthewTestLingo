Document Schedules

Once a schedule is selected for use, the Document Schedule page opens, where the schedule is manually managed.

: Document Schedule with planned and completed operations

The Document Schedule page will open each time the “Document Schedule” function is run from the sales/purchase line.

If a Contact No. is assigned to a line in the Document Schedule , the “To-dos” and “Create Interact” actions become available in the ribbon, which create a connection to Marketing capabilities for that specific line.

In the Planned Starting Date and Planned Ending Date fields, enter the planned date range for each milestone. In the first Completion % field, enter the percentage of which the milestone is completed. The next Completion % field provides a graphical representation of completion.

In the Actual Starting Date and Actual Ending Date fields, enter the actual dates that the milestone was performed. When values are entered in either of these fields, the Expected Starting Date and Expected Ending Date fields are calculated based on the actual dates.

In the ribbon, the “Forward” and “Backward” actions can be used to change the scheduling method. The “Show Document” action opens Sales Lines/Purchase Lines page, which shows the document to which the selected line is related. Additional information about the sales/purchase line is available and the sales/purchase document can be opened from these pages.

Document Schedules in Document Lines

Fields in the sales/purchase order lines indicate whether a document schedule exists for the line, as well as the maximum planned and expected ending dates that show the latest dates from the linked Document Schedule, which helps identify if there is a date conflict.

Document Schedule Sales/Purchase

All Document Schedules that exist for sales and purchase documents can be viewed in the Document Schedule Sales and Document Schedule Purchase pages, respectively.

All functions available in the respective document orders are available in the Document Schedule Sales/Purchase pages when the “Edit List” action is activated.
