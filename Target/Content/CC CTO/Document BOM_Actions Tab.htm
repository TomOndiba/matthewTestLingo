Actions Tab

The following menu items appear on the Actions tab of the Document BOM ribbon.

  • Request BOM: Opens the master BOM for selection if any are assigned to the item. The following page appears.

: Request of the Master BOM and/or Production BOM and Routing

Here the user can decide whether only one or all levels need to be opened. If the item is also assigned to a production BOM or a routing, these can be opened instead of the master BOM by selecting the Production BOM and/or the Routing check boxes. The BOM is assigned to the sales line as the document BOM.

When the configuration feature in cc|configure to order is used, the BOM is calculated using formulas when it is opened and configured on an order-related basis (refer to topic “Dynamic Master BOM” on page 1).

  • Request BOM from Line: Opens the Master BOM for the component in a document BOM line, if it has its own master BOM.
  • Schedule: With this function, scheduling occurs based on the document BOM. In this case, scheduling considers all the times for resources, work centers and groups of work centers. The items are always scheduled so that they must be available to an assembly at the start of the first series of operations.

Limited capacities are taken into consideration.

: Scheduled Document BOM

  • The following types of scheduling are possible:
    • Forward – Here scheduling of the document BOM is performed for the future starting at the start date. The Starting Date field must be completed manually.
    • Backward – Here scheduling of the document BOM is performed backward starting at the end date. The Ending Date field must be completed manually.
      Example: Backward scheduling is used in Fig. 2. The end date is 3/26/2019. Accordingly, all components marked yellow must be available at the start of the series of operations marked yellow. The series of operations have a processing time of 16 hours. The Front Wheel assembly is therefore available for the subsequent series of operations on 3/25/2019. That is also the due date for item 1900, Frame.
  • Transfer Due Date in Shipment Date: Enables the schedule result to be transferred to the Shipment Date field on the document BOM lines.
  • Release: Changes the status of all Document BOM lines to Released. Additionally, a variant code will be created automatically for released lines with Part Type = Purchase.
  • Reopen: Changes the status of all Document BOM lines back to Open.
  • Cut: Enables individual items, multiple items and complete structures from the same or a different document BOM to be cut and copied to the clipboard.
  • Copy: Enables individual items, multiple items and complete structures from the same or a different document BOM to be copied to the clipboard.
  • Paste: Enables individual items, multiple items and complete structures that were copied to the clipboard from the same or a different document BOM to be inserted here.
  • Assign New Line No.: Enables the document BOM to be renumbered. This is necessary when no further new lines can be added to the document BOM using the “Auto Split Key” function of Microsoft Dynamics NAV.
  • Order Promising: Enables, for example, critical parts or long lead time items to be planned in advance. This function is based on the standard order promising functionality. The system calculates the earliest possible procurement date. If you accept the proposal, planning proposals are generated immediately which are connected with the document BOM line or the construction line via the reservation management of Microsoft Dynamics NAV.
  • Update Unit Cost & Price: Updates the unit cost and unit price of lines (Type <> G/L Account, Purchase Status <> Purchase Posted) with the current costs and prices. If something has really changed, the Changed field will be set to TRUE while the page is open. Refer to topic “Example – Update Unit Costs/Prices in CTO BOMs” on page 1 for more information.
  • Reserve: Enables reservations to be made manually (e.g. from the warehouse stock).
  • Reservation Entries: Displays the reserved quantity of reservation items that have been generated for this position (e.g. via the date of promised delivery).
  • Purchase G/L Acc./Item: Creates a purchase quote/order or requisition line for a component or against a G/L account for an item that does not yet exist for demand and normally would never be created (refer to topic “Create Purchase Documents for G/L Account and Components” on page 1).
  • Structure Representation: Displays the document BOM in a tree-structure where the user can expand or collapse the different levels.
  • Switch Const.-/Prod. View: Switches between the construction and the production view (refer to topic “Example – Production View” on page 1).
  • Delete Release: Enables the document BOM lines that were marked via the delete reservation of the PDM interface to be released for deletion in Microsoft Dynamics NAV. Further information on the ProfiDB interface is provided in the separate interface description document.
  • Create Change Order: Creates an order to notify the design department of change suggestions from the Document BOM (refer to topic “Change Order ” on page 1).
  • Open Change Order: Opens the Change Orders page from where existing change orders for the Document BOM can be viewed (refer to topic “Change Order ” on page 1).
  • Transfer Comp. to Sales Document: Transfers all Document BOM Lines with Transfer to Sales Document = Yes to the current Sales Quote/Order lines. If the marked component is an assembly group, a new Document BOM (with the copied assembly group) will be assigned to the new Sales Line. The source Document BOM Lines are deleted after transferring to the Sales Lines. Additionally, existing reservations are transferred to the target lines.

Refer to topic “Example – Transfer Components to Sales Order” on page 1 for more information.
