Example – Sub-Checklists

Sub-checklists are checklist templates that can be used, for example, when a special option in a checklist must be described more clearly, or separate due to the complexity of the checklist. The checklist is structured better and is more concise by the use of sub-checklists.

Two types of sub-checklists can be used:

  • Simple sub-checklist
  • Dynamic sub-checklist

: Setup of Sub-Checklists in the Checklist Template

Simple Sub-Checklists

With the simple sub-checklist, the Sub-Checklist Template Code field is filled in and the Input Type field is set <> Integer (refer to Fig. 1).

When a checklist that is defined with a simple sub-checklist is opened (for example on a sales order), the sub-checklist (refer to Fig. 3) is directly added, as indicated in the Sub-Checklist Exists field on the line (refer to Fig. 2). The sub-checklist can be opened from the Lines FastTab by clicking Line > Sub-Checklist or by selecting the Show Sub-Checklists check box on the General FastTab to display the sub-checklists directly in the lines (refer to topic “Show Sub-Checklist” on page 1).

Further information is then entered in the Input field of the Sub-Checklist lines.

: Simple sub-checklist line on document checklist with opened Sub-Checklist page for input

Dynamic Sub-Checklists

Dynamic sub-checklists are used if several of the same assembly groups have different configurations that need to be defined. Dynamic sub-checklists can be used instead of setting up numerous checklists with specific variables for each possible configuration.

Dynamic sub-checklists are set up in the Checklist Template by filling in the Sub-Checklist Template Code field and setting the Input Type field to Integer (refer to Fig. 1).

In the Checklist Templates for the sub-checklists, an internal counting variable “i” is available. To mark these, the syntax “#i#” is entered at the end of the description (refer to Fig. 3). When opening the sub-checklist by this syntax, the variable “i” is replaced with its contents.

: Setup of a dynamic sub-checklist

This checklist line is transferred first to the main checklist, without producing a sub-checklist. If a number between 1 and 99 is entered in the Input field on this line, then as many copies as are entered will be added (refer to Fig. 4). In the description, a counter is added (01, 02, 03 and so on).

: Dynamic Sub-Checklist

In each of these lines, a sub-checklist is inserted, which can be opened from the main checklist by clicking Line > Sub-Checklist on the Lines FastTab.

: Dynamic Sub-Checklist 01 generated from the checklist

If in the sub-checklist, the Variable for Dynamic BOM field is filled in (for example with the variable “Width”), then this is replaced automatically with “Width01”, “Width02”, “Width03”, and so on (refer to Fig. 5). If attribute texts were entered for the “Width” variable, they are then copied for the new variables. When the checklists are generated, the substitute symbol #i# (refer to Fig. 2) is replaced by the description counter (refer to Fig. 3).

Note

The system limits the number of sub-checklists to 99.

Show Sub-Checklist

Instead of entering values into the Sub-Checklist page, select the Show Sub-Checklists check box on the Checklist header to automatically show the sub-checklist lines on the Checklist page for direct entry in the lines.

: Simple Sub-Checklist and Dynamic Sub-Checklists 01–03 shown in the Checklist lines

The values entered directly on the Checklist lines are also entered in the Sub-Checklist lines, and vice versa.

: Values entered on the Checklist lines appear in the Sub-Checklist page

Clear the Show Sub-Checklist check box to hide the lines on the Checklist page.

Retain Input Values of Changed Dynamic Sub-Checklist Quantity

If the input quantity is changed on the main sub-checklist line, a message appears asking what should happen to the existing sub-checklist lines.

: Message to change sub-checklist value from 3 to 2

Depending on whether the quantity is increased or decreased, the sub-checklist lines are updated as follows:

Input Change

Message Action

Result

Decrease

No

The last sub-checklist lines are removed and the remaining lines retain the original input.

Decrease

Yes

The last sub-checklist lines are removed and the original input is deleted from the remaining lines.

Increase

No

Additional sub-checklist lines are added with zero values and the remaining lines retain the original input.

Increase

Yes

Additional sub-checklist lines are added with zero values and the original input is deleted from the remaining lines.

In this example, No is selected in the message. Therefore, Sub-Checklist 03 lines are deleted and the values for Sub-Checklists 01 and 02 are retained.

Macros: Use Formulas without Parameters

When dynamic sub-checklists are used, array variables like LENGTH01 and LENGTH02 are generated. The existing MIN and MAX function can be used with only one parameter so then the whole array of this variable will be analyzed (e.g. MIN(LENGTH) will check all variables like LENGTH00, LENGTH01, LENGTH02, ... , LENGTH99).

Additionally, the following functions are available in the Formula Editor:

  • SUM(<Var>): Add all values of the variable array (SUM(LENGTH) = LENGTH00 + LENGTH01 + ... + LENGTH99)
  • COUNT(<Var>;<Value>): Counts the elements that are equal to the value (COUNT(LENGTH;5) will count the number of all LENGTH elements (LENGTH00, LENGTH01, ...,LENGTH99) which have the value 5)

Note

The SUM, MIN, MAX and COUNT functions are available in any area that uses the Formula Editor.

In this example, a sub-checklist is created where Length measurements can be inserted.

: Dynamic sub-checklist for array function example

This sub-checklist is then assigned to a checklist for calculating length using a variety of formulas.

: Checklist Template for array function example

When this checklist is opened from a document, the user enters 3 in the Input field in the line to create sub-checklists. The LENGTH in sub-checklist lines 01, 02, and 03 is defined as 10, 10, and 5, respectively. Finally, the user enters 10 in the line for “LENGTH to be counted”.

: Document Checklist with manual input of sub-checklist and length values

After the “Recalculate” function is run, the following values are calculated based on the manual input:

  • “Sum of all LENGTH” = 25 (10+10+5)
  • “Number of LENGTH = Input” = 2 (because in sub-checklist 01 and 02 the input is 10)
  • “Maximum of all LENGTHs” = 10
  • “Minimum of all LENGTHs” = 5
