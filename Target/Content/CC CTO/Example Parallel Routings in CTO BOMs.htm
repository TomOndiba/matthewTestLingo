Example – Parallel Routings in CTO BOMs

The routing fields (Operation No., Previous Operation No. and Next Operation No.) in the Master BOM lines and the setting in the Routing Type field in the Master BOM header determine how routings are processed in cc|configure to order (refer to topic "Master BOM Lines" on page 1).

: Routing Type in header and routing fields in lines

As opposed to creating the operations on certification, the “Set Operation Codes” action on the Master BOM can be used to automatically populate the fields in the CTO BOM lines.

: Operations updated automatically on the Master BOM line using the “Set Operation Codes” function

Additionally, the Parallel Operation Code field on the CTO BOM lines provides a simplified process for defining parallel routings. For example, if Work Center 100 and 400 should be done parallel, both lines are assigned the same Parallel Operation Code, such as 1.

: Parallel Operation Code added to work center 100 and 400

When the “Set Operation Codes” function is run, the Operation No., Previous Operation No. and Next Operation No. fields will be set automatically.

: Parallel Operations updated by running the “Set Operation Codes” function

The Operation No., Previous Operation No. and Next Operation No. fields can be populated manually as well.

If some other routings should work parallel, a higher code must be used, such as 2. For example, if the next group of Work Centers (400, 100 and 200) should be done parallel, each line is assigned Parallel Operation Code “2”.

: Parallel Operation Code “2” added to work centers 400, 100 and 200

When the “Set Operation Codes” function is run, the Operation No., Previous Operation No. and Next Operation No. fields will be set automatically.

: Parallel Operations updated by running the “Set Operation Codes” function

The fields in the Master BOM line are also available in Document BOM, Construction Lines and Service BOMs. Fields populated in the Master BOM Lines are copied to Document BOM/Construction Lines/Service BOM Line; otherwise these fields are empty.

In the Document BOM and Construction Line, the fields are populated when the status is changed to Released. At the latest, these fields are calculated when the “Create Prod. Order” function is run.

The values in the Operation No., Next Operation No. and Previous Operation No. fields are copied to the Prod. Order Routing Lines.
