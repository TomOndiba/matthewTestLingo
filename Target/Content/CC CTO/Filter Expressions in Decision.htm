Filter Expressions in Decision Table Lines

It is also possible to use filter expressions like "<100" or "250..300" or "BLUE|GREEN|RED" in the Decision Table lines (used within the GETVALUE function).

With these setup options, it is possible that more than one line will match all conditions. The system will always use the first Decision Table line that matches all dimension values so the Search Sequence field can be used to indicate the proper order for the search.

Each Dimension Value… field must be marked if the filter should be used as an alphanumeric filter or as a numeric filter.

Example – Get Values with Filters

The examples in this section show how filters can be used with the GETVALUE function in the checklist lines.

As illustrated in Fig. 1, three lines are defined with mostly numeric values in the Decision Table. However, the third line contains also an alphanumeric filter for Dimension Value 1.

A Checklist Template has been defined so that different values can be entered in the three Dimension lines, which will be assigned to the variables “DIM1”, “DIM2” and “DIM3”. In the last line, the GETVALUE function will be started using the decision table DT00001, Value Code = PRICE and the three dimensions.

: Checklist Template with filtering for Decision Table

Note

If any of the dimension values are outside of the specified values in the Decision Table, no result value is returned.

In the first example, numerical values are entered in the Input field for the Dimension lines on the document checklist and then the “Recalculate” function is run to calculate the GETVALUE formula.

: Document Checklist with results from numeric values of Decision Table

The value 100 of the first line of the decision table is transferred to the last checklist line because 15 is between 10..20, 16 is between 10..30 and -7 is between -10..-5.

In the second example, alphanumeric and numeric values are entered.
