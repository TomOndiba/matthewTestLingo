Lines FastTab

Master BOMs represent the product structure as a type of resource list, therefore there is no separation between BOMs and routing.

The fields on the Lines FastTab are described in the following table.

Field

Description

Type

Identifies the type of BOM line. The options are:

  • <blank> – This option is mainly used to insert a comment in a manufacturing BOM.
  • Account (G/L) – This option allows for the selection of a G/L account.
  • Item – With this option an item from the item list can be registered as a component of a BOM.
    • If an item possesses a Master BOM, this is considered automatically and transferred to the Document BOM / Construction Line, if the Part Type field is set to Production in the item card.
    • If the Part Type field is set to Standard, the item is treated as the standard.
    • If the Part Type is set to Purchase, the item is treated as in standard Microsoft Dynamics NAV but a variant code will be created automatically when the status of a Document BOM or Construction Line is changed to Released.

If the Information Line check box is selected in the Document BOM setup, then the components from items with the Part Type field of Standard or Purchase are copied into a Document BOM.

  • Resource – With this option, resources can be selected. The use of resources in the Master BOM is useful when indirect ranges (e.g. construction) in the parts list are captured and related to the value to be considered. If no Manufacturing modules are used, resources replace the function of work centers and/or machine centers.
  • Work Center – Selection of a job for a processing step.
  • Machine Center – Selection of a group of jobs for a processing step.
  • Master BOM – Selection of another Master BOM. The components of this Master BOM will not be transferred as its own assembly group, but will be copied directly into the current BOM level.
  • Production BOM – Used with the integration to cc|process manufacturing (refer to section “cc|process manufacturing” on page 1.

Delete Status

Delete reservation, if this instruction from the PDM interface was deleted (only relevant when using a PDM system).

Position – Position 3

Reference to the design item number of a CAD design. The fields are for informational purposes only.

No.

Number of the appropriate items, resources, work center, machine center, or a G/L account.

Variant

Number of the item variant, if a variant item is selected.

Standard Task Code

Indicates the Standard Task Code for the master BOM line.

Description – Description 5

Descriptions from the Item Card. The item descriptions are copied into the Master BOM and can be changed if necessary.

Calculation Group Code

Groups items, work centers, machine centers, resources, or G/L accounts for calculation purposes (refer to topic “Calculation Groups in Order-Related BOMs” on page 1).

Routing Link Code

Connects material with a processing step. In the order completion process, the Routing Link Code is used to assign material submitted to an external manufacturing processing step.

Enclosure

Indicates material submitted for an external manufacturer (refer to topic “Subcontracting with Supply of Material” on page 1).

Enclosure Drop Shipment

Indicates material provided for external processing that must be drop shipped to the vendor.

Formula Valid

Refer to topic “Configuration” on page 1.

Formula Quantity

Refer to topic “Configuration” on page 1.

Quantity

Indicates the quantity of this component. If nothing is entered in the Formula Quantity field, then this field is used in the calculation (refer to the Formula Quantity field).

With work centers and machine centers, this field is used for the calculation of the time requirement according to the following formula:

Time requirement = Tr + Te/ Lot size * Quantity

Note

When opening the Master BOM in a Document BOM, the contents of this field are transferred to the Exp. Quantity field.

Unit of Measure Code

Quantity unit of measure from the components.

Calculation Formula

Determines how to calculate the Quantity field. The options are:

  • <blank>

Quantity = Quantity per

  • Length

Quantity = Length * Quantity per

  • Length * Width

Quantity = Length * Width * Quantity per

  • Length * Width * Depth

Quantity = Length * Width * Depth * Quantity per

  • Weight

Quantity = Weight * Quantity per 

Length

Identifies a length measurement. If this measurement should be calculated with the help of the configuration feature, it is necessary to insert an attribute with a formula. The attribute is defined on the Dyn. BOM FastTab of the Configure to Order Setup page in the Variable for Length field.

Width

Identifies a width measurement. If this measurement should be calculated with the help of the configuration feature, it is necessary to insert an attribute with a formula. The attribute is defined on the Dyn. BOM FastTab of the Configure to Order Setup page in the Variable for Width field.

Depth

Identifies a depth measurement. If this measurement should be calculated with the help of the configuration feature, it is necessary to insert an attribute with a formula. The attribute is defined on the Dyn. BOM FastTab of the Configure to Order Setup page in the Variable for Depth field.

Weight

Identifies a weight measurement. If this measurement should be calculated with the help of the configuration feature, it is necessary to insert an attribute with a formula. The attribute is defined on the Dyn. BOM FastTab of the Configure to Order Setup page in the Variable for Weight field.

Formula Quantity per

Refer to topic “Configuration” on page 1.

Quantity per

Identifies a quantity per value. If this value should be calculated with the help of the configuration feature, it is necessary to insert a formula in the Formula quantity per field.

Recalculate

Refer to topic “Configuration” on page 1.

Setup Time

Preparation time for the processing step (only relevant with the Type field set to Work Center and/or Machine Center).

Setup Time Unit of Meas. Code

Time unit, in which the setup time is planned (only relevant with the Type field set to Work Center and/or Machine Center).

Formula Setup Time

Refer to topic "Configuration" on page 1 (only relevant with the Type field set to Work Center and/or Machine Center).

Run Time

Operating time for the processing step (only relevant with the Type field set to Work Center and/or Machine Center).

Run Time Unit of Meas. Code

Time unit, in which the run time is planned (only relevant with the Type field set to Work Center and/or Machine Center).

Formula Run Time

Refer to topic "Configuration" on page 1 (only relevant with the Type field set to Work Center and/or Machine Center).

Lot Size

Lot size in which the components are manufactured, and refers to the run time (only relevant with the Type field set to Work Center and/or Machine Center).

Part Type

The part type of the item (does not apply to capacity positions and resources). The part type defaults from the item master and can be modified here on a BOM basis.

Spare Part

Spare part setting from the Item Card. If necessary the setting can be changed here.

Wear Part

Wear part setting from the Item Card. If necessary the setting can be changed here.

Location Code

Main warehouse location from the Main Location Code field on the Item Card. If necessary the setting can be changed here.

Qty. per Base Unit of Measure

Indicates the component quantity in the base quantity unit of the item.

Quantity (Base)

The quantity in this field is the expressed quantity of the item in the instruction, in fundamental units. Microsoft Dynamics NAV automatically populates this field.

BOM

Identifies whether this component has its own Master BOM.

Separate Prod. Order

Setting from the Item Card. If necessary the setting can be changed here.

Production Lead Time

Indicates the difference between date of completion and date of readiness.

Scrap %

Defines the percentage of scrap for the master BOM line.

Line Discount %

Indicates the line discount percentage that is valid for the item on the line.

Base Material

Identifies the assignment of a Base Material to the item. The Base Material will be used for the work package (refer to topic “Work Packages” on page 1).

Production View Link

Identifies the production view code for the assembly unit that will be linked to the component (refer to topic “Example – Production View” on page 1).

Belongs to Prod. View Link

Indicates which assembly unit this component is linked to and thus will become part of its configuration when viewing the construction order in the production view (refer to topic “Example – Production View” on page 1).

Separate Delivery

Indicates if the component is planned for a separate delivery (refer to topic “Separate Delivery of Semi-Finished Goods” on page 1).

Text Set ID

Indicates the ID of any text from the cc|text module that has been entered for the master BOM line.

Copy BOM to Text

Determines if the BOM will be copied to the purchase line text for a purchased item that is internally designed (refer to topic “Purchasing of Internally Designed Items” on page 1).

Move Time

Move time for the series of operations (only relevant with the Type of Work Center or Machine Center).

If this move time should be calculated with the help of the configuration feature, it is necessary to insert an attribute with a formula. The attribute is defined on the Dyn. BOM FastTab of the Configure to Order Setup page in the Variable for Move Time field.

Move Time Unit of Meas. Code

Time unit in which the move time is planned (only relevant with the Type of Work Center or Machine Center).

Wait Time

Wait time for the series of operations (only relevant with the Type of Work Center or Machine Center).

If this wait time should be calculated with the help of the configuration feature, it is necessary to insert an attribute with a formula. The attribute is defined on the Dyn. BOM FastTab of the Configure to Order Setup page in the Variable for Wait Time field.

Wait Time Unit of Meas. Code

Time unit in which the wait time is planned (only relevant with the Type of Work Center or Machine Center).

Operation No. / Previous Operation No. / Next Operation No.

Have the same function as in standard Microsoft Dynamics NAV routings. In the Master BOM line, these fields only work if Type = Work Center or Machine Center.

If each field in the line is empty, no checks are done when the Master BOM is certified. If any field is populated and:

  • Routing Type = Serial, the Operation No., Previous Operation No. and Next Operation No. fields will be populated automatically.
  • Routing Type = Parallel, the Operation No. and Next Operation No. fields must be populated manually. The Previous Operation No. field will be populated automatically (standard Microsoft Dynamics NAV).

Refer to “Example – Parallel Routings in CTO BOMs” on page 1 for more information.

Parallel Operation No.

Provides a simplified process for defining parallel routings (refer to “Example – Parallel Routings in CTO BOMs” on page 1).

Transfer to Sales Document

Determines if the Document BOM (where Type = Item) will be transferred to the current Sales Quote/Order lines (refer to topic “Example – Transfer Components to Sales Order” on page 1).

Type Production Structure

Used with the integration to cc|process manufacturing (refer to section “cc|process manufacturing” on page 1).

Lines FastTab, Line Menu

The following menu items appear on the Line menu, accessed from the Lines FastTab.

  • Attributes: Opens the Master BOM Attributes page, which displays the attributes that were assigned to the Master BOM line. Attributes can also be assigned manually here.
  • Item - Master BOM: Opens the Item - Master BOM page when line Type = Item.
  • Characteristic Search Tree: Opens the Characteristic Search Tree page where an item can be searched using characteristics (refer to topic “Search Characteristics” on page 1).
  • Cut: Enables individual items, multiple items and complete structures from the same or a different Master BOM to be cut and copied to the clipboard.
  • Copy: Enables individual items, multiple items and complete structures from the same or a different Master BOM to be copied to the clipboard.
  • Paste: Enables individual items, multiple items and complete structures that were copied to the clipboard from the same or a different Master BOM to be inserted here.
  • Comments: Comments can be informatively entered. The meaning and handling correspond to standard Microsoft Dynamics NAV.
  • Tools: Tools information can be informatively entered. The meaning and handling correspond to standard Microsoft Dynamics NAV.
  • Personnel: Personnel information can be informatively entered. The meaning and handling correspond to standard Microsoft Dynamics NAV.
  • Quality Measures: Quality measures can be informatively entered. The meaning and handling correspond to standard Microsoft Dynamics NAV.
  • Line Texts: Opens the Edit Text page, where line-related text can be stored. Refer to topic “cc|text module” on page 1 for more information.
