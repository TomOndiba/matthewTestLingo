Example – Dynamic Linkage of Work and Machine Centers

Similar to the dynamic linkage of items, a dummy work and machine center in the Checklist Template and Dynamic Master BOM can be replaced by actual work and machine centers that are selected in the checklist.

In this example, component types Work Center and Machine Center (along with others) are added to a Checklist Template and Master BOM. After a sales order is created, selections for each component are made in the Document Checklist which are then transferred to the Document BOM.

: New Component Type = Work Center/Machine Center assigned along with other types to Checklist Template

The components are also added to the related Master BOM; the numbers selected are irrelevant as they are only placeholders.

: New Component Type = Work Center/Machine Center assigned along with other types to Checklist Template

Each master BOM line must be set up with the attribute assigned in the Variable for Component field in Configure to Order Setup (in this case, COMP) and assigned the corresponding variable from the Checklist Template. Therefore, for the Work Center line, Attribute = COMP and Formula Value = WORKCENTER is entered in the Master BOM Attributes page, and so on for the other component lines.

: Master BOM Attribute assigned to the Work Center line

Next the sales order is created and the Document Checklist is filled out with the necessary components either manually or with the lookup in the Input field.

: Document Checklist with Machine Center component lookup in Input field

When the Document BOM is opened, the placeholders in the Master BOM are replaced with the codes selected in the Document Checklist.

: Document BOM showing components assigned in the Document Checklist

The same results occur in the Construction Order lines and the Service BOM lines, either from the Document BOM or when the “Request BOM” function is run.
