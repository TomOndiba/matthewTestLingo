Procurement Processing with Execution Units

Procurement processing from PaC Structural Plans created with Execution Units is generally handled the same way as other jobs in cc|project. This section briefly shows the procurement processing for work packages and then more detailed usage of procurement processing of the placing units.

Procurement Processing with Work Packages

PaC Structural Plans created from work packages uses the standard cc|project planning and procurement functionality. On the PaC Structural Plan page for the Work Package PaC job, click Planning and Processing.

: Accessing the Job Planning and Procurement for the PaC work package

The Job Planning and Procurement page opens with the complete structure from the PaC Structural Plan, ready for procurement processing. The procurement planning lines can be created based on different procurement channels. For example, planning can be started for construction or engineering services , or any material procurements.

: Job Planning and Processing with PaC Structural Plan from work package

Procurement Processing with Placing Units

Procurement processing with placing units also uses the standard cc|project functionality, but some defaults can be set up in the Execution Unit before planning (or the PaC Structural Plan) begins. It is not manadatory to set up the defaults, it simply automates some of the steps in the procurement processing.

On the Execution Unit page for the placing unit, expand the Procurement FastTab.

: Procurement defaults added to placing unit

In this example, the same vendor used in the PQM placing process is used for the ordering procurement process.

Note

If the procurement defaults are added after the PaC Structural Plan was created, the existing Structural Plan will need to be deleted first and then recreated using the “Insert Execution Unit” function.

On the PaC Structural Plan page for the Placing Unit PaC job, click Planning and Processing.

: Accessing the Job Planning and Procurement for the PaC placing unit

The Job Planning and Procurement page opens with the complete structure from the PaC Structural Plan, ready for procurement processing. The procurement defaults from the Execution Unit are also available.

For the purpose of this example, the planning and procurement processes are run to generate a Purchase Order. First, the procurement “Planning” function is run.

: Running the procurement Planning function

In this example, planning needs to be done for the current line and sub-structure. Click OK to create the procurement planning lines. The Procurement Planning field is updated accordingly.

: Procurement Planning field updated after Planning process

Now, click Release to create the Purchase Order for the current line and sub-structure.

: Running the procurement Release function

Click OK to release the procurement planning lines. A message appears asking to create the Procurement Documents.

: Message to create Procurement Documents from released procurement lines

Click Yes to create the Purchase Order. The Procurement Planning and Procurement Release fields are updated accordingly.

: Procurement Planning and Procurement Release fields updated after Release process

Click the Yes value in one of the released planning lines to open the Purchase Order based on the BoQ position structure. Note that the Execution Code is assigned in the General FastTab.

: Purchase Order created based on the BoQ Position structure
