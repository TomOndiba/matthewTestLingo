Jobs Setup

The Jobs Setup page includes two FastTabs that are used with Construction Works Processing: Measurement & Construction Works and BoQ Manager. Each is described in this section.

Measurement & Construction Works FastTab

The Measurement & Construction Works FastTab contains defaults used in Construction Works Processing.

: Jobs Setup, Measurement & Construction Works

The fields in this FastTab are described in the following table.

Field

Description

Auto Create PaC

Determines whether PaC Jobs are created automatically when a BoQ Job is created. The options are:

  • Never – Option will never be used
  • Optional – Option will be used on request
  • Always – Option will always be used

PaC Auto Insert Calc. Group Totals

Determines whether Calculation Group Totals are created automatically when the PaC Job is created. The options are:

  • Never – Option will never be used
  • Optional – Option will be used on request
  • Always – Option will always be used

PaC Default Job Type

Indicates the default Job Type to be assigned to PaC Jobs.

Refer to section “Job Types Setup” on page 1 for more information about the Job Types used with Construction Works Processing.

Exemption Certificate Code

Indicates the Certificate Code for Exemption Certificates that are used for validity checks on Tax No. and Tax State No.

Calc. Base Calculation with Status Order

Determines whether the Base Calculation is calculated automatically when the Job Status is set to Order. The options are:

  • Never – Option will never be used
  • Optional – Option will be used on request
  • Always – Option will always be used

Autom. Project Dimension

Indicates the dimension to be automatically created for main Projects.

Project Account Dimension

Indicates the dimension to be assigned to the Project account.

Autom. Proj. Acc. Dimension

Determines how the Project Account dimension value is created. The options are:

  • Job No. – The Job No. is used as the dimension value.
  • No. Series – The number series entered in the Project Account Nos. field is used as the dimension value.
  • Project No. + Index (2-digit) – The Project No. plus a 2-digit index is used as the dimension value (e.g. CON00001-01).

Project Account Nos.

Indicates the No. Series to be used as the Project Account dimension value if Autom. Proj. Acc. Dimension = No. Series.

Project Nos.

 Indicates the number series to be used for new Projects.

Measurement Nos.

 Indicates the number series to be used for new Measurements.

Measurement Summary Nos.

Indicates the number series to be used for new Measurement Summaries.

CS Invoicing Templ. Nos.

Indicates the number series to be used for the Construction Invoicing Template. This is the general template that will always be copied to a Construction Invoicing Schedule on Jobs or Sales Invoices.

CS Invoicing Schedule Nos.

Indicates the number series assigned to the Construction Invoicing Schedule that is used on Sales Invoices, e.g. for Share in Cost or other deductions.

Component Code Wages

Indicates the component code that will be entered in the No. field of the BoQ Budget Calculation when budget lines for Wages are created automatically from the Fast Calculation page (refer to section “Fast Calculation” on page 1 for more information) or by the GAEB Unit Price Parts functionality (refer to section “Order Award and Comparison” on page 1 for more information).

Component Code Material

Indicates the component code that will be entered in the No. field of the BoQ Budget Calculation when budget lines for Materials are created automatically from the Fast Calculation page (refer to section “Fast Calculation” on page 1 for more information) or by the GAEB Unit Price Parts functionality (refer to section “BoQ Job Calculations” on page 1 for more information).

Component Code Ext. Service

Indicates the component code that will be entered in the No. field of the BoQ Budget Calculation when budget lines for External Services are created automatically from the Fast Calculation page (refer to section “Fast Calculation” on page 1 for more information) or by the GAEB Unit Price Parts functionality (refer to section “Order Award and Comparison” on page 1 for more information).

Component Code Equipment

Indicates the component code that will be entered in the No. field of the BoQ Budget Calculation when budget lines for Equipment are created automatically from the Fast Calculation page (refer to section “Fast Calculation” on page 1 for more information) or by the GAEB Unit Price Parts functionality (refer to section “Order Award and Comparison” on page 1 for more information).

Component Code Other

Indicates the component code that will be entered in the No. field of the BoQ Budget Calculation when budget lines for Other Costs are created automatically from the Fast Calculation page (refer to section “Fast Calculation” on page 1 for more information) or by the GAEB Unit Price Parts functionality (refer to section “Order Award and Comparison” on page 1 for more information).

Component Code Other 2

Indicates the component code that will be entered in the No. field of the BoQ Budget Calculation when budget lines for Other Costs 2 are created automatically from the Fast Calculation page (refer to section “Fast Calculation” on page 1 for more information) or by the GAEB Unit Price Parts functionality (refer to section “Order Award and Comparison” on page 1 for more information).

Alloy Surch. Raise Unit Cost

If selected, Alloy Surcharges directly increase the Unit Cost in the Budget Calculation.

Unit Price Rounding Precision

Indicates the rounding precision for Unit Prices of sales positions. This is defaulted to the Job Card.

Show BoQ Short Text

If selected, BoQ Short Text is shown in the Budget Calculation.

Show BoQ Long Text

If selected, BoQ Long Text is shown in the Budget Calculation.

Measurement Level Required

If selected, Measurement Levels are required to be selected in Measurement Documents before registration. This is defaulted to the Job Card.

Measurement Location Required

If selected, Measurement Locations are required to be selected in Measurement Documents before registration. This is defaulted to the Job Card.

Measurement Price Source

Indicates if Measurement pricing originates from the Sales Order or the BoQ Budget Calculation.

Measurement Report ID

Specifies the report that is used for printing a Measurement Document.

 

BoQ Manager FastTab

The BoQ Manager FastTab contains defaults used in BoQ processing.

: Jobs Setup, BoQ Manager FastTab

The fields in this FastTab are described in the following table.

Field

Description

GAEB No.

Indicates the number series to be used for new GAEB files created in Microsoft Dynamics NAV.

GAEB Debugger On

If selected, the GAEB Debugger is activated and runs when files are imported/exported for error tracking.

GAEB Debugger Client Folder

Indicates the folder where the GAEB Debugger Client is stored.

GAEB Log File Mode

Determines how GAEB log files are handled. The options are:

  • Off – Log files are not created.
  • Append – New GAEB imports/exports are added to existing log files.
  • New – New log files are created for new GAEB imports/exports.

Default UoM Allocation Type

Indicates the default to be entered in the Type field of the GAEB Unit of Measure Assign page when a unit of measure is missing and added during the GAEB file import (refer to section “GAEB Unit of Measure Allocation” on page 1).

Default UoM Allocation No.

Indicates the default to be entered in the No. field of the GAEB Unit of Measure Assign page when a unit of measure is missing and added during the GAEB file import (refer to section “Construction Setup” on page 1).

Default UoM Allocation Sales Code

Indicates the default to be entered in the Sales Code field of the GAEB Unit of Measure Assign page when a unit of measure is missing and added during the GAEB file import (refer to section “Construction Setup” on page 1).

Default OZ Mask

Defines the default BoQ Position No. format for the “Set BoQ Position No.” function; value defaults to the OZ Mask field in the Job Card.

OZ Pos. Increase with

Defines the default steps between each positions; value defaults to the OZ Pos. Increase with field in the Job Card.

OZ Space instead of Null

If selected, BoQ Positions have a space instead of a leading zero.

Defaults to the OZ Space instead of Null field in the Job Card.

Free OZ Allowed

If selected, BoQ Position numbers can be changed manually.

Defaults to the Free OZ Allowed field in the Job Card.
