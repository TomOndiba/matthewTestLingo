Execution Unit Page

Execution Units are managed in the Execution Unit page. This page is opened from the Execution Units page, which is accessible from the Job Card, Jobs list or the BoQ Position List by clicking Navigate > Execution Units.

: Execution Units list page

The Execution Units page displays all Execution Units that have been assigned to a job, along with some overview information.

Click New or Edit to create or open an Execution Unit.

: Execution Unit page

The fields and actions on the Execution Unit page are described in this topic.

General FastTab

The fields on the General FastTab are described in the following table.

Field

Description

Job No.

Automatically populated with the associated job number.

Code

User-defined, unique code for the Execution Unit.

Type

Identifies the type of Execution Unit. The options are:

  • Placing Unit – vendor requests created during the calculation phase, or later for processing the subcontractor purchase order.
  • Work Package – to-dos to be processed internally, work done by different departments.

Description

Brief description of the Execution Unit.

Cost Type

Identifies the cost type that is considered in the Execution Unit. The options are:

  • <blank> – all components are considered
  • Wage – only wage components are considered
  • Material – only material components are considered
  • External Service – only external service components are considered
  • Equipment – only equipment components are considered
  • Other – only other components are considered

Only relevant for Type = Placing Unit.

Status

Indicates the current status of the Execution Unit. The options are:

  • Open – Execution Units with this status can be modified can be modified.
  • Released – Execution Units with this status can be used in the procurement process.
  • Completed – Execution Units with this status cannot be edited or used in the procurement process.

Purchase Quote Manager

Indicates whether the integrated cc|purchase quotes manager (PQM) was used for selecting a subcontractor based on quoted costs.

Only relevant for Type = Placing Unit.

Refer to topic for more information about using the PQM with Execution Units.

Assigned PaC Job No.

Indicates the PaC job number to which this execution unit has been assigned when preparing to start the procurement process.

 

Lines FastTab

The Lines FastTab is populated from the BoQ Position List. Therefore, only the fields that are specific to the Execution Units are described in the following table.

Field

Description

Assigned Quantity

Displays the complete quantity that has been assigned. When the Quantity field is changed, the Assigned Quantity is updated.

The field is formatted as italicized red when the assigned quantity is greater than the available quantity, and bold green when the assigned quantity matches the quantity.

Direct Unit Cost

Displays the Direct Unit Cost in foreign currency.

Purchase Line Amount

Displays the Purchase Line Amount in foreign currency.

Unit Price (LCY)

Displays the Unit Price in local currency.

Sales Line Amount (LCY)

Displays the Sales Line Amount in local currency.

Orig. Unit of Measure Code / Original Quantity / Original Sales Line Amount (LCY) / Original Total Cost (LCY)

Original values from the BoQ Budget Calculation. These fields provide a comparison between what was calculated, planned, and allocated to the PaC job.

Proc. Currency Factor

Displays the exchange factor between local currency and foreign currency, which is selected in the Purch. Currency Code field.

Purch. Currency Code

Displays the purchase currency, which is defaulted from the BoQ Budget Calculation. When the Currency Code is changed, the value in the Proc. Currency Factor field is updated.

Direct Unit Cost (LCY) / Total Cost (LCY) / Total Direct Cost (LCY) / Unit Cost (LCY)

Cost values in local currency.

The sorting of the lines can be refreshed by clicking Functions > Refresh Sorting on the Lines FastTab. This can be useful when additional positions that should apply between existing lines are added afterwards.

GAEB FastTab

The fields in the GAEB FastTab are populated with cost types included in the GAEB files.

: Execution Unit page, GAEB FastTab

Procurement FastTab

The fields in the Procurement FastTab are used for setting defaults for procurement processing. Typically these fields are set when the Execution Unit Type is set to Placing Unit.

: Execution Unit page, Procurement FastTab

Controlling FastTab

The fields on the Controlling FastTab provide current and budgeted cost/sales information based on the Execution Unit lines.

: Execution Unit page, Controlling FastTab

Each field is described in the following table.

Field

Description

Last Update

Date and time the Execution Unit was updated using the “Update Controlling Values” function.

Total Cost Original (LCY)

Calculated cost from the BoQ Budget Calculation.

Total Cost Budget (LCY)

Planned cost based on the Quantity and Direct Unit Cost fields in the Execution Unit line.

PaC Total Cost Budget (LCY)

Allocated cost budgeted to one PaC job.

Total Cost Usage (LCY)

Posted costs to the PaC job; opens the Job Ledger Entries filtered on the Execution Unit.

Sales Line Amount Orig. (LCY)

Calculated price from the BoQ Budget Calculation.

Sales Line Amount Budget (LCY)

Planned cost based on the Quantity and Unit Price fields in the Execution Unit line.

PaC Sales Line Amount Budget (LCY)

Allocated sales budgeted to one PaC job.

 

Actions Tab

The functions on the Actions tab are described in the following table. These functions are available on the Home tab.

Action

Description

Release

Sets the Status to Released.

Reopen

Sets the Status to Open from a released Execution Unit, or to Release from a finished Execution Unit.

Finish

Sets the Status to Completed.

Update Controlling Values

Updates the values in the fields on the Controlling FastTab.

 

Navigate Tab

The functions on the Navigate tab are used with the integrated PQM functionality. Each is described in the following table.

Action

Description

Create PQM

Opens a new PQM page for PQM processing.

Show PQM

Opens an existing PQM for the Execution Unit.
