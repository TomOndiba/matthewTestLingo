Construction Setup

This topic describes setup pages located within the Setup and GAEB menu groups in the Construction, Administration menu.

EFB Surcharges

The EFB Surcharges page is used to define the surcharges that need to be included in a calculation document (refer to topic “EFB Calculation” on page 1).

: EFB Surcharge

The fields on this page are described in the following table.

Field

Description

BoQ Job No.

Indicates a BoQ Job No. if the EFB Surcharge is specific to a job.

Cost Type

Identifies the Cost Type to which the EFB Surcharges are to be applied.

Starting Date

Indicates the starting date from which the EFB Surcharges are valid.

Preliminaries %

Indicates the initial surcharges for on-site overhead relating to the Cost Type.

Business Expenses %

Indicates the surcharge for business expenses relating to the Cost Type.

Risk %

Indicates the surcharge for project risk relating to the Cost Type.

Profit %

Indicates the surcharge for profits relating to the Cost Type.

Total Surcharge %

Calculated total of all surcharges relating to the Cost Type.

CS Invoicing Templates

The CS Invoicing Templates page displays a list of all CS Invoicing Templates, which are similar to calculation schedule templates in structure but are used to create Invoicing Schedules for Sales Invoices. To create a new template or open an existing template, click New or Edit on the ribbon.

: CS Invoicing Template

In the General FastTab, the Invoice Type field determines the type of invoice for which the calculation template will apply. The options are:

  • Invoice – used for Sales Invoices and Sales Orders
  • Advance Payment – used for Advance Payment Invoices (which are part of the German-specific product cc|advance payment. Refer to your COSMO CONSULT consultant for more information about cc|advance payment.)
  • Final Invoice – used for Final Invoices
  • Part Invoice – used for Partial Sales Invoices

The fields in the Lines FastTab are described in the following table.

Field

Description

Row No.

Unique code field that is used to identify a corresponding calculation line in the Totaling and Reference to Row No. fields.

Description

Short description of the calculation.

Totaling Type

Specifies what the calculation line is based on. The options are:

  • Sales Line – used to get the Total Sales Line Amount of all related Sales Lines.
  • Measurement – used to get the Total Sales Line Amount based on all related Measurement Lines.
  • Formula – used on lines that total other calculation lines with a mathematical formula. For example:
    • To use the value of the line with Row No. 10 multiplied by 2, enter R10 * 2.
    • To calculate the cost in Row No. 10, with a Charge % of 10, plus the costs in row 20, enter R10 * 10/100 + R20.
  • Charge % – used to define the charge in percent to calculate within the line
  • Sum – used on lines that total other calculation lines. Standard Microsoft Dynamics NAV filters can be used, such as range (1000..2000), either/or (1000|2000), or any of the following: * ? > < = @.

Totaling

Values relating to the selection in the Totaling Type field.

Charge Type

Indicates a Job Charge Type to find the correct G/L Account when the charge is transferred to the sales line. Used only on lines with a Totaling Type of Charge %.

Reference to Row No.

Refers to the row number that applies to the charge calculation. Used only on lines with a Totaling Type of Charge %.

Value Transfer Option

Determines if and to where on a Sales Order or Sales Invoice the calculation value can be transferred when the “Transfer Value” function is run:

  • Unit Price
  • Sales Line Discount
  • Construction Charge

Print Line

If selected, the template line is printed in the Sales Invoice Report.

 

GAEB Unit of Measure Allocation

The GAEB Unit of Measure Assignment page is used to map units of measure from a GAEB file to units of measure in Microsoft Dynamics NAV during import.

: GAEB Unit of Measure Assignment page with various UOMs from GAEB files

The fields in the GAEB Unit of Measure Assignment page are described in the following table.

Field

Description

GAEB Unit of Measure

Indicates the unit of measure in the imported GAEB file. This field is case-sensitive.

Unit of Measure Code

Indicates the corresponding unit of measure in Microsoft Dynamics NAV.

Type

Indicates if the unit of measure is from an Item or a General Component (Misc.).

Automatically populated from the Default UoM Allocation Type field in the BoQ Manager FastTab of the Jobs Setup page if set up during GAEB file import.

No.

Indicates the Item No. or General Component, based on the selection in the Type field.

Automatically populated from the Default UoM Allocation No. field in the BoQ Manager FastTab of the Jobs Setup page if set up during GAEB file import.

Sales Code

Indicates the Job Sales Code that relates to the unit of measure.

Automatically populated from the Default UoM Allocation Sales Code field in the BoQ Manager FastTab of the Jobs Setup page if set up during GAEB file import.

If the units in the GAEB file do not exist in Microsoft Dynamics NAV when a GAEB file is imported into a Job, it is possible to enter them into this page during import (refer to section “Import GAEB File” on page 1).
