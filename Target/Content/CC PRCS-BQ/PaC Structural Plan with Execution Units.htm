PaC Structural Plan with Execution Units

Execution Units for both work packages and placing units can be used to create the PaC Structural Plan. The difference is the creation method. In the PaC Structural Plan, there are three options:

  • Get Calculation Groups – creates the PaC Structural Plan in different levels: the first is the Execution Units of type Work Package, and the second is cost types with the associated calculation groups. This is the most common way to create the work packages in PaC jobs.
  • Get Material Demands – adds specified calculated materials to the PaC structural plan for each work package. This step is done after the “Get Calculation Groups function” is run.
  • Insert Execution Unit – creates structure based on the BoQ structure. This is typically used for placing units.

This section describes how to create the PaC Structural Plans for work packages and placing units.

Example – Create PaC Structural Plan for Work Packages

In this example, a PaC job has been set up for the work package. Once created, this job is allocated to the Execution Units for the work packages. In the PaC Structural Plan page, the “Get Calculation Groups” function is run to create the structure per execution unit, and then the “Get Material Demands” function is run to add the material (item D-9008) from the Power Distribution position.

Assign PaC Job to Execution Units

Execution Units must be released to be used in the PaC Structural Plan. Therefore, each work package is released after the PaC job is assigned.

: Execution Unit work packages to be assigned a PaC job and then released

Note

The “Update Controlling Values” function should also be run prior to release; however, this step was previously done and all values are current.

The first work package Execution Unit for WP 01 is opened. In the Assigned PaC Job No. field, click the drop-down and select the PaC job.

: Selecting PaC job in work package Execution Unit

On the ribbon, click Release to release the Execution Unit for PaC Structural Plan usage.

: Work package Execution Unit released with assigned PaC job

On the ribbon, click Next to open work package Execution Unit WP 02.

: Preparing to assign PaC job and release work package Execution Unit

Repeat the steps to assign the Work Package PaC job and then release the Execution Unit.

: Work package Execution Unit released with assigned PaC job

Click OK to close the Execution Unit pages and return to the Execution Units page. Note that the Assigned PaC Job No. and Status fields have been updated.

: Execution Units page with updated work package information

Create PaC Structural Plan

In the Job Card for the Work Package PaC job, click Structural Plan on the ribbon.

: Accessing the Structural Plan from the PaC job for the work package

As with any cc|project structure, a new position must be created before any lines can be added.

: Position added from the New Position function

On the ribbon, click Get Calculation Groups to create the PaC Structural Plan based on the work package Execution Units.

: PaC Structural Plan created from work package Execution Unit

The first level of the structure is based on the work package Execution Units: WP 01 Complete Step 1 and WP 02 Step 2. The next levels are the substructures that are based cost types and calculation groups. The substructures include the calculated components from the Execution Unit assignment.

Get Material Demand

The current PaC Structural Plan only includes PaC job lines based on calculation groups, but no specific material that was calculated. The BoQ Budget Calculation specified an entry for item D-9008 within the Power Distribution position.

: Material demand specified in the BoQ Budget Calculation

To add the material demand to the PaC Structural Plan, click Get Material Demands on the ribbon.

: Material demand added to the PaC Structural Plan

The material demand list will be added to each work package area. In this example, item D-9008 was added, which was calculated and assigned to work package WP 01.

Example – Create PaC Structural Plan for Placing Units

In this example, a PaC job has been set up for the placing unit. In the PaC Structural Plan page, the “Insert Execution Unit” function is run to create the structure directly from an Execution Unit.

Note

The Execution Units for placing units do not need to be assigned to a PaC job prior to creating the Structural Plan. After the “Insert Execution Unit” function is run, the PaC job is automatically assigned to the Execution Unit.

Assign PaC Job to Execution Units

Execution Units must be released to be used in the PaC Structural Plan.

: Execution Unit work packages to be assigned a PaC job and then released

Note

The “Update Controlling Values” function should also be run prior to release; however, this step was previously done and all values are current.

The Execution Unit for placing unit PU 01 is opened. On the ribbon, click Release to release the Execution Unit for PaC Structural Plan usage.

: Execution Unit for placing unit released

Click OK to close the Execution Unit page and return to the Execution Units page. Note that the Status fields has been updated.

: Execution Units page with updated work package information

Create PaC Structural Plan

In the Job Card for the Placing Unit PaC job, click Structural Plan on the ribbon.

: Accessing the Structural Plan from the PaC job for the placing unit

As with any cc|project structure, a new position must be created before any lines can be added.

: Position added from the New Position function

On the ribbon, click Insert Execution Unit to create the PaC Structural Plan based on the Execution Units. The Execution Units page opens for placing unit selection.

: Selecting Execution Unit from Insert Execution Unit function

With the Placing Unit line selected, click OK. A message appears to expand the placing unit like a BOM. Click Yes to expand the placing unit.

: Message to expand placing unit

The structure is created as a one-to-one copy of the BoQ Budget Calculation, including all components that were used for the calculation.

: PaC Structural Plan created from Execution Unit

For purposes of review, click Navigate > Execution Units. Note that the Assigned PaC Job No. field is populated with the current PaC job number

: Execution Unit for placing unit updated with PaC job number
