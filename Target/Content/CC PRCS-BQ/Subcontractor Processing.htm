Subcontractor Processing with Placing Units

Subcontractor quotes can be brought into the BoQ Calculation through the use of the cc|purchase quotes manager (PQM) with the placing units. With PQM, different vendors can be selected for quoting the external services/items, and then the selected vendor and costs are transferred automatically to the Execution Units and BoQ Budget Calculation. Once the PQM process is complete, further processing of the BoQ job can continue, such as creating the PaC job and beginning the procurement processes.

This section describes how to use placing units and the PQM to choose a vendor.

Example – Subcontractor Quoting with PQM

In this example, a job has been set up with various BoQ Budget Calculation entries for a construction site. The cables and lines need to be subcontracted to a vendor with the materials and skills for installation. CRONUS International Ltd. has two vendors for this purpose: 10000 and 20000. To get the best price and selection, the PQM will be used to send a quote to both vendors.

Create Execution Unit

The first step of the subcontractor quoting process is to create the Execution Unit for the BoQ positions that will be subcontracted to a vendor. This example starts in the BoQ Position List, but note that the Execution Units can also be accessed from the Job Card/Jobs list.

: BoQ Position List with subcontractor quoting needs

On the Navigate tab, click Execution Units to open the Execution Units page.

: Execution Units page

Click New to create an Execution Unit.

: Execution Unit page

In the Code field, enter a unique code for this execution unit. Leave the Type field set to Placing Unit, and then enter a Description of the execution unit.

: Execution Unit header populated

To add the BoQ Positions to the Execution Unit, use the lookup in the BoQ Pos. No. field in the Lines FastTab.

: Opening the BoQ Position List for addition to Execution Unit lines

Select the BoQ Positions to add to the Execution Unit. To add a position with sub-positions, select the main position. In this example, the Cables & Lines position is selected, which will include the two sub-positions.

: Cables & Lines selection for addition to execution unit

A message appears asking if the found substructure should be expanded.

: Substructure expansion message

Click Yes. The Execution Unit lines are populated with the BoQ Positions.

: BoQ Positions added to the Execution Unit

Create Quotes with PQM

Now that the lines have been updated with the BoQ positions, the quoting process can begin.

Note

This example shows basic processing using PQM. It does not provide an overview of or any setup/detailed processing information. Therefore, business practices will typically be more intensive than what is demonstrated in this exercise. Refer to topic “cc|purchase quotes manager” on page 1 for more information about using the PQM.

On the ribbon, click Create PQM.

: Preparing to start the quoting process with PQM

The PQM page opens with relevant information from the Execution Unit.

: PQM page created from execution unit

Vendors must be assigned before quotes can be created. In the General FastTab, click the Assigned Vendors field to open the PQM Vendors page.

: Assigning vendors to the PQM

Click New to add each vendor to be included in the quoting process.

: Vendors assigned to the PQM 

Click OK to complete the vendor assignment. The value in the Assigned Vendors field is updated accordingly.

: Preparing to create purchase quotes to assigned vendors

On the ribbon, click Create Purch. Quotes.The quotes are created and the PQM Matrix page opens. Note that this page can be opened from the ribbon also by clicking Purchase Quotes Matrix.

: PQM Matrix opened by running the Create Purch. Quotes function

Click Show Matrix to open the PQM Placing Matrix page, which lists the BoQ Positions as well as the two vendors for quoting.

: PQM Placing Matrix

At this point, in a typical process, the quotes have been sent to the vendors and now each will be updated with the quoted costs from the vendor. On either “Normal Position” line, click the value in the field for vendor 10000. The related purchase quote opens.

Note that the project number and execution unit code appear in the General FastTab.

: Purchase Quote highlighting project and execution unit codes, and empty cost in lines

In the Direct Unit Cost field on the substructure lines, enter the quoted costs from the vendor.

: Purchase Quote lines updated with costs from vendor 10000

Click OK to close the Purchase Quote. In the PQM Placing Matrix page, click Refresh to update the line for vendor 10000 with the individual costs. If lead time calculations were used, the dates would indicate material delivery.

: Costs from vendor 10000 updated for both BoQ Position lines

Next, repeat the same process for vendor 20000. Click the value in either field and in the Purchase Quote, enter the quoted costs.

: Purchase Quote lines updated with costs from vendor 20000

Click OK to close the Purchase Quote. In the PQM Placing Matrix page, click Refresh to update the line for vendor 20000 with the individual costs.

: Costs from vendor 20000 updated for both BoQ Position lines

Now that the quotes have been updated, the placing (vendor selection) process can begin. On the ribbon, click Placing. A message appears asking to use all lines or selected lines only.

: Placing line selection message

With the default set to All lines, click OK. The PQM Placing page opens.

In the middle section, the preferred vendor can be selected manually in the Vendor No. field, or the best price can be selected automatically. To choose the vendor manually, clear the Select Best Price check box.

: PQM Placing page

In this example, vendor 10000 is the preferred vendor and had the best price, so the Select Best Price check box remains selected.

In the Options FastTab, the Update Source Data check box is selected in order to transfer the costs to the execution unit and BoQ positions.

Click OK to complete the placing process. The PQM Placing Matrix is updated to indicate the winning vendor (information cleared for vendor 20000) and that the lines were placed.

: PQM Placing Matrix with placing information

Close the open pages to return to the Execution Unit page. The costs and purchase amounts from the quoting process have been added.

: Unit Cost and Purchase Line Amounts added to Execution Unit

On the ribbon, click Update Controlling Values to update the values in the Controlling FastTab. Note that there are differences between the original and budgeted values because no changes were made.

: Updated values in the Controlling FastTab

Click OK to close the Execution Unit and return to the BoQ Position List. Note that the prices have been updated, based on the new unit costs and settings in the BoQ Budget Calculation.

: Prices updated in BoQ Position List based on costs in the BoQ Budget Calculation

On the ribbon, click Budget Calculation. The placing process created new BoQ Budget Calculation lines with the external service component, which have the transferred Direct Unit Costs.

: New external service component lines in BoQ Budget Calculation with costs from the PQM placing process
