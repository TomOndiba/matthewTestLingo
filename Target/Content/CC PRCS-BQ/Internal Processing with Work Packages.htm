Internal Processing with Work Packages

In cc|project, Work Packages are used to create the work breakdown structure of PaC Jobs. The process of creating the Execution Units for work packages is similar to the placing units, in that an Execution Unit is created, the Type is set to Work Package and then entries from the BoQ Position List are assigned in the lines.

Often, different work package will contain the same BoQ Positions, to allow for different procurement processing.

The work packages are created, a PaC job is assigned to each Execution Unit. From there, a Structural Plan is created in the PaC job, and the standard procurement planning of Construction Work Processing can begin.

This section describes how to create work packages using Execution Units.

Example – Create Work Packages

This example uses the same job that was set up with various BoQ Budget Calculation entries for a construction site. The cables and lines were already processed for subcontracting. Now, two work packages are created for the remaining BoQ Positions: one with the complete entries, and one for only a portion of the Network Distribution, which is handled differently in procurement.

Create First Work Package

The first step is to create the Execution Units for the BoQ positions. This example starts in the BoQ Position List, but note that the Execution Units can also be accessed from the Job Card/Jobs list.

: BoQ Position List with subcontractor quoting needs

On the Navigate tab, click Execution Units to open the Execution Units page.

: Execution Units page with existing Placing Unit

Click New to create an Execution Unit.

: Execution Unit page

In the Code field, enter a unique code for this execution unit. Change the Type field to Work Package, and then enter a Description of the execution unit.

: Execution Unit header populated

To add the BoQ Positions to the Execution Unit, use the lookup in the BoQ Pos. No. field in the Lines FastTab.

: Opening the BoQ Position List for addition to Execution Unit lines

Select the BoQ Positions to add to the Execution Unit. To add a position with sub-positions, select the main position. In this example, the Construction Site Setup and Distributions positions are selected, which will include the sub-positions.

: Construction Site Setup selection for addition to execution unit

A message appears asking if the found substructure should be expanded.

: Substructure expansion message

Click Yes. The Execution Unit lines are populated with the BoQ Positions.

: BoQ Positions added to the Execution Unit

Now, add the Distributions position to the work package.

: Distributions selection for addition to execution unit

The Distributions substructures are expanded and added to the work package.

: : BoQ Positions added to the Execution Unit

If necessary, the planned quantity can be changed if calculations were done based on the specification, but more or less time and/or materials are required. The Assigned Quantity field shows the quantities that have been assigned to the Execution Unit. Therefore, it is possible to split BoQ Positions into different Execution Units. In this example, the Network Distribution position is split into two work packages. A quantity of 30 PCS will be assigned to the current work package, and a new work package will be created for the remaining 20 PCS.

: Quantity changed from 50 to 30 for Network Distribution position

On the ribbon, click Update Controlling Values and then expand the Controlling FastTab. Note that the budgeted cost/price values are lower than the original cost/price values from the BoQ Calculation, due to the changed quantity in the work package.

: Updated values on the Controlling FastTab

Create Second Work Package

Create another work package Execution Unit for the Network Distribution.

: New work package Execution Unit

Add the Network Distribution position only to the work package.

: Network Distribution selection for addition to execution unit

The position is added to the work package, with no message to expand the substructure.

: Network Distribution added to work package

Note that the Assigned Quantity field is red because 30 has already been assigned in the previous Execution Unit, and therefore has been over-allocated. In this example, the remaining 20 PCS are assigned to this Execution Unit. To see what has been assigned to this BoQ Position, click the value in the Assigned Quantity field.

: Viewing assigned quantities for the BoQ Position

Close the assigned quantity lines.

On the ribbon, click Update Controlling Values and then expand the Controlling FastTab. Note that the budgeted cost/price values are lower than the original cost/price values from the BoQ Calculation, due to the changed quantity in the work package.

: Updated values on the Controlling FastTab
