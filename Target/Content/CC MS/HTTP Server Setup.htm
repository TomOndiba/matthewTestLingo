HTTP-Server Setup

This section explains how to configure the HTTP-Server to enable the communication to the Microsoft Dynamics NAV database.

General Information

The HTTP-Server can be performed as Windows Forms Application or as Windows Service.

Notes for Windows Forms vs Windows Service

Since both variants use the same config file (“HttpServer.config.xml”), both can be executed – but not performed – at the same time. The config file specifies an IP port to listen on, therefore the server cannot be started if the service is running because the server is already listening to the port.  If performing the server as Service is preferred, the Windows Forms Application can be used for applying relevant settings. If the server is run while the service is started, an error message occurs.

Licensing

During the installation of the HTTP-Server, the product key must be entered in Page 2 of the Configuration Wizard.

: Product Key page of HTTP-Server Configuration Wizard

If the Product Key field is left empty, the HTTP-Server can be used as a 14-day trial version, limited to one device. The product key can also be entered after the installation using the configuration assistant in the installation folder of the HTTP-Server.

If the current license does not contain a limitation of the number of process dialogues, the No. of Dialogues field can be left empty or a “0” can be entered.

The current license can be displayed in the HTTP-Server application by clicking Help > (Info) About.

: Current license information displayed in HTTP-Server application

HTTP-Server Window

This section describes the HTTP-Server window.

: HTTP-Server

In the Server menu, the server can be started or stopped. Alternatively the buttons on the toolbar below can be used.

The Tools menu contains links to the “Mobile Emulator”, where handheld requests can be simulated for testing. Additionally the “Terminal App” can be run from here.

In the Extras menu, the HTTP-Server Settings can be accessed.

HTTP-Server Settings

Before starting the HTTP-Server, several settings should be applied and checked. Since the HTTP-Server communicates with Microsoft Dynamics NAV, the connection settings must be made correctly.

Ensure that the Windows User – who runs the HTTP-Server – is registered as user in Microsoft Dynamics NAV and granted the required rights and roles.

General Tab

The fields on this tab are described in the following table.

: HTTP-Server Settings, General tab

Field

Description

Port

Specifies the port to which the HTTP-Server should listen.

Note

This port must not be used by other applications.

HTTPS

If selected, HTTPS is used instead of HTTP.

Root Directory

Indicates the root directory of the HTTP-Server. In this folder, Microsoft Dynamics NAV creates subfolders for handhelds and stores generated files. This folder must be identical with the Root Folder in the Mobile Setup of Microsoft Dynamics NAV (refer to topic "General FastTab " on page 1).

Log File Path

Contains the path to a log file that the HTTP-Server uses when a value exists. The content of the log file is the same as the content of the console in the HTTP-Server application. Logging is useful when running the HTTP-Server as a Windows Service.

Use Ajax

If selected, enables the activation of an AJAX preloader with status bar.

Note

The browser of the handheld must be AJAX compatible. In most cases, handheld operating systems only provide a simple browser without AJAX support.

AJAX Request Prefix

Indicates the prefix to be used for AJAX requests; should be left to default.

Wait Time (Sec.)

Specifies how long the HTTP-Server should wait for a response from Microsoft Dynamics NAV before a timeout occurs. On timeout, the HTTP-Server delivers an error message to the handheld/client.

Request MQ Name

Defines the Windows Message Queue for requests in Microsoft Dynamics NAV.

Response MQ Name

Defines the Windows Message Queue for responses of the HTTP-Server.

Request Admin MQ Name

Defines the Windows Message Queue for status messages to Microsoft Dynamics NAV.

Terminal App Path

Specifies the path to the Terminal Application that can be launched via the menu. This path is set by default.

Emulator Path

Specifies the path to the Mobile Emulator that can be launched via the menu. This path is set by default.

Set Up HTTP-Server as Service

During the installation of the HTTP-Server, a Windows Service named “cc|mobile solution – HTTP-Server Service” is automatically registered. For establishing a connection to Microsoft Dynamics NAV, the Service must be run as a Windows User that is registered to Microsoft Dynamics NAV and has the needed permissions.

The Startup Type should be set to “Automatic”.

Example – Setup/Cleaning of Additional Server Service

For each additional setup of a Server Service, a copy of the entire HTTP-Server Directory has to be created. In this example, the contents of the HTTP-Server folder is copied to “HTTP-Server”. Each copy contains a unique config file.

Additonally, the “C:\CCMS_NAV” folder has to be copied (e.g. “CCMS_NAV2”). According to this, the settings of the folders in the Microsoft Dynamics NAV Mobile Setup and the HTTP-Server Setup have to be changed.

The Windows Command Prompt must be run as administrator.

Important

The following commands must contain the space character after the “=”.

Register an additional Windows Service:

sc \\<Hostname> create CCMSHTTPServer2 binpath= "%programfiles[(x86)]%\Cosmo Consult AG\cc mobile solution\HTTP-Server 2\CosmoConsult.MobileSolution.Server.HttpServer.exe" DisplayName= "cc mobile solution - HTTP-Server Service 2" start= auto

Add description:

sc \\<Hostname> description CCMSHTTPServer2 " cc|mobile solution - HTTP-Server for Microsoft Dynamics NAV, No. 2"

Delete Windows Service:

sc \\<Hostname> delete CCMSHTTPServer2

HTTP-Server Start (as Windows Service)

Look for the Windows Service with the name “cc|mobile solution – HTTP-Server Service” in the list of Windows Services. Click Start Service on the toolbar, or Action > Start on the menu bar, or right-click and select Start.

HTTP-Server Start (as Windows Forms Application)

To start the HTTP-Server, click Start on the toolbar or Server > Start on the menu bar.

: HTTP-Server Start

On receiving the first requests, the HTTP-Server displays the responses in the console.

The Registered Devices tab displays the devices that are connected to or attempted to connect to the HTTP-Server. Depending on the license, the HTTP-Server allows only a certain number of devices. For example, if 11 devices send a request but only 10 devices are allowed, then the 11th device will receive an error message stating that only 10 devices are licensed.

The Process Dialogues tab displays the available process dialogues and contains information about which process dialogue is in use.
