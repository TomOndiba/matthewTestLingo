Document Editor

This lesson explains the page editor that is used for creating and editing pages.

: Document Editor

The General, Information and Universal Dlg. Settings FastTabs contain information and settings for the respective page. The data for creating the page can be found in the Lines FastTab.

General FastTab

The fields on this FastTab are described in the following table.

Field

Description

Name

Indicates the name of the document. The document is called up by the HTTP request using this name.

Note

The document name is specified without a file extension.

Codeunit

Specifies the object number of the codeunit to be assigned to this document, which is used to fill the layout with data.

Note

Documents that do not have to be filled with dynamic data can be generated via codeunit 5308021, MS - Call Fix Document.

Codeunit Name

Displays the name of the codeunit specified in the Codeunit field.

Description

An optional description of the document

Status

Shows the validation status of the page. The options are:

  • <blank> – The document contains errors or has not yet been validated and can therefore not be called up.
  • Released – The document has only been validated internally (without DOM) and is free from errors. It is available for calling up.
  • Validated – The document has been validated internally and according to DOM and is free from errors. It is available for calling up.

Note

The status can be changed manually to “<blank>” or “Released” only.

Module

Indicates the name of a module to which the page is to be assigned.

Style Sheet File

Assigns a .css file to the document.

Lines FastTab

The layout of the page is specified in the Lines FastTab. The structure corresponds to a tree structure transposed to a line structure. The individual levels are indicated by indentations.

A page is set up according to the parent-child principle. The top element is always the deck. The next refinement level is the data sheet. A data sheet in turn can then have various elements such as a label, text field or table. Depending on which element the child belongs to, it is indented in the display, i.e., the children of an element are always indented one level further than the element itself. All elements on the same level within an element are therefore siblings.

The fields on the Lines FastTab are described in the following table.

Field

Description

Object

Contains the name of the element or attribute. Elements are shown in bold text; attributes are shown in default text.

Value

Indicates the value for the element or attribute. For certain attributes or elements, values can be selected via the lookup.

The system indicates which type of input is expected through the following color formats:

  • Unique values are marked in green
  • Values marked in blue require a selection from a predefined list
  • Static values are shown in light gray

Usage of the Document Editor

When a new deck is created, the main element (deck) is automatically inserted. Starting with this element, further elements and attributes can be added by setting the focus to the Object field. The lookup provides a menu of elements and attributes that can be added to this element as the child node. If the cursor is placed in the Object field, a Lookup page can be opened, which lists all elements and attributes that can be added to the structure. Universal Dlg. Settings FastTab

The settings of this FastTab determine the individual configuration of the Universal Dialogue for the current document.

: Document Editor, Universal Dlg. Settings FastTab

The fields on this FastTab are described in the following table.

Field

Description

Channel

Identifies a four-character code field that is a unique assignment of a channel for which the Universal Dialogue will use. Depending on this value the dialogue associates its calling codeunit.

Changing the value of existing dialogues is not recommended.

Title

Identfies a labeltext to specify the title that will be displayed when using the Universal Dialogue.

Confirm Post

Specifies if it is necessary to confirm the data before posting.

Post per Line

Specifies if the posting occurs on each captured record or after collecting multiple records.

NOTE

This feature is not supported by every dialogue.

Print

Specifies if a document should be printed after successful posting.

NOTE

This feature is not supported by every dialogue.

Actions Tab

The Actions tab of the ribbon contains five actions. Each is described in the following table.

Action

Description

Import/Export

Imports or exports selected document structures to or from an ASCII file using a dataport.

Validate

Checks the current document for correctness according to the schema. Depending on the settings in the editor setup, the document is checked against the schema in a DOM validation.

Validate All

Checks several documents for correctness according to the schema using appropriate filters.

Copy

Duplicates complex document structures. Following the creation of a new document header, a report is started in Microsoft Dynamics NAV in which the source and target document are selected. Within the report, the document lines are copied to the target document.
