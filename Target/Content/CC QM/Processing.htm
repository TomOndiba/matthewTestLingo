Processing

This section explains how and where verifications are processed.

Verify Lines

Besides the protection of critical master data, the cc|regulatory affairs module enables the protection of all critical business processes by selectively assigning password and documentation duties.

The verification of transaction data occurs exclusively in the Verify Lines page, which is accessed from the cc|regulatory affairs menu by clicking Verification > Verify Lines (or Tasks > Verify Lines).

: Verify Lines page

In the General section, users can filter the table according to different areas or the Created by user. Depending on the setup, an entry in the following fields may be required:

Verify Value

Verify Value 2

Verific. Description

Verific. Description 2

After entering the verification value – and verification description, if required – in the line, click Verify on the ribbon to open the Login page, where the user ID and password is entered to complete the verification process. Completely verified lines are removed from the Verify Lines page.

If second user verification is required, the line in the Verify Lines page for the first user will become non-editable and grayed out but the Verify Value 2 and Verific. Description 2 fields remain editable for all other authorized users. Refer to topic "Four-Eyes Principle" on page 1 for more information about second user verification.

If any verified data is changed (on the originating document, journal, etc.), the verification must be executed again.

Verification of Master Data

This type of verification can be processed for the following master data records:

  • Production BOM
  • Routing

If the verification has been activated for the above master data, a modification of the status of a record (New, Certified, Under Development, or Closed) must be verified by the user.

Based on the settings in the Verification Setup page, the user must enter his or her password, a description (= reason for modification) or the User ID and password of a different user (= approval by different user).

: Password and description verification required to certify the Production BOM

After verification, the status of the according document is changed automatically.

Note

An additional Verification Comment can be added to the Verification Entry authorization by clicking Navigate > Comment.

Verification Entries

Processed master data verification entries can be viewed in the Verification Entries page, accessible on the cc|regulatory affairs menu by clicking Verification > Verification Entries (or Lists > Verification Entries).

: Processed verification entries

Verification Entries for specific master data lines (including production order components and routings) can be accessed by clicking Verific. Entries on the ribbon.

: Processed verification entries for changes to the COSM-001 Production BOM

Four-Eyes Principle

cc|regulatory affairs requires two authorized users to execute verification processes if the different user verification (option +Diff. User) has been selected in the Verification Setup. Therefore, the system saves the information of the first user who verifies the process or action and then demands that a different user also verify the process or action.

The verification process is handled differently in each verification page:

  • Verify Lines – Two verification value and description fields exist and the Diff. User Required check box is selected on lines that require double user verification.
    • The first user enters the value and description in the Verify Value and Verific. Description fields.
    • The second user can edit the Verify Value 2 and Verific. Description 2 fields.
      The line is not grayed out for the second user if logged into his/her account. Once verification is complete, the line is removed from the Verify Lines page.
  • Verification Entry – Opens twice for user verification.
    • The first user enters his/her password and description.
    • A message appears that a second user is required to complete verification.
    • The page re-opens for the second user to enter his/her password and description.

Once the double user verification is completed, the user can continue with the process or action.

Exercise – Process a Manufactured Item with Verifications

Item Q-40006, Shower Gel Bulk, is ready to be sold to customers directly. The net weight for this item’s test plan needs to be changed from an upper limit of 205 ML to 204 ML, which requires a different user and documentation verification.

Some raw materials need to be purchased from vendor V00020, which requires password verification and then 500 units of item Q-40006 need to be produced, which requires password and documentation verification on output.

After a Test Order is created, the lot needs to be released, which requires password and documentation verification.

The final step is to ship 50 units of this lot to customer 49858585, which requires password verification. 

Important

Prior to starting this exercise, delete any Retain Sample Instructions for Item Category Code = SEMI or Item No. Q-40006 (refer to topic "Retain Sample Instructions" on page 1).

Modify the Test Plan and Assign to Item

Follow these steps to modify a test plan and assign to item Q-40006:

  1. In the navigation pane, click cc|quality management > cc|quality management > Test Plans.
  2. Locate test plan TP00005 and open the Test Plan Card.
  3. Change the Status to Under Development.
  4. On the Lines FastTab, select the first line for Final Inspection and then click Property Set > Card.
  5. For Test Property Code TP00001, change the value in the Upper Limit field to 204.
  6. Click OK to close the Test Plan Prop. Set Card page.
  7. Certify the Test Plan.
    The Verification Entry page opens.

: Verification Entry

  1. In the User ID field, click the arrow and select NT.
  2. In the Password field, type 12S4%.
  3. Click OK (or press CTRL+ENTER) to close the page without entering a description.
    A message appears stating that the Description must have a value.

: Message received when a required description is not entered with the verification

  1. Click OK to close the verification description message.
  2. In the Description field, type “Change net weight upper limit”.
  3. Click OK.
    A message appears stating that a second user is required to complete the verification.

: Different user verification requirement to certify the test plan

  1. Click OK to close the second user verification message.
    The Verification Entry page reopens for the second user verification.
  2. In the User ID field, click the arrow and select the OSCAR (blocked) user.
  3. Press ENTER or TAB.
    A message appears stating that the user must not be Blocked.

: Message received for entering a blocked user ID

  1. In the User ID field, click the arrow and select the EMIL user.
  2. In the Password field, type @bCD3.
  3. In the Description field, type “Change net weight upper limit”.

: Verification User ID, Password and Description provided to certify the Test Plan

  1. Click OK.
    The Test Plan is certified.
  2. Click OK to close the Test Plan Card page.
