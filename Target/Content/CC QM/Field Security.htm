Field Security

With field permissions, it is possible to specify modification authorizations down to the field level. These settings can be established for a certain user and/or security role. The Field Permissions page is accessed from the cc|regulatory affairs menu by clicking Administration > Field Permissions.

: Field Permissions page

In this page, define which fields are to be editable by which user and/or role. The fields in this page are described in the following table.

Field

Description

Type

The options are <blank>, User and Role.

With the respective option, specify if this permission is to be valid for a certain user or role – a group of users. If this option is left blank, the permission / restriction will be applicable for all users.

ID

Depending on the chosen option in the Type field, select either a certain User ID or Role. If this field is left blank, the permission or restriction will be applicable for all users and roles.

Table ID

Identification number of the table for which to specify the permissions / restrictions.

Field No.

Identification number of the field for which to specify the permissions / restrictions. The displayed list depends on the chosen table in the Table ID field.

Modification Allowed

If selected, the selected user/role is allowed to modify the field.

Table Caption

Caption of the table that has been entered in the Table ID field.

Field Caption

Caption of the field that has been entered in the Field No. field.
