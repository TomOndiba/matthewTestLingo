QM Permissions

QM permissions specify the authorizations for the execution of Good Manufacturing Practice (GMP)-critical functions, such as the changing of lot data, the release of test orders or the release of production orders. Each of these activities can be cleared or blocked for a certain user (User ID) or a role (Role ID).

: QM Permissions page

The fields in this page are defined in the following table.

Field

Description

Action Type

Identifies the type of lot data change that can occur, including Status, Expiration Date, Entry Date, External Lot Number, Quarantine Date, Retest Date and Warranty Date.

Furthermore, authorizations can be defined for the change of Lot Number and Serial Number; the release of a Test Order; the release of a Production Order; and the certification of a Stability Test.

User ID

User ID, for which this data record is validated. If this field is empty, the setting applies for all users.

Role ID

Role ID, for which this data record is validated. If this field is empty, the setting applies for all roles.

Item No.

Specific item for which this data record is validated.

Variant Code

Depending on the item, defines an item variant for which this data record is validated.

Item Category Code

Specific item category for which this data record is validated.

From Status / To Status

Defines what status changes can be executed if the Activity Type field is set to Status Change.

Allowed

If selected, specified changes can be executed in the data record.
