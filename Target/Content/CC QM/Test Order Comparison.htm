Test Order Comparison

When the testing from a Test Order has failed, the lot may be sold to a specific customer that has different specifications. To run the comparison from a Test Order to see which customer’s specifications match the testing results, click Actions > Compare Test Results on the ribbon.

: Test Order Properties Compare page for a failed test order for item Q-40014, LOT0005 with measurement properties that are acceptable for customer 20000

For example, customers 20000 and 30000 have different measurement specifications. Different test plans are set up with the lower and upper limits for each measurement that is acceptable for each customer and are then assigned to the item’s Test Plan Setup (Source Type = Sales Order and Source Code = customer number). 

A Test Order is processed with Width measurements that fail the lower and upper limits defined for both the item’s test plan and customer 30000 test plan (19.00..21.00). However, the determined value is within the acceptable limits for customer 20000 (19.00..22.00).

When the Test Order Properties Compare page is opened, all measurement properties for customer 30000 contain an “X” while the acceptable ranges for customer 20000 appear (refer to Fig. 28).
