Quality Management Setup

In the Quality Management Setup page, basic cc|quality management settings are defined as well as number series for the individual areas of the test order process.

: Quality Management Setup page

The fields on the General FastTab are described in the following table.

Field

Description

Test Order Unblocks Test Device

If selected, test devices are automatically released immediately after finishing a test order. By finishing the test order, the status of the test device is set to Certified.

Use Without Expected Value

If selected, the Valid Determined Value field on the Value Recording page is calculated based on the Upper Limit and/or Lower Limit fields and therefore, the value in the Expected Value field is not considered.

Refer to topic “Interpret Determined Value” on page 1for more information.

Prod. Test Order Creation

Determines whether and when Test Orders for production are created automatically. The options are:

  • Manually – Test Orders are not created automatically.
  • Prod. Order Release – Test Orders are automatically created when a production order is released.

This function does not work when a Released Production Order is created manually. Additionally, item tracking must exist on the main production order line and any component lines (if required) in order to release the order.

  • First Output – Test Orders are automatically created when the first output is posted for a released production order.

NOTE

This function requires the Test Plan Setup for production-related test orders (refer to topic “Test Plan Setup on Items” on page 1).

Storage Path - Ext. Documents

Specifies a default path where Test Devices (Certificates), Test Orders (External Document) and Test Plans (External Documents) are to be stored. The lookup opens a folder selection page to identify where all external documents should be stored.

Document Storage Point

Determines if the document is stored as either a BLOB (internal) in the records or in a path on the network (external).

Default Blocked Item Location Code

Blocked location where a lot will be automatically transferred if the Inspection Result on its test order is Items are not OK.

Lot-/Serial No.-Status on Test Order Release

Status that a lot- or serial number must inherit when a test order is Released; used with the Adjust Lot Status on Test field in Test Plan Setup (refer to topic “Test Plan Setup on Items” on page 1).

Lot-/Serial No.-Status on Test Order Finish

Status that a lot- or serial number must inherit when a test order is Finished; used with the Adjust Lot Status on Test field in Test Plan Setup (refer to topic “Test Plan Setup on Items” on page 1).

Inventory Overview Display

Determines whether the Quantity field on the Inventory Overview page is displayed in the current Unit of Measure or the Base Unit of Measure from the Item Card.

Default 8D Template No.

Indicates the default template to be used when creating 8D Reports (refer to topic “8D Template” on page 1).

Use Unspecified Status on Test Order

If selected, unspecified statuses can be used on Test Orders (refer to topic “Unspecified Inspection Result on Test Order” on page 1).

Retain Sample Status Code

Indicates the status that will be used for the retained sample, which is created with a new lot number from the posted lot output/shipment (refer to topic “Retain Sample Management” on page 1).

Test Sched. Dummy Item No.

Indicates the dummy item number that will be used for the Test Order Scheduling process (refer to topic “Test Order Scheduling” on page 1).

Default Sched. Work Center No.

Indicates the default work center number that will be used for the Test Order Scheduling process (refer to topic “Test Order Scheduling” on page 1). This Work Center will be selected for rescheduling Test Orders based on system date plus Wait Time , when no testing activity is assigned.

Update From Certified Plans

Determines whether the latest valid Certified Test Plan is used for copying or all other Statuses are considered as well when running the Update Test Plans function in Test Properties or Test Property Sets. Refer to topic“Mass Updates of Test Plans” on page 1 for more information.

On the Numbering FastTab, number series are defined for test properties, test property sets, test devices, test plans, test orders, stability tests, 8D reports, sample occurrences, test incidents and simulated orders.
