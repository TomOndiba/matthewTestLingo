Exercise – Set Up and Test Field Permissions

As the database administrator, you are responsible for setting up the field permissions. Before completing a full field permission setup, you decide to set up a test environment to ensure the field permissions are controlled as needed.

You first enter the database-wide access and Field Permission table to your user ID. You then complete settings to test the following scenarios:

  • To test the individual user setup, grant only Description field modification access on the Item table to user SUSAN.
  • To test the SUPER role setup, grant Item Tracking Code field and Description field modification access on the Item table.
  • To test the User/Role setup, remove the Item Tracking Code modification access on the Item table to user SUSAN, who is a member of the SUPER role.

After you enter all the settings, you test the scenarios.

Note

Ensure that both users set up in this demonstration are assigned to the SUPER security role.

Enter Field Permissions

Follow these steps to enter field permissions:

  1. Login as NT.
  2. In the navigation pane, click cc|regulatory affairs > Administration > Field Permissions.
  3. On the first line, tab through all the fields to have all fields blank, except the Modification Allowed check box selected.
  4. On the next line, enter the following:
    1. Type = User
    2. ID = NT
    3. Table ID = 5060475
    4. Modification Allowed = selected
  5. Go to the next line and enter the following to prevent all users from changing the Description field on the Item table:
    1. Table ID = 27
    2. Field No. = 3
    3. Modification Allowed = not selected
  6. Go to the next line and enter the following to prevent all users from changing the Item Tracking Code field on the Item table:
    1. Table ID = 27
    2. Field No. = 6500
    3. Modification Allowed = not selected
  7. Go to the next line and enter the following to allow user SUSAN to change the Description field on the Item table:
    1. Type = User
    2. ID = SUSAN
    3. Table ID = 27
    4. Field No. = 3
    5. Modification Allowed = selected
  8. Go to the next line and enter the following to prevent user SUSAN from changing the Item Tracking Code field on the Item table:
    1. Type = User
    2. ID = SUSAN
    3. Table ID = 27
    4. Field No. = 6500
    5. Modification Allowed = not selected
  9. Go to the next line and enter the following to allow role SUPER to change the Description field on the Item table:
    1. Type = Role
    2. ID = SUPER
    3. Table ID = 27
    4. Field No. = 3
    5. Modification Allowed = selected
  10. Go to the next line and enter the following to allow role SUPER to change the Item Tracking Code field on the Item table:
    1. Type = Role
    2. ID = SUPER
    3. Table ID = 27
    4. Field No. = 6500
    5. Modification Allowed = selected

: Field Permissions with Item table restrictions/access

  1. Click OK to close the Field Permissions page.

Test Field Permissions for User SUSAN

Follow these steps to ensure that user SUSAN can change the Description field but not the Item Tracking Code field on the Item Card:

  1. Login as SUSAN.
  2. In the navigation pane, click Items (on the Home button).
  3. Locate item Q-20005 and open the Item Card.
  4. Change the value in the Description field to “Zinc Chloride”.
  5. Click OK to close the Item Card page.
    The Item Card closes and the description is changed.
  6. Reopen the Item Card.

: Description field changed on Item Card

  1. Expand the Item Tracking FastTab.
  2. In the Item Tracking Code field, enter LOTALL.
  3. Click OK to close the Item Card page.
    The following message appears that the user does not have permission to modify the field.

: Field change permission error

  1. Click Yes to discard the change.
    The Item Card closes and the Item Tracking Code field is not changed.
  2. Reopen the Item Card and review the Item Tracking Code field.

: Item Tracking Code field unchanged on Item Card

  1. Close the Item Card page.
