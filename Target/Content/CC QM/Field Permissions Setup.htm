Field Permissions Setup

The following procedure can be regarded as a reasonable Field Permission setup:

  1. On the first line, create a data record which allows all modifications with the following settings:

Type = <blank>; ID = <blank>; Table ID = 0; Field No. = 0; Modification Allowed = selected

  1. Grant permission to one user or role to change everything (same as step 1 except with a user or role assigned) or assign the Field Permission table (5060475) to a user or role with the Modification Allowed check box selected.

IMPORTANT

If this step is not done, no field permission changes can be made once the Field Permissions page is closed or after Microsoft Dynamics NAV is restarted.

  1. All critical fields should generally be blocked for modifications.
  2. These fields can then be released for modifications by authorized users or roles.

Field Permission Levels

Field permission set up can be accomplished on different levels of detail that are checked by the system in the following order. The check occurs if the user leaves the respective data record. Example

The following example shows how users set up with and without field permissions are allowed to modify fields on the Item Card based on the system level checks in the above table.

: Field Permissions page example setup

Based on the setup in Fig. 1, the following results occur:

  1. The Gen. Prod. Posting Group field on the Item Card is modified by the user LISA, who has not been set up with field permissions and who is not a member of the SUPER role. When doing a check on level 3 (refer to table above), the system locates the data record in the third line of the Field Permissions page. The user LISA is not allowed to modify the field. The system returns a message that she does not have Field Permissions and the field keeps its original value.
  2. The Gen. Prod. Posting Group field on the Item Card is modified by the user SUSAN. When doing a check on level 1 (refer to table above), the system locates the data record in the seventh line of the Field Permissions page. The user SUSAN is allowed to modify the field.
  3. The Costing Method field on the Item Card is modified by the user SUSAN. When doing a check on level 4 (refer to table above), the system locates the data record in the sixth line of the Field Permissions page. The user SUSAN is not allowed to modify the field, because user SUSAN is only permitted to change the Gen. Prod. Posting Group field on the Item Card.
  4. The Gen. Prod. Posting Group field on the Item Card is modified by the user NT, who is a member of the SUPER role. When doing a check on level 2 (refer to table above), the system locates the data record in the ninth line of the Field Permissions page. The user NT is allowed to modify the field.
