Inventory Posting Permissions

In the Inventory Posting Permissions page, define users and roles (groups of users) for lot- or serial number-related postings and specify their posting authorizations in detail.

: Inventory Posting Permissions page showing a portion of the complete setup

The fields in this page are defined in the following table.

Field

Description

Posting Allowed

If selected, this data record is about permission or restriction. It specifies, for example, if a user or role is allowed to execute a posting or not.

Item Tracking Assignment Allowed

In contrast to the Posting Allowed field, this field determines if the creation of an Item Tracking line or the execution of a reservation is allowed.  This means – that with the Posting Allowed and Item Tracking Assignment Allowed fields – a two-stage check can be set up:

  1. Item Tracking Assignment Allowed – determines whether a lot-/serial number assignment is permitted for the selected conditions.
  2. Posting Allowed – determines which restrictions apply to the posting of a material movement.

Entry Type

Determines for which entry type (document type) restrictions are valid. The options are:

  • Purchase (Order, Invoice, Credit Memo, Return Order, Warehouse Receipt)
  • Sale (Order, Invoice, Credit Memo, Return Order, Warehouse Shipment)
  • Positive Adjustment (Item Journal, Warehouse Item Journal, Physical Inventory, etc.)
  • Negative Adjustment (Item Journal, Warehouse Item Journal, Physical Inventory, etc.)
  • Transfer (Picking, Put-away, Item Reclassification Journal, Transfer Order)
  • Consumption (Consumption Journal, Production Journal)
  • Output (Output Journal, Production Journal)
  • Whse. Transfer (Bin transfers – Picking, Put-away, Item Reclassification Journal)

If this field is left blank, then the record is applicable for all entry types unless the settings are overruled by a record using a more detailed filtering.

Action Type

Specifies if the record is about an inbound or an outbound material movement. Inbound material movements always describe a positive adjustment whereas outbound material movements describe a negative adjustment (for example, Purchase – Outbound = Purchase credit memo; Purchase – Inbound = Receipt).

Lot-/Serial No. Status

Limits the posting responsibility assignment of lot- or serial numbers to a certain lot or serial number status. Each status defined in the Lot-/Serial No. Status page are available.

Item No.

Identifies an item for which a posting authorization is required. For example, it can be set that a certain user is not authorized to post this item. 

Variant Code

Identifies an item variant for which a posting authorization of lot- or serial numbers is required. For example, it can be set that a certain user is only authorized to post this item variant.

Item Category Code

Identifies an item category for which a posting authorization is required. For example, it can be set that this item category can only be posted by users with a certain role.  

Location Code

Identifies location-specific restrictions and thus prevents or allows material movements to a certain location. For example, it can be set that a lot or serial number with the status Blocked cannot be posted to the picking location.

Zone Code

If locations are used that have the Directed Put away and Pick check box selected on the Location Card and thus zones can be entered for this location, a permission or restriction for the posting or assignment of lot- or serial numbers can be set up for certain zones.

Bin Code

If a warehouse is handled in bins, postings to a certain bin code can be restricted. Furthermore, a bin can be excluded from postings.

For example, it can be set that a lot- or serial no. with the status Blocked is only to be posted to the quarantine bin. It can also be set that blocked material must not be transferred to Manufacturing.

Role ID

Identifies a specific role that the stated specifications apply to. If no specific role and user is assigned, this authorization applies for all users of the system.

User ID

Identifies a specific user that the stated specifications apply to. If no specific user and role is assigned, this authorization applies for all users of the system.

Inventory Posting Permissions Setup

For the ability to create posting responsibility lot- or serial number assignments (by entering an item tracking line or a reservation) it is necessary to complete setup in the Inventory Posting Permissions page. Blank fields indicate that there are no restrictions on posting authorizations. If a line does not specify any role or user, the setting is valid for all users in the system.

System Checks – Single Data Record

If a user executes a posting or assigns a lot- or serial number, the system checks all actions that the role or user is permitted to execute. Restrictions can be defined on several levels of detail. The system checks the settings according to the following order, whereby the filters are set with “field content and/or blank” (e.g. 4711|’’):

Examples

  1. User “WS” tries to post an inbound lot on location code BLUE.
    During the authorization check on level 1, the system finds a data record in the setup table. As a result, user “WS” is not permitted to execute lot postings on location BLUE. The transaction will be cancelled with an according error message.
  1. User “COSMO” tries to post a lot of the item category Raw Materials to warehouse bin S-08-0003 in warehouse location SILVER.
    During the authorization check on level 5, the system finds a data record in the setup table. As a result, user “COSMO” is permitted to post the respective item on bin S-08-0003 in location SILVER. The posting can be executed without issue.
  1. User “COSMO” tries to post an Outbound Consumption for a lot with the status Blocked. During the authorization check, the system does not find an appropriate data record within the table. As a result, the posting is interpreted as not permitted and cancelled with an according error message.
System Checks – Multiple Data Records

If several data records are detected after setting all filter options according to the recent posting activity, the system gives points corresponding to the priorities. The data record with the maximum number of points is the accurate one for this posting activity.
