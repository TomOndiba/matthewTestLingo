Mass Updates of Test Plans

Changes made to Test Plan Properties and Test Plan Property Sets can be updated at once in all related Test Plans where changed Test Plan Properties/Test Plan Property Sets are assigned.

When this function is run, the current, valid Test Plan may be marked with a Valid to date and a new Test Plan version may be created with a Valid from date. The new Test Plan version is a copy of the original Test Plan version. It will include all information for all other Property Sets/Properties and the updated information for the current Property Set or Property.

In the Test Plan Properties and Test Plan Property Set pages, the “Update Test Plans” action opens a request page with several options.

: Update Test Plan Property and Update Test Plan Property Set request pages

The fields in the Validity FastTab are described in the following table.

Field

Description

Active Version “Valid To”

Indicates the date to which the current Test Plan Version will be valid.

Typically this will be the day before the New Version “Valid from” date.

New Version “Valid from”

Indicates the date from which the new Test Plan Version will be valid.

Typically this will be the day after the Active Version “Valid to” date.

Use Record Selection

Opens a page that lists all test plan properties and test plan property sets that can be selected for updating.

Copy From Certified Test Plans

If selected, the latest, valid certified Test Plans will be included in the update process. Otherwise, the latest version will be selected and status together with valid date period is not relevant.

This setting defaults from the Quality Management Setup but can be changed in the request page.

 

Updating Test Plans from Test Properties

Although most settings in Test Properties are typically defined in the Test Property Sets, there are instances where a standard criteria is valid for all tests. If one of these criterion is changed, the “Update Test Plan” function will handle this change in all related Test Plan Property Sets in the Test Plans.

Note

The actual Test Plan Property Sets are not changed with this function.

Example

Test Property TP0000109 has been set up with an expected value and upper/lower limits.

: Expected value and Upper/Lower limits in Test Property Card

Due to changes in the lab, the Upper Limit has been raised to 1.10. The change is made to the Upper Limit field and then the “Update Test Plans” function is run.

: Changed Upper Limit and preparing to run Update Test Plans function

Before completely running the update function, the Use Record Selection and Copy From Certified Test Plans check boxes are selected to view the certified test plans that will be affected by this change.

: Preparing to view the Test Plan Properties record selection page

This Test Property appears in one Test Property Set (TPS00015) that is used in one version of Test Plan (TP00014). Also note that the Upper Limit is still set to 1.09 for this test property/plan.

: Reviewing the entry in the Test Plan Properties record selection page

Since no validity dates were added in the Update Test Plan Property request page, the page is closed by clicking Cancel.

Note

If validity dates were added in the request page prior to using the record selection page, the update process can be run by clicking OK.

The “Update Test Plan” function is run again, this time dates are added in the request page and the process runs without opening the record selection page. The Copy From Certified Test Plans check box is selected to ensure that currently certified Test Plans are included.

: Preparing to run the Update Test Plans function

The update process runs by clicking OK. A message appears stating the number of records that were updated and the record selection page opens again to show the records that were updated.

The Test Plans are reviewed: the original test plan (version 1) has a Valid to date of 1/31/2019 and the new test plan (version 2) has a Valid from date of 2/1/2019. The new Test Plan is set to Status = New so that it can be checked and set to certified by a valid user.

: Original and new versions of test plan TP00014

In the new Test Plan, the changed Test Property is confirmed in the Test Property Set.

: Updated Test Property in Test Property Set in new version of test plan TP00014

The Test Plan is certified once any further review or changes are made.

Updating Test Plans from Test Property Sets

In most situations, Test Plans should be updated from the Test Property Set because of the specifically defined properties. Making changes to settings in Test Property Sets to be updated in Test Plans is the same process as described for Test Properties.

Example

Test Property Set TPS00001 has been set up with three Test Properties with expected values.

: Test Properties with Expected Values in Test Property Set Card

Due to changes in the lab, the Expected Value of Test Property TP000041 has been changed from BLUE to GREEN. The change is made to the Expected Value field and then the “Update Test Plans” function is run.

: Changed Expected Value and preparing to run Update Test Plans function

Since the update function can be run after using the record selection page, validity date are added to the request page, along with the selected Use Record Selection and Copy From Certified Test Plans check boxes to view the certified test plans that will be affected by this change.

: Preparing to view the Test Plan Properties record selection page

This Test Property Set appears in two Test Plans (TP00006 and TP00021).

: Reviewing the entries in the Test Plan Property Set record selection page

Since changes are ready to be made, both lines are selected, the page is closed by clicking OK and the update process runs. A message appears stating the number of records that were updated.

Note

When updating multiple Test Plans, each line to be included in the update must be selected.

The Test Plans are reviewed: the original test plans (version 1) have Valid to dates of 2/27/2019 and the new test plans (version 2) have Valid from dates of 2/28/2019. The new Test Plans are set to Status = New so that each can be checked and set to certified by a valid user.

: Original and new versions of test plans TP00006 and TP00014

In the new Test Plans, the changed Test Property is confirmed in the Test Property Set.

:Updated Test Property Sets in new version of test plans TP00006 and TP00021

The Test Plans are certified once any further reviews or changes are made.
