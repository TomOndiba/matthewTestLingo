Test Plan Setup on Items

The Item Test Order Option, Sampling Method and Inspection Severity Code can be allocated to the respective item in the Test Plan Setup page, which is accessed from the ribbon on the Item Card.

: Test Plan Setup for item Q-20001

In this page, assign test plans to different Source Types (such as Purchase Orders, Production Order and so on) item (group) specifically. Furthermore, this page offers the opportunity to enter customer-specific or vendor-specific test plans.

The fields in the header of this page are described in the following table.

Field

Description

Source Type Filter

Source type to allocate the test plan to. The options are Inventory, Purchase Order, Purchase Receipt, Sales Order, Sales Return Order, Sales Return Receipt, Production Order, Operation, None, Order-specific and Component.

Source No. Filter

Number of the source type selected in the Source Type Filter field.

Item No. Filter

Item to be affected by these settings.

Item Category Filter

Item category to be affected by these settings.

The fields in the lines are described in the following table.

Field

Description

Item Test Order Option

Defines the level on which an item should be tested. The options are:

  • No Test – no test orders are created.
  • Test on Item Level – a single test order is created for the item, regardless of lot- or serial numbers.
  • Test on Lot No. Trading Unit – a test order is created for each lot number.
  • Test on Serial No. – a test order is created for each serial number.
  • Test on Trading Unit – a test order is created for each trading unit of a lot.

For example, if a user posts a Receipt with two lots (Lot A and Lot B) the following tests are created, based on the Item Test Order Option:

  • Test on Item Level:

One Test Order – Item No. is populated in the General FastTab; lot information is empty on the Test Plan Source FastTab

  • Test on Lot No. Trading Unit:

Two Test Orders (one for each lot) – Item No. and Lot No. are populated

Note

In cc|process manufacturing, trading units are an integral part of the storage unit functionality. Therefore, the “Test on Lot No.” option was renamed to “Test on Lot No. Trading Unit” to accommodate the integration of cc|quality management with cc|process manufacturing.

Sampling Method

Defines for the item if the sampling method according to Oeser/Strohecker or DIN ISO 2859 should be applied.

If the DIN ISO 2859 option is selected, the Inspection Severity Code and Sampling Method fields must be filled with the ISO2859 or DIN2859 values. In this case, the system will apply an acceptance sample testing based on faulty parts or errors.

Inspection Severity Code

Required if the DIN ISO 2859 option is selected in the Sampling Method field.

Insp. Severity on Test Prop.

Indicates whether an inspection severity has been assigned to a test property (refer to topic “Inspection Severities of Test Properties” on page 1).

Inspection Level

Indicates the Inspection Level for counting (AQL) to be used for testing; if this field is empty, Inspection Level II is used.

Test Plan No.

Number of the test plan that should be allocated to the item depending on the source type.

Adjust Lot Status on Test

If selected, the lot status will automatically be set to:

  • The value in the Lot-/Serial No.-Status on Test Order Release field in the Quality Management Setup page when a test order is Released.
  • The value in the Lot-/Serial No.-Status on Test Order Finish field in the Quality Management Setup page when a test order is Finished.

No Test Order on Subcontr.

If selected, prevents duplicate testing requirements when creating a Purchase Order for a Production Order line, which may already contain Test Plan Setup (Source Type = Purchase Receipt) for the subcontracting order.

Test Wait Time

Indicates the time needed to start testing, beginning from the Test Order creation. For example, it may define offset time to start testing after goods receipt posting. The Wait Time Unit of Meas. Code field is required to specify this time.

Refer to topic “Test Order Scheduling” on page 1 for more information.

Wait Time Unit of Meas. Code

Used with the Test Wait Time field to complete the time specification. Codes of Capacity Units of Measure (from manufacturing) can be assigned here.

Refer to topic “Test Order Scheduling” on page 1 for more information.

Copy Item Test Plan Setup

Existing Test Plan Setup records can be copied from a selected item to a selection of one or more items. This can be used from the Item Test Plan Setup or the Periodic Activities on the cc|quality management menu and provides the ability to copy the Test Plan Setup for a specific Source Type or for all existing Source Types of the “Template” Item.

Inspection Severities of Test Properties

Different schedules can be assigned to each test property so that testing is done based on the Inspection Severities of a property as opposed to the Inspection Severity of the overall test plan.

Note

Properties without a specific Inspection Severity will use the one assigned to the Test Plan Setup line.

On the ribbon of the Test Plan Setup page, click Property Inspection Severities to open the Item Test Prop. Insp. Severities page.

: Inspection Severities for Test Properties

As Fig. 6 illustrates, the first line has the Test Each Used check box selected, which means that a value has been entered in the Test Each (Times) field for the test property (refer to Fig. 2), and therefore an Inspection Severity cannot be entered on this line. A different Inspection Severity Code has been entered for the last property. When the page is closed, the Insp. Severity on Test Prop. field is set to Yes on the Test Plan Setup page.
