Setup

The two setup pages used for verification functionality are:

  • Verification Setup – defines the general settings and processes that require verification
  • User Authorization – defines which users are authorized to verify which processes

Each page (accessed from cc|regulatory affairs > Administration) is described in this section.

Verification Setup

In the Verification Setup page, general settings are defined, such as:

  • Verification activation
  • Maximum quantity of failed logins
  • Password requirements

Furthermore, users can specify all sensitive processes or actions that will be subject to verification.

: Verification Setup

General FastTab

The Verification Activated check box enables or disables the Verification functionality.

Area FastTabs

The area-specific FastTabs in the Verification Setup page contains several processes which can be included in the verification. The type of verification to be applied to each process can be specified as well. The available options are described in the following table.

Important

All corresponding documents MUST be posted before one of the options in the Verification Setup page is enabled. For example, to use verification on sales orders, all existing Sales Orders must be posted prior to selecting any options in the Verify Sales Order field.

Option

Description

None

No verification will be applied.

Password Check

An authorized user’s password must be entered.

Password Check + Documentation

An authorized user’s password and an additional description must be entered.

Password Check + Diff. User

The verification with User ID and password must be done by two authorized users.

Refer to topic "Four-Eyes Principle" on page 1 for more information.

Password Check + Documentation + Diff. User

The verification with User ID, password and documentation must be done by two authorized users.

Refer to topic "Four-Eyes Principle" on page 1 for more information.

Password Policies FastTab

On this FastTab, define the rules for verification passwords.

Field

Description

Max. Qty. Failed Logins

Number of incorrect logins that a user is permitted before his/her login is blocked.

Minimum Password Length

Minimum length of the password.

Password Change Interval

Number of days when a password expires; entered as a standard Microsoft Dynamics NAV date formula.

For example, every user must change his/her password every three months (3M) from the date it was first entered or last changed.

Password Change Warning Period

Number of days before a password expires. Entered as a standard Microsoft Dynamics NAV date formula, the system will issue a warning to remind the user to change his/her password soon.

For example, seven days (7D) before the password expires, a user will receive a warning that his/her password will expire in 7 days.

Prevent Use of Prev. Passwords

Indicates the number (up to six) of previously used passwords that can be issued by the user.

Min. No. of Uppercase Letters

Minimum number of uppercase letters that a password must contain.

Min. No. of Lowercase Letters

Minimum number of lowercase letters that a password must contain.

Min. No. of Numbers

Minimum number of numbers that a password must contain.

Min. No. of Non-Alphanumeric Characters

Minimum number of non-alphanumeric characters that a password must contain.

Exercise – Set Up Verifications and Password Policies

For the purpose of the other exercises in this manual, set up the following verifications:

  • Change Lot Information = password check + documentation
  • Post a Sales Order = password check
  • Post a Purchase Order = password check
  • Post Output = password check + documentation
  • Certify a Test Plan = password check + documentation + different user

Additionally, define the following password policies:

  • Max. Qty. Failed Logins = 3
  • Minimum Password Length = 5
  • Min. No. of Uppercase Letters = 1
  • Min. No. of Numbers = 1
  • Min. No. of Non-Alphanumeric Characters = 1

Set up Verifications

Follow these steps to set up the necessary sales and manufacturing verifications:

  1. In the navigation pane, click Departments > cc|regulatory affairs > Administration > Verification Setup.
  2. On the General FastTab, select the Verification Activated check box.
  3. Expand the Inventory FastTab and enter Password Check+Documentation in the Verify Status Change field.

: Verifications set up on General and Inventory FastTabs

  1. Expand the Sales FastTab and enter Password Check in the Verify Sales Order field.
  2. Expand the Purchase FastTab and enter Password Check in the Verify Purchase Order field.
  3. Expand the Manufacturing FastTab and enter Password Check+Documentation in the Verify Output Journal field.
  4. Expand the Quality Management FastTab and enter Password Check+Documentation+Diff. User in the Verify Certify Test Plan field.

Set up Password Policies

Follow these steps to enter the password policies:

  1. Expand the Password Policies FastTab.
  2. In the Max. Qty. Failed Logins field, type 3.
  3. In the Minimum Password Length field, type 5.
  4. In the Min. No. of Uppercase Letters field, type 1.
  5. In the Min. No. of Numbers field, type 1.
  6. In the Min. No. of Non-Alphanumeric Characters field, type 1.

: Verifications set up on Password Policies FastTab

  1. Click OK to close the Verification Setup page.

User Authorization

Define the sensitive processes and actions that a user is allowed to execute on the User Authorizations page.

: User Authorizations

Field

Description

User ID

Windows login for the authorized user.

Qty. Failed Logins

System-tracked number of failed logins.

If the user is released after having been blocked or types the correct password – before the Max. Qty. Failed Logins is reached – this field is reset to zero (0).

Blocked

If selected, the user cannot verify any specified processes.

This check box is automatically selected if the user reaches the Max. Qty. Failed Logins.

Expiration Date

Indicates the date until which the user is permitted to verify the specified processes.

Area settings

Defines the specific processes – from the area verifications on the Verification Setup page – to which the user is permitted to complete verifications.

For example, if verification is entered in the Verify Sales Order field on the Sales FastTab of the Verification Setup page, then verification is required. For a specific user to provide the verification, he or she must have the Verify Sales Order check box selected for his/her User ID in this page.

Set up Verification Passwords

Follow these steps to set up the verification passwords for the users:

  1. In the navigation pane, click cc|regulatory affairs > Verification > Change Verification Password.

: Change Verification Password page

  1. In the New Password field, type 123$.
  2. Press ENTER.
    Note the error that the password must be at least five characters long.
  3. Go back to the New Password field and type 1234%.
  4. Press ENTER.
    Note the error that at least one uppercase character must be used.
  5. Go back to the New Password field and type 12S4%.
  6. In the Reenter New Password fields, type 12S4%.
  7. Click OK.
    The password is accepted and the Change Verification Password page closes.
  8. Login as EMIL.
  9. In the navigation pane, click Departments > cc|regulatory affairs > Verification > Change Verification Password.
  10. In the New Password and Reenter New Password fields, type @bCD3.
  11. Click OK.
    The password is accepted and the Change Verification Password page closes.
  12. Login as OSCAR.
  13. In the navigation pane, click Departments > cc|regulatory affairs > Verification > Change Verification Password.
  14. In the New Password and Reenter New Password fields, type @Bcd5.
  15. Click OK.
    The password is accepted and the Change Verification Password page closes.
