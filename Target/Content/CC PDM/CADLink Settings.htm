CADLink Settings

The CADLink settings has various options for defining the import behavior and the BOM display.

General Settings

General settings, such as drawing type or standard values for unit or revision code, are entered in this tab.

: CADLink General Settings

The fields on the General Settings tab are described in the following table.

Field

Description

Show Preview Image

Defines whether the drawing or image describing the item is displayed directly in CADLink.

Drawing Type

Default drawing type, which is transferred to the ERP if one is not transferred from the CAD.

Default Base Uom

Default unit, if no unit is transferred from CAD.

Default New Revision

Default revision code, which is stored if one was not transferred from the CAD.

Create New BOM Version

If selected, no changes are made in the ERP system when the BOM is changed or items are deleted, but a new BOM version is created directly.

Use Default Category Code

Defines whether the default category code should be used, if it does not exist.

 

Field Mappings

The field mapping between CADLink and the ERP system determines exactly which values are transferred from a field in CADLink to the correct field in the ERP system.

: Field Mapping in CADLink

Color Codes

In the Color Codes tab, the colors and their meaning are defined for use in the BOM.

: Definition of Color Codes in CADLink

By default, the colors have the following meaning:

  • Yellow – There are changes to the part, which are adopted during import.
  • Green – The part does not yet exist in ERP and is created again.
  • Grey – The corresponding part no longer exists in the BOM and is removed from the document BOM in the ERP system.
