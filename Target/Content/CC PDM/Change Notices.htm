Change Notices

During the manufacturing process there are circumstances that lead to certain parts within a BOM not meeting the desired requirements. This can be, for example, a generator that does not provide enough power or a bicycle chain that cannot be used for a model because it is not working.

In such a case, a change must be made during manufacturing.

Example: Create a Change Notice

Open the production order in which the assembly is affected. Then choose Functions > Create Eng. Change Notice on the Lines FastTab.

: Create an Eng. Change Notice from a Production Order

Engineering Change Notice Card

The General and Comments FastTabs are described below. The Source Drawing Set and Source Document FastTabs only show which current drawing is affected and from which production order this change notice was created.

: Eng. Change Notice

General FastTab

The General FastTab shows the most important information about a change notice at a glance.

: Eng. Change Notice, General FastTab

The fields on this FastTab are described in the following table.

Field

Description

Change Reason Code

Specifies the reason for the change notice.

Change Reason Description

Basic description that defaults from the Change Reason Code.

Item No.

Number of the item affected from the production order.

Item Description

Description of the item affected in the production order.

BOM Comment Exists

Shows whether a comment exists in the area Eng. Change Notice BOM lines or not. This is independent of the number of comments. As soon as a comment exists, “Yes” is displayed. The BOM of the originating document is always copied to the change notice.

Drawing No.

Number of the affected drawing.

Note

The drawing number does not have to be that of the item. The user can select any drawing here.

Drawing Type Code

Type of affected drawing.

Status

Status of Eng. Change Notice:

  • Open
  • Pending Approval
  • On Hold
  • Approved
  • Rejected

Creation Date

Date of creation of data record.

Created By

User ID from user list, which has created the data record.

Comments FastTab

In the Comments FastTab, add information to a change notice.

: Eng. Change Notice, Comments FastTab

The fields on this FastTab are described in the following table.

Field

Description

Comment

Text field to describe the problem or reason why an item needs to be changed, at maximum 80 characters.

Date

Date on which the comment was entered.

User ID

User ID from the user list that is to make the change and must approve it.

Note

It is possible to delete or change lines only with your own User ID. The lines of other users cannot be edited.

Actions for Change Notices

Various functions are available on the Home or Actions tab for editing change notices.

BOM Lines

After the relevant information has been entered in the change notice, the BOM Lines of the Eng. Change Notice can be opened on the Home or Actions tab using the BOM button.

: Eng. Change Notice, BOM Lines with Comment

The fields on the Eng. Change Notice BOM Lines page are described in the following table.

Field

Description

Type

The type of data record to be addressed, e.g. Item.

No.

The number of the record that is affected, depending on the type, e.g. item number.

Description

Description of the data record.

Quantity

The quantity affected by the change.

Comment

Field for the change reason, specification of the type of material or assembly affected.

 

Send

As soon as all information has been entered on the card and anchored in the BOM lines of the Eng. change notice, the responsible person must choose Send on the Home or Actions tab in the Eng. change notice. The status changes to Pending Approval.

The change notice now appears in the “Pending Approval” Activities of the Eng. Change Notices.

During the processing, change notices can have the following statuses:

  • On Hold – If a change notice has been sent, the responsible engineer can set the status to Wait. This means that he or she accepts the change notice but will not process it until a later date.
  • Approved – If a change notice has been sent, the responsible engineer can set the status to Approved. This means that he or she accepts the change notice.
  • Rejected – If a change notice has been sent, the responsible engineer can set the status.

Open Item

This function opens the item card of the relevant item.

Open Source Document

This function opens the source document of the change notice.
