Drawings

In the Search box, enter Drawings, then choose the related link to open the Drawings page.

Drawings are the construction plans linked with the item. In Microsoft Dynamics NAV, these consist of four components:

  • List of Drawings
  • Drawing Card
  • Link Drawings with Documents (in header and in lines)
  • History

List of Drawings

The drawing list is the overview of all drawings. If an item uses different drawings, all drawings are displayed in connection with the No. field to the Type Code field.

: Drawings

The fields in the Drawings page are described in the following table.

Field

Description

No.

Number of drawing.

Type Code

Unique description of a drawing type.

Revision Code

Displays the current revision of the drawing.

Description

Brief description of the drawing.

Status Code

Displays the current status of the drawing.

Created from No.

Displays the number of the drawing from which the current status has emerged. This field is used to make revision statuses traceable.

Created from Revision Code

Displays the revision status of the drawing from which the current status has emerged. This field is used to make revision statuses traceable.

Note

It is possible to create a new revision status D from revision status A. B and C will then be outdated.

The drawing picture can be imported and exported normally. However, this is not handled the same as in standard Microsoft Dynamics NAV. The differences are:

  • Saving a picture in standard Microsoft Dynamics NAV – The respective picture is saved directly in the table of the item between the other fields.
  • Saving a picture in cc|product data management – The respective picture is regarded as a reference and retrieved from its storage structure.

This means that the underlying storage structure in the drawings must not be changed. If this should happen, no more drawings are shown in picture form here.

Drawing Card

The fields in the Drawing Card are used to specify each individual drawing.

Open the Drawing Card by selecting a data set in the drawing list and then choose Edit or View.

: Drawing Card

General FastTab

In this area the general drawing data is recorded, some of which can also be seen in the list. The fields on this FastTab are described in the following table.

Field

Description

No.

Number of drawing.

Type Code

Unique description of a drawing type.

Revision Code

Displays the current revision of the drawing.

Description

Brief description of the drawing.

Status Code

Displays the current status of the drawing.

Comment

An internal comment can be written for the drawing. This has no relevant effect on the further procedure.

 

History FastTab

All data regarding the last change is displayed on this tab. This makes it possible to trace exactly when a drawing was changed by whom at each revision status.

The fields on this FastTab are described in the following table.

Field

Description

Created from No.

Displays the number of the drawing from which the current revision status has emerged.

Created from Revision Code

Displays the revision status of the drawing from which the current status has emerged.

Creation Date

Date on which the data record was created.

Created by User ID

User ID from the user list that created the record.

Last Date Modified

Date on which a change was made to the current record.

Modified by User ID

User ID that last changed the record.

 

Link Drawings with Documents

Drawings are used in manufacturing to guide the assembly of parts to ensure that the correct drawing is used for a manufacturing part. Therefore, cc|product data management offers the ability to trace the currently used drawing status with links in the documents.

: Production BOM

As shown in Fig. 3, the Drawing Set and Line Drawing Set FactBoxes show the drawing status of the header item and the status of the item in the line, respectively. These cc|product data management FactBoxes are available in many pages to provide immediate access to the drawing information. The Drawing Set FactBox shows the drawing status of the header item, and the Line Drawing Set FactBox shows the status of the item in the row.

In addition to this information, you always have the option of checking the components of an item on its drawing.

: Component of a Production Order

Note

If a component for an assembly of a production order contains a component whose status code is not valid, an error message is issued when you post or process the production BOM or any other document.

Example: Assigning a Drawing Set to an Item

In this example, a new drawing is to be assigned to item 1001 “Touring Bicycle”. First a drawing is created. The corresponding item number is selected from the item master.

: Create Drawing

The type of drawing is then selected in the Type Code field and a revision is assigned. The revision number is checked against the PDM Setup (see topic “Product Data Management Setup” on page 1). If a revision code is required, it is possible to assign it manually or automatically.

: Assign Revision and Status Code

In addition, a drawing file can be assigned in the Links FactBox. Finally, a status is assigned to the drawing.

: Created Drawing

The created drawings are automatically assigned to the item if the Use in Item check box is selected for the status in the Drawing Status Setup page (see topic “Drawing Status” on page 1).

: Drawing Set for Item Card

Using the Drawings action on the Navigate tab, all drawings for the item can be displayed, regardless of whether they are valid or not.

: Drawing List for an Item

To edit the current drawing set, choose Drawing Set on the Navigate tab.

: Drawing Set for an Item
