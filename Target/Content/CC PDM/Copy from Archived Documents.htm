Copy from Archived Documents

The situation is different when copying documents from archives: If a document is restored from the archives, the drawings that were assigned to the document at that time are NOT copied, but the revisions that are always valid are.

Fig. 1 shows the creation of a sales order with a valid drawing and revision level A.

: Sales order with drawing per line

The sales order is transferred to the archive because it has been completely processed. The drawing set is also transferred to the archive.

: Sales Order Archive, Archived Drawing

After some time, there is a new drawing for the item and thus a revision level. In the Drawing Card, the revision level is set to “blocked”. This must be done before the new revision is released. Otherwise a message will open stating that a valid drawing already exists for the item and it is not possible to assign the new revision to the item.

: Drawing with multiple valid number

In this case, the drawing set must be replaced manually in the Item Card.

The drawing set still contains the blocked drawing revision.

: Drawing Set for Item

This is manually replaced by the new drawing revision.

: Drawing Set Exchanged to Item

The sales order archive shows that revision level A is assigned to the drawing used at that time, but that the status was changed to “BLOCKED”.

: Drawing Set Status Change

Now create a new sales order and copy the archived order.

: Copy from Archive in Sales Order

By confirming the selection, all information from the archived sales order is now transferred. With one exception: The drawing set is always the current one, but never the obsolete one.

: Copied Sales Order with New Drawing
