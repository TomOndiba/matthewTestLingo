Replace, Exchange Drawings - Navigate

Each drawing of a type may only be assigned to an item (and automatically to a BOM) if this drawing has a valid status. If a change is made, the drawing can be exchanged. However, the user must note that he or she sets the old revision status to not valid (see topic “Drawing Status” on page 1).

To change drawings, use the “Navigate” action to get an overview of the current uses of the drawing. To do this, select a drawing in the Drawings list and then choose Navigate on the Home or Actions tab. This opens the predefined Drawing Navigate page, in which the documents assigned to the drawing are immediately displayed.

Alternatively, enter Drawing Navigate in the Search box, and then choose the related link to open the Drawing Navigate page.

: Navigate Action in the Drawings

: Drawings Navigate Page

In the Drawing Navigate page, both the related documents and their number is displayed.

For a detailed check, the No. of Documents field can be used, as with the items, to open a list of all documents with their related number, in which a change can be made. This can be done manually or in all documents using the “Replace” function on the Home tab.

The fields on this FastTab are described in the following table.

Group / Field

Description

Options / Incl. Finished Documents

If selected, those documents with the status Completed, Closed or similar finished or archived statuses are also listed when executing the “Find” function.

Drawing / Drawing No.

Number of the drawing to be searched for or replaced.

Drawing / Drawing Type Code

Code from the drawing types: Used to display the drawing type.

Drawing / Revision Code

Drawing number, which must be reassigned. Also contains the code of the drawing type.

Replace with / Drawing No.

Drawing number, which must be reassigned. Also contains the code of the drawing type.

Replace with / Revision Code

Revision code of new drawing.

Related Documents

Documents to which the drawing to be searched for or to be replaced is assigned.

No. of Documents

Sum of the documents found.

Important

The drawing to be replaced must not have a status that defines it as valid. However, the drawing that is to be replaced must have this. Only then an exact exchange can be carried out.

Actions on the Home Tab

The following describes the functions on the Home tab for the Drawing Navigate page.

: Functions on the Home Tab of the Drawing Navigate Page

Action

Description

Show Related Docs

Displays the documents found on a separate page.

Find

Applies a filter to search on this page. It thus locates all the related documents / entries based on the set criteria.

Replace

Opens a separate page for selecting the documents where the drawing should be exchanged.

Find by Drawing

The search mask offers a filter option for searching documents by drawing.

Find by Item

The search mask offers a filter option for searching documents by item number.

 

Example: Replace Drawing Set Using Navigate

Your company has a sales order or other document with revision level B which is “valid” and revision level A which is “blocked”.

Your customer informs you that he would rather have the design type with revision level A. However, you have already created an order and assigned the revision level to it. Now it is possible to exchange the drawings manually. Unfortunately, however, it is often the case that there are several other documents behind the sales order, such as a production order, for example. In addition, the component to be changed can also be contained in a BOM structure. The “Replace” function is then used for such changes.

: Sales Order

To check the drawing usage, open the Drawing Navigate page using the Search box.

: Drawing Navigate in the Search box (global)

In the Drawing Navigate page, enter the drawing with the revision status to be replaced and the drawing to replace the old version.

Then choose Find to display the corresponding documents.

: Drawing Navigate with Result

The Drawing Navigate page shows an item line and a sales line, in which the drawing “PDM-1000” with the revision level “B” and the drawing type code “MECHANICAL” is located.

This should be replaced by the revision code “A”.

Note

For “Replace with” and the drawing number, the drawing type code is included in the drawing number.

The “Show Related Docs” function on the Navigate tab displays a list of all documents in a list.

: Documents from Drawing Navigate displayed in the list

In the Drawing Navigate page, choose Replace on the Home or Actions tab to start the exchange. Note, however, that the status of the drawing to be replaced is no longer valid, but the drawing to be replaced is valid.

Fig. 8 shows a message that is displayed when the drawing to be replaced is not valid.

: Generated error when replacing drawings

If, however, the drawing to be replaced has a status that allows it to be used in Sales lines, the document type selected previously is opened with the “Replace” function. For example, all orders that must be selected for change are displayed here.

: Selection in Document List for Drawing Navigate Lines

By choosing the OK button, the exchange is carried out for the selected documents. A message opens asking to confirm this exchange.

After replacing, documents that have the old drawing values are searched for again. The sales document is no longer displayed here.

: Drawing Navigate after Replacing

The sales order now shows that the revision level A of the drawing is in use.

: Drawing in Sales Order after “Replacing”

Since the drawing with revision level B was only blocked and replaced in the sales order, it is still in the drawing record of a production order, although revision level A is used in the corresponding item.

: Production order with old drawing revision

In this case, the drawing status must also be replaced on the production order.

Important

An item should only have ONE drawing, which has a status code marked “Valid” (see Fig. 1). When replacing the revision status, always change the item first.
