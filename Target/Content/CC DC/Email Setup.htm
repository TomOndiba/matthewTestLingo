Setup

The email setup starts with defining Email Distribution Codes to which multiple email settings can be assigned. The Email Distribution Codes page is accessed from the Departments menu by clicking Administration > Application Setup > General.

: Email Distribution Codes

As illustrated in Fig. 1, separate codes can be set up for different departments or processes. The different email settings are then defined in the Email Distribution Setup page, accessed by clicking Setup on the ribbon.

: Email Distribution Setup list view

In the Email Distribution Setup page, the usage of Outlook or SMTP Mail can be defined as well as standard Recipients, Subject line format, or special usage filters like Customer/Vendor/Contact /Responsibility Center and Language Code.

Email Distribution Setup Card

On the ribbon, click New to enter the settings in the Email Distribution Setup card view.

: Email Distribution Setup card view

Indicates the format to be used for the subject line. The first two options are generated automatically while the third is manually defined.

  • Recipient Name - Source Type - Source No.

(e.g. CoolWood Technologies - Quote SQ1001)

  • Subject Text – defined in the Subject Text field.

 

Subject Text

User-defined text, may contain placeholders to use dynamic parts in the Subject line and as well as in the Email Body Text. The possible placeholders are:

  • %1 – Item No.
  • %4 – From Status (Lot change)
  • %2 – Variant Code
  • %5 – To Status (Lot change)
  • %3 – Lot No.
  • %6 – Serial No.

SMTP Setup in Email Distribution Setup

For easier handling of SMTP delivery, the Email Distribution Setup page contains the necessary SMTP fields for each record when Mail Type = SMTP and Sender = Fixed.

: Email Distribution Setup page highlighting actions and fields related to SMTP

Therefore, if Mail Type = SMTP and the Sender field = Fixed, then the information in the Email Distribution Setup page is used. If the Sender field = Current User, then the information in the standard Microsoft Dynamics NAV SMTP Mail Setup table is used.

Additionally, the Email Distribution Setup (card/list) pages contains two SMTP actions (“Apply Office 365 Server Settings” and “Test Email Setup”) to assist with SMTP mail setup.

Attachments

Attachments can be assigned to the specific email distribution setup entries. In the Email Attachment Setup page, enter a new line for each attachment that needs to be assigned to the email distribution entry. Along with a description, optional starting and ending dates can be set to indicate the length of time for which the attachment should be included in emails.

: Email Attachment Setup with attached Word documents and graphic

The File Name field is populated with the name of the uploaded file but can be changed. Once uploaded, attachments can be downloaded for review.

Attached file(s) will be included in the emails sent using this distribution, along with the Microsoft Dynamics NAV document.

: Email to be sent with PDF and attachments

cc|text module

cc|text module functionality can be used on each email distribution entry to define the text to appear in the body of the email; this includes text formatting options like bold, underline, italic and some colors as well as placeholders (refer to section “Placeholders in Body Text” on page 1).

To create or open an existing Text Module Card for a line, click Navigate > Text Module > New or Edit.

: cc|text module text added to the PURCHORDER Email Distribution code with no filters

If an Email Distribution Setup is allocated, the data in the setup is used for email creation. Otherwise, the schedule that exists in standard Microsoft Dynamics NAV is used. In both cases, the Send E-Mail dialog box opens with settings from Email Distribution Setup (or Outlook/SMTP Mail Setup). If the Email Distribution Setup is used, the dialog box will also display the cc|text module text added to the setup pages. From the dialog box, the email can be sent or if using Outlook, can be opened for editing before sending.

Refer to the current “cc|text module” documentation for more information about setting up and using cc|text module functionality.

Refer to topic “Example – Add Formatted Text to Sales Order Confirmation” on page 1 for information about using cc|formatted documents functionality with cc|text module.

Report Selections

To create an email using the “Send PDF as Email” function with text from the cc|text module, an Email Distribution Code must be entered on the relevant Report Selection entry. This can be done in the Report Selection page for Sales, Purchase, Reminder, Commission, and DACH functions.

: Email Distribution Code assigned to the Purchase Order in Report Selection - Purchase

Custom Report Selections

The custom report selections in standard Microsoft Dynamics NAV can also be set up to use the Email Distribution functionality. On the Customer Card or Vendor Card, click Navigate > Document Layouts. The various sales or purchase documents can be set up with different email addresses and different Email Distribution Codes, each code with different cc|text module text.

: Standard Microsoft Dynamics NAV Document Layouts page with regular (TOCUSTOMER) and special (CUST30000) Email Distribution Codes for customer 30000
