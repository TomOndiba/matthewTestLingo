Add Carry Overs for Document Sums

Carry Overs are used in cc|document configurator reports to carry values over multiple pages. The carry overs are added in the RDL in the reports header and footer section.

The carry over header and footer text that appears in the report header and footer is determined by the Document Text settings in the Document Configurator Setup page.

: Header and Footer text for carry overs in Document Layouts

The carry over’s font type and size are defined by the document layout headers configuration. Custom formatting of a decimal value can be defined in the Carry Over Format field in the General FastTab of the Document Layout. An RDL formatting string must be used in order to change the default value.

: Font and Format fields in Document Layout, currently using defaults not custom settings

In the Document Layout Fields, the Carry Over field determines if the carry over is starting (Create) or ending (Stop).

: Carry over creation on “Lines” row for Sales Line in Document Layout section

The Create option can only be used on only one field in the Document Layout, while the Stop option can be used on multiple fields.

: Two Carry over stops on “Total Section” row in Document Layout section

The left position is used according to the layout field's setting. All other markups and positions need to be changed inside RDL. An example for the Carry Over Format value could be “#,0.00' €'”. Refer to https://msdn.microsoft.com/en-us/library/4fb56f4y(v=vs.90).aspx for more information.

When a document is generated which carries over to multiple pages, the footer text and value appears on the first page of the document when using the document layout OC-03-TM.

: Carry Over Footer Text and Value on first page of document

On the following page(s), the header text and value appears on the document when using the document layout OC-03-TM.

: Carry Over Header Text and Value on next page of document
