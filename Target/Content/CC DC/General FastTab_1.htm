General FastTab

The General FastTab contains general information about the Document Layout. The fields on this FastTab are described in the following table.

Field

Description

No.

Unique identifiers for the Document Layout.

Description

Short description of the Document Layout.

Top DataItem Table No.

Indicates the table from where the report will be run. This should always be the same as the first DataItem in the report.

Top DataItem Table Name

Displays the name of the top DataItem table.

Report No.

Indicates the number for the report that should use this layout.

Report Name

Displays the name of the report.

Render Report No.

Indicates the number of the report that should be used to render the report, which defines the page size, margins, logos, etc. cc|document configurator includes the following predefined render reports:

  • 12032027, DC - A4 Portrait
  • 12032028, DC - A4 Landscape
  • 12032029, DC - Letter Portrait
  • 12032030, DC - Letter Landscape
  • 12032031, DC - Excel Render
  • 12032032, DC - Label (57 x 102 mm)
  • 12032040, DC - A4 Portrait FlowLayout

Render Report Name

Displays the name of the render report.

Document Label Code

Indicates the code for the Document Text that is used for printing the Label at the top of the page (e.g., the label on a Sales Quote is where it says “Sales Quote” on the top).

The Document Text that this is referring to should end with %1; this is for the system to be able to insert a COPY label when printing copies of the same document.

Phantom Layout

Indicates that the layout is a phantom layout, i.e., a layout that is not printed by itself, but used for printing sections in other layouts.

Refer to topic “Sections FastTab” on page 1 for more information.

Font Family

Identifies the Font Family for the layout. If this field is empty, the system will use the Font Family assigned in the Document Configurator Setup.

Font Size

Identifies the Font Size for the layout. If this field is empty, the system will use the Font Size assigned in the Document Configurator Setup.

Carry Over Format

Enables custom formatting of a decimal value (refer to topic “Add Carry Overs for Document Sums” on page 1).

Last Modified Date

Displays the last date the Document Layout was modified.

Last Modification Time

Displays the time when the Document Layout was last modified.

Last Modified by User ID

Displays the User ID of the user that last made a change to the Document Layout.

Starting Date

Indicates the starting date from which the Document Layout should be valid. If left blank, then it is valid from any date.

Date = work date of the client.

Ending Date

Indicates the ending date until which the Document Layout should be valid. If left blank, then it is valid until any date.

Date = work date of the client.

Status

Indicates the status of the Document Layout. The options are:

  • New
  • Certified
  • Under Development
  • Closed

Only Certified layouts can be used for printing. Testing the layout by running the report directly from the layout can be done at any status.

When changing the Status to Certified, some checks are performed to validate the layout.
