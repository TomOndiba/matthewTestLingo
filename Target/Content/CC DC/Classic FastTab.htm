Classic FastTab

The fields on this FastTab are described in the following table.

Field

Description

Max Pages per Document

Specifies the maximum number of pages that could be generated for a single document. This is used to avoid a configuration where the system goes into an infinite loop and start generating an infinite number of pages.

The recommended setting is 1000. When a document reaches 1000 pages, it will do a hard stop and display an error message.

Max Iterations per Section

Specifies the maximum number of iterations of a table. This is used to avoid a configuration where a filter is missing and the system generates thousands of sections. The recommended setting is 10000.

Note

This field only works in combination with classic layouts, not with the flow layouts.
