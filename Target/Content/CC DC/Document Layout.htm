Document Layout

As illustrated in Fig. 1, the Document Layout for a Sales Quote has five sections:

  1. Header of the Document
  2. Lines
  3. Total Section
  4. Footer with Responsibility Center information
  5. Footer with Company information

: Document Layout for Sales Quotes

These sections refer to the structure of the printout.

: Sales Quote printout, noting the five corresponding sections in the Document Layout

Note

Sections can have criteria that specifies whether it should be printed or not. This is the case with sections 4 and 5. Sections can also be defined for iterations of tables only without printing anything.

In addition to the sections defined in the Document Layout, the system always prints a Report Top section on each page. This section includes the logo, page number and report label.

Each section has a list of fields that will be printed. The orientations of the fields within each section are defined as the Line and Indentation.

: Document Layout Fields for Sales Quote header with indentation

: Sales Quote printout, indicating line and indentation section settings

In addition to the fields, there are also Filters, Criteria, Codeunits and Variables that can be defined for each section. These are used as follows:

  • Filters – specify the relations between the tables
  • Criteria – define if data should be shown or not
  • Codeunits – write custom code using C/AL
  • Variables – declare variables with formulas

: Document Layout highlighting Line menu items on the Sections FastTab

Filters, Criteria, and Variables can also be used in conjunction with the Fields (refer to topic “Document Layout” on page 1 for more information).
