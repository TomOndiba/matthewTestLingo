Document Entries

Document Entries represent the history of actions performed from Microsoft Dynamics NAV. It is possible to deactivate the document entry tracking or saving in the fields on the History FastTab in the DMS Setup page.

To view all Document Entries in the system, click Departments > cc|dms > History > DMS Document Entries. Additionally, the DMS Document Entries page can be opened to filtered views on Document Libraries, Content Types, MetaData, SharePoint Table Filters and NAV Table Filters.

: DMS Document Entries page opened from History menu

The “Run as Client” and “Run as Server” functions provide the ability to process a document entry in the specific context. When an error occurs, click Actions > Reset Error to change the Service Result to <blank> and process it manually. The actions regarding documents are only visible if any BLOB (document, service xml or result xml) are stored in the document entry.

The fields in the DMS Document Entries page are described in the following table.

Field

Description

Entry No.

Indicates the system-generated number of the entry. When aborting the MetaData dialog for a document, the entry will be deleted, which might lead to gaps between the numbering.

Entry Type

Defines the processed action. The options are:

  • Upload New Document – A document will be uploaded as soon as the MetaData are defined (saved).
  • Edit MetaData – The existing MetaData of a SharePoint existing document will be changed.
  • Show MetaData – The MetaData of an existing document will be displayed in Microsoft Dynamics NAV.
  • Delete Document – A document has been deleted from the document library.
  • Check In – Manages the check in of documents in SharePoint.
  • Check Out – Manages the check out of documents in SharePoint.
  • Undo Check Out – Reverts the check out of documents in SharePoint to the previous state.
  • Download Document – The document has been downloaded from the document library.

Upload Type

Indicates the type of upload, either Client or Queue (refer to topic "Document Actions" on page 1).

Service Result

Displays the current status to perform the action in SharePoint. The options are:

  • <blank> – Action was created but not processed in SharePoint yet.
  • Success – Entry was successfully processed in SharePoint.
  • Error – Error occurred while processing the action in SharePoint. The actual error will be displayed in the Service Error Message field.

Service Error Message

Displays error messages from the communication with SharePoint.

Document Library No.

Identifies the number of the document library to perform the entry.

Content Type Code

Displays the code of the content type to process the action. When uploading a document, the code might change because of the content type selection on the MetaData definition dialog.

Table No.

Indicates the table number for the content type and document library.

Document

If selected, the document (BLOB) is saved in Microsoft Dynamics NAV for the entry.

Record ID

Displays the Microsoft Dynamics NAV record with which this action is performed.

File Name

Displays the file name of the document, including timestamps.

MetaData Saved

Indicates if the MetaData for this document entry have been saved or are not yet defined.

MetaData Saving Date

Displays the date of defining the MetaData.

MetaData Saving Time

Shows the time of defining the MetaData.

MetaData Saved by User

Displays the user who has created the document entry and changed the MetaData.

Service Node String

If selected, the XML that triggered the SharePoint ServiceAdapter to process the action (Entry Type, like uploading or deleting) is stored for this entry. This field is used for debugging purposes only, to check what cc|dms is sending to the SharePoint ServiceAdapter.

Result Node String

If selected, the XML returned from the SharePoint Service Adapter is saved for this entry. This field is used for debugging purposes only, to check what the SharePoint ServiceAdapter is returning to cc|dms.

Upload Counter

Displays a system-generated value that counts the upload attempts.
