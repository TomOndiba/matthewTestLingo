Document Actions

The upload and modification (including delete) of documents can be done directly by the user or with the help of a queue. The upload type is configured on the content type, or in the DMS Setup. The entry types “Edit MetaData” and “Delete Documents” depend on this configuration for the upload type, as well (refer to topic “Document Entries” on page 1). Both actions are processed directly by the user or with the help of a queue. The user rights for the document library (New, Modify, Delete) are checked when performing the action.

: Client and Queue

The direct upload of the DocumentBox is asynchronous, in order to let the user modify and work with the record. Thereby the page should not be closed while uploading the document into SharePoint to avoid unexpected errors with the RoleTailored Client. The direct upload processes the document in consideration of the user’s context, therefore changes by a user signed into Microsoft Dynamics NAV can be overwritten by User Credential settings in cc|dms. The user should have appropriate rights to upload, modify or delete the documents in SharePoint.

When uploading a document with the queue, the document will be copied into a Microsoft Dynamics NAV table (Document Entry) and Navision Application Server (NAS) must be configured in the Microsoft Dynamics NAV Server Management Administration to run the “DMS - NAS Start Up” codeunit. The service user (NAS) context will be used for processing, and is not translated by any user DMS credential configuration. Any actions of the NAS will be tracked by DMS Protocol Entries (refer to topic “DMS Setup” on page 1).

When uploading a new document or modifying the MetaData, depending on the configuration (refer to the “Define MetaData” field in topic “Content Type” on page 1), a dialog opens to set the MetaData before performing the action. When defining and editing the MetaData, colors are used for field descriptions and field values to highlight the possibilities. With the “Show Extended Fields” function, all fields that have configurations with Visibility = Extended are shown.

: Define, Show and Edit MetaData Dialog

The color codes are defined as follows:

  • Red – A value is necessary, the field must not be blank
  • Blue – An option is configured to select any value (lookup)
  • Grey – The value is not editable

Errors while uploading, modifying or deleting are tracked in the document entries.

If a value is not editable, must contain a value and the default value is empty, the field becomes editable automatically to let the user enter any value before uploading or updating.
