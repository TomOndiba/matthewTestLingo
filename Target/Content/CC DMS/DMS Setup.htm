DMS Setup

Basic setup is performed in the DMS Setup page, accessed from the navigation pane by clicking Departments > cc|dms > Setup > DMS Setup.

: DMS Setup

When the DMS Setup page is opened for the first time, the setup initializes itself. The setup contains all important global configuration for cc|dms.

General FastTab

The fields on the General FastTab are described in the following table.

Field

Description

Doc. Library Nos.

Displays the number series used to create new document libraries. The number series should allow manual numbering.

Upload Type

Defines how to upload documents. The options are:

  • Client – Documents will immediately be uploaded in the current Microsoft Dynamics NAV user’s context after defining the MetaData.
  • Queue – After defining the MetaData, all documents will be copied into a Queue (Document Entry), which must be processed by a Navision Application Server (NAS) or a user manually. The upload will use the NAS’ user context.

Refer to topic “Document Actions” on page 1 for more information.

SharePoint File Duplicate

Determines how duplicate file names will be handled in the target document library. The options are:

  • New Version – Creates a new Version for the uploaded document. If Versioning is disabled in the document library, the existing document will be overwritten.
  • Add Timestamp on File Name – Appends a timestamp at the end of the new file’s name and before the extension: <Year>-<Month>-<Day> - <Hour>-<Minute>-<Second>, for example Bicycle_Image 2014-04-03 11-23-54.bmp.

Report Upload Type

Indicates additional actions when uploading a report via automated “Upload PDF” or “Post and Upload PDF” functions.

  • Only Upload – The report will be uploaded into SharePoint without any further action.
  • Upload and Open – After uploading the report, the local copy (from a temporary path) will be opened, e.g. to manually print it.
  • Upload and E-Mail – A new Email will be created by the default application with the PDF automatically attached.

Refer to topic “Report Upload” on page 1 for more information.

Report File Name Formula

Defines the formula to generate the file name when automatically uploading a report. Possible variables are [RECORDID], [REPORT.ID], [REPORT.NAME], [REPORT.CAPTION], [TODAY], [TIME], [COMPANY] and [USERID]. An example for a formula would be “[RECORDID] - [REPORT.ID].pdf”.

Service Timeout

Determines the timeout in milliseconds until the service aborts an action (such as retrieve document list for DocumentBox). “0” will use the default settings.

Save Xml Nodes (Debug)

If selected, the XML files are saved for processing in the corresponding document entry fields. This field should only be activated for Debugging purposes.

Document Box FastTab

The fields on the Document Box FastTab are described in the following table.

Field

Description

Document Box Height

Sets the height of the DocumentBox. The user cannot resize the height on each page. Therefore, it should be configured to a suitable value for all display resolutions and number of document libraries.

Document Box Sorting

Sorts the order of documents in the DocumentBox, either ascending or descending, for the creation date or file name.

Doc.-Box Layout

Defines the layout of the DocumentBox. The options are:

  • Logo and TreeView – Shows the Logo on the top section of the DocumentBox and the document list below.
  • TreeView Only – Displays only the document list, without the logo. This option is useful for a narrow implementation.

Document Box Bitmap

Stores the picture to be displayed on the top section of the DocumentBox. Uses standard Microsoft Dynamics NAV picture insertion functionality.

For an optimized performance, the picture size should not be more than 200 kb.

History FastTab

The fields on the History FastTab are described in the following table.

Field

Description

Save Document Entries

If selected, document entries will not be deleted after successful processing. This enables a tracking of document actions from Microsoft Dynamics NAV. Tracked actions can be upload, delete and modify.

Remove Documents from Doc. Entries

Determines if the document (binary, BLOB) is saved for each upload in the Microsoft Dynamics NAV database.

  • If selected, the needed database volume increases with the saved document.
  • If not selected, the document BLOB is removed after successful upload.

Navision Application Server FastTab

The fields on the Navision Application Server FastTab are described in the following table.

Field

Description

Protocol Entry Delete Formula

Defines a formula to delete the protocol entries, created by NAS, which are older than the calculated formula compared to the current date.

Document Entry Delete Formula

Defines a formula to delete the document entries, created by Upload Type = Queue, which are older than the calculated formula compared to the current date.

A running NAS for cc|dms is necessary to perform this auto delete.

NAS Poll Interval (Milliseconds)

Sets a time interval between the end of the previous and the start of the next NAS processing. A high interval results in longer breaks between the definition and the processed action (upload, modify, delete) of a document, if Upload Type = Queue.

Log Document Entry Heartbeat

If selected, a heartbeat protocol entry will be created after each poll interval to check the vital sign of the NAS.

Navigate Tab

The Navigate tab includes items that are relevant to report selections and user rights in cc|dms.

: Actions on the Navigate tab in DMS Setup

Each is described in the following table.

Action

Description

Report Selection - Sales / Report Selection - Purchase / Report Selection - Service

Opens the report selections for sales, purchase and service; also includes fields for configuring the document library and content type to upload the PDF to SharePoint.

Refer to topic “Report Upload” on page 1 for more information.

User Credentials

Opens the DMS User Credentials page, where users without access to uploading files in SharePoint can be assigned a “SharePoint User Account” that can upload files. Without any configuration, the Microsoft Dynamics NAV user account communicates with SharePoint.

Settings here affect all Document Libraries, while settings on the Document Libraries affect only those libraries.

A global function in the DMS - Management codeunit verifies the “global” User ID entries (entered here) and the User ID entries for a special document library. This function also allows for the usage of a “default user” by entering an asterisk (*) in the User ID (NAV) field.

Additionally, the user credentials set here are used for service adapter pattern queries, which allows the service adapter pattern to run for SharePoint Online. For information about the Service Adapter Pattern, refer to the “cc|dms 2015 SP1 - LibrarySet User Manual” documentation.
