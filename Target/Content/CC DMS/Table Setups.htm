Table Setup

The table setup splits the content type settings to different tables (records) and is configured on the DMS Table Setup FastTab in the DMS Document Library page. It defines for which records (table number and Microsoft Dynamics NAV table filter) the content type should be visible in the DocumentBox. Different MetaData (refer to topic “MetaData” on page 1) and different table filters (refer to topic “SharePoint Table Filter” on page 1 and topic “Microsoft Dynamics NAV Table Filter” on page 1) can be defined for each table setup. Additionally, the table setup holds a release status certification for plausibility and integrity. Only released table setup will be displayed in the DocumentBox.

: Table Setup of a Content Type on the DMS Document Library page

The fields on the DMS Table Setup FastTab are described in the following table.

Field

Description

Table ID

Indicates the table number for which the setup will be configured.

Table Caption

Displays the caption of the related table.

Status

Indicates the status of the table setup. The options are:

  • Open – Changes on MetaData or table filters are possible.
  • Released – Changes on MetaData or table filters are not possible. The DocumentBox shows only released table setups. During release, the configuration data are validated for plausibility and integrity, displaying any errors.

MetaData

Indicates whether a MetaData definition (refer to topic “MetaData” on page 1) is configured for the current table setup.

SharePoint Table Filter

Indicates whether a SharePoint table filter (refer to topic “SharePoint Table Filter” on page 1) is configured for the current table setup.

Microsoft Dynamics NAV Table Filter

Indicates whether a Microsoft Dynamics NAV table filter (refer to topic “Microsoft Dynamics NAV Table Filter” on page 1) is configured for the current table setup.

Folder Formula

Enables the storage of files in a folder and is a formula field that can contain variables to be used as folder definition. Refer to topic “Folders for Libraries” on page 1 for more information.

Last Modification Date

Displays the last date of modification of any table setup’s sub-tables, such as MetaData or table filters.

Last Modification Time

Displays the last time of modification according to the modification date.

Modified by User

Displays the user according to the last modification.

Library Relative Url Formula

Indicates a dynamic, record based URL instead of the fixed URL of a Document Library. Enter the full relative URL and then use "[Field Name]" to get values of the corresponding field of the currently selected record.

For example, if a customer has a project website (e.g. websites/dms/projects), that contains sub-websites for each project based on the job's No. field (e.g. PR1234), there are two ways to enter the Library Relative Url Formula:

  1. "websites/dms/projects/[No.]"
  2. "[DMS Website Url]", and create a new field in the Job table that contains the value "websites/dms/projects/<Job No.>" (where <Job No.> = e.g. "PR1234")

 

Folders for Libraries

After there are more than 5,000 elements in a SharePoint list, the filtered fields must be managed by indexed columns. Document libraries are technically lists, with a document field to store the files. Managing the indexed columns is fine, but can still create problems with the file upload via the SOAP web service, which DMS is using. A way to avoid the limitation of 5,000 elements is to upload files into folders. Each folder can again contain 5,000 elements, and so on. However, since a “folder” is an element, there should only be a maximum of 5,000 folders inside a folder, again.

In the DMS Table Setup FastTab, the Folder Formula field enables the storage of files in a folder. Thereby, more than 5,000 files can be managed (especially uploaded) together with DMS. This is a formula field that can contain variables to be used as folder definition. Because this field is applied on the table setup configuration, all fields from the record can be used to create folders, as well as some system variables:

  • “[MM]” can be used for the current month (e.g. “02” or “12”)
  • “[DD]” can be used for the current day (e.g. “01” or “27”)
  • “[YYYY]” can be used for the current year (e.g. “2017”)
  • “[No.]” is an example to retrieve the information from the current “No.” field of the current record in the current table
  • “[Description]” is an example to retrieve the information of the “Description” field

Variables can also be added to more complex structures. “[YYYY]/[MM]” will create a folder “2017” and a subfolder “03” (for March) within. Values of the fields will be translated into valid URL path names by replacing the characters \/.?&%#{}~ with an underline (“_”). Before uploading the file, the existence of the folders is tested. Folders that do not exist will be created. Therefore, the user must have enough rights to create folders if necessary and upload a file into the folder.

The Document Box filters all the files, regardless of where they are stored in the SharePoint list’s folder structure. All documents that apply the DMS’ Table Setup SharePoint filter will be displayed plainly under the content type inside the Document Box. When editing the metadata, the absolute file path on the top of the page displays the correct path to the file, including folders.

More information about folder limitations can be found in the following links:

  • SharePoint folder limitations that must be observed inside projects: https://technet.microsoft.com/en-us/library/cc262787(v=office.16).aspx#ListLibrary
  • Column indexing that should be made within SharePoint for each filtered field in document libraries with a huge number of documents (more than 5,000): https://www.codeproject.com/Articles/1076854/Overcoming-the-List-View-Threshold-in-SharePoint-C

Note

No column indexing was done when the current Library Set was created because it is technically impossible with SOAP. Therefore, users must manage the column indexing of the filtered fields within SharePoint or the folder structure will not be visible. Column indexing must be completed before 5000 elements are reached; subsequent adjustments are not possible.

MetaData

MetaData defines the fields (columns) of a document library for the content type and a specific table setup. For the DocumentBox’s actions “File Upload”, “Show MetaData” and “Edit MetaData”, only the configured fields will be shown in the dialog (refer to topic “Document Actions” on page 1).

Furthermore, MetaData provides the document with additional information to filter and find the documents for the DocumentBox and in SharePoint. The information sources (Field Class) can either be record specific, looked up or system values.

: MetaData Configuration Worksheet

The fields in the Edit DMS MetaData page are described in the following table.

Field

Description

Field Name

Designates the internal (static) SharePoint field (column) name that is used for MetaData information upload.

Field Caption

Shows the display name (caption) of the SharePoint column. This value is not editable in Microsoft Dynamics NAV.

Field Type

Displays the type of the SharePoint column. This value is not editable in Microsoft Dynamics NAV.

Field Length

Displays the length of characters of the SharePoint column. This value is not editable in Microsoft Dynamics NAV.

Field Class

Defines the source of information, as further indicated in the Default Value / Field field. The options are:

  • Value – Enter a user-defined value (constant) that will be used for upload as the default.
  • Field – Use the lookup in the Default Value / Field field to select a field value of the current record (table setup) that will be used as the default value.
  • Option – Use the lookup in the Default Value / Field field to enter option values. Later, those values can be used for selection. The Option Code is a unique value, whereas the Option Value represents the actually uploaded value.
  • Date Formula – Enter a date formula that is calculated before uploading a document.
  • System – Use the lookup in the Default Value / Field field to select a system value from a list. The options are: User ID, Workdate, System Date, System Time, Company Code.
  • File – Use the lookup in the Default Value / Field field to select file information from a list. The options are: File Name, File Path, File Extension, Absolute File Name.
  • Lookup – Use the lookup in the Default Value / Field field to retrieve a value from another record from any Microsoft Dynamics NAV table. The target record can either be a filtered single result, or selected by the user from a page.

After entering the target Table ID in the Lookup Table No. field, the Default Value / Field field shows all fields of the target table. Select the field and then click the AssistEdit to filter the target table.

Enter a value in the Lookup Page ID field to show the filtered result in a page and make the user select the target record. Without a value in the Lookup Page ID field, depending on the Lookup Action field, the value of the first record or the last record in the filtered list will be used as default value, or no value at all.

Default Value / Field

Specifies the actual information based on the selection in the Field Class field:

  • Value – User-defined constant (fixed value)
  • Field – Field number for the current table setup
  • Option – Option code to be used as the default selected option
  • Date Formula – Microsoft Dynamics NAV formula to calculate the date
  • System – Code of the system value
  • File – Code of the file information
  • Lookup – Field number of the referenced lookup table number

Value Caption

Shows additional information regarding the default value. This field is not editable.

Editable

If selected, the user can make changes before uploading the document or changing the MetaData.

Not Blank

Determines whether this column must contain a value or can be empty. If the SharePoint field must contain a value, this field cannot be selected.

Visibility

Defines the visibility of the MetaData item. The options are:

  • Standard – The field (and thereby its value) are always visible.
  • Extended – The field will be invisible unless the user clicks Show Extended Fields on the Define, Show and Edit MetaData dialogs.
  • Hidden – The field is always invisible but the value will be uploaded or modified in the background.

Format Property

Defines the formatting of the default value. The number is equal to the Microsoft Dynamics NAV format definitions (C/AL “Format Property”).

Lookup Table No.

Identifies the table number from which to look up the value; used with Field Class = Lookup.

Lookup Table Caption

Displays the caption from the table number from which to look up the value. This field is not editable.

Lookup Filter

Indicates if a lookup filter is defined for the current MetaData item. This field is not editable.

Lookup Page ID

Identifies the page number to open when the user clicks the lookup on the Define or Edit MetaData dialogs. The page’s source table must match the lookup table to avoid errors.

Lookup Page Caption

Displays the caption from where the page number is to look up the value. This field is not editable.

Lookup Action

Defines the selected record from where to retrieve the looked up value. The options are:

  • First Record  – The field value of the first record from the filtered list will be used.
  • Last Record – The field value of the last record from the filtered list will be used.
  • No Record – The value will be empty and no record will be selected after filtering.

Lookup Field No.

Identifies the field number from which to look up the value; used with Field Class = Lookup. Refer to section “Example – Create a Linked Lookup” on page 1 for more information.

Lookup Field Caption

Displays the caption from the field number from which to look up the value. This field is not editable. Refer to section “Example – Create a Linked Lookup” on page 1 for more information.

Field No. Linked Lookup

Identifies which field of the looked up value to be received; used with the Field Name Linked Lookup field. Refer to section “Example – Create a Linked Lookup” on page 1 for more information.

Field Name Linked Lookup

Indicates another field name from this configuration to retrieve additional values if a lookup is processed. This field links the current metadata field with the field where the lookup is pressed. Refer to section “Example – Create a Linked Lookup” on page 1 for more information.

Value by Lookup only

If selected, the user cannot enter a value by keyboard in the metadata dialog; the Lookup must be used to select a value. Refer to section “Example – Create a Linked Lookup” on page 1 for more information.

Sorting Position

Displays an ongoing number that represents the sort order for the dialog. This field is system-generated and should only be changed due to data inconsistencies.

 

Example – Create a Linked Lookup

This example explains how to create a linked lookup of a customer name to a customer number when uploading a new document.

To set the customer name to the metadata definition dialog when the user clicks on the lookup for the customer number during upload:

  • Define a metadata field to send the contact number to (SharePoint field name = CCDMS_Customer_No) and set the fields to Class = Lookup, Lookup Table No. = 5050, Lookup Field No. = 1, Lookup Page ID = 5052, Value by Lookup only = true
  • Define a metadata field to send the contact name to and set the fields to Class = Lookup, Editable = false, Lookup Table No. = 5050, Lookup Field No. = 2, Linked Lookup Field Name = CCDMS_Customer_No, Linked Lookup Field No. = 2

SharePoint Table Filter

A SharePoint Table Filter defines which files in the document library are displayed in the DocumentBox. Only the documents from the document library that meet the filter criteria are listed.

: SharePoint Table Filter Worksheet

The fields in the Edit DMS SharePoint Table Filter page are described in the following table.

Field

Description

Field Name

Identifies the internal (static) SharePoint field (column) name to be filtered.

Field Type

Displays the type of the SharePoint column. This value is not editable in Microsoft Dynamics NAV.

Field Caption

Shows the display name (caption) of the SharePoint column. This value is not editable in Microsoft Dynamics NAV.

Field Class

Defines the source of the filter value, as further indicated in the Default Value / Field field. The options are:

  • Const – Enter a user-defined value (constant) that will be used as a fixed filter value (such as document type).
  • Field – Use the lookup in the Default Value / Field field to select a field value of the current record (table setup) that will be used as the filter value.
  • Date Formula – Enter a date formula that is calculated before filtering the document list.
  • System – Use the lookup in the Default Value / Field field to select a system value from a list. The options are: User ID, Workdate, System Date, System Time, Company Code.

Default Value / Field

Specifies the actual information based on the selection in the Field Class field:

  • Const – User-defined, fixed value
  • Field – Field number for the current table setup
  • Date Formula – Microsoft Dynamics NAV formula to calculate the date
  • System – Code of the system value

Value Caption

Shows additional information regarding the default value. This field is not editable.

Field Length

Displays the length of characters of the SharePoint column. This value is not editable in Microsoft Dynamics NAV.

Format Property

Defines the formatting of the default value. The number is equal to the Microsoft Dynamics NAV format definitions (C/AL “Format Property”).

 

Microsoft Dynamics NAV Table Filter

A Microsoft Dynamics NAV Table Filter defines the display of the content type in the DocumentBox only for appropriate records that meet the filter criteria. Thereby is it possible to define different content types for sales orders and sales quotes that are both in the same Microsoft Dynamics NAV table, or a separation for different item categories.

: Microsoft Dynamics NAV Table Filter Worksheet

The fields in the Edit DMS Microsoft Dynamics NAV Table Filter page are described in the following table.

Field

Description

Field No.

Identifies the field number for the Microsoft Dynamics NAV table to be filtered.

Field Type

Displays the type of the Microsoft Dynamics NAV field.

Field Caption

Displays the caption of the Microsoft Dynamics NAV field.

Field Class

Defines the source of the filter value, as further indicated in the Default Value / Field field. The options are:

  • Const – Enter a user-defined value (constant) that will be used as a fixed filter value (such as document type). The class corresponds as SETRANGE.
  • Field – Use the lookup in the Default Value / Field field to select a field value of the current record (table setup) that will be used as the filter value.
  • Filter – Enter a Microsoft Dynamics NAV filter value to define ranges. The class corresponds as SETFILTER.
  • Date Formula – Enter a date formula that is calculated before filtering the record list.
  • System – Use the lookup in the Default Value / Field field to select a system value from a list. The options are: User ID, Workdate, System Date, System Time, Company Code.

Default Value / Field

Specifies the actual information based on the selection in the Field Class field:

  • Const – User-defined, fixed value
  • Field – Field number for the current table setup
  • Filter – Filter expression
  • Date Formula – Microsoft Dynamics NAV formula to calculate the date
  • System – Code of the system value

Value Caption

Shows additional information regarding the default value.

Field Length

Displays the length of the Microsoft Dynamics NAV field.
