FDC Shift Structure

FDC Shift Structures store the defined working hours for the company and are extremely flexible in cc|factory data capture. In the lines, all shifts can be assigned, which can be classified to various combinations of weekdays. Furthermore, a break time pattern (refer to topic "FDC Break Time Patterns" on page 1) can be assigned to a shift.

: FDC Shift Structure Card

When a Start Date / Time and End Date / Time is specified during FDC posting, a corresponding shift structure, a layer and a break model must be located to create the FDC Entry.

The intersection of the entry (start / end) with the defined shifts (start / end) with the FDC Resource (last shift date of the assigned shift structure) is made to create the layer. The layer with the largest intersection is stored as a shift in the FDC Entries.

The order used to search for the corresponding shift structure is Day ð Monday to Friday ð All. For example, if no shift is found for “Monday”, a shift defined with “Monday to Friday” is sought, and so on.

The fields in the FDC Shift Structure page are described in the following table.

Field

Description

Code

Unique identifier of the shift structure; maximum of 20 alphanumeric characters.

Description

Brief description of the shift structure.

Weekdays

Indicates the valid days of the shift. For multiple days like Monday and Tuesday, multiple lines with the same shift code have to be entered.

FDC Break Time Pattern

Identifies the FDC Break Time Pattern for the shift structure.

Shift Code

Unique identifier of the shift (e.g. numbering).

Start Time

Begin time of a shift.

End Time

End time of a shift.

Start Time Next Day

Must be activated when the shift begins on the following day.

End Time Next Day

Must be activated when the shift ends on the following day.

Net Shift Time

Indicates the duration of the shift, deducting breaks.

Gross Shift Time

Indicates the duration of the shift.
