FDC Setup Page

The setup of cc|factory data capture takes place in the navigation pane by clicking Departments > COSMO CONSULT > cc|bde (FDC) > Setup > FDC Setup.

The FDC Setup page contains all the necessary configurations for cc|factory data capture. Some of the settings are the default, e.g. settings are copied on creating work orders and can then be adjusted later as needed.

On opening the FDC Setup page, several settings are checked and initialized, if nonexistent. The journal templates for invoicing, allocation and processing are created automatically and can be shown or hidden via the FDC Setup. Additionally, the number series are generated.

: FDC Setup

The fields in the FDC Setup page are described in the following sections.

General FastTab

The fields on this FastTab are described in the following table.

Field

Description

Order Bundle Time Distribution

Distribution of time for new FDC work orders (refer to topic "Distribution" on page 1).

OB Qty. Distribution Default

Distribution of quantity for new FDC work orders (refer to topic "Distribution" on page 1).

Recording Time Assignment Lapse (h)

Summarizes different FDC postings in the same FDC entry. Therefore, FDC postings with subtype Recording will have begin time and end time that look for other FDC detailed entries in the same range of time. For example, a value of 1 (hour): a posting of 12:30 to 15:00 would be appended to an existing FDC detailed entry from 09:00 to 12:00.

Interrupt Time Assignment Lapse (h)

Summarizes different FDC postings in the same FDC entry. Therefore, FDC postings with subtype Break will have begin time and end time that look for other FDC detailed entries in the same range of time. So breaks can be posted to existing FDC entries with subtype Recording automatically.

Create Missing FDC Resources

When activated, creates a new FDC resource when an FDC resource cannot be associated with a source on creating FDC work orders.

If not activated, no resource is set if no existing resource is found.

Posting Permission Init Type

Specifies the posting permissions on creating an FDC work order line. When set to None, no posting permissions are deposited.

Order Stock Type

Used for initialization on creating an FDC resource.

Concurrent Beginnings

Specifies the initial concurrent beginning on creating an FDC resource (refer to topic "Concurrent Beginning" on page 1).

FDC Work Area Assignment Lapse (h)

Specifies the number of hours an FDC resource was assigned to the last FDC work area without changing the work area (refer to topic "FDC Work Areas" on page 1).

Create FDC WO Automatically

Creates FDC work orders automatically for source documents.

Allow Changes on FDC WO Src.

Locks/unlocks changes in source documents when transferred to FDC work orders, unless posted entries exist for the FDC Work Order.

Allow Posting for changed FDC Work Order Sources

Locks/unlocks changes in source documents, even if posted entries exist for the FDC Work Order.

Note

Not all changes are transferred from the source to the FDC Work Order automatically.

Source Doc. Key as Description

When activated, the source document key is transferred to the FDC work order line as the description.

Input Entry FastTab

The fields on this FastTab are described in the following table.

Field

Description

Processing FDC Journal Template Name

Specifies the FDC journal template name for automatic processing of FDC.

Processing FDC Journal Batch Name

Specifies the FDC journal batch name for automatic processing of FDC.

Processing STDC Journal Template Name

Specifies the STDC journal template name for automatic processing of FDC.

Processing STDC Journal Batch Name

Specifies the STDC journal batch name for automatic processing of FDC.

Automatic Processing Input Journal When Posting

When active, input entries are processed automatically after posting (refer to topic "Processing" on page 1). Activating this field is recommended for an automated run of FDC and STDC.

Autom. Posting When Processing Input Journal

Specifies if processed input entries in FDC and STDC journal templates should be posted instantly (refer to topic "Processing" on page 1). Activating this field is recommended for an automated run of FDC and STDC.

Dim. Values Mandatory on End

FDC Dimensions with “Code Mandatory” have to be specified only at “End” (refer to topic "Input Entries" on page 1). In the FDC terminal, the corresponding dialogues are shown for capture.

Link Input Entries While Posting

Specifies if automatically linked input entries should be found and connected during posting. Activation is recommended for an automated run of FDC.
Run as Limited User

When cc|fdc is run as Limited User, FDC Input Entries will not be processed automatically – this should be done by a job routine or a manual processing.

This option must be active when, e.g., the FDC Terminal is used by Limited Users.
