NAV-Client-Terminal (Classic Client and RTC)

The Terminal is a RoleTailored Client (RTC) Microsoft Dynamics NAV Client Add-In and has to be installed on every machine running the RTC (refer to topic "FDC Terminal" on page 1).

: Microsoft Dynamics NAV Client Add-in Folder

It is also possible to run the Terminal in the Classic Client. The Classic Client does not require a DLL or an Add-in.

The Terminal consists of a Main Menu and a couple of input dialogues (e.g. FDC Terminal Resource dialogue).

: Terminal Main Menu

“Come” and “Leave” captures STDC Check ins and Check outs. For that, Input Journal lines without reference to an FDC Work Order are created and posted in the background. “Break” captures an STDC Interruption for the FDC Resource which can be finished with an additional “Come”.

Recordings for FDC Work Order Operations can be done with “Start” and “Stop” commands. In addition, “Stop” asks for output quantity, which can also be entered by using “Output”. “Consumption” captures consumed quantities and “Work Area” offers the change of the current FDC Work Area.

: Terminal - Choose FDC Resource

In the FDC Terminal Resource dialogue the FDC Resource can be entered via Barcode Scan, using the NumPad on the right or by lookup “…” at the end of the input field. The lookup also shows some additional information about the FDC Resource.

: Terminal - Output

The setting in the Dim. Values Mandatory on End field (in FDC Setup) forces the dialogue to ask for all FDC Dimension values that are set with “Code Mandatory” after entering an output or logging out. This functionality can be used for result codes in the Terminal.

In FDC Setup, the Terminal Lookup Type field determines how lookup pages appear in the Terminal. The options are:

  • Terminal List – applies a list view that is optimized for the Terminal.
  • Microsoft Dynamics NAV Page – opens the default Microsoft Dynamics NAV pages of lists, e.g., for items or FDC Resources.

Interfaces

It possible to connect interfaces to hardware-oriented terminals (e.g. Kaba or Datafox) or MES-Systems. Please refer to your COSMO CONSULT consultant for more information.
