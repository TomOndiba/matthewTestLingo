FDC Work Areas

FDC Work Areas are used for:

  • Localization / detection of relevant work orders (order stock)
  • Permission of posting on FDC Work Orders
  • Default diverse tables to allow different assignments / settings

Examples:

  • An employee in “Steel” will not see the order stock of “Woodworking”.
  • The “Steel” employee should not be allowed to post on work orders of “Woodworking”.
  • Concurrent beginnings are permitted only in the assigned Work Area for the employee.

: FDC Work Areas

The fields in the FDC Work Areas page are described in the following table.

Field

Description

Code

Unique identifier of the work area; maximum 20 alphanumeric characters.

Name

Brief description of the work area.

It is crucial that each FDC Resource can change from one work area into different work areas. For this reason, the FDC Resource has a regular work area and the current FDC Work Area. The work area changes can be posted using the Input Journal.

The current work area of an FDC Resource at a desired date is found through- the last posted work area change. The FDC Work Area Assignment Lapse (h) field in FDC Setup defines how long the FDC Resource remains in the associated FDC Work Area with no further changes. Accordingly, the FDC Resource “leaves” the work area automatically after the noted time has lapsed. It is also possible to post an empty work area change (equivalent to “leave”).

From the FDC Resource card:

  • The Regular Work Area field serves as the default value of the FDC Work Area field in an FDC Work Order Line.
  • The Current Work Area field serves as the default value of the FDC Work Area field in an FDC/STDC Journal Line.
