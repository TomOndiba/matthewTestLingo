Invoicing

Invoicing is a function of STDC to transfer STDC Breaks into FDC. Breaks can be posted in STDC automatically using shift structures, and can be transferred into FDC (Automatic Invoice Break When Posting field in FDC Setup) by using Invoicing.

On creating a (new) FDC Entry, cc|factory data capture checks on given Start Time and End Time if a shift structure has been assigned to the FDC Resource. If a relevant shift is found, the shift structure, the shift date (Start Date of the Shift) and – if existing – a break time pattern is added to the FDC Entry. If a break time pattern is deposited, the break time pattern is posted into STDC on automatic Break Invoicing after posting of FDC. Through Invoicing, those recorded Interruptions will be transferred afterwards as breaks into the FDC. This ensures that Interruptions of STDC – recorded independent of break time patterns – get transferred to FDC. Therefore, a record with Start Time and End Time is necessary so that entries can find each other.
