Input Journal

In the Input Journal, capturing can be done for FDC as well as for STDC (Simple Time Data Capture). A capturing of STDC differs from FDC capturing because no FDC Work Order No. (and Line) is entered.

: FDC Input Journal

Beside the capturing of presence, break, setup, idle and unplanned failure time – which are entered with Entry Type = Begin – it is possible to capture output, consumption and work area change in the Input Journal.

: STDC Capturing in Input Journal

An Input Journal line results in exactly one Input Entry. The Posting Date, Document No. and Posting Time fields are always required.

The fields in the FDC Input Journal page are described in the following table.

Field

Description

Posting Date

Specifies the date on which the posting should be processed. The date is used among others for permission, order stock or work area assignment of the selected FDC Resource. A value is required.

Document No.

Code field for entering a number.

Entry Type

Specifies the type of capturing. The options and the related/required fields are:

  • Begin and End – together result in a capturing pair, including the entry subtype, the FDC Resource and the FDC Work Order Operation. For STDC Postings, only Begin and End are valid entry types.
  • Output – requires entries in the Item No. and Output Quantity fields.
  • Consumption – requires an entry in the Consumption Quantity field.
  • Work Area Change – requires an entry in the FDC Work Area No. field.

Entry Subtype

Determined based on the Entry Type:

  • Begin and End – requires an Entry Subtype.
  • Consumption and Output – no requirement.
  • Work Area Change – requires an empty Entry Subtype.

FDC resource No.

Specifies the FDC Resource of the entry.

FDC Work Area No.

Specifies the new FDC Work Area for a Work Area Change, or the Work Area of the posting (informational, no further checking).

FDC Work Order No.

If empty, the entry is processed for STDC.

Posting Time

Specifies the time on which the posting should be processed. The time together with date is used among others for permission, order stock or work area assignment of the selected FDC Resource. A value is required.

Item No.

Required for Entry Type = Consumption and Output, is optional for Entry Type = End.

Output Quantity

Specifies the quantity of output. A value is required for Entry Type = Output.

Scrap Quantity

Specifies the quantity of scrap. A value is relevant for Entry Type = Output.

Consumption Quantity

Specifies the consumed quantity of the item entered in the Item No. field. A value is required for Entry Type = Consumption.

Cancel FDC Input Entry No.

Cancels an existing input entry with the new entry.

Finished

If selected for an Entry Type = End, the related operation of the production order receives the status “Finished” during the export.
