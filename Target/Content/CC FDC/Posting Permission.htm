Posting Permission

Postings of an FDC Resource on an FDC Work Order Operation can be permitted or declined through the use of posting permissions. An FDC Resource may be permitted directly or via an FDC Work Area to capture times or quantities for an FDC Work Order Operation. On the Lines FastTab of the FDC Work Order, click Line > Posting Permission to configure the permissions. If no permission is configured, all FDC Resources are available for capture.

: Posting Permissions

The “Assign to all FDC Work Order Lines” function copies the current permissions to all FDC Work Order Operations of the FDC Work Order. On creating an FDC Work Order Operation, a posting permission can be set by default. This default value is set in FDC Setup in the Posting Permission Init Type field. The options are:

  • None – does not create any permissions so no limitations are set.
  • FDC Resource – permits only the corresponding FDC Resource to post to the work order operation.
  • Regular FDC Work Area – permits the current FDC Work Area of the corresponding FDC Resource to post to the work order operation.
  • Current Applied FDC Work Areas of FDC Resource – permits all FDC Work Areas that the FDC Resource is currently assigned to, including the regular work area, to post to the work order operation.
