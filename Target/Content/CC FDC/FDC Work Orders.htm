FDC Work Orders

FDC Work Orders – like FDC Resources – are essential for the use of cc|factory data capture. The list of FDC Work Orders is accessible from the navigation pane by clicking Departments > COSMO CONSULT > cc|bde (FDC) > Planning & Execution or Settings.

FDC Work Orders with source (refer to topic "Order Source" on page 1) connect cc|factory data capture with standard Microsoft Dynamics NAV and can consist of a job, or of a production, service or assembly order. Additionally, when not linked to a source, FDC Work Orders can be processed only in cc|factory data capture. Every FDC Work Order contains one or more FDC Work Order lines, to which postings can be applied.

: FDC Work Order

The fields in the FDC Work Order page are described in the following table.

Field

Description

No.

Unique identifier of the work order, can be set manually or automatically.

Description

Brief description of the work order; automatically filled when selecting a source.

Status

Indicates the current status of the FDC Work Order. The options are:

  • Open – Only changes can be applied to FDC Work Orders.
  • Released – Required for posting to FDC Work Orders.
  • Finished – Completed, non-editable FDC Work Orders; available in the navigation pane by clicking COSMO CONSULT > cc|bde (FDC) > History > Lists > Finished FDC Work Orders.

FDC Work Order Lines Finished

Indicates if all FDC Work Order lines are finished.

Export Blocked

When active, postings of this work order must not be exported.

Type

Indicates the type of the order. The options are:

  • No Source – only used in cc|factory data capture
  • With Source – connected to an order in standard Microsoft Dynamics NAV
  • Order Bundle (refer to topic "Order Bundles" on page 1)

This field is set automatically.

Order Bundle Time Distribution

Indicates how the captured time should be distributed to the work order lines. The options are: Uniform, Expected Time / Quantity, Actual Quantity and Custom Distribution / Factor Calculation (refer to topic "Order Bundles" on page 1). The default distribution is set in the FDC Setup.

Order Bundle Qty. Distribution

Indicates how the captured quantity should be distributed to the work order lines. The options are Uniform, Expected Time / Quantity, Actual Time and Custom Distribution / Factor Calculation (refer to topic "Order Bundles" on page 1). The default distribution is set in the FDC Setup.

Order Bundle FDC Res. Filter

Used for filtering FDC Resources when running the “Assemble Routing Lines” function.

Total Actual Gross Time (h)

Displays the sum of the gross time of all posted entries of the order.

Total Actual Net Time (h)

Displays the sum of the net time of all posted entries of the order.

Distributed

Indicates if the “Distribute” function was run on the last posting of the order bundle.

Distribute Date

Indicates the last date when the distribution was used.

Distribute Time

Indicates the last time when the distribution was used.

Distributed by User

Indicates the User ID who last used the distribution.

Distribute Error

Displays the error message if an error occurred during distribution.

The functions in the Line tab on the Lines FastTab are described in the following table.

Action

Description

FDC Entries / FDC Input Entries

Displays a list of corresponding FDC (Input) Entries of the routing line.

Posting Permission

Used to set the posting permissions of a routing line. If not set, everyone is permitted to post for this routing line.

FDC Dimensions

Used to assign FDC Dimensions to the routing line.

Permitted FDC Dimension Values

Used to assign Permitted FDC Dimension Values to the routing line.

Source Document

Displays the source document of the FDC Work Order (e.g., the corresponding production order).

Source Document Line

Displays the source document line of the FDC Work Order Line (e.g., the corresponding Prod. Order Routing).

Navigate

Displays an overview of entries of the FDC Work Order, FDC Work Order Line and the source document.

FDC Work Order Lines are shown partially in different styles:

  • Bold – Order bundle (Line with Line No. 0)
  • Grey – Redirecting is active
  • Orange – An released order bundle exists, which is linked to this line
