FDC Journal

The FDC Journal generates FDC Entries and FDC Detailed Entries. From every FDC Journal Line results a maximum of one FDC Entry and at least one FDC Detailed Entry. In contrast to the Input Journal, here only a line with Begin Time and End Time is captured that creates a complete time entry; no entry for Begin and End is produced (refer to topic "FDC Entries" on page 1).

: FDC Journal

The capturing in the FDC Journal is applied with an Entry Type (FDC Entry) and a detailed Entry Type (FDC Detailed Entry). With Entry Type = Recording, it is possible to capture times as well as quantities in one line. The selection of the Detailed Entry Type field specifies if the Total Time (h) is handled as a work time (“Input”), setup time or idle time (posted as positive), or if it is a break time or unplanned failure time (posted as negative).

Beside Total Time (h) there is the possibility to create simultaneously another FDC Detailed Entry of the corresponding Type for the FDC Entry, using the Contained Break Time (h) and Contained Idle Time (h) fields. Therefore, it is possible to enter in a single FDC Journal Line a time capture and a break capture. Contained times are deducted from the total time.

However, postings containing times should be processed without Begin Time and End Time, because otherwise they do not correspond to the Total Time of the capture.

In case of posting an interruption afterwards on an FDC Entry, either a recording and an interruption with concrete times have to be posted or a chargeback (Applies-to FDC Entry field) must be applied.

In case of posting an interruption afterwards on an FDC Entry, either a recording and an interruption with concrete times have to be posted or a chargeback (Applies-to FDC Entry field) must be applied. Finding the capture time for an interruption time occurs by using the entered time of the line, extended by the Interrupt Time Assignment Lapse (h) field. This means that a “capture” from 08:00 to 12:00 and a “break” from 13:00 to 14:00 are combined as one common FDC Entry if the Interrupt Time Assignment Lapse (h) is greater than 1 (h).

The fields in the FDC Journal are described in the following table.

Field

Description

Posting Date

Specifies the date on which the posting should be processed.

Document No.

Code field for entering a number.

Entry Type

Defines the type of capturing. The options are:

  • Recording – allows entering times and output quantities.
  • Output – requires entries in the Item No. and Output Quantity fields.
  • Consumption – requires an entry in the Consumption Quantity field.

Detailed Entry Type

Determined based on Entry Type:

  • If Entry Type = Output and Consumption – the Detailed Entry Type field has to be set to Recording.
  • If Entry Type = Recording – the Detailed Entry Type field corresponds to the kind of Total Time:
    • Recording, Setup, and Idle result in positive Total Time
    • Break and Unplanned Failure Time result in negative entries

Applies-to FDC Entry

Creates the new detailed entry within the FDC Entry that has to be applied.

FDC Resource No.

Specifies the FDC Resource for the entry.

FDC Work Area No.

Specifies the FDC Work Area (informational, no further checking).

FDC Work Order No.

Specifies the FDC Work Order for which the entry should be processed.

FDC Work Order Line No.

Specifies the FDC Work Order Line for which the entry should be processed. The value “0” is only valid for Order Bundles.

Total Time (h)

Contains the captured time of Detailed Entry Type.

Contained Break Time (h)

Reduces the Total Time (h) and generates an additional Detailed Entry Type = Break.

Contained Setup Time (h)

Reduces the Total Time (h) and generates an additional Detailed Entry Type = Setup.

Contained Idle Time (h)

Reduces the Total Time (h) and generates an additional Detailed Entry Type = Idle.

Start Date/ Start Time

Specifies the Start Date and Start Time of the Detailed Entry.

Automatic assigning of break times or shift invoicing requires values for Start Time and End Time.

End Date/ End Time

Specifies the End Date and End Time of the Detailed Entry. If a Start Time is entered, entering an End Time is also required.

Output Quantity (Base)

Specifies the quantity for an output. The value is relevant for Entry Type = Output.

Scrap Quantity (Base)

Specifies the scrap quantity for an output. The value is relevant for Entry Type = Output.

Consumption Quantity (Base)

Specifies the consumed quantity for an item, entered in the Item No. field. The value is relevant for Entry Type = Consumption.

Cancel FDC Detailed Entry

Cancels an existing FDC Detailed Entry while posting a new Detailed Entry.

Finished

If selected for an Entry Type = End, the corresponding operation of the Production Order receives the status “Finished” on export.

Force New FDC Entry

Forces the creation of a new FDC Entry. When using this e.g. the assignment of breaks with Recording Time Assignment Lapse (h) is ignored.

Invoice Entry

Specifies if the generated Detailed Entry is an Invoice Entry.

Distribute Entry

Specifies if the generated Detailed Entry is an Distribution Entry.
