Concurrent Beginning

Concurrent Beginning enables simultaneous checking in of an FDC Resource at different FDC Work Order Operations. A Concurrent Beginning of the same FDC Resource at the same FDC Work Order Operation is always declined. This function is determined by the value in the Concurrent Beginning field in FDC Setup. The options are:

  • Prohibited – allows only one (active) check in (Input Journal Type = Begin) of the FDC Resource. This includes “Login” and “Come” in the FDC Terminal. The beginning of a break (Entry Subtype “Break Time”) is still possible.
  • Only in the same FDC Work Area – allows a Concurrent Beginning of FDC Work Order Operations in the same FDC Work Area of the FDC Resource.
  • Always – allows beginning FDC Work Order Operations in foreign work areas.
