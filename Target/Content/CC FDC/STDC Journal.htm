STDC Journal

In the STDC Journal, just like the FDC Journal, STDC captures can be made. In STDC, every STDC Entry has at least one STDC Detailed Entry.

: STDC Journal

Every STDC Journal line results in maximum one STDC Entry and at least one STDC Detailed Entry. Similar to the FDC Journal, Start Time and End Time is captured, resulting in a full time entry (refer to topic "STDC Entries" on page 1).

Capturing in the STDC Journal occurs using a detailed entry type. The selection of detailed entry type defines, if the Time (h) field is work time (Recording), setup time or idle time (posted as positive value), or if it is a break time or unplanned failure time (posted as negative value).

Beside Time (h), an additional FDC Detailed Entry Type = Interruption may be generated for the FDC Entry while posting, using the Contained Break Time (h) field. That way it is e.g. possible to enter a recording and a time of interruption in one single FDC Journal Line. Contained times are deducted from the duration. However, postings containing times should be processed without Start and End Time, because they will not correspond to the total time.

If an interruption of a STDC Entry should be posted afterwards, either the “Recording” and “Interruption” with time details must be posted, or a chargeback must occur (Applies-to STDC Entry field). Finding the interruption time of a recording time is done by the time details of a line extended by the Recording Time Assignment Lapse (h) field. That means that a “Recording” from 08:00 to 12:00 and an “Interruption” from 13:00 to 14:00 is combined to the same STDC Entry, if an Assignment Lapse is greater than “1” (h).

The fields in the STDC Journal page are described in the following table.

Field

Description

Posting Date

Specifies the date on which the processing should be posted.

Document No.

A code field for entering a number.

Detailed Entry Type

Determined based on Entry Type:

  • Recording is a positive duration
  • Break and Unplanned Failure result in a negative posting

Applies-to STDC Entry

Creates the new detailed Entry within the STDC Entry that is to be applied.

FDC Resource No.

Identifies the FDC resource for the entry to be created.

FDC Shift Structure

Indicates the shift structure of the STDC Entry.

Work Shift

Indicates the shift of the STDC Entry.

FDC Break Time Pattern

Defines the deposited Break Time Pattern, to which the break should be posted.

Shift Start Date

Indicates the shift start date on which the shift has begun.

Contained Break Time (h)

Reduces the Time (h) and generates an additional Detailed Entry Type = Break.

Time (h)

Contains the time of the Detailed Entry Type.

Start Date-Time

Specifies the Start Date and Time of the detailed entry. Only breaks with Start Time and End Time is to be transferred to FDC when using automatic invoicing.

End Date-Time

Specifies the End Date and Time of the detailed entry. If a Start Time is entered, entering an End Time is required.

Cancel STDC Detailed Entry

Cancels an existing STDC Detailed Entry while posting a new Detailed Entry.
