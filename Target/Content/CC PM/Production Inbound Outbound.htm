Production Inbound/Outbound Location Code and Bin Code

Default Location/Bin Codes can be assigned on the Planning FastTab of the Item Card in these fields:

  • Prod. Inbound Location Code / Prod. Inbound Bin Code
  • Prod. Outbound Location Code / Prod. Outbound Bin Code

: Location/Bin Code defaults on Item Card

The defaults are transferred to production order lines and components based on these conditions:

  • The Use Location and Bin Defaults check box is selected on the Process Manufacturing Setup page.

: Use Location and Bin Defaults setting on Process Manufacturing Setup page

  • The Comp. Location Code field is empty on the Production BOM lines (refer to topic “Default Location/Bin Codes” on page 1).
  • The default fields are populated on the Item Card.

Production Order

Default Transfer Conditions

Header / Lines

When the production order is created and the Prod. Inbound Location Code on the Item Card is not empty, then the values in the Prod. Inbound Location Code and Prod. Inbound Bin Code fields from the Item Card are transferred to the Location Code and Bin Code fields on the Production Order header / line.

Components

When the production order is created and the:

  • Comp. Location Code field on the Production BOM line is not empty, then the values in the Comp. Location Code and Comp. Bin Code fields from the Production BOM line are transferred to the Location Code and Bin Code fields on the Prod. Order Component line.
  • Comp. Location Code field on the Production BOM line is empty and the Prod. Outbound Location Code field on the Item Card is not empty, then the values in the Prod. Outbound Location Code and Prod. Outbound Bin Code fields from the Item Card are transferred to the Location Code and Bin Code fields on the Prod. Order Component line.

Note

Settings on the Production BOM override settings on the Item.
