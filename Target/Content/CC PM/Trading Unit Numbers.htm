Trading Unit Numbers

In standard Microsoft Dynamics NAV, users can assign lot numbers to inventory. In cc|process manufacturing, the standard lot number is split into Lot No. Trading Unit and Trading Unit No. fields, which can be shown in item tracking. If a user enters values in both fields, the values are concatenated and copied into the Lot No. field. This function allows users to track each trading unit of one lot.

For example, a warehouse receives ten barrels of a liquid, each are from lot L1234. The company wants to track each barrel, so the receiving manager enters ten lines in item tracking. Each line has lot number trading unit L1234 but in the Trading Unit No. field, he or she enters 001, 002, 003 and so on. Now the company can track each barrel.

The Trading Unit No. Required field on the Item Card determines whether a Trading Unit No. can be used (Optional) / is required (Always) / must not be used (Never) on posting.

To make it easier to fill in Trading Unit Numbers in the Item Tracking, the Default Start Trading Unit No. field has been added to the Numbering FastTab in the Inventory Setup page. The value entered here will be automatically transferred to the Start Trading Unit No. field in the Assign Lot/Trading Unit page, accessed from the Item Tracking Lines page.
