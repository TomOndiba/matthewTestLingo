Test Plan Setup on Items

cc|process manufacturing provides the ability to test items based on the Lot No. Trading Unit or the Trading Unit Number, as indicated in the Test Plan Setup page, which is accessed from the ribbon on the Item Card.

: Test Plan Setup for item Q-20001

The Item Test Order Options for Test on Lot No. Trading Unit and Test on Trading Units are described in the following table.

Field

Description

Item Test Order Option

Defines the level on which an item should be tested. The cc|process manufacturing options are:

  • Test on Lot No. Trading Unit – a test order is created for each lot/trading unit.
  • Test on Trading Unit – a test order is created for each trading unit.

If this option is selected and no trading unit is entered in the item tracking, the test will be performed per lot number.

For example, if a user posts a Receipt with two lots (Lot A and Lot B) the following tests are created, based on the Item Test Order Option:

  • Test on Lot No. Trading Unit:

Two Test Orders (one for each lot) – Item No. field is populated in the General FastTab, Lot No. and Lot No. Trading Unit No. fields are populated on the Test Plan Source FastTab.

  • Test on Trading Unit:

Two Test Orders (one for each lot) – Item No., Lot No., Lot No. Trading Unit and Trading Unit No. fields are populated.
