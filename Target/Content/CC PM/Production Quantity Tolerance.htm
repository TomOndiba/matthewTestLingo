Production Quantity Tolerance

The use of the Production Quantity Tolerances is slightly different than the Delivery Tolerances because it is used internally.

Initially, a production tolerance, a replenishment tolerance or a sales tolerance is entered for each item on the Item Card.

: Production quantity tolerance assigned to item P-10500

The production quantity tolerance does not apply to creating a production order for an item that has a production tolerance; the quantity can be adjusted without considering the planning tolerances.

 

: Order to produce 1000 units of item P-10500

Production quantity tolerance is used when posting output.

: Message received when changing the posting quantity from 1000 to 985 units of output for item P-10500

After changing production quantities, the related production order line is updated accordingly.

: Updated production order lines based on output change

With production tolerance, users can reduce the quantity within the range of the tolerance and the system will reduce the quantity that must be posted. This ensures that there is no remaining quantity after posting, which avoids the need to start a partial production for a small quantity with new setup times.

: Production Journal

The original quantity entered is stored in the Original Quantity field in the Delivery Tolerance Entries page from the Item Card; this value is never changed. If the quantity to handle is changed, the quantity in the document line is changed, but not the original quantity in the delivery tolerance entries. If the quantity is re-entered in the document line, the delivery tolerance entry is deleted.
