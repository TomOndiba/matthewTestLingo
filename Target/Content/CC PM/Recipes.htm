Recipes

Each recipe is contained in the Recipe Header page. In this page, users can maintain the percentage of the division of ingredients (raw materials) for the manufactured items.

: Recipe Header for item P-10002, Panel

General FastTab

The fields on this FastTab are described in the following table.

Field

Description

No.

Unique identifier for the recipe.

Description of the recipe.

Unit of measure to which the recipe refers.

Lines FastTab

In the recipe line, the Item Type field distinguishes between:

  • Share of Recipe – an item entered in percent or proportion.
  • Item – an item that can be entered in component-value and unit of measure.

Due to this, it is possible to maintain in the recipe not only items with percentages but also to enter, for example, the packaging.

Recipe lines can be calculated using the following methods:

  • Share % – total shares must equal 100 percent.
  • Shares – total share amounts must equal 100 and automatically calculate the percentage in the Share % field when the recipe is certified.
  • Quantity per – absolute values.

Refer to online help for more information about the other fields on this FastTab.

Copy BOM Function

On the Actions tab, click Copy BOM to copy lines from existing production BOMs/recipes into the current recipe. When complete, all necessary changes can be made.

Navigate Tab

Click the Navigate tab to access actions specific to the entire recipe.

Action

Description

List

Opens a list of all recipes.

Comments

Displays comments for the selected recipe. Comments are for internal use only; they are not used on any documents.

Versions

Opens the list of recipe versions for the selected recipe. Versions allow users to create different versions of a recipe which is useful if changes should be documented. Every version of the recipe can have a unique start date in the header and a unique start and end date in the lines.

All production orders that start after the start date of a recipe are linked to the corresponding version of the recipe. It is possible to create two versions of one recipe with the same start date. In this case, the correct version can be chosen over the production order lines.

Matrix per Version

Opens the Prod BOM Matrix per Version page, which enables users to compare all versions of a recipe.  This page provides an overview of all components on one or more layers and shows the percentile shares of recipe components.

Where-used

Shows where a recipe is used in a production structure. Single and multilevel features are helpful for maintenance or troubleshooting. The multilevel feature displays all effected BOMs.

Ingredients

Opens the Production BOM Header Ingredients page, which displays the sum of recipe ingredients (refer to topic “Ingredients Management” on page 1).

Incorporation Items

Allows users to determine the incorporation items for this recipe (refer to topic “Incorporation” on page 1).

Recipe Comparison

Opens the Recipe Comparison page (refer to topic “Recipe Comparison” on page 1).

Recipe Search

Opens the Recipe Search page (refer to topic “Recipe Search” on page 1).

Multilevel Recipes

In the production structures, different items or recipe shares can be entered or a new main recipe can be added. If a user enters a recipe as the main or base recipe, it can be selected in the production structure.

: Main recipe used on production structure

If the production structure is used in production, the lines of the main recipe are transferred to the production order component lines and are divided accordingly.

: Components of the main recipe

This makes it possible to use a main recipe that is only a part of an overall recipe.

For example, an overall recipe has a main recipe of 90% and a color share of 10%. Therefore, the components of the main recipe are divided to 90% of the entire recipe and the color share holds the remaining 10%.

Using a main recipe is helpful because it only needs to be configured once. For example, if the raw materials change, they are used in all new production orders.

Change Recipe

With the “Change Production BOM No.” function, the used production BOM for released production orders can be changed even if a finished quantity was posted.

: Released Production Order with partially finished quantities

On the Lines FastTab, click Line > Change Production BOM No. The Production BOM List page appears, from which a new production BOM can be selected.

Note

To run the “Change Production BOM No.” function, the Allow Product Design Change in Prod. Order Line check box must be selected on the Process Manufacturing Setup page.

: Production BOM List with new BOM selected to be transferred to production order line

After selecting the new production BOM, or recipe, the quantity of the existing production order line is set to the finished quantity and a new production order line is added and calculated.

: Released Production Order with new Production BOM line

The order tracking of the remaining quantity is transferred to the new production order line. Existing reservations attached to the old production order components are transferred.

: Prod. Order Components on the new Production BOM line
