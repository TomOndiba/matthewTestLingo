Sales Delivery Tolerances in Warehouse and Inventory Picks

Using sales delivery tolerances with Picks is like Sales Orders and Warehouse Shipments except for the following behaviors:

  1. The confirmation dialog for quantity changes is always executed, regardless if the Change Quantity field is set to Question in the Delivery Tolerances page. This is necessary to perform the split line function.
  2. If a split pick line is deleted, the original quantity is restored on the first remaining pick line.

The Quantity to Handle field on Pick lines can be increased or reduced but not exceed the tolerance.

: Message to increase Qty. to Handle to 105 from shipment quantity of 100

The change on the Pick line is transferred to the Shipment line and originating Sales line.

: Quantities increased on Sales and Shipment lines based on Pick line tolerance change

Pick Line Split Tolerance Check

If a split Pick line is deleted, the original quantity is restored on the first remaining Pick line.

For example, a Pick line with a quantity of 100 is split into two equal quantity lines of 50. If the second line is increased to 60, a tolerance message is generated to increase the quantity.

: Message to increase Qty. to Handle to 60 from split pick line of 50

Note

If the Qty. to Handle field had been set to 65, an error message would have been generated stating the exceeded quantity and the prevented change.

: Quantity increased within the tolerance on second Pick line

The change on the Pick line is transferred to the Shipment line and originating Sales line.

When one of the split Pick lines is deleted, the original quantity of 100 is set on the remaining Pick line and the Qty. to Handle field is unchanged.

: Original quantity restored on remaining Pick line (second line with Qty. to Handle = 60)
