QC Status on Locations

When a QC Status change occurs, the history can be tracked with the item tracing functionality if the QC Status is used with locations.

For each QC status, a fixed location can be specified. This status indicates in which location an item must be stored during certain processes.

: QC Status with standard location codes highlighted

For example, items that are set up to default to the Blocked QC status must be placed in the BLOCKED location during transfers, items with QC Status = FREE must be placed in the MAINWH location during all processing, and so on.

This information not only controls whether certain goods with a certain status will be properly stored, but also when changing from one status to another.

In the QC Status Change page, if the Transfer check box is selected, the location code is also changed in the item ledger entries to match the standard location code connected to the status entered in the New QC Status field.

The Transfer check box can only be selected when the New QC Status field is filled in. When the Transfer check box is selected, the Posting Date and Document Date fields become available and indicate the dates used during the transfer posting.

 

: QC Status set to be changed from Blocked to Free

As Fig. 2 illustrates, the QC Status is set to be changed from Blocked to Free and the current Location Code is set to BLOCKED. Since the Transfer check box is selected, the Location Code will change to MAINWH when the function is run.

: QC Status changed to Free and Location Code set to MAINWH

As previously stated, the item ledger entries are updated with the QC Status, date and Location Code changes.

: Item Ledger Entries updated with date, QC status and location changes

The “Transfer” function is also available in the Batch QC Status Change page.
