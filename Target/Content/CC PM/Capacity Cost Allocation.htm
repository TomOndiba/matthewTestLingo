Capacity Cost Allocation

By-products that are added to production orders are allocated to the production order lines. The output of both lines are posted separately. The capacity costs (run and setup time) are only posted with the output of the normal production order line.

There are three ways in which the capacity costs can be calculated:

  1. Costs are shared proportionally.
  2. All costs are calculated on major output.
  3. Costs are assigned by the item Costing Method = Standard; item unit cost is preset based on estimation.

In the Process Manufacturing Setup page, the By-Product Cap. Cost Alloc. field determines if the first or second option is used:

  • Quantity Proportional – the total resulting capacity costs are allocated quantity proportional to each output posting.
  • None – the capacity cost is allocated to the specific output that was posted with run or setup time.

: By-Product Capacity Cost Allocation options in Process Manufacturing Setup

Calculating the cost based on the item’s costing method of Standard uses the standard Microsoft Dynamics NAV costing functionality.

This section describes the different allocations based on the settings in the By-Product Cap. Cost Allocation field.

Quantity Proportional Cost Allocation

The Quantity Proportional setting is the default for by-product capacity cost allocation in Process Manufacturing Setup.

In this example, the following items have been set up as described for by-production:

  • P-BP01 – finished product
  • P-BP02 – by-product
  • P-BP03 – raw material

The production structure for the finished product uses 1.5 quantity per of the raw material and includes work center M008 with a run time of 1 hour. For every 1 KG of the finished product, .5 KG of the by-product is created. The raw material has a direct unit cost of 2.00.

After a production order for 10 units of the finished product was processed and finished, the “Adjust Cost - Item Entries” function was run for all related items. The posted capacities are then reviewed.

: Capacity Ledger Entries for finished production order

As illustrated in Fig. 2, the posted capacity cost is 10 (Direct Cost = 9.5 + Overhead Cost = 0.5).

The actual cost allocation is visible in the Item Ledger Entries.

: Item Ledger Entries with quantity proportional allocated capacity costs

As illustrated in Fig. 3, the capacity cost is allocated quantity proportional as follows:

  • Finished product = 36.67 Cost Amount (Actual)
    • Raw material = 30.00
    • Capacity = 6.67
    • (10 / 15 * 10) (total cap. cost / total output qty. * qty. of finished product)
  • By-product = 3.33
    • Raw material = 0
    • Capacity = 3.33
    • (10 / 15 * 5) (total cap. cost / total output qty. * qty. of by-product)

Cost Allocation to Finished Good

In this example, the same item by-product configuration is used, and the None option is selected as the by-product capacity cost allocation in Process Manufacturing Setup.

After a production order for 10 units of the finished product was processed and finished, the “Adjust Cost - Item Entries” function was run for all related items. The posted capacities are then reviewed.

: Capacity Ledger Entries for finished production order

As illustrated in Fig. 4, the posted capacity cost is 10 (Direct Cost = 9.5 + Overhead Cost = 0.5).

The actual cost allocation is visible in the Item Ledger Entries.

: Item Ledger Entries with capacity costs allocated to finished good only

As illustrated in Fig. 5, the posted capacity cost is allocated to the finished product only :

  • Finished product = 40.00 Cost Amount (Actual)
    • Raw material = 30.00
    • Capacity = 10.00
  • By-product = 0.00
    • Raw material = 0.00
    • Capacity = 0.00
