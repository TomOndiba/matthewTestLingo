Scheduling on Due Date in Sequence Planning

When a sales order is created with a desired delivery date and a quantity of an item which the order can be scheduled, the planning function is used to direct the production.

This option is especially useful for fast processing of smaller work orders, such as when customers cut lengths of standard products from the bearing.

In this example, a sales order for 1000 KG of item P500 is created with the desired delivery date of March 14, 2019. On the ribbon, click Actions > Planning. The standard Sales Order Planning page appears. To create the production order, click Create Prod. Order.

: Planning on Specified Date check box and Specified Date field populated

When the Planning on Specified Date check box is selected, the system checks whether a scheduling order on the work center is possible at the specified time. Depending on the option, either a production order is created, scheduled and fixed at the same time, or the user gets a message that a schedule at this time is not possible.

Note

If the shipment date on the sales line is before the specified date, then it is set to the specified date during the “Create Order From Sales” process.

: Confirmation message

In the Sequence Planning for this work center and production order, the end date is set to 3/7/2019, although the end date on the production order is 3/12/2019.

: Production order in sequence planning and highlighting the Ending Date on the production order

The difference in ending dates is because the desired delivery date is calculated backward from the machine and the other routing operations.

 

: Production order routing operations

In this example, there are six production order routing lines for one production order line. The third line is the sequence work center.

When the sequence planning is calculated:

  1. The starting date of the third production order routing line is updated with the End Date Sequence Planning field of the previous production order line in sequence.
  2. The ending date of the production order routing line is calculated (run and setup time).
  3. The second and the first routing lines are calculated backward based on the starting date of the third routing line.
  4. The fourth, fifth and sixth routing lines are calculated forward based on the ending date of the third routing line.

Now the values of the routing lines are transferred to the production order/sequence planning lines:

  • Starting Date: Starting date of the first routing line
  • Start Date Sequence Planning: Starting date of the third routing line
  • End Date Sequence Planning: Ending date of the third routing line
  • Ending Date: Ending date of the last routing line

Note

This also applies to the time fields (Starting Time, Starting Date-Time, Ending Date-Time and Ending Time).

To use this functionality, the Allow Fix. from Sales Order check box must be selected on the Scheduling FastTab of the work center.
