Update Multilevel Production Orders

If a production order is created using multilevel production, the scheduling for each production step is independent from each other if the starting or ending date is changed manually after the production order was calculated.

: Released production order with Update Multilevel Production Order functionality

As Fig. 1 illustrates, item P-10502 must be produced first to be bottled in a second production step in the item P-10502-F200. The production line P-10502 has been scheduled and is finished at the same time the bottling item is starting to be produced.

The setup of the scheduling using multilevel productions is configured in the Planning Extension Setup page.
