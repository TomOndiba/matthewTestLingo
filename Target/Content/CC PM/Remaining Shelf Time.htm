Remaining Shelf Time

In relation to a customer or customer group there might be delivery arrangements for the shelf life of an item. This guaranteed remaining shelf time is usually arranged between manufacturers and wholesalers.

When creating a sales line, the system checks if a remaining shelf time has been defined for the combination of customer and item / item category. If the item to be sold will expire within the denoted period, the system displays a warning message when assigning the lot number. The remaining shelf time is added to the Planned Delivery Date or – if existing – the Promised Delivery Date.

To use this functionality, the shelf time must be defined on the Remain. Shelf Life Setup page, accessed from the Navigate tab on either the Customer Card or Item Card.

: Remain. Shelf Life Setup page opened from Customer Card

Similar to Sales Prices or Line Discounts, customer and item filtering can be set in the header and then the lines contain the customer/item combination with the specified remaining shelf time.

As illustrated in Fig. 1, when sold to customer 10000, item P-10503 has a remaining shelf time of 60 days. This means that if a lot of item P-10503 is set to expire within 60 days of delivery to customer 10000, a warning message will appear when the lot is entered for the sales line. The warning message is calculated by Warning Date (Expiration Date - Remaining Shelf Time) < Planned Delivery Date.

: Warning message received when planned delivery date is within specified warning date range

Important

This is only a warning message; it does not prevent the delivery to the customer.
