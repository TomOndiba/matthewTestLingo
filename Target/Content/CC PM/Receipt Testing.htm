Receipt Testing

The determination of the actual ingredients of a lot occurs in the goods receipt by cc|quality management. Therefore, a test order with the according test properties is created in Microsoft Dynamics NAV. The potency of the respective ingredients is allocated to the according test properties in the form of the determined actual values.

The link between cc|quality management and the ingredients of the lot is established through the test property that has been allocated to the ingredient (refer to topic “Ingredient Card” on page 1) and the Alloc. Unit of Measure Code assigned to the Test Property.

: Ingredient Card and related Test Property with assigned allocation unit of measure

When the test order is finished, the determined ingredient proportions (actual values from the test order properties) are transferred to the tested lot. Therefore, a lot ingredient record exists for each lot. If a test property is to be tested several times (re-test), the test result of the latest test is always transferred to the according lot.

The Lot No. Ingredients page can be accessed from the Lot No. Information Card by clicking Navigate > Ingredients.

: Actual ingredients transferred to tested lot

The fields on this page are described in the following table.

Field

Description

Ingredient No. / Description

Indicates the ingredient number and description.

Quantity (Expected)

Indicates the calculated and posted value from the source document ingredient, e.g. production order.

Quantity (Actual)

Indicates the real value from the finished test order.

Unit of Measure Code

Indicates the unit of measure code assigned to the item ingredient.

Alloc. Unit of Measure Code

Indicates the allocation unit of measure code assigned to the item ingredient.

Actual Value

If selected, values have been measured through a test order.
