Restricted Sales

Sales restrictions refer to items / finished products that are only to be sold to certain customers; for example, because of legal trade restrictions, a company is not allowed to deliver pharmaceuticals to the US market that are not accepted by the Food and Drug Administration (FDA).

Restricted sales are created from the Item Card or Customer Card. On both cards, the Restricted Sales Setup page can be accessed from the Navigate tab by clicking Restricted Sales.

: Sales restrictions assigned to an item

In the Restricted Sales Setup page, enter the required sales restrictions. As Fig. 1 illustrates, item P-50001 has been assigned as a restricted sale to customer 12000 and all customers in Germany.

The fields in the Restricted Sales Setup page are described in the following table.

Field

Description

Item No.

Indicates the item number to allocate a sales restriction.

If this page was opened from the Item Card, this field is already filled with the item number from the Item Card.

Variant Code

Indicates the item variant to allocate a sales restriction.

Item Category Code

Indicates the item category to allocate a sales restriction.

Note

The insertion of the Item No. / Item Variant Code fields and Item Category Code excludes one another.

Customer Filter

Indicates the customer number for the item to allocate a sales restriction, thus allowing the sale of this item to this customer.

If this page was opened from the customer card, this field is already filled with the customer number from the Customer Card.

Users can also enter filter options. For example, it is permitted to sell item P-50001 to all customers except 30000.

Customer Group Filter

Indicates the Customer Group Code for the item to allocate a sales restriction. This can, for example, be segmentation according to pharmacies, wholesalers, doctors and so on. Filter options can be used here as well.

Country Filter

Indicates the country code for the sales restrictions. Filter options can be used here as well.

Expiration Date

Indicates the expiration date until the sales restriction should be valid.

Combination

Determines if restrictions using the same filter can be combined. The options are OR and AND. For example, the following Customer Filters are set for item P-50004.

If a sales order is created to Customer No. 20000 for item P-50004, the OR filters are tested first. If one of the OR conditions is valid (the second line), then item P-50004 can be added to the sales order for Customer No. 20000.

If a sales order is created to Customer No. 61000, none of the OR conditions meets the Customer No. restrictions so the system tests the AND conditions. The AND conditions must be met all at the same time. Therefore, based on the settings above, if the Customer No. is > 50000 it must not be 61000 and not 62000 so item P-50004 cannot be added to the sales order for Customer No. 61000. If the sales order had been created to Customer No. 60000, then item P-50004 could be added to the sales order.

Additionally, the Sales Restrictions Exist field on the Shipping FastTab of the Customer Card is automatically set to Yes when sales restrictions are assigned for a customer.

: Sales restriction assignment indicated on customer

When a sales order is created for the customer that an item sales restriction has been allocated to, the item can be entered in the sales line.

If the respective item number is entered for a customer who the item has not been allocated to in the Restricted Sales Setup page, the system displays a warning massage.

: Error message when assigning a restricted item to an invalid sales line

The crucial factor regarding the validity of restricted sales in the Expiration Date field is always the Shipment Date in the sales order.
