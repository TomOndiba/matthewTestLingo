Expiration Determining

The Expiration Determining check box identifies all components of a production order that are relevant for calculating the expiration date. It is possible to set or change this setting in the Production Structure, BOM, Recipes or Prod. Order Components.

: Production order component with expiration determining setting

The calculation of the expiration date of the produced item happens as follows:

  • If the Expiration Determining check box is selected in none of the component lines, the calculation takes place according to the “standard” logic. That means that the Expiration Calculation specified on the Item Card is always used for the calculation.
  • It the Expiration Determining check box is selected in all component lines, the calculation takes place according to the “shortest component” logic. That means that the component with the shortest expiration date determines the expiration date. If the components do not have an expiration date or if the produced item expires earlier than the shortest component (according to its expiration calculation), the expiration date of the produced item is used.
  • If the Expiration Determining check box is marked in some component lines, the calculation takes place according to the “shortest component” logic, except that only the selected components are observed.
