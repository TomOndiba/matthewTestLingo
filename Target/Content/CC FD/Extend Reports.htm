Extend Reports

The reports described in this section have been extended for use with cc|formatted documents.

Data Structure of Reports

For a report to be used by cc|formatted documents, the following two new columns must be created below a DataItem:

  1. FD_BLOB

Column of datatype “BLOB” – contains the formatted text of the record as binary data.

  1. FD_RectangleForNo

Column of datatype “Text” – contains a unique identifier value for the record. This column usually does not exist as a field in the corresponding table, but contains (recommended) the primary key fields of the corresponding table as a combined text. The value must be unique.

Both columns provide required information to the subreport and are constrained by the following naming convention:

  1. [<SubreportIdentifier>_]FD_BLOB
  2. [<SubreportIdentifier>_]FD_RectangleForNo

The prefix of the subreport is only required if multiple subreports are embedded in the main report. The “<SubreportIdentifier>” name needs to correspond to the subreport of the two fields.

Summarizing explanation (as example):

  1. [<SubreportIdentifier>_]FD_BLOB

Field of the corresponding table with Data Type = BLOB.

Name: subreport2_FD_BLOB

SourceExpr: "FD Blob"

  1. [<SubreportIdentifier>_]FD_RectangleForNo

Primary key fields of the corresponding table as combined text.

Name: subreport2_FD_RectangleForNo

SourceExpr: FORMAT("Sales Line"."Document Type") + "Sales Line"."Document No." + FORMAT("Sales Line"."Line No.")

Layout of the Report (in Report Designer)

Subreports can be placed anywhere in the main report, except in the header of the report. The name of the report can be user-defined; however, the name must match the prefix of the column names.

The following parameters must (mandatory) and can (optionally) be given to a report:

  • Mandatory parameter (required):
    • RectangleForNo
  • Optional parameters (optional):
    • FontFamily
      • Default value: “Arial”
    • FontSize
      • Default value: “9pt”
    • TextColor
      • Default value: “black”
    •  PaddingLeft
      • Default value: “0pt”
    • PaddingRight
      • Default value: “0pt”
    •  PaddingTop
      • Default value: “0pt”
    •  PaddingBottom
      • Default value: “0pt”

: Integrating a Subreport

Note

If the subreport is placed in a table or matrix, it will be rendered for each line. In this case, the value of the parameter “RectangleForNo” for each line is relevant.

  • Expression for value:

=Fields![<SubreportIdentifier>_]FD_RectangleForNo.Value

If the subreport is placed outside of a table or matrix, the First function must be used. The First function returns the first value in a set of records, after sorting and filtering of a specified area is applied.

  • Expression for value:

=First(Fields![<SubreportIdentifier>_]FD_RectangleForNo.Value)

Important

It is required to calculate the field of Data Type = BLOB via the C/AL function CALCFIELDS in the trigger OnAfterGetRecord of the DataItem because the BLOB content is provided to the DataSet.

Visibility Settings for Elements of a Subreport

The subreport control allocates a small area on the report, even if no formatted text is printed. To disable the allocation, always hide the subreport control by default.

: Order Confirmation with Default Report Output

To ensure that the visibility of integrated subreport elements is set to “Hide,” right-click the subreport and select Subreport Properties.

: Accessing Subreport Properties

In the Subreport Properties page, select the Hide option.

: Hide Subreport

Visibility Settings for Elements of a Subreport in Tables and Matrices

For avoiding empty subreport elements (subreports, for those where no content created in cc|formatted documents exists) to reserve space, it is necessary to set the visibility of a line containing the subreport element to “Hide.”

: Subreport as Empty Line

To ensure that the visibility of lines containing subreport elements is set to “Hide,” right-click the row and select Row Visibility. In the Row Visibility page, select the Hide option.

: Accessing Row Visibility and Row Visibility page

If the settings were made correctly, the output of the result is optimal. cc|formatted documents shows hidden elements independently, as required.
