Server Installation

The installation routine of the “cc|formatted documents – Server” is usually run on the host, where the Microsoft Dynamics NAV Middle Tier is located. That way the “cc|formatted documents – Server” gets direct contact to Microsoft Dynamics NAV (no network access required). Alternatively, the “cc|formatted documents – Server” can be run on any host in your network, but that host should remain in the same domain in this case.

The server provides a (required) service to the “cc|formatted documents – Clients”. Without the server, the clients cannot operate (especially the report output).

: Three Tier Architecture

Note

The installation routing of the server registers a Windows Service named “cc formatted documents – Server Service” to enable a high availability of the server without interaction of the user. The Windows Service needs a service account for cc|formatted documents that must be a domain account having full access to the Microsoft Dynamics NAV Instance. Ensure that the Service Account (or the Windows User) is registered in Microsoft Dynamics NAV and has the required permissions (Roles, Access Rights) for Microsoft Dynamics NAV.

Requirements

Requirements for the correct operation of “cc|formatted documents – Server” are:

  • .NET Framework 3.5 or higher
  • Microsoft Dynamics NAV Client

Installing the server includes the following processes:

  1. Run the file “cc formatted documents <NAV Version ID> - Server Setup.msi”.
  2. Follow the installation wizard.
  3. Follow the configuration wizard.

Each process is described in the following sections.

Installation Wizard – Server

The installation of the server requires administrator rights.

Follow these steps to run the “cc|formatted documents – Server” installation:

  1. Locate and run the “cc formatted documents <NAV Version ID> - Server Setup.msi” file to start the installation.

: cc|formatted documents Server Setup Wizard, Welcome window

  1. Click Next. The information window opens.

: cc|formatted documents Server Setup Wizard, Information window

  1. Click Next. The License Agreement window opens.

: cc|formatted documents Server Setup Wizard, License Agreement window

  1. Accept the License Agreement and then click Next.

: cc|formatted documents Server Setup Wizard, Installation Folder selection

  1. Confirm or select the installation folder and then click Next.

: cc|formatted documents Server Setup Wizard, domain account and password window

A service account must be entered for the new Windows Service. The service account must be assigned to a domain account that has full access to the Microsoft Dynamics NAV Instance.

  1. Enter a domain account and password for the service and then click Next.

: cc|formatted documents Server Setup Wizard, settings and features selection

  1. Confirm the default setting or (de-)select additional features and then click Install.
    The required files/components are copied and registered. When the installation is finished, the Installation Complete window opens.
  2. When the installation process is finished, click Finish.

Configuration Wizard – Server

When the installation is complete, the cc|formatted documents - Configuration Wizard window opens. The Wizard assists with setting up the connection to Microsoft Dynamics NAV.

: cc|formatted documents Configuration Wizard, Welcome window

  1. Click Next. The Product Key window opens.

: cc|formatted documents Configuration Wizard, Product Key window

  1. Enter the company name and the product key provided by COSMO CONSULT.

Note

The product key is company specific. For companies of affiliated enterprises more product keys are issued free of charge.

  1. Click Next. The Connection Settings window opens.

: cc|formatted documents Configuration Wizard, Connection Settings window

 

  1. Confirm/correct the address of the server hosting the “cc|formatted documents – Server” and adjust the NAV Server, Port, Service Name and Company Name to the Microsoft Dynamics NAV Service Tier.
    • The address of “cc|formatted documents – Server” (FD Server) corresponds to the network address of the host, on which the installation is run.
    • The Default NAV Service URL and Default NAV Company Name fields must correspond to your Default NAV Instance.
    • It is very important to specify the NAV Client Directory.

    • The fields NAV User Name (all) and NAV Password (all) have to be populated if the Microsoft Dynamics NAV middle tier operates with the Credential Type “NavUserPassword”. For this setting, it is important that this cc|formatted documents installation is used for all connections with the middle tier/NST. This means that the user has to be set up in every NST with the same password which are using cc|formatted documents. If several middle tiers/NST differs, multiple cc|formatted document entities/instances must be set up.

  2. Click Next. The Connection Test window opens.
    This window offers the ability to test the connection of the “cc|formatted documents – Server” to Microsoft Dynamics NAV.

: cc|formatted documents Configuration Wizard, Connection Test window

  1. Perform a connection test by clicking Test Connection.
    If the connection is established successfully, the “cc|formatted documents – Server” indicates the company that it has opened.
  2. Click Next. The Deployment window opens.

: cc|formatted documents Configuration Wizard, Deployment window

To use cc|formatted documents in Microsoft Dynamics NAV, the Add-Ins path for Microsoft Dynamics NAV is required.

  1. Specify the path to the Add-In Folder of the Microsoft Dynamics NAV Client, which is used for the connection to the correct Microsoft Dynamics NAV Instance and then click Finish to complete the configuration wizard.

On finishing the Wizard, the Service “cc formatted documents – Server Service” is started automatically.

Microsoft Dynamics NAV Client Directory – Hints for Administrators

If multiple Microsoft Dynamics NAV Clients are used on this host, whether it is the installed Microsoft Dynamics NAV Client or not, continue reading this and topic "Example – Installation/Uninstallation of Additional Server Service" on page 1. If the installed Microsoft Dynamics NAV Client is the only and correct Microsoft Dynamics NAV Client, there is no need to continue reading this topic or topic "Example – Installation/Uninstallation of Additional Server Service" on page 1.

As mentioned in the requirements of the server, it is necessary to have a Microsoft Dynamics NAV Client installed on the host, where the “cc|formatted documents – Server” is run. The Installation Wizard recognizes the installed Microsoft Dynamics NAV Client and assumes that this Build Status is used for connecting to Microsoft Dynamics NAV. If this is not the case, it is important to adjust the path to the Microsoft Dynamics NAV Client (with the correct Build Status).

Important

Ensure that the specification of the Microsoft Dynamics NAV Client Directory on Page 3 (Connection Settings) of the Configuration Wizard is correct.

Example – Installation/Uninstallation of Additional Server Service

As preparation for further steps, the “Service” folder of “%programfiles(x86)%\Cosmo Consult AG“ needs to be copied and renamed to “Service 2”.

Now run the Windows Command Prompt as administrator.

Important

The space after the “=” of the following commands must be included.

Register an additional Server Service:

sc \\<Hostname> create CCFDServer2 binpath=

"%programfiles(x86)%\Cosmo Consult AG\cc formatted documents\ Service 2\ CosmoConsult.FormattedDocuments.FdRServerService.exe" DisplayName= "cc formatted documents – Server Service 2" start= auto

Add description:

sc \\<Hostname> description CCFDServer2 "cc formatted documents – Server Service for Microsoft Dynamics NAV, No. 2"

Delete Server Service:

sc \\<Hostname> delete CCFDServer2

Note

Every “cc|formatted documents – Server Service” is able to switch between Microsoft Dynamics NAV Services and Microsoft Dynamics NAV Companies. However, an additional Server Service is required conceivably, e.g. when permissions or the default connection settings should be different.
