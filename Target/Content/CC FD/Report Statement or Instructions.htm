Report Statement or Instructions (Variants/Possibilities)

The report instruction is used by cc|formatted documents to call a main report or to define the source of a subreport. Certain specifications are optional and can be skipped.

In the case of subreports, the report instructions should be assembled in C/AL code. The specification of SORTING and WHERE can be applied exactly from the return value of the C/AL function GETVIEW(FALSE).

Example of a report instruction:

SERVICE('localhost:7046/DynamicsNAV90') COMPANY('CRONUS AG') REPORT(205) TABLE(36) SORTING(Field1,Field3) WHERE(Field3=1(1001)) USAGE(0)

Basic structure:

All arguments in [] are optional.

[SERVICE('localhost:7046/DynamicsNAV90')]

[COMPANY('CRONUS AG')]

REPORT(205)

TABLE(36)

[SORTING(Field1,Field3)]

WHERE(Field3=1(1001))

[USAGE(0)]

[FILE('C:\Temp\document.pdf')]

[RENDER(PDF)]

 

Any optional argument (in []) can be skipped. But if any are used, they should be declared in the right order.

Text values should be included in single quotes; otherwise spaces, special characters, etc. may invalidate the report instruction.

Arguments:

Argument

Data Type

Description
[SERVICE]

Text

Service URL of the Microsoft Dynamics NAV Service (in single quotes). If skipped, the default Service URL of the “cc|formatted documents – Server” is used.

[COMPANY]

Text

If skipped, the default Microsoft Dynamics NAV Company Name of the “cc|formatted documents – Server” is used.

   REPORT

Integer

ID of the Microsoft Dynamics NAV Report.

   TABLE

Integer

ID of the corresponding table of the Microsoft Dynamics NAV Report.

   [SORTING]

Text

Key to be used for sorting the corresponding Microsoft Dynamics NAV Table.

   WHERE

Text

Filter of the corresponding Microsoft Dynamics NAV Table.

   [USAGE]

Integer

Specifies how the report is generated. The options are:

  • 0 = Preview (Default value)

Displays the report as page view

  • 1 = Print

Prints the report with a printer

  • 2 = File

Saves the report to a file

  • 3 = E-Mail

Generates an email with the report as attachment

Note

Will be ignored in case of a report instruction of a subreport.

     [FILE]

Text

Name of the file to be saved (in single quotes). Only relevant using USAGE option 2 or 3.

Note

Will be ignored in case of a report instruction of a subreport.

     [RENDER]

Text

Specifies the type of rendering. Only relevant using USAGE option 2 or 3. The options are:

  • PDF (Default value)

Renders a report in Adobe Acrobat Reader (PDF File)

  • EXCELOPENXML

Renders a report in Microsoft Excel (XLS File)

 

  • WORDOPENXML

Renders a report in Microsoft Word (DOC File)

  • IMAGE

Renders a report in a site based format (TIFF File)

Note

Will be ignored in case of a report instruction of a subreport.
