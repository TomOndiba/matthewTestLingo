Data Structure of Reports

Set up the following two new columns below the DataItem for those that should use one or more subreports with cc|formatted documents:

  1. FD_ReportStatement

Column of datatype “Text” – contains the report instructions for the subreport (refer to topic "Report Statement or Instructions (Variants/Possibilities)" on page 1).

  1. FD_RectangleForNo

Column of datatype “Text” – contains the unique identifier value of a record. This column contains (recommended) the primary key fields of the corresponding table as a combined text. The value must be unique.

Both columns provide the required information to the subreport to be integrated by the following naming convention:

  1. [<SubreportIdentifier>_]FD_ReportStatement
  2. [<SubreportIdentifier>_]FD_RectangleForNo

The prefix of the subreport is only required if multiple subreports are embedded in the main report. The “<SubreportIdentifier>” name must correspond to the subreport of the two fields.

Summarizing explanation (as example):

  1. [<SubreportIdentifier>_]FD_ReportStatement

Global report variable having the name “Subreport1ReportStatement”.

Name: subreport1_FD_ReportStatement

SourceExpr: Subreport1ReportStatement

  1. [<SubreportIdentifier>_]FD_RectangleForNo

Primary key fields of the corresponding table as combined text.

Name: subreport1_FD_RectangleForNo

SourceExpr: FORMAT("Sales Line"."Document Type") + "Sales Line"."Document No." + FORMAT("Sales Line"."Line No.")
