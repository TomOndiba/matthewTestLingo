Rent Orders

In the Rent Order page, enter the subject to be rented, the period of renting and invoicing, and the price for the rate period. In addition to the Rate Code, the Rental Start Date, Rental End Date (optional), Rate Unit Price, and Invoicing Period can be entered. The appropriate current period for invoicing is retrieved from the information entered in the Invoice Period, Month Based, and Prepaid fields. The price for this period depends on its length and the price for the rate period.

In the navigation pane, click cc|rental > Contract Management > Rent Orders to view existing or create new Rent Orders.

: Rent Order page

This page contains the same fields as the sales order, plus the rental-specific fields, which are described with other relevant fields in the following sections.

Lines FastTab

The fields on the Lines FastTab are described in the following table.

Field

Description

Rental Position

Identifies if the line is a Sales or Rental item or a Rate Component. The Sales option is the default, but it automatically changes to Rental when a rental item is entered.

While it is unnecessary to select a value in this field when entering lines, it can be useful to change it to Rental so that the Item No. list is filtered by rent items.

When the line is to be used as a Rate Component, then this field must be set to Rate Part or One-Time Invoicing. A Rate Component can be manually entered below a Rental position and will be linked to that item line automatically.

Type

Identifies what is being rented or sold. The options are G/L Account, Item and Resource.

No.

Number of the G/L Account, Item or Resource.

Note

This field also contains an AssistEdit button that opens the Disposition Calendar (refer to topic “Disposition Calendar” on page 1).

Rate Component No.

Contains the number of the Rate Component if the Rental Position field is set to Rate Part or One-Time Invoicing. These lines are related to the item line above.

Description

Description of the G/L Account, Item or Resource.

Location Code

Defaults from the Location Code field on the Shipping FastTab. If a Rental Location Code is specified on the Rental FastTab on the Customer Card, it defaults to this field.

For Rental line types, this field must indicate the proper rental location to process the rent order.

Customer Location Code

Defaults from the Customer Location Code field on the Customer Card. This field must be filled in to process the rent order.

Quantity

Number of units to be rented or sold.

Unit of Measure Code

Measurement of units to be rented or sold.

Rental Rate Code

Determines the Rate Unit Price on the rental line as set up in the Rent Prices or Line Discounts pages.

Invoice Period

Determines how the periods on the invoice are created. The options are:

Month

Half Year

Two Months

Year

Quarters

None

For more information, refer to topic “Invoicing Parameters” on page 1.

Calendar Code

If populated, the system uses a Calendar Code for the invoicing of Rental periods.

For more information, refer to topic “Calendar Codes” on page 1.

Month Based

If selected, invoices are geared toward the calendar month. For example, if the contract period is 05/15/2019 through 08/31/2019 and this check box is selected, the first invoice period will be from 05/15/2019 through 05/31/2019, and then the next period will be 06/01/2019 through 06/30/2019, and so on. The price on the first period will be prorated accordingly.

For more information, refer to topic “Invoice Periods and Month Based Date Calculations” on page 1.

Prepaid

If selected, the line is processed by the date it is invoiced and the account to which it is posted.

For more information, refer to topic “Invoice Periods and Prepaid Date Calculations” on page 1.

Rental Start Date

Beginning of the rental contract period. This date is included in the item availability and is integral with the invoicing functionality (refer to topic “Invoicing” on page 1).

Rental End Date

End of the rental contract period; this date is optional.

If left empty, the Total Price is calculated for one Rate Period on the document line and in page totals.

Rental Return Date

Used with the return of the rental item (refer to topic “Returns” on page 1).

Next Invoice Period

Displays the next period to be invoiced after an invoice has been created. This field must be displayed using the “Choose Columns” function.

Rate Unit Price

Price from the Rental Rate Code/Rent Price combination set up for the rental item.

Unit Price Total Rental Time

Total price for the complete rental duration:

Number of Rate Periods resulting from the
Rental Start Date and Rental End Date fields * Rate Unit Price

For example:

  • If Rental Rate Code = MONTH, Rental Period = 2/1/2019 – 2/28/2019, and Rate Unit Price = 300, then the Unit Price Total Rental Time = 300.
  • If Rental Rate Code = DAY for the same period (28 days), and Rate Unit Price = 9.5, then Unit Price Total Rental Time = 266 (28 * 9.5).

The total is also based on whether the Month Based check box is selected (refer to topic “Invoicing” on page 1).

Amount Total Rental Time

Total price based on the following calculation:

Quantity * Unit Price Total Rental Time - Rental Line Discount Amount

Return Qty. to Receive / Return Qty. Received

Used with the return of the rental item (refer to topic “Returns” on page 1).

Qty. Extensions

Displays the number of extensions that have been processed for the order line (refer to topic “Extensions ” on page 1).

Qty. Failures

Displays the number of failures that have been processed for the order line (refer to topic “Failures” on page 1).

Next Price Update Date

Date the next price update can be run on the line (refer to topic “Update Rental Contract Prices” on page 1). Display this field with the “Choose Columns” function.

Price Update Period

Date calculation formula that drives the Next Price Update Date field (refer to topic “Update Rental Contract Prices” on page 1). Display this field with the “Choose Columns” function.

Last Price Update Date

Date the price update was last run for the line (refer to topic “Update Rental Contract Prices” on page 1). Display this field with the “Choose Columns” function.

Last Price Update %

Percentage of the last price update run for the line (refer to topic “Update Rental Contract Prices” on page 1). Display this field with the “Choose Columns” function.

Invoice Multiple

If selected, invoices a portion of the complete rental period without using a fixed invoicing period; used with Invoice Period = None.

Invoice to Date

Indicates the date to which the next invoice will be created; used with Invoice Period = None.

Total No. of Rate Units

Indicates the total rental time calculation and the quantity of rate units as determined by the Rental Start Date and Rental End Date.

With graduated rental pricing, this field determines the valid price level in the sales line for non-cascading scale price when the Minimum Qty. Rate Units field is populated in the Rental Prices table (refer to topics “Rental Rate Prices” on page 1 and “Exercise – Process a Rent Order with Non-Cascaded Scale Price” on page 1).

With Lump Sum Invoicing, this field can be changed manually or recalculated by a change to the Unit Price Total Rental Time and Amount Total Rental Time fields (refer to topic “Lump Sum Invoicing” on page 1).

Display this field with the “Choose Columns” function.

Cascaded Price

Indicates if the sales line is using a cascading scale price (refer to topic “Exercise – Process a Rent Order with Cascaded Scale Price” on page 1).

Display this field with the “Choose Columns” function.

Include Counters

If selected, activates the counter-based invoicing for this rental line.

In addition to the rate based invoicing, the system will determine whether a contingent of hours or miles (which are included in the Rate Price) is exhausted or not and create an extra line on the invoice for the difference in usage (refer to topic “Counter-Based Invoicing” on page 1).

Display this field with the “Choose Columns” function.

Machine No.

Automatically populates the Item No. field on the rental contract line and the Serial No. in Item Tracking Lines with the corresponding numbers of the selected Machine. The Quantity is also automatically set to 1, which is required when using this field.

Refer to topic “Machines” on page 1 for more information about setting up and working with Machines.

Rental Shipment Time

Used with hour-based invoicing to indicate the time that the shipment is posted (starting time of rental). If this time is before the Start Time Working Day defined in the Rental Mgt. Setup, the calculation starts with the Start Time Working Day.

Refer to topic “Hour-Based Invoicing” on page 1 for more information about hour-based invoicing.

Rental Return Time

Used with hour-based invoicing to indicate the time that the return is posted (ending time of rental). If this time is past the End Time Working Day defined in the Rental Mgt. Setup, the calculation ends at the End Time Working Day.

Refer to topic “Hour-Based Invoicing” on page 1 for more information about hour-based invoicing.

Rental FastTab

As with other information entered at the header level, the information entered in the fields on the Rental FastTab (except the Closed check box) will default to the lines, making data entry simpler when entering multiple rental lines. The Closed check box is automatically selected when the rent order has been fully received and invoiced; it cannot be edited. The fields on this FastTab have no effect on sales positions.
