Hour Based Rental FastTab

On this FastTab, defaults for hour--based invoicing are defined.

: Rental Mgt. Setup page, Hour Based Rental FastTab

The fields on this FastTab are described in the following table.

Field

Description

Start Time of Working Day

Indicates the default start time of each working day.

End Time of Working Day

Indicates the default end time of each working day.

Hour Rounding Precision

Indicates the hourly rounding precision; uses an hour calculation. Therefore, if 65 minutes is entered in the field, it changes to 1 hour and 5 minutes. If only 65 is entered in the field, it changes to 2 days 17 hours.

Note

Decimal entries, such as .25 to represent 15 minutes, are prohibited. To round to the nearest 15 minutes, enter 15 min (or minutes) in the field.

Hour Rounding Direction

Indicates the direction of the hourly rounding, based on the Hour Rounding Precision field. The options are:

  • 5|4 – rounds the hourly value up or down to the nearest value as follows:
    • the rounding digit is rounded up one digit if the digit to the right is greater than or equal to 5.
    • the rounding digit is rounded down one digit if the digit to the right is less than 5.
  • Up – hourly values are rounded up to the nearest value.
  • Down – hourly values are rounded down to the nearest value.

Rate Codes

cc|rental uses a period of time for which a price is to be entered. For this period of time, a Rate Code has to be set up to which a period (day, month, week, and so on) must be assigned.

: Rental Rate Codes page

The fields in this page are described in the following table.

Field

Description

Code

Unique identifier for the rental rate.

Type

Period of time that must consist of days; hours or minutes cannot be used for rental rates.

Description

Short description of the rental rate code.

Rental Rate Code

If selected, this code can be used as a rental rate. This check box is automatically selected.

 

Rent Rate Components

To enable the invoicing of amounts that are related to the rental of items but have to be reported as separate positions, cc|rental offers the use of Rate Components. This can be a G/L Account as well as a Resource and has a specific Price for each Rate.

Rate Components are divided into two groups:

  • Rate Part – Invoiced periodically.
  • One-Time Invoicing – Invoiced once at the beginning, end or a selected time of a renting period.

Rate Components are inserted to the Rent Order automatically after entering the item number.

: Rent Rate Components page

The fields in the Rent Rate Components page are described in the following table.

Field

Description

Type

Identifies the type of rate component. The options are Rate Part and One-Time Invoicing.

No.

Unique identifier for the rate component.

Description

Short description of the rate component.

Gen. Prod. Posting Group

General Product Posting Group that will be used when inserting the rate component to the Rent Order.

VAT Prod. Posting Group

VAT Product Posting Group that will be used when inserting the rate component to the Rent Order.

Search Description

Search Description for the rate component.

Invoicing at

Determines whether a One-Time Invoicing rate component is invoiced on shipment, on return of the related item, or a selected time during the rental period. The options are Start of Rental, End of Rental, and Selected.

Usage

Identifies how the rate component is used. The options are:

  • <blank>
  • Insurance – used with the Self-Insured Customer function, which prevents the addition of the rent component to the rental contract line if the customer is set up as Self-Insured (refer to topic “Self-Insured Customers” on page 1).
  • VAT
  • Fees
  • Surcharges

 

Failure Types

When an event – such as planned maintenance – prevents a customer from using a rental item, a failure can be processed on the rent order line. When the “Enter Failure” function is run, Failure Types direct how the failure is to be processed.

: Failure Types page

The fields in the Failure Types page are described in the following table.

Field

Description

Failure Type Code

Unique identifier for the failure type.

Description

Short description of the failure type.

Credit Memo

If selected, a credit memo will be created for the period specified on the Rental Failure Card.

Discount %

Percentage of the amount to be credited.

Reduce Free Counter

Determines if the calculated free counter is reduced when a specific type of failure is entered. The options are:

  • <blank> – the counter is not reduced for the failure type.
  • <Full> – the counter is reduced completely for the failure type.
  • <Percent> – the counter is reduced the same as the discounted percentage for the failure type.

 

Rental Preconditions

Prior to completing a rental shipment, some items may have preconditions that must be met to drive or operate a rented machine. In the Rental Preconditions page, the types of preconditions are defined.

: Rental Preconditions

The fields in the Rental Preconditions page are described in the following table.

Field

Description

Code

Unique identifier for the rental precondition.

Description

Short description of the rental precondition.

Action Type

Indicates if the precondition requires a type of confirmation (Confirmation) or is only a message to the customer (Message).

If a precondition has been defined for a confirmation, a Name or Name and Date of Birth must be entered before the shipment can be posted. Precondition messages appear prior to posting but require no action.

Confirmation Type

Determines if the person responsible for completing the precondition must provide a name or name and date of birth. This field is only used with Action Type = Confirmation.
Minimum Age

Calculates the minimum age of the person responsible for completing the precondition. This field uses a date formula and is calculated by the Rental Start Date, e.g., if 16 years is the minimum age, then the date of birth entered must be 16 years before the Rental Start Date.

This field is only used with Confirmation Type = Date of Birth.

Refer to topic “Item Shipment Preconditions” on page 1 for more information.
