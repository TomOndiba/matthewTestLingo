Shipping

Shipments in cc|rental differentiate between rental and sales positions:

  • Rental positions will be shipped using transfers. They need both a Location Code and a Customer Location Code as the source and destination locations because warehouse shipment and receipt occurs automatically with the shipment.
  • Sales positions do not require the source and destination locations.

Both rental and sales positions can be partly shipped; however, a complete shipment is required for invoicing of rental positions.

The values indicated in the Qty. to Ship field are processed in the shipment. A posted shipment is created as usual. With this shipment, the “Undo Shipment” function can be used.

When shipping the complete quantity of a line (regardless of whether this is done in full or partial shipments), a service contract is created that contains a line for each entity of the rented position with its Item No., Serial No. and Service Item No. (or both if applicable).

Exercise – Ship the Rent Order

Item R-30000-01 has a one day shipping time. On January 31, 2019, you ship the order to customer R-30000.

Ship the Order

Follow these steps to ship the order:

  1. Open the rent order created in the “Record a Rental Position on a Rent Order” exercise.
  2. On the ribbon, click Post.
  3. With the Ship option selected, click OK. The Qty. to Ship field is cleared and the Quantity Shipped field contains 1.

Review Availability by Locations and Transfers

Follow these steps to review the shipment posting:

  1. In the Rent Line Details FactBox, click item number R-30000-01.
  2. In the Item Card, click Navigate > Items by Location.
  3. On the ribbon, click Show Matrix.
  4. In the Items by Location Matrix page, filter on item number R-30000-01.
