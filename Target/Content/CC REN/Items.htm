Items

Typically, items are the main transaction on a rent order. Because items play multiple roles within Microsoft Dynamics NAV (consumption for Service or Manufacturing, Sales item, and so on) users can decide which items can be used for rental purposes. To do so, select Yes in the Rent Item field on the Rental FastTab. With this setting, the item can also be used for renting.

If the item is serial number tracked and a service item is required before a serial number can be assigned on a rent order, then select the Service Item for SN needed check box (refer to topic “Exercise – Extend a Rental Period” on page 1 for an example of this requirement).

Note

The Costing Method must be set to Specific for items using the Service Item for SN needed functionality.

: Item set as rental item with service item requirements

Pricing and preconditions related to rental items is defined in the Rental action accessed on the Navigate tab. Each is described in the following sections.

Rental Rate Prices

Rental prices are assigned to items. Rental rates can be set up for a single customer, all customers or customer price groups. Rates can be based on unit of measure, minimum quantity, starting/ending dates and currency.

Generally, Rental Rate Prices are the same as Sales Prices in standard Microsoft Dynamics NAV in that multiple rental prices can be entered for each item, based on the different criteria. However, there are several fields that are used for different rental processes. These Rental-specific fields are described in the following table; refer to online for more information about the standard Sales Prices setup.

: Rental Prices page

Field

Description

Rental Rate Code

Indicates the time period to which the rental price applies.

Minimum Qty. Rate Units

Indicates the minimum number of rate units for which the Unit Price is valid; used to determine if cascading or non-cascading scale pricing is used in the rental contract line:

  • Non-cascading scale pricing: One line is entered with a value in this field, or multiple lines with prices per Minimum Qty. Rate Units, and without the Cascading check box selected. The Total No. of Rate Units from the Sales Line determines the valid price level (as calculated by the Rental Starting Date and Rental Ending Date), which is used throughout the complete rental unless the price is changed manually or by the price update report.
  • Cascading scale pricing: Multiple lines are entered with a value in this field and with the Cascading check box selected. The first line must have Minimum Qty. Rate Units = 0 and Unit Price = [starting price] to create the price structure. The remaining lines are then set up with minimum rate units/unit price for each level in the pricing structure.

During invoicing, the Rental Price table is checked for graduated rent after a line is created on the Service Invoice. If a total of Rate Units is reached during invoicing, the Rent Order line is updated with the new price from the Rental Price table before invoicing the next Invoicing period.

Refer to topic “Graduated Rental Pricing” on page 1 for more information about both types of pricing.

Cascading

If selected, cascading scale pricing is used to determine pricing in the Rental Contract.

When this field is set or cleared, a message will appear and automatically set or clear this field on all rental price lines with the same Sales Type/Sales Code/Rental Rate Code combination to ensure that all or none are set for cascading pricing.

Refer to topic “Exercise – Process a Rent Order with Cascaded Scale Price” on page 1 for more information.

Rental Line Discounts

Line discounts can also be defined on rental items, using similar criteria. This functionality is the same as standard Microsoft Dynamics NAV except the Rental Rate Code field determines if the line discount is specific to rentals.

: Sales Line Discounts page

Multiple rental line discounts can be entered for each item, based on the different criteria. Refer to online help for more information about sales line discount setup.

Item Rent Rate Components

Rate Components can be defined on rental items. The Item Rate Components are selected from the Rent Rate Components list (refer to topic “Rent Rate Components” on page 1). They are inserted automatically into the Rent Order line when the Item No. is entered if the Insert Rate Components field is set to Always in the Rental Mgt. Setup page. Otherwise the components are inserted if a valid rent price is entered for the Rate Component and the selected Rate Code.

Rate Components can also be selected manually in the Rent Order line by setting the Rental Position field to Rate Part for a line that is to be invoiced periodically or One-Time Invoicing for a position to be invoiced once.

: Item Rent Rate Components page

The fields on this page are described in the following table.

Field

Description

Type

Identifies the type of item rent rate component. The options are Rate Part and One-Time Invoicing.

Rate Component No.

Identifies the Rate Component for the selected type.

Source Type

Determines whether a G/L Account or a Resource must be used for the Rate Component.

Source No.

Identifies the G/L Account No. if the Source Type = G/L Account or a Resource No. if the Source Type = Resource.

Note

If the Source Type = G/L Account, the Source No. field can be left blank. The system then uses the G/L Account from the General Posting Setup.

Rent Component Prices

Rent Component Prices are used to set up prices for the Rate Components of an item and for a specific Rate Code.

This may be a Tax or Insurance as well as a one-time invoicing, such as cleaning on return. This functionality is the same as Rental Rate Prices except the Rate Component Type and Rate Component No. fields determine the pricing for the rental components. Additionally, the Percentage of Item Rate field allows for components to be set up as a percentage of the item’s rental price instead of defining a unit price (refer to topic “Exercise – Use Rate Component Percentages in Rent Order” on page 1.)

: Rent Component Prices page

Item Rental Preconditions

Preconditions can be assigned to specific items or item categories. These preconditions are used to ensure that the proper conditions are met prior to shipping a rental item to a customer. The base preconditions are set up in the Rental Preconditions page (refer to topic “Rental Preconditions” on page 1).

: Rental preconditions assigned to an item

Refer to topic “Item Shipment Preconditions” on page 1 for more information.

Preconditions on Item Categories

In the Item Categories page, click Actions > Rental Preconditions to assign a precondition to a specific item category.

: Item Rental Precondition added to Audio Item Category

Any shipments posted with a precondition for the item category will be visible in the Item Shipment Precondition Log.
