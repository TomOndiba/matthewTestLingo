Machines

A special approach may be used if a serial number-based item that is handled as a specific entity with special attributes is used in Purchasing, Sales, Service or Rental. These items:

  • Are valued as an entity (Costing Method = Specific)
  • Use serial numbers (Item Tracking Code contains SN-specific Tracking)
  • May have separate attributes such as equipment

To differentiate these items from each other, cc|rental categorizes items types. Each is described in the following table.

Type

Description

Item

No attributes are pre-allocated from the Type field.

Machine

SN Specific Tracking, Costing Method Specific, Unit of Measure Code from Machines Setup (should be like PCS).

Supplementary Part

SN Specific Tracking, Unit of Measure Code from the Machines Setup (should be PCS).

Machine Component

Unit of Measure Code from the Machines Setup (should be PCS).

For each purchased machine-based Type item, a Card is available in the Machines menu that is auto-populated with the type-specific fields.

If Machines or Supplementary Parts are bought or posted to inventory, a Machine Card or Supplementary Part Card is created automatically. These cards are used to collect the data from commercial transactions on a Serial No. basis.

Additionally, Machines can be connected to Fixed Assets and purchased and sold like a Fixed Asset.
