Invoicing

Invoicing of the Rent Order starts with the shipment of the document lines. If the complete quantity is shipped, the contract is ready for invoicing with its particular due date. This due date is dependent on the specific Invoice Period (refer to topic “Invoicing Parameters” on page 1).

The amount that will be invoiced is calculated from the Rate Unit Price. If the rate period is Quarter, Year or any other period that can be assembled by months, the period is divided into months in case no full rate period can be invoiced. If a division has to be done by date, the system uses the actual number of days to determine the price per day.

Example

If the Invoice Period is set to Month and the Month Based check box is selected, the first invoice for a contract line from May 15th to August 31st will contain a first invoicing period from May 15th to May 31st. This means that if the Rate Unit Price is based on the monthly rate of 310.00 then:

  • the daily price is 310 / 31 (number of days in May) = 10.00
  • the invoicing amount calculates from 10.00 * 17 (days) = 170.00

Invoice Periods and Month Based Date Calculations

For an Invoice Period of Month that does not use the Month Based option, the period always uses the Microsoft Dynamics NAV date formula of +1M-1D. This means it starts at the Next Invoice Period Start Date and adds this date formula to determine the Next Invoice Period End Date.

This is useful for calendar months, when the rental begins January 1st, the invoice will end on January 31st because one month is added (February 1st) and then backed up one day.

The same is done if it not the calendar month: the start date for invoicing is the Rental Start Date if no invoicing has occurred or it is the last invoiced date +1.

However, the system does not stop at the Rental End Date automatically, so if no return has been posted when the invoice is created, the system creates an invoice for a full invoicing period (refer to topic “Returns” on page 1).

If a Calendar Code is entered to the Rent Order line the invoice amounts are calculated from the Rental Price for the invoicing period minus the price for free days within the period.

Invoice Periods and Prepaid Date Calculations

If the Prepaid check box is selected on the line, the due date is the Next Invoice Period starting date. If the Prepaid check box is not selected, the due date is the Next Invoice Period end date.

For example, if a rental period is April 15th through December 31st, the Invoice Period is Month, the Prepaid check box is selected, and the invoice is to be created through May 15th:

  • The first invoice period starts on April 15th and ends on May 14th. This is because the Start Date of the period is the due date of it.
  • The second starts on May 15th and ends on June 14th. This period is due on May 15th so the system creates invoices up to June 14th.

Calendar Codes

Calendar Codes are linked to the Base Calendar Card, where a company sets up a base calendar on a general level and defines each date as either a working or nonworking day. For example, a base calendar for a particular country/region would define weekends and national holidays as nonworking days. All other days would be working days.

cc|rental uses one day as a base unit of measure. Prices are calculated for the whole or a part of the Invoicing Period that is shown on the Rent Order (or Quote) or are transferred to the invoice.

If a Calendar Code is entered on a Rent Order line (or on the Rental FastTab and defaulted to the lines) and nonworking days are identified for the period to be calculated, the price for the period is calculated and then cc|rental subtracts one day’s price for each nonworking day.

Specific calendar changes can be set up for each line by clicking Line > Cust. Calendar on the Lines FastTab.

Invoice Processing

After selecting the Invoice option in the Post page, the “Create Rental Contract Invoice” report opens, filtered on the rent order. A posting date and date filter for the last due date included must be entered. Both fields are filled with the work date by default.

Note

Ensure that the Posting Date / Work Date are correct before running the batch job.

: Create Rental Contract Invoice report request page opened from the rent order

The Action field determines if invoices are created or if a preview invoice is generated. If the Print Only option is selected, the “Rental Contract Invoicing - Test” report opens with the ability to Print or Preview the invoice without posting.

: Rental Contract Invoicing - Test report request page opened from Create Rental Contract Invoice report

The invoicing date must be entered so the preview uses the same preconditions as the invoice to be created. This is only a preview, with no effect on the on existing data.

Rental Ledger Entries

Each rent order line gets Rental Ledger Entries on creation of the Invoice (Service Invoice). Rental Ledger Entries are accessed from the Rent Order by clicking Navigate > Entries > Rental Ledger Entries.

: Rental Ledger Entries with posted and un-posted invoice lines for a sales item, resource, and rental item

These ledger entries are marked as Open until they are closed by posting the invoice. They contain the Entry Type, No. (Item, Resource, G/L account or Service Contract = rental item) and serial numbers where applicable. They also carry information about the period that is invoiced with that entry.

Prepaid Positions

In the case of prepaid positions, the ledger entries are divided according to calendar months.

: Rental Ledger Entries for a prepaid order from 1/15/2019 – 3/15/2019, not month based

The profit from the invoice is posted to a Rental Prepayment Account. The amounts have to be posted one-by-one to the Rental Account using the “Post Prepaid Rent Entries” function, accessed from the Rental Periodic Activities menu. Once this function is run, the Moved from Prepaid Acc. check box is selected on the applicable lines.

Refer to topic “Post Prepaid Rent Entries” on page 1 for more information about running this function.

Exercise – Invoice the Rent Order

On February 1, 2019, you invoice the order to customer R-30000.

Invoice the Order

Follow these steps to invoice the order:

  1. Open the rent order created in the “Record a Rental Position on a Rent Order” exercise.
  2. On the ribbon, click Post.
  3. Select the Invoice option and then click OK.
  4. In the Posting Date field, enter 02/01/19.
  5. In the Invoice to Date field, enter 02/28/19.
  6. Click OK to create the invoice and then accept all confirmation messages.

Review the Ledger Entries and Invoice

Follow these steps to review the rental ledger entries:

  1. On the ribbon, click Navigate > Entries > Rental Ledger Entries.
  2. Notice the following:
    • The rental invoice information appears.
    • The Open check box is selected because the invoice is still pending.

: Rental Ledger Entries from the non-posted invoice

  1. Close the Rental Ledger Entries page.
  2. On the Navigate tab, click Invoices.
  3. Double-click the line to open the Service Invoice Rental page.

: Service Invoice for the rent order

  1. Notice that the rental invoice will be posted to the Rental Account, as specified in the General Posting Setup page.

Post the Invoice and Review the Entries

Follow these steps to post the invoice and review the entries:

  1. On the ribbon, click Post and then click Yes to post the invoice.
  2. Escape back to the rent order.
  3. On the Navigate tab, click Entries > Rental Ledger Entries.
  4. Notice that the Open check box is cleared because the invoice is posted.

: Rental Ledger Entries for a posted invoice

  1. Close the Rental Ledger Entries page.
  2. On the Lines FastTab of the Rent Order, notice that the Quantity Invoiced field is not updated for the rental position. This field is updated when the return is posted.
