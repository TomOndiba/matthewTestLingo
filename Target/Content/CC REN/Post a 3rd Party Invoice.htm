Post a 3rd Party Invoice

Invoices from the vendor are posted from the created Purchase Order. The following information can be entered:

  • Period invoiced by the Vendor
  • Price for one day
  • Number of days
  • Rental Rate Code

The Invoice Period and the number of days are filled automatically on creation of the Purchase Order.

: Purchase Order page, 3rd Party Invoicing

The rent-related fields on the Purchase Order line are described in the following table.

Field

Description

Rental Mgt. Document

If selected, the purchase document was created from a rental document.

Rental Document Type

Indicates the Document Type of the rental document from which the purchase document was created.

Rental Document No. / Rental Document Line No.

Indicates the Document No. / Document Line No. of the rental document from which the purchase document was created.

Rent Cost per Rent Base Unit Excl. VAT

Cost of the rental for a single day. The Direct Unit Cost Excl. VAT field is calculated as:

No. of Rent Base Units * Rent Cost per Rent Base Unit

The total amount can be entered in the Direct Unit Cost Excl. VAT field instead if no daily rental price is available.

No. of Rent Base Units

Number of base units (days) that will be invoiced with the next invoice posting.

Rental Start Date / Rental End Date

Rental Start Date / End Date from the 3rd Party rent order; can be modified based on vendor invoice.

Rate Code

Rental Rate Code from the 3rd Party rent order.

After shipment of the Purchase Order, the item is available to the Rent Order because it has inventory in the 3rd Party location. An invoice from the vendor can be posted from the 3rd Party Purchase Order once the No. of Rent Base Units, Rent Cost per Rent Base Unit Excl. VAT, and the Rental Start Date and Rental End Date fields are populated.

Since the Purchase Order is not closed, these fields can be filled with new values to allow the posting of additional invoices, which can occur until the return of the line is posted. After that, one more invoice can be created and the Purchase Order is closed.
