Post Prepaid Rent Entries

The “Post Prepaid Contract Entries” batch job is used to transfer prepaid rental ledger entry amounts from prepaid accounts to income accounts. It can also be used to print a test report of the potential transfers.

: Post Prepaid Rent Entries batch job

When the batch job is run to post prepaid rental entries, the program posts the ledger entries with the rental prepaid accounts to the rental income accounts. It also updates the Rental Ledger Entries by selecting the Moved from Prepaid Acc. Check box for the lines that have been processed.

The fields on the Options FastTab are described in the following table.

Field

Description

Post until Date

Date up to which the prepaid entries are to be posted. The batch job will include rental ledger entries with posting dates on or before this date.

Posting Date

Date to be used as the posting date on the rental ledger entries and the G/L entries.

Action

Determines the action that occurs when the batch job is run:

  • Post Prepaid Transactions – Posts the prepaid entries.
  • Print Only – Prints a test report based on the selected criteria. This report shows the service contract numbers, dates, items, service contract account group codes, and amounts posted.

Exercise – Run the Post Prepaid Rent Entries Batch Job

On January 1, 2019, you post a prepaid rent order for the January – February 2019 time frame. When performing monthly closing procedures, the accounting manager runs the “Post Prepaid Rent Entries” batch job as of January 31, 2019 to move the amounts from the prepayment rental account (5792) to the rental account (6111).

Create and Post a Prepaid Rent Order

Follow these steps to create and post the rent order:

  1. Ensure that the Work Date is set to 01/01/19.
  2. Insert a new rent order for customer R-30000.
  3. In the lines, enter the following:
    1. Type = Item
    2. No. = R-30000-01
    3. Quantity = 1
    4. Rental Rate Code = Month
    5. Prepaid = Yes
    6. Rental Start Date = 01/01/19
    7. Rental End Date = 02/28/19
  4. On the ribbon, click Post.
  5. With the Ship option selected, click OK.
  6. On the ribbon, click Post.
  7. Select the Invoice option and then click OK.
  8. In the Posting Date field enter 01/01/19.
  9. In the Invoice to Date field, enter 02/28/19.
  10. Click OK to create the invoice and then accept all confirmation messages.
  11. On the ribbon, click Navigate > Entries > Rental Ledger Entries. Notice the following:
    • The Prepaid check box is selected.
    • The Moved from Prepaid Acc. check box is not selected.

: Rental Ledger Entries for prepaid order, prior to running the Post Prepaid Rent Entries batch job

  1. Escape back to the rent order.
  2. On the Navigate tab, click Invoices.
  3. Open the invoice and notice that the lines, each totaling 350.00, are to be posted to the Rental Prepayment Account (5792).

: Invoice for prepaid order

  1. On the ribbon, click Post and then click Yes to post the invoice.
  2. Close the open pages.

Run the Post Prepaid Rent Entries Batch Job

Follow these steps to run the batch job:

  1. In the navigation pane, under Rental, click Periodic Activities > Post Prepaid Rent Entries.
  2. In the Post until Date field, enter 01/31/19.
  3. In the Posting Date field, enter 01/31/19.
  4. In the Rent Order No. filter, enter the number of the rent order created in this exercise.

: Post Prepaid Rent Entries batch job

  1. Click OK.
  2. In the navigation pane, under cc|rental, click Contract Management > Rent Orders.
  3. Open the rent order created in this exercise.
  4. On the ribbon, click Navigate > Entries > Rental Ledger Entries.
  5. Notice that the Moved from Prepaid Acc. check box is selected on only the first line, which has an Invoice End Date of 01/31/2019.

: Rental Ledger Entries for prepaid order, after running the Post Prepaid Rent Entries batch job

  1. On the ribbon, click Navigate.
  2. On the Document FastTab of the Navigate page, select the G/L Entry line and then click Show Related Entries on the ribbon.
  3. On the first line, for the Prepaid Rental Contracts account (5792), click the G/L Account No. field, and then click Advanced.
  4. In the G/L Account List page, click Navigate > Ledger Entries. Notice that on 01/31/2019, an invoice total of 350.00 is debited.

: Debited prepayment rental account

  1. Select the debit line, click Navigate and then open the G/L Entry.
  2. Notice that the rental account is credited the prepaid amount.

: Rental account credited for the prepayment amount

  1. Close the open pages.

Note

The amount posted to the prepaid account is the total amount of the invoice. In cases when it is not full calendar month, such as 4/15/2019 to 5/14/2019, the amount is divided into monthly rental entries (refer to the screen shot in topic “Prepaid Positions” on page 1). These ledger entries have different posting dates so the system can separate which month they are posted for.

The system posts a cumulated sum from the prepaid account to the rental account based on the filtered Rental Ledger Entries not regarding the G/L entries on the prepaid account. Therefore, if the Post until Date field on the batch job is set to 5/31/2019, then the total of those three lines from the rental ledger entries are moved to the rental account.
