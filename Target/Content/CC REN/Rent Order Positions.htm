Rent Order Positions

In the Rent Order, users can enter positions to be rented (items) and also positions to be sold (items, resources, and G/L accounts) along with the rental.

Only items that have the Rent Item field set to Yes on the Item Card can be rented; items without this attribute can be sold only.

Sales positions are entered the same as a regular sales line while the rental positions have more requirements.

Record a Sales Position on a Rent Order

Follow these steps to fill in a sales position on the rent order:

  1. In the navigation pane, click cc|rental > Contract Management > Rent Orders.
  2. On the ribbon, click New.
  3. In the Sell-to Customer No. field, enter the customer number.
  4. On the Lines FastTab, leave the Rental Position set to Sales.
  5. In the Type field, enter Item, Resource or G/L Account.
  6. In the No. field, enter the relevant sales item, resource or G/L account.

Note

If a rental item is selected that is to be added as a sales line, the Rental Position will need to be reset to Sales. Any Rate Component lines that appeared will be removed when the Rental Position is changed.

  1. In the Location Code field, enter a location if the inventory is posted using locations.
  2. In the Quantity field, enter the number of units to be sold.
  3. Fill in other standard Microsoft Dynamics NAV fields as needed.

Exercise – Record a Rental Position on a Rent Order

On January 31, 2019, customer R-30000 orders one unit of item R-30000-01 for the period of February 1, 2019 through February 28, 2019. The rental price is 350.00 for all customers.

Follow these steps to record the rental position:

  1. Set the Work Date to 01/31/19.
  2. In the navigation pane, click cc|rental > Contract Management > Rent Orders.
  3. On the ribbon, click New.
  4. In the Sell-to Customer No. field, enter R-30000.
  5. In the Lines FastTab, select Item in the Type field.
  6. In the No. field, enter R-30000-01. Notice the following:
    • The Rental Position field changed from Sales to Rental.
    • The RENTAL location defaults to the Location Code field (if assigned to the customer).
    • The customer location (R-30000) defaults to the Customer Location Code field.
    • The Unit Price Excl. VAT field is blank because no standard unit price has been assigned to the item.
    • The Rent Line Details FactBox displays 34 available units and a sales price for the item.

  2. In the Rental Rate Code field, enter MONTH. Notice that the Rate Unit Price field is 350.00, which is from the rent prices setup.
  3. In the Rental Start Date field, enter 02/01/19. Notice that the Rent Line Details FactBox now displays 31 available units of item R-30000-01.

Transportation Periods and Item Availability

For the availability of transporting periods, the outbound and inbound shipping time can be entered.

With entry of the Rental Start Date and the Rental End Date on the rental lines, the system creates a separate Transfer Order that uses the rent order information, such as Shipping Times, Rental Start Date, Rental End Date, Location Codes, and so on. It prepares a transfer from the rental location to the customer location on shipment and back on return.

Shipping Times and Shipment / Receipt Dates of the Transfer Order are grouped to provide availability information on the Item Availability by Periods page for the specific date.

In addition, the availability information in the Rent Line Details FactBox on the Rent Order shows the maximum rental availability for the total renting period after filling in the Rental Start Date and Rental End Date to avoid overlapping. Therefore, it calculates the inventory regarding planned positive and negative stock movements from documents in the period and retrieves the minimum availability of that total period. If this is not the case (such as when the Rental End Date is missing or it is a sales position) it shows the availability information as in a standard Sales Order.

Serial Number Tracked Rental Items

For renting of items with serial numbers, the tracking information must be assigned on Shipment and on Receipt of the serial number. After clicking Line > Item Tracking Lines on the Lines FastTab, a message appears for rent positions where users can indicate whether the shipment or the receipt of the serial number is to be assigned.

After selecting the proper option, the standard Item Tracking Lines page opens:

  • On shipment, the serial numbers can be entered.
  • On receipt, the available serial numbers are filtered by the ones that were shipped from the line.

Refer to topic “Failures on Serial Number Tracked Items” on page 1 for exercises that include serial number-tracked item processing.

Test Report

The Test Report can be run to simulate the results of posting (ship/receive and/or invoice) the rent order. In the Invoice To Date field, enter a date filter for the last due date to be included in the view. Select the Show Dimensions check box to display dimension information for the rent order lines.
