Rate Components

Rate components can be added to Rent Quotes and Rent Orders (refer to topic “Rent Rate Components” on page 1). Depending on the settings in the Rental Mgt. Setup page, the lines are added always or only when the components and rent prices exist for the item (refer to topic “Rental Mgt. Setup” on page 1).

Once a rental item number is added to a rent order line, the related components are added to the line (refer to topic “Item Rent Rate Components” on page 1). Rental parameter changes made to the rental item line are also made to the component lines. Rate Components are always linked to rental item lines; they cannot be used as “stand alone” positions. Rate Components are not automatically credited when a failure is processed (refer to topic “Failures” on page 1).

Rate Components can be set up with a unit price or as a percentage of the item’s rental price (refer to topic “Rent Component Prices” on page 1).

The processing of Rate Components differs depending on the type:

  • Rate Parts
    • Always invoiced with the rental item line and for the same period
    • Automatically link to a rental item line if added to a rent order
    • Must have a Rate Unit Price prior to shipping
  • One-Time Invoicing
    • Handled the same as Sales positions.
    • When set up to invoice at Start of Rental, the lines are shipped with the item and are due on the first day of the Rental (therefore the lines are included on the first invoice).
    • When set up to invoice at End of Rental, the Qty. to Ship field is initially set to 0 (blank).

    When the return is processed, the value in the Return Qty. to Receive field from the rental item line is added to the Quantity Shipped field on the One-Time Invoicing line. This quantity is used when the line is invoiced.
