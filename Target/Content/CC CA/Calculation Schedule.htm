Calculation Schedule

After the setup of calculation groups is complete, the Calculation Schedules need to be defined.

A default calculation schedule can be assigned for all items in the Standard Calc. Schedule No. field on the Calculation Setup page and then modified as needed on the Invoicing FastTab of the individual Item Cards.

: Calculation Schedule Card for the Default Calculation with all line fields

The General FastTab contains the code, description, start date, status, and how concurrent calculations are created. Active schedules must have a Status of Certified.

The fields in the Lines FastTab are described in the following table.

Field

Description

Row No.

Optional code field that is used to identify a corresponding calculation line in the Reference to Row No. field.

Note

With the SQL Server, it is useful to create row numbers with leading zeros (0100, 0200, and so on) to prevent sequencing from changing when numbers greater than 999 are used.

Description

Description of the calculation, which can be indented by clicking Functions > Increase Indentation or Decrease Indentation on the Lines FastTab.

Totaling Type

Specifies what the calculation line is based on. The options are:

  • <blank> – used on lines that have no calculation value.
  • Charge – used on lines that calculate an increase or decrease of another calculation line.
  • Sum – used on lines that total other calculation lines. Standard Microsoft Dynamics NAV filters can be used, such as range (1000..2000), either/or (1000|2000), or any of the following: * ? > < = @.
  • Arithm. Formula – used on lines that total other calculation lines with a mathematical formula. To use the values from other calculation lines, the "Row No." must begin with an "R". For example:
    • To use the value of the line with Row No. 010 multiplied by 2, enter R010 * 2.
    • To calculate the material costs in Row No. 010, with an additional 10% for material, plus the resource costs in row 020, enter R010 * 10/100 + R020.

Note

With this option, the Formula Editor can be opened from the Totaling field.

  • Item Calc. Group – used on calculation lines that gather totals from item calculation groups.
  • Capacity Calc. Group – used on calculation lines that gather totals from capacity calculation groups.

Totaling

Values relating to the selection in the Totaling Type field.

When Totaling Type = Arithm. Formula, this field can open the Formula Editor that displays which functions can be used and if the formula is correct.

Reference to Row No.

Refers to the row number that applies to the charge calculation. Used only on lines with a Totaling Type of Charge.

Charge Editable

If selected, the value for the price charges in the following fields in the Calculation Card can be overwritten:

  • Charge Type
  • Charge Rate

Line calculations are updated as changes are made to these fields.

Used only on lines with a Totaling Type of Charge and can only be performed on Calculation Cards with the Status set to New or Under Development.

Calculate Foreign Currency

If selected, values are calculated in foreign currency, as seen in the Value (LCY) and Value fields (which display only for foreign currency). This is useful in lines with prices.

Note

A currency code must be assigned to the calculation – in the Foreign Trade FastTab or from the sales line – to use this feature.

Value Transfer Option

Determines if and to where on an item the calculation value can be transferred when the “Transfer Value” function is run:

  • Standard Cost/Unit Cost
  • Unit Price
  • Sales Line Discount

Refer to topic "Transfer Value" on page 1 for more information.

Bold

If selected, the Description on the line is bolded.

Calculation View Authorization

Determines if certain calculation lines only be viewed by certain users (refer to topic “Calculation Views per User” on page 1).

 

Update Sum Calculation

The “Update Sum Calculation” function shows which calculation lines are included in Charge, Sum and Arithm. Formula totaling lines by changing the text color of the Row No. and Description fields to bold blue on all lines within the calculation. To run this function, select a totaling line and then on the Lines FastTab, click Functions > Update Sum Calculation, or press F5.

This is helpful when setting up Calculation Schedules to confirm totaling. For example, to see the lines included in the “Total production costs” calculation, click line 0500 and then run the “Update Sum Calculation” function. Note that lines 0320, 0400 and 0450 are included, as indicated by the setup in the Totaling field on line 0500.
