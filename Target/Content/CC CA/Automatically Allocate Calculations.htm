Automatically Allocate Calculations to Concurrent Calculations

Concurrent calculations can be transferred from existing documents to new documents during creation processes.

Sales Orders Created from Quotes or Blanket Orders

If a sales order is created from a sales quote or a blanket sales order and a calculation with a concurrent calculation number exists for the document line, a sales order line calculation is created automatically. The calculation number is transferred to the concurrent calculation line.

Example

The CA-047 calculation was created for Sales Quote No. 1001. Then, the CC-0001 concurrent calculation was assigned to the CA-047 calculation, which has now been linked to and inserted into the concurrent calculation.

: Concurrent Calculation assigned to the CA-047 calculation

When the “Make Order” function is run on the quote, a new sales order is created.

: Make Order process run on Sales Quote No. 1001 to create Sales Order No. 1020

A new calculation is created for the order and automatically attached to the sales order line and inserted into the Concurrent Calculation.

: New calculation inserted in the concurrent calculation

Production Orders Created from Sales Orders

If a production order is created from a sales order and a calculation with a concurrent calculation number exists for the document line, a production order line calculation is created automatically. The calculation number is transferred to the concurrent calculation line.

An expected cost calculation for production order lines is created. If the production order status is changed to Released or a released production order is created directly, an actual cost calculation is provided automatically.

Example

The CA-055 calculation for Sales Order No. 1020 was created from the calculation of the sales quote and inserted in the CC-0001 concurrent calculation (refer to Fig. 3).

: Calculation Card for Sales Order created from Sales Quote

Run the “Create Prod. Order” function to create a production order from the sales order planning.

: Create Prod. Order process run on Sales Order No. 1020 to create a Released Production Order

Since a released production order was created, Expected and Actual calculations are created (copied from the sales order calculation), attached to the released production order line, and inserted in the Concurrent Calculation.

: New calculations inserted in the concurrent calculation

Assembly Orders Created from Sales Orders

If an assembly order is created from a sales order and a calculation with a concurrent calculation number exists for the document line, an expected assembly order line calculation is created automatically. The calculation number is transferred to the concurrent calculation line.

When the assembly order status is changed to Released, an actual cost calculation is created automatically and is transferred to the concurrent calculation line.

Example

The CA-058 calculation for Sales Order No. 1021 was created and inserted in the CC-0009 concurrent calculation.

: Calculation Card for Sales Order

An assembly order can be created manually from the sales order or created automatically by populating the Qty. to Assemble to Order field on the sales order line.

: Assembly Order A00002 created from Sales Order No. 1021

Since the assembly order has not been released, only an Expected calculation is automatically created (copied from the sales order calculation) and inserted into the Concurrent Calculation.

: New expected calculation created from Assembly Order

When the Assembly Order is released, the Actual calculation is created and assigned to the Concurrent Calculation.

: New calculations inserted in the concurrent calculation
