Item Calculation Groups

Item Calculation Groups identify items (in a production BOM) that need to be combined in a separate calculation line.

: Item Calculation Groups

Note

It is only necessary to use Item Calculation Groups if items or work centers in the calculation need to be separately represented.

Define the calculation to be performed on a group of items in the Calculation Group Code field on the Item Card.

: Calculation Group Code defined on item 1100

With this function, costing calculation groups can be created.

For example, an item’s production BOM contains raw materials, semi-finished materials, and packing materials. In the Item Card, a user enters the RAW item calculation group for the raw materials, SEMI for the semi-finished materials, and PACK for the packaging materials.

In the calculation schedule, the user adds the following lines:

  • First for the raw material costs and enters the RAW item calculation group.
  • Second for the semi-finished material and enters the SEMI item calculation group.
  • Third for the packaging materials and enters the PACK item calculation group.

: Sales Calculation with item calculation group setup

When a calculation is created for the item, the costs of the raw materials are totaled in the first calculation line, the costs of the semi-finished lines are totaled in the second one, and the costs of the packaging lines are totaled in the third line.

In this example, the system checks the assigned BOM of the calculated item and differentiates between the items by those that have the Calculation Group Code of RAW, SEMI, and PACK and then any other lines will be calculated in the Misc. material line (refer to Fig. 3).

: Sales Calculation for item 1100 with raw material and semi-finished material calculations

Note

To ensure that the values for the material costs are correct after calculation, all items included in the production BOM must have a value in the Standard Cost field. The calculation is generally based on the item’s base unit of measure.
