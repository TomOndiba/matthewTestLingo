Allocate Calculations to Concurrent Calculations

Concurrent calculations can be used to compare different calculations for sales document, production order and assembly order lines, all of which are created in an order tracking chain.

The concurrent calculation setup is opened from the Sales & Marketing, Administration menu. From the Concurrent Calculation List page, open an existing calculation or create a calculation by clicking New on the ribbon.

The lines for a concurrent calculation – Item, Quote, Blanket Order, Order, Prod. Order (Expected), Prod. Order (Actual), Asm. Order (Expected) and Asm. Order (Actual) – are inserted automatically after creating a new concurrent calculation.

The Calculation Schedule No. field must be populated in the concurrent calculation. This calculation schedule is used as the base calculation schedule for all calculations within one concurrent calculation, otherwise the calculations are incomparable.

: New Concurrent Calculation for the Sales Calculation

The number series for the concurrent calculation are entered in the Calculation Setup page.

: Number series defined for concurrent calculations

There are four ways to use a calculation for a concurrent calculation:

  1. Selecting calculations directly from a concurrent calculation line.
  2. Entering the concurrent calculation number in the Concurrent Calculation No. field on the Document FastTab of the Calculation Card.
  3. Creating a concurrent calculation from an existing calculation (empty Concurrent Calculation No. field).
  4. Creating a concurrent calculation automatically when a new calculation is created.

Select Calculation from a Concurrent Calculation Line

Calculations can be selected directly from a concurrent calculation line by clicking the arrow in the Calculation No. field and selecting the appropriate calculation.

: List of calculations from a Concurrent Calculation

Once selected, the calculation information populates the fields on the line in the Concurrent Calculation page.

Enter the Concurrent Calculation Number on the Calculation Card

A concurrent calculation can be assigned to a Calculation Card by entering the number in the Concurrent Calculation Code field on the Document FastTab.

Once selected, the concurrent calculation is assigned to the calculation.

Create a Concurrent Calculation from an Existing Calculation

A new concurrent calculation can be created from an existing calculation if the Concurrent Calculation No. field is blank on the Document FastTab of the Calculation Card.

On the ribbon, click Concurrent Calculation. The following message appears.

: Concurrent Calculation creation message for existing calculation

Click Yes to create the concurrent calculation or No to stop the creation process. The Concurrent Calculation page opens when the new concurrent calculation is created.

: Newly created Concurrent Calculation

Automatically Create a Concurrent Calculation with a New Calculation

Calculation Schedules can be set up to have a concurrent calculation created automatically when a calculation is created. The Create Concurrent Calculation field on the General FastTab of the Calculation Schedule page has three options that determine if a concurrent calculation is created for the specific calculation schedule:

  • Never – concurrent calculations are not created automatically; one of the previously listed procedures must be followed to create the concurrent calculation
  • Question – user is prompted to create the concurrent calculation
  • Always – concurrent calculations are created automatically without user interaction

: Create Concurrent Calculation field on Calculation Schedule Card

When the Create Concurrent Calculation field is set to Question, the user will receive a message when they select the calculation schedule (or press ENTER if a standard template is used).

: Message received upon creating a new calculation for a Calculation Schedule with Create Concurrent Calculation set to Question

Click Yes to create the concurrent calculation or No to stop the creation process.

: Concurrent calculation link to calculation

The Description field on the automatic concurrent calculation is populated from the header of the first calculation used in the concurrent calculation.

: Concurrent calculation created automatically when clicking Yes to creation question

When the Always option is selected on the calculation schedule, the concurrent calculation and link to the calculation are created automatically when the calculation schedule is selected on the new calculation.

: Concurrent calculation created with new calculation for a Calculation Schedule with Create Concurrent Calculation set to Always

On the ribbon, click Concurrent Calculation to open the Concurrent Calculation for the calculation.
